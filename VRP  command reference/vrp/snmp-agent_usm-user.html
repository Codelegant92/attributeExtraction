snmp-agent usm-user

命令功能

snmp-agent usm-user命令用来为一个SNMP组添加一个新用户。

undo snmp-agent usm-user命令用来删除SNMP组的一个用户。

命令格式

snmp-agent usm-user v3 user-name group-name [ authentication-mode { md5 | sha } password [ privacy-mode des56 password ] ] [ acl acl-number ]

undo snmp-agent usm-user v3 user-name group-name { engineid engineid-string | local }

参数说明

v3：用户使用V3的安全模式。

user-name：用户名。字符串形式，不支持空格，长度范围是1～32。

group-name：该用户对应的组名。字符串形式，长度范围是1～32。

authentication-mode：指明安全级别为需要认证。

md5：指定认证协议为HMAC-MD5-96。

sha：指定认证协议为HMAC-SHA-96。

privacy-mode：指明安全级别为加密。

des56：指定加密协议为DES加密协议。

password：密码。字符串形式，长度范围是1～64。

acl：设置访问视图的列表。

acl-number：基本访问列表，取值范围是2000～2999。

engineid：指定与该用户相关联的引擎ID。

engineid-string：引擎ID字符串，长度范围10～64。

local：本地实体用户。

使用指南

用户在为代理配置了远端用户后，当代理和远端用户通信时，需要用到engineID进行校验。如果配置用户后引擎ID（engineID）改变，则原引擎ID相应的用户无效。

使用实例

# 为SNMP组Johngroup加入一个用户John，安全级别为需要认证、指定认证协议为HMAC-MD5-96、认证密码为hello。

<Quidway> system-view
[Quidway] snmp-agent usm-user v3 John Johngroup authentication-mode md5 hello
