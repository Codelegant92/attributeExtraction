label-type

命令功能

label-type命令用来在NQA视图下配置报文的封装类型。

undo label-type命令用来取消在NQA视图下配置的报文封装类型。

缺省情况下，报文封装类型使用控制字control-word方式。

命令格式

label-type { control-word | label-alert | normal }

undo label-type

参数说明

control-word：指定在报文中封装control word控制字信息。

label-alert：指定在报文中封装router alert标签。

normal：普通方式，MPLS echo request报文不封装control word和router alert选项。只能用在NQA PWE3 Tracet中。

视图

NQA视图

缺省级别

2：配置级

使用指南

使用label-type命令前，需要先在NQA视图下配置NQA测试例的测试类型为PWE3trace或者PWE3ping，否则无法配置label-type命令。

封装control word后，多跳的情况下，在switch节点，如果标签TTL值不超时，则MPLS echo request报文不会被上送CPU处理。这种方式下，源端获得的PW信息较少，但对设备的性能影响较小，并且源端看不到该switch节点的下游信息。当报文多的时候，推荐用此方式。

封装router alert后，多跳的情况下，在switch节点，MPLS echo request报文被上送CPU处理。这种方式下，源端获得的PW信息较多，但如果报文太多是，对设备的性能影响较大。如果测试例比较少时，推荐采用此方式，以获取switch节点的详细信息。

使用实例

# 在NQA视图下配置报文的封装类型为label-alert。

<Quidway> system-view
[Quidway] nqa test-instance admin  pwe3
[Quidway-admin-pwe3] test-type  pwe3trace
[Quidway-admin-pwe3] label-type label-alert
