increment-config synchronization

命令功能

increment-config synchronization命令用来把成员交换机的配置文件同步到FTP服务器。

命令格式

increment-config synchronization [ group-by { device-type device-type | ip {ip-addres [ to ip-addres ] } &<1-10> | member-number { member-number [ to member-number ] } &<1-10> } ]

参数说明

group-by：指定成员交换机的分类方式。目前支持三种分类方式：设备类型、IP地址和成员编号。不选择该参数时，默认对所有的成员交换机进行操作。

device-type device-type：指定成员交换机的设备类型。字符串形式，不区分大小写，不支持空格，长度范围是1～100。一次只能指定一种设备类型。目前支持的设备类型由NTDP协议收集得到，例如NE40E，CX200B等。

ip-address：指定成员交换机的IPv4地址。可以同时指定一个或者多个成员交换机的地址和地址范围。

member-number：指定成员交换机的成员编号。十进制整数形式，取值范围是1～255。可以同时指定一个或者多个成员交换机的成员编号。

file-name：配置文件、版本文件、PAF文件、License文件或者补丁文件的名称。字符串形式，不区分大小写，长度范围是1～50。

ip ftp-ip-address：FTP服务器的IPv4地址，点分十进制形式。

username user-name：FTP服务器的用户名称。字符串形式，区分大小写，不支持空格，长度范围是1～20。

password password：FTP服务器的用户密码。字符串形式，区分大小写，不能输入空格和问号等命令行专用字符，长度范围是1～8。

使用指南

本命令只能在命令交换机上执行。

同步成员交换机的配置文件的目的是把成员交换机的配置保存到FTP服务器，以便设备更换后，可以根据原有设备的配置来恢复新设备配置，以减少人工再次配置工作量。

指定成员交换机的成员编号时不能超过集群支持的最大编号。

成员交换机的配置在FTP服务器上的存储方式是以成员交换机MAC地址命名的配置文件。

使用本命令时，如果不指定FTP服务器的地址、用户名称和用户密码，则成员交换机将使用命令交换机上配置的默认FTP服务器信息。

说明：

命令交换机上默认的FTP服务器配置用来支持HGMP的即插即用功能。具体配置方法请参考cluster-plug-play ip命令。

使用实例

# 指定集群中IP地址是10.1.1.1～10.1.1.100的设备同步配置文件至FTP服务器100.1.1.1。FTP服务器的用户名称是user1，密码是1234。

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] increment-config synchronization group-by ip 10.1.1.1 to 10.1.1.100 ip 100.1.1.1 user user1 password 1234
