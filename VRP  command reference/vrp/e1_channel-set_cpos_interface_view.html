e1 channel-set（CPOS接口视图）

命令功能

e1 channel-set命令用来对E1通道的时隙进行捆绑。

undo e1 channel-set命令用来取消指定的捆绑。

命令格式

e1 e1-number channel-set set-number timeslot-list slot-list

undo e1 e1-number channel-set set-number

参数说明

e1-number：CPOS的E1通道号。整数形式，取值范围是1～63。

set-number：捆绑集的编号。整数形式，取值范围是0～30。

slot-list：用于捆绑的时隙编号或时隙范围。取值范围是1～31。指定捆绑时隙时，可以指定单个时隙，也可以指定时隙范围。使用“,”分隔多个时隙，使用“-”表示时隙范围。

视图

CPOS接口视图

缺省级别

2：配置级

使用指南

当CPOS的E1应用在通道化模式（Channelized）时，除时隙0用于同步外，其它31个时隙可任意捆绑为一个或多个串口。

说明：

为保证系统处理能力，一个CPOS物理端口上最多可以捆绑256个串口。

使用实例

# 对E1通道63进行捆绑。

<Quidway> system-view
[Quidway] controller cpos 1/0/0
[Quidway-Cpos1/0/0] e1 63 channel-set 1 timeslot-list 1-31

# 进入捆绑后形成的串口的视图。

[Quidway-Cpos1/0/0] quit
[Quidway] interface serial 1/0/0/63:1
[Quidway-Serial1/0/0/63:1]

项目

描述

Serial1/0/0/63:1 current state :
串口当前的物理状态：
  • UP：接口的物理层处于正常启动的状态。
  • DOWN：接口的物理层出现故障。
  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Line protocol current state :

显示该接口的链路协议状态：
  • UP：接口的链路协议处于正常的启动状态。
  • DOWN：接口的链路协议层出现故障，或者没有在此接口配置IP地址。

The Maximum Transmit Unit is

接口的最大传输单元（MTU）。例如串口缺省值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

Hold timer is

报文的生命周期。报文如果在生命周期内没有被送出去，则将丢弃。

Derived from

形成串口的CPOS接口号和E1通道号。

Timeslot(s) Used:

形成串口的时隙号。

baudrate is

串口的带宽。波特率。

Internet protocol processing : disabled

接口没有配置IP地址。

Link layer protocol is

串口的数据链路层协议。

LCP initial

LCP初始化。

Physical layer is

物理层链路信息。

Last 300 seconds input rate

Last 300 seconds output rate

errors

这一字段显示了在前300秒内通过接口发送和接收的字节速率和报文速率。在物理层检测时发现的错误报文数目。

Input

接口收到的总报文数和总字节数。

giants

接收到长度大于规定长度的报文数目。此接口接收报文的最大长度为1500字节。

errors

在物理层检测时发现的错误报文数目。

CRC

收到了长度正常但CRC校验错误的报文数目。

giants

接收到长度大于规定长度的报文数目。此接口接收报文的最大长度为1500字节。

CRC

收到了长度正常但CRC校验错误的报文数目。

overruns

接收的报文速度大于转发处理能力导致无法处理的报文。

aborts

接收报文的异常错误。

Output:

接口发送的报文数和总字节数。

underruns

因为接口读取内存的速度小于转发的速度而无法发送报文数目。
