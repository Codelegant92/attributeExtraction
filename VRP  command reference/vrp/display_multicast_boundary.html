display multicast boundary

命令功能

display multicast boundary命令用来查看接口上配置的组播边界信息。

命令格式

display multicast [ vpn-instance vpn-instance-name | all-instance ] boundary [ group-address [ mask | mask-length ] ] [ interface interface-type interface-number ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

group-address：表示组播组地址。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

mask：表示组播组地址的掩码。点分十进制形式。

mask-length：表示组播组地址掩码长度。整数形式，取值范围是4～32。

interface-type interface-number：表示路由器的接口类型和接口号，用来指定具体的接口。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display multicast boundary命令：

  • 若不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 如果指定group-address，则显示为指定组配置的边界信息。

  • 如果指定group-address mask或group-address mask-length，则显示为指定范围内的组地址配置的边界信息。

  • 如果指定interface，则仅显示在指定接口上配置的组播边界。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
