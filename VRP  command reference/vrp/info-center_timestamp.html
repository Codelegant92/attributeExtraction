info-center timestamp

命令功能

info-center timestamp命令用来设置输出的调试/告警/日志信息中时间戳格式。

undo info-center timestamp命令用来恢复缺省设置。

缺省情况下，告警信息和日志信息采用date时间戳，调试信息采用boot时间戳。

命令格式

info-center timestamp { debugging | trap } { boot | date | none | short-date }

info-center timestamp log { boot | date | none | { short-date | format-date } [ precision-time { tenth-second | millisecond }] }

undo info-center timestamp { log | debugging | trap }

参数说明

debugging：调试信息。

log：日志信息。

trap：告警信息。

boot：时间戳采用相对时间类型，即系统启动后经过的时间。格式是xxxxxx.yyyyyy，xxxxxx为系统启动后经过时间的毫秒数高32位，yyyyyy为低32位。

date：时间戳采用系统当前日期和时间，中文环境下为yyyy/mm/dd hh:mm:ss；英文环境下为mm dd hh:mm:ss yyyy。

short-date：时间戳采用短日期格式。这种格式的时间戳与date类型的时间戳基本相同，唯一区别是短日期格式取消了年份的显示。

format-date：时间戳采用另一种时间表示方式。按照年、月、日、时、分、秒的格式显示：YYYY-MM-DD hh:mm:ss

none：输出的信息不包含时间戳。

precision-time：时间戳精度。

tenth-second：时间戳精确到0.1秒。

millisecond：时间戳精确到毫秒。

使用指南

当采用date类型时，时间戳各部分介绍如下表所示。

表1 date类型的时间戳各部分描述

字段

含义

取值

yyyy

年份

4位数字格式

mm

月份

中文环境：1～12西文环境：Jan，Feb，Mar，Apr，May，Jun，Jul，Aug，Sep，Oct，Nov，Dec

dd

日期

1～31西文环境下，如果日期的值小于10，则在日期前面补一个空格，例如“7”

hh:mm:ss

本地时间，时:分:秒

hh采用24小时制，取值范围是00～23mm和ss取值范围都是00～59

在时间戳的精度精确到0.1秒或毫秒时，同时间产生的日志会按照日志的产生顺序加入相应的标示。

使用实例

# 设置日志（log）信息时间戳格式为boot。

[Quidway] info-center timestamp trap boot
