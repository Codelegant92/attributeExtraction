load-balance（Eth-Trunk接口视图）

命令功能

load-balance命令用来配置转发表的散列算法。

undo load-balance命令用来恢复缺省设置。

缺省情况下，三层Eth-Trunk接口根据ip进行散列，二层Eth-Trunk接口根据mac进行散列。

命令格式

load-balance { ip | mac | packet-all }

undo load-balance

参数说明

ip：根据源IP地址和目的IP地址进行散列。

mac：根据MAC地址进行散列。

packet-all：根据所有包进行散列。

视图

Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

二层Eth-Trunk接口可以按照MAC地址和所有包进行散列，三层Eth-Trunk接口只能按照IP地址和所有包进行散列。

对于NE40E、NE80E、NE5000E产品，二层Eth-Trunk接口根据IP地址进行散列。

IP、MAC两种方式适用于按照流进行负载分担的场景，而packet-all适用于逐包进行负载分担的场景。

基于包的散列算法能保证带宽利用率，但不能保证包的顺序。

基于IP地址或MAC地址的散列算法能保证包顺序，但不能保证带宽利用率。

使用实例

# 配置Eth-Trunk 1根据IP地址进行散列。

<Quidway> system-view
[Quidway] interface eth-trunk 1
[Quidway-Eth-Trunk1] load-balance ip
