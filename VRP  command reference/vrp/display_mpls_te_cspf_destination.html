display mpls te cspf destination

命令功能

display mpls te cspf destination命令用来检查是否存在满足指定约束条件的路径。这里的约束条件由该命令的参数指定。

命令格式

display mpls te cspf destination ip-address [ affinity properties [ mask mask-value ] | bandwidth { ct0 ct0-bandwidth | ct1 ct1-bandwidth | ct2 ct2-bandwidth | ct3 ct3-bandwidth | ct4 ct4-bandwidth | ct5 ct5-bandwidth | ct6 ct6-bandwidth | ct7 ct7-bandwidth }* | explicit-path path-name | hop-limit hop-limit-number | metric-type { igp | te } | priority setup-priority | srlg-strict exclude-path-name | tie-breaking { random | most-fill | least-fill } ]*

参数说明

destination ip-address：指定TE隧道的目的IP地址。

affinity properties：指定TE隧道的亲和属性。

mask mask-value：指定亲和属性的掩码。

bandwidth：指定TE隧道的带宽。

ct0 ct0-bandwidth～ct7 ct7-bandwidth：指定CT0～CT7的带宽值。

explicit-path path-name：指定TE隧道的显式路径。

hop-limit hop-limit-number：指定TE隧道的最大跳数限制值。

metric-type：指定CSPF计算时使用的度量类型。

igp：指定度量类型为内部网关协议。

te：指定度量类型为流量工程。

priority setup-priority：指定TE隧道的建立优先级。

srlg-strict exclude-path-name：指定SRLG排除路径（SRLG disjoint path）。SRLG排除路径是指配置了排除地址（exclude address）的严格显式路径，并且严格显式路径包含了一个或多个配置了SRLG的接口。如果指定SRLG排除路径，则CSPF检查是否存在这样的路径，该路径没有包含排除路径的任何SRLG属性。

tie-breaking：指定CSPF计算的仲裁策略。

random：指定仲裁策略为随机选择路径。

most-fill：指定仲裁策略为选择已用带宽和最大可预留带宽的比值最大的链路。

least-fill：指定仲裁策略为选择已用带宽和最大可预留带宽的比值最小的链路。

使用指南

执行该命令时，设备使用CSPF算法进行计算，检查是否存在满足指定约束条件的路径。如果有，则显示该TE隧道的路径，否则显示结果为空。

使用实例

# 查看通过CSPF算法得到的，满足命令约束条件的路径。

<Quidway> display mpls te cspf destination 2.2.2.9

项目

描述

Path for the given constraints is

计算得到的路径。后面有*表示指定的显式路径中，通过CSPF计算不能到达的节点；后面没有*表示通过CSPF计算得到的路径

The total metrics for the calculated path is

计算得到的路径的度量值

System is busy try it later

系统忙稍后再试。表示CSPF接收了大量的计算请求，任务队列满，无法接收当前的计算请求

CSPF compute time out

CSPF计算超时。表示CSPF计算的时间太长，超过了定时器的值
