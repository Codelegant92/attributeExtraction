rip metricout

命令功能

rip metricout命令用来设置接口发送RIP路由增加的度量值。

undo rip metricout命令用来恢复该度量值为缺省值。

命令格式

rip metricout { value | { acl-number | ip-prefix ip-prefix-name } value1 }

undo rip metricout

参数说明

value：对发送的路由增加的度量值。整数类型，取值范围是1～15。缺省值为1。

acl-number：指定基本ACL的编号。整数类型，取值范围是2000～2999。

ip-prefix ip-prefix-name：指定IP地址前缀列表的名称。该名称必须唯一。字符串类型，不支持空格，取值范围是1～169。

value1：对通过ACL或ip-prefix方式过滤的路由增加度量值。整数类型，取值范围是2～15。

使用指南

当发布一条路由时，发送度量值会在发布该路由之前附加在这条路由上。因此，增加一个接口的发送度量值，该接口发送的RIP路由权值也会相应增加。但路由表中的度量值不会发生改变。

当用ACL或ip-prefix方式来设置接口发送RIP路由增加的度量值时，指定value1为通过过滤策略的RIP路由增加的度量值，没有通过过滤的RIP路由增加的度量值为1。

使用实例

# 设置RIP发送度量值为12。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] rip metricout 12

# 设置用编号为2050的ACL过滤的RIP路由的发送度量值为12。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] rip metricout 2050 12

# 设置用IP地址前缀列表p1过滤的RIP路由的发送度量值为12。

<Quidway> system-view
[Quidway]ip ip-prefix p1 permit 10.10.10.1 24
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] rip metricout ip-prefix p1 12
