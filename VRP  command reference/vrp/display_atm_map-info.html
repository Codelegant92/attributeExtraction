display atm map-info

命令功能

display atm map-info命令用来显示ATM的上层协议映射表信息。

命令格式

display atm map-info [ interface atm interface-number [ pvc { pvc-name [ vpi/vci ] | vpi/vci } ] ]

参数说明

interface-number：接口编号。

pvc-name：PVC名，字符串形式，长度为1～16。如果不指定PVC名且不指定VPI/VCI值对，则缺省显示指定ATM接口范围内所有PVC的上层协议映射表信息。

vpi/vci：VPI/VCI值对。详见pvc命令中的参数说明。

缺省级别

1：监控级

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

使用实例

# 显示设备中所有ATM接口的上层协议映射表信息。

<Quidway> display atm map-info
Atm1/0/0, PVC 1/33, IP, State UP
100.11.1.1, vlink 1
Atm1/0/0, PVC 1/32, PPP, Virtual-Template10, UP
Atm1/0/0, PVC 2/101, ETH, Virtual-Ethernet1/0/0, UP

项目

描述

Atm1/0/0, PVC 1/33, IP, State UP100.11.1.1, vlink 1

Atm 1/0/0接口的PVC 1/33配置了IPoA映射，配置映射的静态IP地址为100.11.1.1，占用了1号vlink，其状态为已激活（Up）。

Atm1/0/0, PVC 1/32, PPP, Virtual-Template10, UP

Atm 1/0/0接口的PVC 1/32配置了PPPoA映射，采用了编号为10的VT接口，其状态为已激活（Up）。

Atm1/0/0, PVC 2/101, ETH, Virtual-Ethernet1/0/0, UP

Atm 1/0/0接口的PVC 2/101配置了PPPoEoA映射或者IPoEoA映射，采用了编号为1/1/1的VE接口，其状态为已激活（Up）。
