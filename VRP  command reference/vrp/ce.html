ce

命令功能

ce命令用来在VPN内创建一个CE，并进入MPLS-L2VPN-CE视图。

undo ce命令用来删除配置的CE。

缺省情况下，系统在VPN内没有创建任何CE。

命令格式

ce ce-name [ id ce-id [ range ce-range ] [ default-offset ce-offset ] ]

undo ce ce-name

参数说明

ce-name：在当前PE的当前VPN上指定CE名称。字符串形式，不支持空格，区分大小写，长度范围是1～20。

id ce-id：VPN内CE的ID值。十进制整数形式，取值范围是0～249。CE ID用于在一个VPN中唯一确定一个CE。为了方便配置，建议CE ID从1开始，采用连续自然数编号。

range ce-range：当前PE上最多可连接的CE数量。十进制整数形式，取值范围是1～250。缺省值为10。

default-offset ce-offset：缺省的初始ce-offset（ce-offset是指与本CE建立连接的本地其他CE或远端CE的CE ID）。十进制整数形式，取值为0或1。缺省值为0。

使用指南

配置此命令前，先要配置L2VPN实例的路由标志Router-Distinguisher。

修改CE range只能把CE range变大，不能变小。一台设备为其所有Kompella方式L2 VPN实例和VPLS VSI实例的Range分配标签时共用一个标签块，因此Kompella方式L2 VPN实例和VPLS VSI实例的Range总和不能超过该标签块的大小，否则系统会提示分配的标签数量超过了系统允许的最大值，无法获得标签，创建CE或分配VSI实例的Site ID失败。

系统允许使用的最大标签块大小与产品相关。

此命令设置的default-offset、CE range与connection命令中的ce-offset之间约束关系如下：

  • 如果default-offset为0，ce-offset的值必须小于CE rang。

  • 如果default-offset为1，ce-offset的值必须小于或等于CE rang，且不能为0。

使用实例

# 在VPN内创建一个CE。

<Quidway> system-view
[Quidway] mpls l2vpn vpn1 encapsulation ethernet
[Quidway-mpls-l2vpn-vpn1] route-distinguisher 100:1
[Quidway-mpls-l2vpn-vpn1] ce ce1 id 1
