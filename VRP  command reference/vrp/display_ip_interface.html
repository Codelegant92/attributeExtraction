display ip interface

命令功能

display ip interface命令用来查看接口与IP相关的配置和统计信息，包括接口接收和发送的报文数、字节数和组播报文数，以及接口接收、发送、转发和丢弃的广播报文数。

display ip interface brief命令用来查看接口与IP相关的简要信息，包括IP地址、子网掩码、物理链路和协议的Up/Down状态以及处于不同状态的接口数目。

命令格式

display ip interface [ interface-type interface-number ] [ | count ] [ | { begin | include | exclude } regular-expression ]

display ip interface brief [ interface-type [ interface-number ] | interface-type &<1-8> | slot slot-number [ card card-number ] | ip-configured [ except interface-type ] ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

interface-type：接口类型。如果不指定接口类型，将显示所有接口与IP相关的配置和统计信息。

interface-number：接口编号，与interface-type共同确定一个接口。如果不指定接口编号，将显示所有该类型接口与IP相关的配置和统计信息。

interface-type &<1-8>：表示可以输入多种接口类型，最多可输入8种。

brief：表示显示接口与IP相关摘要信息，包括IP地址、子网掩码、物理链路和协议的Up/Down状态以及处于不同状态的数目。

slot slot-number：板号。如果不指定板号，将显示所有接口板和主控板的所有接口上的与IP地址相关的简要信息。

card card-number：卡号。如果不指定卡号，将显示指定板子上的所有卡上的所有接口的与IP地址相关的简要信息。

ip-configured：查看配置了IP地址的接口的IP相关信息。

except interface-type：查看除了指定接口类型的配置了IP地址的接口的IP相关信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

通过display ip interface brief命令可以查看：
  • 所有接口的IP相关信息
  • 指定接口类型、指定接口、指定槽位号或指定槽位号和板号的接口的IP相关信息
  • 配置了IP地址的接口的IP相关信息
不能通过该命令查看二层接口，也不能查看Trunk（Eth-trunk或IP-trunk）的成员端口。

如果接口下绑定了同种介质VLL或VSI，则执行该命令时不会显示该接口的任何信息。

通过对显示信息中某些类型报文收发情况的分析，可以初步判断网络是否遭到攻击和攻击的可能来源。

说明：

  • 接口的描述信息可以通过命令display interface description查看。

  • 接口的详细运行信息和统计信息可以通过命令display interface查看。

项目

描述

GigabitEthernet1/0/0 current state :
显示接口GigabitEthernet1/0/0的物理状态。

  • UP：接口物理层处于正常启动的状态。

  • DOWN：接口物理层出现故障。

  • Administratively down：如果网络管理员在该接口执行shutdown（接口视图）命令，将显示状态为Administratively down。

Line protocol current state :

显示该接口的链路协议状态。

  • UP：接口的链路协议处于正常的启动状态。

  • DOWN：接口的链路协议层出现故障，或者没有在此接口配置IP地址。

The Maximum Transmit Unit :

接口的最大传输单元（MTU）。例如以太网接口或串口的缺省值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

input packets : 1431, bytes : 97912, multicasts : 1426

接口接收的总报文数、总字节数和组播数。

output packets : 2716, bytes : 179760, multicasts : 2710

接口发送的总报文数、总字节数和组播数。

Directed-broadcast packets:

接口直接广播的报文数。

received packets:

接收到的报文总数。

sent packets:

发送的报文总数。

forwarded packets:

转发的报文总数。

dropped packets:

丢弃的报文总数。

ARP packet input number:

接收到的ARP报文数。

Request packet:

请求的报文数。

Reply packet:

应答的报文数。

Unknown packet:

未知的报文数。

Internet Address is

在接口上配置的IP地址和掩码长度。

Broadcast address :

接口的广播地址。

TTL being 1 packet number:

TTL值为1的报文数。

TTL无效报文数。

项目

描述

*down:

显示接口GigabitEthernet3/2/0的物理状态位DOWN的原因。Administratively down：如果网络管理员在该接口执行shutdown（接口视图）命令，将显示状态为Administratively down。

!down

由于接口板的FIB路由前缀容量超限，接口处于Down状态。

(l): loopback

l代表环回。

(s): spoofing

s代表哄骗。

The number of interface that is up/down in Physical is

物理链路的UP/DOWN状态以及处于不同状态的接口数目。

The number of interface that is up/down in Protocol is

协议的UP/DOWN状态以及处于不同状态的接口数目。

Interface

接口类型和接口编号。

IP Address/Mask

接口的IP地址/子网掩码。

Physical

接口的物理状态。可能出现的物理状态有：

  • Up：接口处于正常启动的状态。其中(l)表示在接口上设置了环回功能。
  • Down：接口在物理上出现故障。
  • *down：网络管理员在该接口执行了shutdown命令。其中(l)表示在接口上设置了环回功能。
  • !down：FIB模块超限挂起。此时接口的协议状态是Down。

Protocol

接口的链路协议状态。可能出现的协议状态有：

  • Up：接口的链路协议处于正常的启动状态。其中(s)表示接口创建时不用配置IP地址协议状态就显示为Up，是接口的固有属性，接口上配置了IP地址仍会被显示。
  • Down：接口的链路协议不正常，或者没有在此接口配置IP地址。

其中(l)表示在接口上设置了环回功能。
