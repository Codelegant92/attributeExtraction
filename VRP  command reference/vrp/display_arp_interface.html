display arp interface

命令功能

display arp interface命令用来查看指定接口的ARP映射表。

命令格式

在NE40E、NE80E和NE5000E上，该命令格式如下：

display arp interface interface-type interface-number [ vid vlan-id [ cevid ce-vid ] ] [ | { begin | exclude | include } regular-expression ]

在NE16E、NE08E、NE05、NE20E和NE20上，该命令格式如下：

display arp interface interface-type interface-number [ vid vlan-id ] [ | { begin | exclude | include } regular-expression ]

在NE40、NE80和S8016上，该命令格式如下：

display arp interface interface-type interface-number [ | { begin | exclude | include } regular-expression ]

参数说明

interface-type interface-number：指定接口类型和接口编号。

vid vlan-id：指定VLAN ID。用来标识一个VLAN，取值范围是1～4094。

cevid ce-vid：指定CE VLAN ID。

|：使用正则表达式对输出信息进行过滤。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

在监控ARP表项或APP出现故障时，可以使用此命令查看ARP表项的内容。

在不同设备上，该命令执行后得到的显示信息可能存在差别，详细内容请参考相关设备分册。

使用实例

# 显示以太接口GigabitEthernet1/0/0的所有ARP表项。

<Quidway> display arp interface gigabitethernet 1/0/0

项目

描述

IP ADDRESS

ARP表项中的IP地址

MAC ADDRESS

ARP表项中的MAC地址

EXPIRE(M)

ARP表项的剩余存活时间（单位是分钟）

TYPE
表项类型及获取该表项的槽位号。类型共有三位。第一位包括以下标志：

  • I：Interface，接口本身的MAC地址；

  • D：Dynamic，通过ARP协议报文获取的动态表项；

  • S：Static，通过静态配置获取的静态表项。

第二位只有标志F，表示该表项已上报路由模块，计算到该IP地址的路由并且更新FIB表项。
说明：

对于没有上报路由模块的表项将显示“-”。对于I类型的ARP表项，该标志位不存在。

第三位表示获取该表项的槽位号。
说明：

对于I类型和S类型的ARP表项将显示“-”。

VPN-INSTANCE

ARP表项所属的VPN实例的名称

INTERFACE

学到ARP表项的接口类型和编号

VLAN/CEVLAN

ARP表项所属的VLAN ID

PVC

PVC所在的接口与VPI/VCI

Total：2

ARP映射表中共有2条ARP表项

说明：

可能会出现该计数与实际输出信息中的表项个数不一致的现象，这是因为该表项个数统计是对实际ARP表项的统计，而不是对使用正则表达式过滤后的ARP表项的统计。

Dynamic：1

ARP映射表中共有1条动态ARP表项

Static：0

ARP映射表中共有0条静态ARP表项

Interface：1

ARP映射表中共有1条本身接口的ARP表项
