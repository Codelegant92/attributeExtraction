display ipv6 fib

命令功能

display ipv6 fib命令用来查看路由设备的FIB（Forwarding Information Base）表项。

在分布式环境中，可以通过指定槽位号来显示主控板或特定接口板上的FIB表项。

缺省情况下，仅显示主控板的FIB表项。

命令格式

在NE40、NE80、NE40E、NE80E、NE5000E上，该命令格式如下：

display ipv6 fib [ [ spt slot-id | slot-id ] [ vpn6-instance vpn6-instance-name ] ] [ | { begin | include | exclude } regular-expression ]

在其他设备上，该命令格式如下：

display ipv6 fib [ slot-id ] [ vpn6-instance vpn6-instance-name ] [ | { begin | include | exclude } regular-expression ]

参数说明

spt：查看下一跳的信息。

slot-id：指定槽位号，仅在分布式环境中可见。

vpn6-instance vpn6-instance-name：IPv6 VPN实例的名称，用来查看指定IPv6 VPN实例的FIB表项。字符串形式，区分大小写，长度范围是1～31。

|：使用正则表达式对输出信息进行过滤。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：指定对输出信息进行匹配的正则表达式。

表1 display ipv6 fib命令输出信息描述

项目

描述

Destination

路由目的地址

PrefixLength

发送到目的地址的前缀长度

NextHop

向目的地址转发报文的下一跳路由设备

Flag

使用S/U/G/H/B/D来描述路由特征，例如S- Static，U- Up，G-Gateway，H-Host，B-BlackHole和D-Dynamic

Label

标签值。如果数据到下一跳需通过MPLS转发，应携带标签。

Tunnel ID

隧道标识符。如果报文到下一跳需通过MPLS转发，其Tunnel ID不为0。如果报文通过IP转发，其Tunnel ID为0。

TimeStamp

FIB中生成路由的时间

reference

用于FIB6消息合并或FIB6老化

Interface

转发报文的出接口

IP6Token

当路由为Gateway路由时，系统为适配模块分配的ND表项的索引。

当路由为非Gateway路由时，该字段取值为0。
