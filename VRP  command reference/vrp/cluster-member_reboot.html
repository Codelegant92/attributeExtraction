cluster-member reboot

命令功能

cluster-member reboot命令用来配置指定的成员交换机进行重启。

命令格式

cluster-member reboot [ group-by { device-type device-type | ip {ip-addres [ to ip-addres ] } &<1-10> | member-number { member-number [ to member-number ] } &<1-10> } ]

参数说明

group-by：指定成员交换机的分类方式。目前支持三种分类方式：设备类型、IP地址和成员编号。不选择该参数时，默认对所有的成员交换机进行操作。

device-type device-type：指定成员交换机的设备类型。字符串形式，不区分大小写，不支持空格，长度范围是1～100。一次只能指定一种设备类型。目前支持的设备类型由NTDP协议收集得到，例如NE40E，CX200B等。

ip-address：指定成员交换机的IPv4地址。可以同时指定一个或者多个成员交换机的地址和地址范围。

member-number：指定成员交换机的成员编号。十进制整数形式，取值范围是1～255。可以同时指定一个或者多个成员交换机的成员编号。

使用指南

本命令只能在命令交换机上执行。

使用本命令时，如果不指定任何设备类型、成员交换机IP地址或者成员交换机编号，则对所有的成员交换机进行重启。

指定成员交换机的成员编号时不能超过集群支持的最大编号。

使用实例

# 配置所有的成员交换机进行重启。

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] cluster-member reboot
