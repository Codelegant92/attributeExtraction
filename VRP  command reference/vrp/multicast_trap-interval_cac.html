multicast trap-interval cac

命令功能

multicast trap-interval cac命令用来配置因组播CAC限制而导致IGMP/PIM加入失败输出Trap信息时的时间间隔。

undo multicast trap-interval cac命令用来取消组播CAC TRAP发送限速，加入失败事件发生时直接发送Trap信息。

命令格式

multicast trap-interval cac min-interval

undo multicast trap-interval cac

参数说明

min-interval：指定因组播CAC限制而导致IGMP/PIM加入失败输出Trap信息时的时间间隔。整数形式，单位是秒，取值范围是0～65535，缺省值是0。

视图

组播MIB视图

缺省级别

2：配置级

使用指南

当multicast trap-interval cac时间间隔配置为0时，PIM/IGMP默认对因组播CAC加入失败的TRAP信息不进行限速，即事件发生时直接发送Trap。

使用实例

# 配置因组播CAC限制而导致IGMP/PIM加入失败输出Trap信息时的时间间隔是50秒。

<Quidway> system-view
[Quidway] multicast-mib
[Quidway-multicast-mib] multicast trap-interval cac 50
