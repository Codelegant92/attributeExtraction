display mpls rsvp-te graceful-restart peer

命令功能

display mpls rsvp-te graceful-restart peer命令用来查看邻居的RSVP-TE GR状态。

命令格式

display mpls rsvp-te graceful-restart peer [ { interface interface-type interface-number | node-id } [ ip-address ] ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

interface interface-type interface-number：显示基于指定接口的邻居的RSVP-TE GR状态。

node-id：显示手工配置的邻居的RSVP-TE GR状态。

ip-address：邻居节点接口的IP地址。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

项目

描述

Neighbor Addr

邻居的IP地址

Last Attribute

邻居属性：

  • Added Usually：接口发现的邻居

  • Added by Node-Id：通过配置Node ID发现的邻居

  • To be deleted：因Hello超时将被删除的邻居

  • Add by Frr：FRR邻居

SrcInstance

源实例

NbrSrcInstance

邻居的源实例

Neighbor Capability
邻居的GR能力：

  • Can Do Self GR：自身具有GR能力。

  • Can Support GR：具有支持GR的能力。

  • Can Transmit RecoveryPath Messages：能够发送RecoveryPath消息。

  • No Gr capabilities：没有GR能力。

GR Status
邻居的GR状态：

  • Normal：邻居没有执行GR

  • Supporting：邻居正在支持本地节点GR

  • Restarting：邻居正在支持本地节点重启

Restart Time

邻居的重启时间，单位是毫秒

Recovery Time

邻居的恢复时间，单位是毫秒

Stored GR message number

支持节点中存储GR信息的数量

PSB Count

PSB数量

RSB Count

RSB数量

Total to be Recover PSB Count

要恢复的PSB的总数

Recovered PSB Count

已恢复的PSB的数量

Total to be Recover RSB Count

要恢复的RSB的总数
