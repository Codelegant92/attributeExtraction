display mld explicit-tracking

命令功能

display mld explicit-tracking命令用来查看使用Include模式加入特定源组的MLDv2主机成员的精确信息。

命令格式

display mld explicit-tracking [ interface interface-type interface-number [ host-address ipv6-host-address ] ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

interface interface-type interface-number：指定要查询接口的类型和编号。

host-address ipv6-host-address：指定主机的链路本地地址。十六进制形式，取值范围是FE80::～FE80:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

该命令对Exclude模式加入组或特定源组的主机成员和MLDv1主机成员都无效。

使用display mld explicit-tracking命令时：

  • 若指定了interface参数则仅显示在该接口上MLD主机成员的精确信息。如果该接口未使能MLD，或接口状态为Down，则出现提示信息。

  • 若指定了host-address，则显示该指定MLD主机成员的信息。

  • 如果同时指定interface和host-address，该命令将显示此接口上指定MLD主机成员的信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

表1 display mld explicit-tracking命令输出信息描述

项目

描述

Explicit-tracking information

精确跟踪信息

Total 2 hosts, 4 entries

总计主机2台，(S, G)表项4个

Host

IPv6主机地址(IPv6链路本地地址)

Uptime
主机加入后的运行时间，时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

Expires
主机超时的预测时间，超时后从MLD主机成员列表里删除。时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

(S, G) List

主机加入的组播源和组播组列表

Group

IPv6组播组地址

Source

组播源的IPv6单播地址

Time since last refresh
距主机上一次加入的时间间隔。时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”
