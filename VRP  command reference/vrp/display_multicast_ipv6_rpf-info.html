display multicast ipv6 rpf-info

命令功能

display multicast ipv6 rpf-info命令用来查看指定的IPv6组播源或源/组的RPF路由信息。

命令格式

display multicast ipv6 rpf-info ipv6-source-address [ ipv6-group-address ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

ipv6-source-address：指定组播源的IPv6地址，显示该源对应的RPF路由信息。

ipv6-group-address：指定组播组的IPv6地址，显示该源/组对应的RPF路由信息。取值范围是FF00::0～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

RPF路由是以ipv6-source-address为目的地址的，从单播路由、组播静态路由、MBGP路由和MIGP路由中选取的一条最优路由。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示组播源2001::101的RPF路由信息。

<Quidway> display multicast ipv6 rpf-info 2001::101
 VPN-Instance: public net
 RPF information about source: 2001::101
     RPF interface: GigabitEthernet6/0/2
     Referenced route/mask: 2001::/64
     Referenced route type: unicast
     Load splitting rule: disable

项目

描述

VPN-Instance

RPF信息所属的VPN实例。

RPF information about source

RPF信息所属的组播源。

RPF interface

RPF接口。

Referenced route/mask

引用的路由及其掩码。

Referenced route type
引用的路由类型。

  • unicast：单播路由。

  • MBGP：MBGP路由。

  • mstatic：组播静态路由。

  • MIGP：MIGP路由。

Load splitting rule
负载分担规则：

  • disable

    ：未使能组播负载分担。

  • balance-preferred：均衡优先负载分担。
