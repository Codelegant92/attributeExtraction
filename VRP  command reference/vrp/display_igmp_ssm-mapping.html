display igmp ssm-mapping

命令功能

display igmp ssm-mapping命令用来查看配置的SSM映射关系和接口是否使能了IGMP SSM Mapping。

命令格式

display igmp [ vpn-instance vpn-instance-name | all-instance ] ssm-mapping { group [ group-address ] | interface [ interface-type interface-number ] } [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

group group-address：显示该路由器上指定组对应的SSM映射信息。group-address表示组播组地址，用于指定一个组播组。点分十进制形式。

interface interface-type interface-number：显示指定接口上是否使能了IGMP SSM Mapping。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

缺省情况下，SSM组范围是232.0.0.0 ～232.255.255.255。可以根据需要使用ssm-mapping（IGMP视图）命令配置超出此范围的SSM映射的组地址。

使用display igmp ssm-mapping命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 若指定了group-address参数，则仅显示和该组相关的SSM映射关系。

  • 若指定了interface interface-type interface-number 参数，则仅显示该接口是否使能IGMP SSM Mapping。如果该接口未使能IGMP SSM Mapping，或接口状态为Down，则出现提示信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示公网实例的所有源/组地址的SSM映射信息。

<Quidway> display igmp ssm-mapping group 
IGMP SSM-Mapping conversion table of VPN-Instance: public net
 Total 2 entries
 00001. (10.1.0.2, 224.1.1.0)
 00002. (10.1.0.2, 239.255.255.0)
 Total 2 entries matched

# 显示公网实例的组地址232.1.1.1对应的SSM映射信息。

<Quidway> display igmp ssm-mapping group 232.1.1.1
IGMP SSM-Mapping conversion table of VPN-Instance: public net
 Total 1 entry
 00001. (10.1.0.2, 232.1.1.1)
 Total 1 entry matched

# 显示公网实例的GE6/2/2接口上是否使能了IGMP SSM Mapping。

<Quidway> display igmp ssm-mapping interface gigabitethernet 6/2/2
 IGMP SSM-Mapping is enabled
