cluster-member unrelated-port

命令功能

cluster-member unrelated-port命令用来配置成员交换机上不需要运行NDP和NTDP协议的接口，使能NDP和NTDP功能。

undo cluster-member unrelated-port命令用来配置成员交换机上不需要运行NDP和NTDP协议的接口，禁止NDP和NTDP功能。

本命令的缺省情况由PAF文件中的定义决定。

命令格式

cluster-member [ group-by { device-type device-type | ip {ip-addres [ to ip-addres ] } &<1-10> | member-number { member-number [ to member-number ] } &<1-10> } ] unrelated-port { ndp | ntdp }

undo cluster-member [ group-by { device-type device-type | ip {ip-addres [ to ip-addres ] } &<1-10> | member-number { member-number [ to member-number ] } &<1-10> } ] unrelated-port { ndp | ntdp }

参数说明

group-by：指定成员交换机的分类方式。目前支持三种分类方式：设备类型、IP地址和成员编号。不选择该参数时，默认对所有的成员交换机进行操作。

device-type device-type：指定成员交换机的设备类型。字符串形式，不区分大小写，不支持空格，长度范围是1～100。一次只能指定一种设备类型。目前支持的设备类型由NTDP协议收集得到，例如NE40E，CX200B等。

ip-address：指定成员交换机的IPv4地址。可以同时指定一个或者多个成员交换机的地址和地址范围。

member-number：指定成员交换机的成员编号。十进制整数形式，取值范围是1～255。可以同时指定一个或者多个成员交换机的成员编号。

使用指南

本命令只能在命令交换机上执行，并且只对Switch类型的接口有效。

不需要运行NDP和NTDP协议的接口是指对整个集群拓扑信息没有作用的接口。

指定成员交换机的成员编号时不能超过集群支持的最大编号。

当集群的拓扑信息收集完成后，使用本命令可以提高整个集群的安全性。关闭成员交换机不相关端口的NDP或NTDP协议后，新的候选交换机接入成员交换机的不相关接口，新的候选交换机不能加入该集群。除非重新使能该成员交换机的不相关端口的NDP或NTDP协议。

使用实例

# 配置集群中所有成员交换机不需要运行NDP协议的接口，禁止NDP功能

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] undo cluster-member unrelated-port ndp

# 配置集群中所有成员交换机不需要运行NTDP协议的接口，禁止NTDP功能

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] undo cluster-member unrelated-port ntdp
