filter-policy import (BGP)

命令功能

filter-policy import命令用来配置对接收的路由信息进行过滤。

undo filter-policy import命令用来取消对接收的路由信息进行过滤。

缺省情况下，不对接收的路由信息进行过滤。

命令格式

filter-policy { acl-number | ip-prefix ip-prefix-name } import

filter-policy { acl6-number | ipv6-prefix ipv6-prefix-name } import

undo filter-policy import

参数说明

acl-number：基本ACL的编号。

ip-prefix ip-prefix-name：IPv4地址前缀列表名。

acl6-number：基本ACL6的编号。

ipv6-prefix ipv6-prefix-name：IPv6地址前缀列表名。

说明：

  • acl-number和ip-prefix-name参数仅在BGP视图、BGP-IPv4单播地址族视图、BGP-VPN实例视图和BGP-VPNv4地址族视图下生效。

  • acl6-number和ipv6-prefix-name参数仅在BGP-IPv6单播地址族视图、BGP-VPNv6地址族视图和BGP-VPN6实例视图下生效。

缺省级别

2：配置级

使用指南

该命令可以对BGP接收路由进行过滤，决定是否将路由添加到BGP路由表中。

使用实例

# 使用ACL 2000对BGP接收的路由信息进行过滤。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] ipv4-family unicast
[Quidway-bgp-af-ipv4] filter-policy 2000 import

# 使用编号为2000的ACL6对所有BGP引入的路由进行过滤。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] ipv6-family unicast
[Quidway-bgp-af-ipv6] filter-policy 2000 import
