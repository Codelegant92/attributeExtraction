peer capability-advertise

命令功能

peer capability-advertise命令用来使能BGP向对等体发布路由刷新功能或常规路由器功能。

undo peer capability-advertise命令用来取消向对等体发布的功能。

缺省情况下，使能BGP路由刷新功能。

命令格式

peer { group-name | ipv4-address | ipv6-address } capability-advertise { route-refresh | conventional }

undo peer { group-name | ipv4-address | ipv6-address } capability-advertise { route-refresh | conventional }

参数说明

group-name：对等体组的名称。

ipv4-address：对等体的IPv4地址。

ipv6-address：对等体的IPv6地址。

说明：

  • ipv4-address参数仅在BGP视图和BGP-VPN实例视图下生效。

  • ipv6-address参数仅在BGP视图和BGP-VPN6实例视图下生效。

route-refresh：路由刷新功能。

conventional：常规路由器功能。对于IPv6对等体，此参数不可见。

视图

BGP视图、BGP-VPN实例视图、BGP-VPN6实例视图

缺省级别

2：配置级

使用指南

在conventional模式下，路由器不具备所有扩展功能（如路由刷新功能、GR能力及多地址族协商），以便于向前兼容版本较早的路由器。

使用实例

# 使能BGP路由刷新功能。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] peer 160.89.2.33 as-number 100
[Quidway-bgp] peer 160.89.2.33 capability-advertise route-refresh
