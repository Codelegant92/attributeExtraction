traffic classifier

命令功能

traffic classifier命令用来定义一个类并进入流分类视图。

undo traffic classifier命令用来删除一个类。

operator缺省情况下为and。

命令格式

traffic classifier classifier-name [ operator { and | or } ]

undo traffic classifier classifier-name

参数说明

classifier-name：指定流分类的名称，字符串形式，区分大小写，长度范围是1～31。类名不允许为系统预定义类。

operator：指定各规则之间的逻辑运算符。

and：指定类下的规则之间是逻辑“与”的关系，即数据包必须匹配全部规则才属于该类。如果没有指定operator，则规则之间为逻辑与的关系。

or：指定类下的规则之间是逻辑“或”的关系，即数据包只要匹配其中任何一个规则就属于该类。

使用指南

类名classifier-name不允许为系统预定义类。

如果在一个流分类中配置了多个匹配规则，可以通过在traffic classifier classifier-name [ operator { and | or } ]中指定参数operator来设置这些规则之间的关系，其中：

  • and：指定类下的规则之间是逻辑“与”的关系，即数据包必须匹配全部规则才属于该类。
  • or：指定类下的规则之间是逻辑“或”的关系，即数据包只要匹配其中任何一个规则就属于该类。

缺省情况下，即如果没有指定operator，则规则之间为逻辑“与”（and）的关系。 一个流分类中最多可以定义16条匹配规则。

使用实例

# 定义一个名为class1的类。

<Quidway> system-view
[Quidway] traffic classifier class1
[Quidway-classifier-class1]
