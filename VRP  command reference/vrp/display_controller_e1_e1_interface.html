display controller e1（E1接口）

命令功能

display controller e1命令用来查看E1接口的状态。

命令格式

display controller e1 [ controller-number ]

参数说明

controller-number：接口号。如果不指定接口编号，将显示所有E1接口的状态。

缺省级别

1：监控级

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和报文统计信息等。

使用实例

# 查看E1接口1/0/0的状态信息。

<Quidway> display controller e1 1/0/0
E1 1/0/0 current state : DOWN
Work Mode is E1 UNFRAME 
Cable type is unknown
    Framing is NONE,Line Code is HDB3,Source Clock is SLAVE
Loopback is not set,    Alarm State is Loss-of-Signal.

项目

描述

E1 1/0/0 current state :
E1接口当前的物理状态：

  • UP：物理状态正常启动。

  • DOWN：物理连接不正常。

  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Work Mode is
线缆工作模式。E1接口有两种工作模式：

  • 净通道化工作模式。

  • 非通道化工作模式。

Cable type is
线缆模式。E1接口的线缆模式有两种：

  • long：长距模式，电缆长度为151～500米。

  • short：短距模式，电缆长度为0～150米。

unknown表示本接口没有插线缆或线缆格式不能识别。

Frameing is
E1接口的帧格式。

  • 当E1接口工作在净通道模式下时没有帧格式，表示为NONE。

  • 当E1接口工作在非通道化模式下，有两帧格式。请参见frame-format（CE1接口视图）。

Line Code is

E1接口的线路编解码格式：请参见code（E1接口视图）。

Source Clock is

时钟模式。可以通过clock（E1接口视图）设置为master或者slave。

loopback is not set

没有使能E1接口或CE1接口环回功能。只有在进行某些特殊功能测试时，才将E1接口设为环回。

Alarm state is
告警类型和错误类型。可能出现的错误类型有以下几类。

  • LOS：Loss of Signal，帧同步丢失。

  • LOF：Loss of Frame，帧丢失。

  • CERR：CRC Error，循环冗余校验错。

  • LCV：Line Code Violation，线路码错误。

  • FEBE：Far End Block Error，远端块错。这种错误只有在E1通道采用CRC4的帧格式时才可能出现。
