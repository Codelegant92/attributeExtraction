priority ingress

命令功能

priority ingress命令用来配置当前业务实例的源端优先级，即对入隧道方向的报文进行封装时I-TAG的优先级。

undo priority ingress命令用来恢复当前业务实例的源端优先级为缺省值。

缺省情况下为信任用户优先级，优先级配置为trust-8021p。当接口类型为transparent时，业务实例默认源端优先级自动修改为0。

命令格式

priority ingress { priority | trust-8021p }

undo priority ingress

参数说明

priority：指定业务实例的源端优先级。整数形式，取值范围是0～7，值越大优先级越高。

trust-8021p：指定业务实例信任用户报文的优先级。

使用指南

业务实例的优先级体现业务在MAC Tunnel内传输的优先级。报文进入MAC Tunnel时，VRP设备首先将按业务实例优先级进行调度处理。如果MAC Tunnel没有配置指定优先级，业务实例的优先级会进一步映射到隧道的骨干VLAN。

业务实例的服务接口类型为transparent（透明传输模式）时，不允许配置信任用户报文优先级。

在同一个视图下重复执行priority ingress命令后，新配置覆盖老配置。

使用实例

# 配置业务实例的优先级为信任用户报文的优先级。

<Quidway> system-view
[Quidway] service-instance instance-name mysi
[Quidway-service-instance-mysi] priority ingress trust-8021p
