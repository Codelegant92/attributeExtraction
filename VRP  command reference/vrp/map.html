map

命令功能

map命令用来配置当前MA关联的VLAN或VSI。

undo map命令用来删除当前MA和VLAN或VSI的关联关系。

缺省情况下，MA未和任何VLAN或VSI建立关联关系。

命令格式

map { vlan vlan-id | vsi vsi-name }}

undo map { vlan | vsi }

参数说明

vlan-id：指定VLAN的ID。整数形式，取值范围是1～4094。

vsi-name：VSI实例名称。字符串形式，长度为1～31个字符。

使用指南

以太网CFM对每个MA分别进行连通性故障检测。配置某个MA和1个VLAN或VSI关联，可实现对该VLAN或VSI内的网络的连通性故障检测。

如果某MA用于两直连设备之间的连通性故障检测，则该MA可以不关联VLAN或VSI，否则该MA应关联VLAN或VSI。

一个MA只能关联一个VLAN或VSI。

注意：

只有当在本设备上，某MA内既未创建MEP也未配置RMEP时，才能成功执行本命令建立或删除该MA和VLAN或VSI的关联关系。

如果要改变当前MA关联的VLAN或VSI，首先需要使用undo map命令删除该MA和原VLAN或VSI的关联关系，然后再执行map命令将MA和另一个VLAN或VSI关联。

使用实例

# 配置MA和VLAN10关联。配置前该MA未关联VLAN。

<Quidway> system-view
[Quidway] cfm enable
[Quidway] cfm md md1
[Quidway-md-md1] ma ma1
[Quidway-md-md1-ma-ma1] map vlan 10
