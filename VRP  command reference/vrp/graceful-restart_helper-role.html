graceful-restart helper-role

命令功能

graceful-restart helper-role用来配置路由器平滑重启Helper模式的策略。

undo graceful-restart helper-role用来取消路由器平滑重启Helper模式的策略。

命令格式

graceful-restart helper-role { [ { ip-prefix ip-prefix-name | acl-number acl-number } | ignore-external-lsa | planned-only ] * | never }

undo graceful-restart helper-role { [ { ip-prefix | acl-number } | ignore-external-lsa | planned-only ] * | never }

参数说明

ip-prefix ip-prefix-name：指定地址前缀列表名称。字符串形式。

acl-number acl-number：指定基本访问控制列表号。整数形式。

ignore-external-lsa：不检查Type-5和Type-7类的LSA。

planned-only：指定路由器只支持Planned-GR。缺省情况下，路由器支持Planned-GR和Unplanned-GR。

never：指定路由器不支持Helper模式。

使用指南

配置路由器平滑重启特性前，需要先通过命令opaque-capability enable使能Opaque LSA功能。使能或禁止Opaque LSA功能可能会删除并重建所有的会话与实例。

使用graceful-restart命令使能路由器的平滑重启功能后，Helper功能也为使能状态。

使用实例

# 使能OSPF的Graceful Restart特性，配置本地Helper策略为只支持planned GR。

<Quidway> system-view
[Quidway] ospf 1
[Quidway-ospf-1] graceful-restart helper-role planned-only
