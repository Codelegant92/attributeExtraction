rmon2 protocoldirtable

命令功能

rmon2 protocoldirtable命令用来创建或修改协议目录表中的一行。

undo rmon2 protocoldirtable命令用来删除协议目录表中的一行，如果不使用可选参数，将删除整个表。

命令格式

rmon2 protocoldirtable protocoldirid protocol-id parameter parameter-value [ descr description-string ] [ host { notsupported | supportedon | supportedoff } ] [ owner owner-name ] [ status { active | inactive } ]

undo rmon2 protocoldirtable [ { protocoldirid protocol-id } { parameter parameter-value } ]

参数说明

protocol-id：目前只实现IP，支持的值为8.0.0.0.1.0.0.8.0。

parameter-value：协议参数信息。目前支持的值为2.0.0。

description-string：协议目录表的描述信息。字符串类型，长度范围是1～64。

notsupported：不支持对本协议的网络层主机表的监控，并且这个值不能改变。

supportedon：支持对本协议的网络层主机表的监控，并且使能监控功能。

supportedoff：支持对本协议的网络层主机表的监控，但是不使能监控功能。

owner owner-name：条目所有者的描述信息。字符串类型，长度范围是1～127。

active：该条目已激活，可以使用。

inactive：该条目没有激活，处于notinservice状态，不能使用。

使用指南

在配置rmon2 protocoldirtable命令前，必须先设置描述信息和主机对协议支持的信息。

使用实例

# 创建协议目录表中的一行。

[Quidway] rmon2 protocoldirtable protocoldirid 8.0.0.0.1.0.0.8.0 parameter 2.0.0 descr huawei host supportedon owner huawei status active

# 修改协议目录表中的一行protocolDirStatus为notinservice。

[Quidway] rmon2 protocoldirtable protocoldirid 8.0.0.0.1.0.0.8.0 parameter 2.0.0 status inactive

# 修改协议目录表中的一行protocolDirStatus为supportedon。

[Quidway] rmon2 protocoldirtable protocoldirid 8.0.0.0.1.0.0.8.0 parameter 2.0.0 host supportedon
