display trunkfwdtbl ip-trunk

命令功能

display trunkfwdtbl ip-trunk命令用来查看IP-Trunk接口的转发表。

命令格式

display trunkfwdtbl ip-trunk trunk-id [ slot slot-id ] [ | count ] [ | { begin | include | exclude } regular-expression ]|

参数说明

trunk-id：IP-Trunk ID，取值范围是0～63。

slot slot-id：接口板的板号。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。字符串格式，支持空格，区分大小写。

使用指南

从捆绑组转发表中可以看到转发负载分担的状态，转发表显示信息主要包括两列内容：“MASTER”所在列表示主用成员端口，“SLAVE”所在列表示备用成员端口。

  • 如果对某端口使用port-standby（IP-Trunk）命令配置了备份端口，“SLAVE”处显示备份端口。

  • 如果没有配置备份端口，而捆绑组内有其他成员端口处于UP状态，将使用这些端口作为组内的备份端口，显示在“SLAVE”处。

  • 如果没有配置备份端口，组内也没有其他处于UP状态的端口，“SLAVE”处显示的端口与“MASTER”处相同，即，端口自己作备份。

另外，由于捆绑组中最多可以支持16条物理成员链路，转发表将显示16条表项。
