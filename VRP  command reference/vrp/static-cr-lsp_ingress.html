static-cr-lsp ingress

命令功能

static-cr-lsp ingress命令用来在入口节点配置静态CR-LSP。

undo static-cr-lsp ingress命令用来在入口节点删除配置的静态CR-LSP。

命令格式

static-cr-lsp ingress { tunnel-interface tunnel interface-number | tunnel-name } destination destination-address { nexthop next-hop-address | outgoing-interface interface-type interface-number } out-label out-label [ bandwidth [ ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 ] bandwidth ]

undo static-cr-lsp ingress { tunnel-interface tunnel interface-number | tunnel-name }

参数说明

tunnel-interface tunnel interface-number：指定静态CR-LSP的隧道接口。interface-number是隧道接口的接口编号。

tunnel-name：静态CR-LSP隧道的名称，字符串形式，区分大小写，不支持空格和缩写，长度范围是1～19。假设使用命令interface tunnel 2/0/0为静态CR-LSP创建了一个的Tunnel接口，则静态CR-LSP入节点中的参数应该写作“Tunnel2/0/0”或者“tunnel2/0/0”，否则隧道将不能正确建立。中间节点和出节点无此限制。

destination destination-address：指定静态CR-LSP的目的地址。

nexthop next-hop-address：指定静态CR-LSP下一跳地址。

outgoing-interface interface-type interface-number：出接口类型和编号，只有点到点链路才能选择配置出接口。

out-label out-label：指定出标签。out-label表示出标签的值，整数形式，取值范围是16～1023。

bandwidth：指定静态CR-LSP的带宽。

ct0～ct7：设置预留给CT0～CT7的带宽。Non-IETF模式只支持CT0和CT1，IETF模式支持CT0～CT7。

bandwidth：CR-LSP所需的带宽，单位为kbit/s。取值范围是0～32000000，缺省值为0。

使用实例

# 在入口节点配置静态CR-LSP。名字为Tunnel1/0/0，目的IP地址为10.1.3.1，下一跳IP地址为10.1.1.2，出标签为237，所需要的带宽20kbit/s，从BC1获取带宽。

<Quidway> system-view
[Quidway] static-cr-lsp ingress Tunnel1/0/0 destination 10.1.3.1 nexthop 10.1.1.2 out-label 237 bandwidth ct1 20
