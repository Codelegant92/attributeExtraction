display tcp status

命令功能

display tcp status命令用来查看TCP状态信息，使用户查看当前的TCP连接状态。

命令格式

display tcp status [ [ task-id task-id ] [ socket-id socket-id ] | [ local-ip ipv4-address ] [ local-port local-port-number ] [ remote-ip ipv4-address ] [ remote-port remote-port-number ] ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

task-id task-id：指定任务ID，显示此任务ID的TCP连接状态。整数形式，取值范围是1～200。

socket-id socket-id：指定Socket ID号，显示此Socket ID的TCP连接状态。整数形式，取值范围是1～14848。

local-ip ipv4-address：指定本端地址，即本台设备的IP地址。显示该IP地址的TCP连接状态。

local-port local-port-number：指定本端端口号，即本台设备的端口号。显示该端口号的TCP连接状态，取值范围是0～65535。

remote-ip ipv4-address：指定远端地址，即其它设备的IP地址。显示该IP地址的TCP连接状态。

remote-port remote-port-number：指定远端端口号，即其它设备的端口号。显示该端口号的TCP连接状态，取值范围是0～65535。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

通过设置参数（Task ID、Socket ID、本端地址、本端端口号、远端地址、远端端口号），只有满足参数条件的信息会被显示出来。这样可以减少显示信息的输出数量，方便查看，提高问题定位的精确度和效率。

使用实例

# 查看本机上的TCP连接状态信息。

<Quidway> display tcp status
TCPCB    Tid/Soid Local Add:port        Foreign Add:port      VPNID  State
0a5d560c 30 /1    0.0.0.0:23            0.0.0.0:0             14849 Listening

# 查看从本端地址0.0.0.0，端口号23发起的TCP连接的状态信息。

<Quidway> display tcp status local-ip 0.0.0.0 local-port 23
TCPCB    Tid/Soid Local Add:port        Foreign Add:port      VPNID  State
0a5d560c 30 /1    0.0.0.0:23            0.0.0.0:0             14849 Listening

# 查看从本端地址0.0.0.0，端口号24发起的TCP连接的状态信息。

<Quidway> display tcp status local-ip 0.0.0.0 local-port 24
TCPCB    Tid/Soid Local Add:port        Foreign Add:port      VPNID  State
表1 display tcp status命令显示信息中各字段的说明

字段

说明

TCPCB

TCP任务控制块编号

Tid/Soid

Task ID 和Socket ID

Local Add: port

TCP连接的本端IP地址和本端端口号

Foreign Add: port

远端IP地址和远端端口号

VPNID

VPN ID

State
TCP的连接状态。具体情况如下：

  • Closed：关闭状态

  • Listening：监听状态

  • Syn_Rcvd：接收到SYN同步报文状态

  • Established：连接建立状态

  • Close_Wait：在Established状态下，用户向服务器发送FIN报文请求终止连接。服务器接收到用户发送的FIN报文后，向用户发送ACK报文，然后进入该状态。

  • Fin_Wait1：用户向服务器发送FIN报文请求终止本方向连接，然后进入该状态

  • Fin_Wait2：用户在收到服务器对自己发出的FIN报文的相应ACK报文后，进入该状态。

  • Time_Wait：TCP在关闭一个连接之后就进入超时等待状态。它在这个状态中停留时间达到最长报文段寿命的两倍时，就删去该连接的记录。

  • Closing：同时关闭
