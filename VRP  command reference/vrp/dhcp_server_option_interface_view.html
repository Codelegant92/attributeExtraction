dhcp server option（接口视图）

命令功能

dhcp server option命令用来配置当前接口的DHCP地址池的自定义选项。

undo dhcp server option命令用来删除配置。

命令格式

dhcp server option code { ascii ascii-string | hex hex-string | ip-address ip-address &<1-8> }

undo dhcp server option code

参数说明

code：指定用户定义的选项Option的数值。整数形式，取值范围是2～254。

ascii ascii-string：ASCII字符串。ascii-string是字符串形式，长度范围是1～63。

hex hex-string：2位或4位长度的16进制数字串（如hh或hhhh）。

ip-address ip-address &<1-8>：执行指定option功能的IP地址，可以输入1～8个。

使用指南

DHCP报文中的Option字段可以用来存放某些普通协议中没有定义的控制信息和参数。如果用户在DHCP服务器端配置了Option，DHCP客户端在申请IP地址的时候，会通过服务器端回应的DHCP报文获得Option字段中的配置信息。

说明：

DHCP自定义选项是可选的，只在需要时才配置。常用的功能，如配置客户端的DNS服务、NetBIOS服务、租期等，不能用Option命令进行配置，只能通过相关命令来实现。

使用实例

# 为GE 1/0/0的DHCP地址池定义code为23的16进制数字为0x11和0x22。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet 1/0/0] dhcp server option 23 hex 11 22
