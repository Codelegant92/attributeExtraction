display isis spf-tree

命令功能

display isis spf-tree命令用来查看SPF树的拓扑信息。

命令格式

display isis spf-tree [ systemid systemid | dname dname ] [ [ level-1 | level-2 ] | ipv6 | verbose ] * [ process-id | [ vpn-instance { vpn-instance-name | default } | vpn6-instance { vpn6-instance-name | default } ] * ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display isis [ process-id ] spf-tree statistics [ [ level-1 | level-2 ] | ipv6 ] * [ vpn-instance { vpn-instance-name | default } | vpn6-instance { vpn6-instance-name | default } ] * [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

systemid systemid：某个系统ID或伪节点ID路由器的SPF树。格式为XXXX.XXXX.XXXX[.XX]。

dname dname：某个动态名称路由器的SPF树。

level-1：Level-1的SPF树。

level-2：Level-2的SPF树。

ipv6：IPv6拓扑的SPF树。

verbose：SPF树的详细信息。

process-id：某个IS-IS进程的SPF树。

vpn-instance vpn-instance-name：指定IPv4 VPN的IS-IS多实例进程。vpn-instance-name为IPv4 VPN实例的名称，区分大小写。

vpn-instance default：除IPv4 VPN实例外的所有IS-IS进程，即所有的IPv6 VPN的IS-IS实例进程和公网IS-IS进程。

vpn6-instance vpn6-instance-name：指定IPv6 VPN的IS-IS多实例进程。vpn6-instance-name为IPv6 VPN实例的名称，区分大小写。

vpn6-instance default：除IPv6 VPN实例外的所有IS-IS进程，即所有的IPv4 VPN的IS-IS实例进程和公网IS-IS进程。

statistics：SPF树当前状态的统计信息，包括SPF树上的节点信息和链路信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

缺省级别

1：监控级

使用指南

使用statistics参数可以查看SPF树的当前状态。只有当SPF的状态为Completed时，I-SPF计算完成，生成的SPF树才能作为路由计算的依据。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

Distance

从根节点到目的节点的最短路径的开销值，不包含TE-Tunnel链路在内

Distance-URT

从根节点到目的节点的最短路径的开销值，包含TE-Tunnel链路在内

Flags

  • SPT/V6_Islt：节点不在IPV6树上

  • SPT/V4_Islt：节点不在IPV4树上

IPv4 Nexthops-URT

该节点在单播路由表中的IPv4下一跳信息

IPv4 Nexthops-MIGP

该节点在MIGP（Multicast IGP）表中的IPv4下一跳信息

IPv6 Nexthops

该节点在单播路由表中的IPv6下一跳信息

Neighbors

包括该节点的所有邻居信息

Cost

从根节点到该节点的链路开销值

Flags
邻居和自身的关系

  • Parent：邻居是父节点

  • Child：邻居是子节点
