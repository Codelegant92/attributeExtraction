igmp max-response-time

命令功能

igmp max-response-time命令用来在接口上配置IGMP查询报文的最大响应时间。

undo igmp max-response-time命令用来恢复该配置参数的缺省值。

缺省情况下，IGMP查询报文的最大响应时间是10秒。

命令格式

igmp max-response-time interval

undo igmp max-response-time

参数说明

interval：表示IGMP查询报文的“最大响应时间”。整数形式，取值范围是1～25，单位为秒。

使用指南

“最大响应时间”用于控制主机反馈组成员关系报告的最后期限。实际主机响应时间是0～“最大响应时间”之间的一个随机值。主机响应时间越小，IGMP路由器获知组播成员的速度越快，但是网络带宽和路由器资源的占用也就越大。

接收到一个IGMP查询报文后，主机会为其所加入的组播组启动一个定时器，超时值在0～“最大响应时间”之间随机选定。当定时器超时，主机发送Report报文。

通过设置最大响应时间，可以控制主机发送组成员关系报告的最后期限。合理的设置最大响应时间，可以使主机快速响应查询报文，同时避免大量主机同时发送响应报文而造成的流量拥塞。

该命令和max-response-time（IGMP视图）命令功能完全相同，差别只是作用范围不同。IGMP视图下的配置具有全局性，对所有接口有效；接口视图下的配置只对当前接口有效。系统优先采用接口视图下的配置。当接口视图下没有进行配置，才采用IGMP视图下的配置。

使用实例

# 配置接口GE1/0/0上IGMP查询报文的最大响应时间为8秒。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp max-response-time 8
