display msdp peer-status

命令功能

display msdp peer-status命令用来查看MSDP对等体状态的详细信息。

命令格式

display msdp [ vpn-instance vpn-instance-name | all-instance ] peer-status [ peer-address ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

peer-address：表示远端MSDP对等体的地址。点分十进制形式。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display msdp peer-status命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 如果指定peer-address，则显示与指定peer-address建立MSDP对等体关系的详细状态信息。

  • 如果不指定peer-address，则列举路由器上建立的所有MSDP对等体关系的详细状态信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

表1 display msdp peer-status命令输出信息描述

项目

描述

MSDP Peer Information of VPN-Instance

MSDP Peer信息所属的VPN实例

MSDP Peer

对等体地址。

AS

MSDP peer的自治号，"?" 表示无法获得自治域号。

State
MSDP会话的状态：

  • Up：连接建立。

  • Listen：本地作为Server，处于监听状态，连接建立。

  • Connect：本地作为Client，处于连接状态，连接未建立。

  • Shutdown：MSDP对等体处于Shutdown状态。

  • Down：连接失败。

Up/Down time
连接建立或失败发生的时间，时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

Connection interface

connect-interface接口地址用于与对端peer地址建立TCP连接。

Number of sent/received messages

MSDP通过该连接发送或接收消息的数目计数。

Number of discarded output messages

丢弃的待发送信息。

Elapsed time since last connection or counters clear

最近一次清除该MSDP对等体的统计信息、重置信息和输入输出信息的时刻距离当前时刻的间隔。

Information about (Source, Group)-based SA filtering policy

  • Import policy：用来接收指定MSDP对等体的SA消息的过滤列表，可通过peer sa-policy import进行配置，通过undo peer sa-policy import取消配置。

  • Export policy：用来转发指定MSDP对等体的SA消息的过滤列表，可通过peer sa-policy export进行配置，通过undo peer sa-policy export取消配置。

缺省情况下，对接收或转发的消息不作过滤，从MSDP对等体接收或转发所有的SA消息，用none表示。

Information about SA-Requests

  • Policy to accept SA-Request messages：用来限制路由器从该MSDP对等体接收的SA请求消息，可通过peer sa-request-policy进行配置，通过undo peer sa-request-policy用来取消该限制。

    缺省情况下，路由器接收来自该MSDP对等体的所有SA请求消息，用none表示。

  • Sending SA-Requests status：是否使能路由器在接收到一个新的组加入消息时，向其指定的MSDP对等体发送SA请求消息。enable表示使能；disable表示未使能。可通过peer request-sa-enable进行配置，通过undo peer request-sa-enable取消该配置。

    缺省情况下，路由器在收到新的组加入消息时，不向其MSDP对等体发送SA请求消息，而是等待下一个SA消息的到来，状态是disable。

Minimum TTL to forward SA with encapsulated data

对于接收到的SA消息，如果封装了组播数据报文，只有该数据报文的TTL大于或等于配置的最小TTL值的情况下，才向其他对等体转发该SA消息。可通过peer minimum-ttl进行配置，通过undo peer minimum-ttl用来取消该配置。

SAs learned from this peer

通过该MSDP对等体的SA消息，路由器产生缓存的数量。

SA-cache maximum for the peer

路由器在接收到该MSDP对等体的SA消息时可产生缓存的最大数量。可通过peer sa-cache-maximum进行配置，通过undo peer sa-cache-maximum用来取消该配置。缺省情况下，SA-cache最多缓存8192个（S，G）表项。

Input queue size

输入缓存中所缓存的数据长度。

Output queue size

输出缓存中所缓存的数据长度。

Counters for MSDP message

  • Count of RPF check failure：未通过RPF检查而被丢弃的SA报文统计数。

  • Incoming/outgoing SA messages：接收或发送的SA报文的统计数。

  • Incoming/outgoing SA requests：接收或发送的SA-Request报文的统计数。

  • Incoming/outgoing SA responses：接收或发送的SA-Response报文的统计数。

  • Incoming/outgoing data packets：接收或发送的携带封装数据的SA报文的统计数。
