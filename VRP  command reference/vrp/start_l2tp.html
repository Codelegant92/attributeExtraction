start l2tp

命令功能

start l2tp命令用来指定本端作为L2TP LAC端时发起呼叫的触发条件。

undo start命令用来删除指定的触发条件。

缺省情况下，路由器上没有配置触发条件。

命令格式

start l2tp { ip ip-address } &<1-5> { domain domain-name | fullusername user-name }

undo start

参数说明

ip ip-address：指定隧道对端（LNS）的IP地址，最多可以设置五个LNS的IP地址，彼此形成备份LNS。

domain domain-name：指定触发连接请求的用户域名，对大小写敏感，字符串形式，长度范围是1～20。

fullusername user-name：指定触发连接请求的用户全名，对大小写敏感，取值为字符串，长度范围是1～64。

使用指南

start l2tp命令在LAC端使用。VRP支持如下2种情况触发连接请求：

  • 根据用户域名发起建立隧道的连接请求。例如：假设用户所在公司的域名为huawei.com，则可以指定包含huawei.com域名的用户为VPN用户。

  • 根据用户全名发起建立隧道的连接请求。此时，可以直接使用用户全名来指定该用户为VPN用户。

LAC在收到LAC客户端发起的呼叫后，根据上面所述的情况判断，如果发现发起呼叫的用户是VPN用户，则LAC按照配置的LNS的先后顺序向各LNS发送建立L2TP隧道的连接请求，当得到LNS的接收应答后，该LNS就作为隧道的对端；否则LAC向下一个LNS发起隧道连接请求。

LAC判定一个用户是否为VPN用户的顺序为：

  1. 检查用户全名是否与配置的VPN用户全名一致，如果一致，则为VPN用户。

  2. 如果用户全名与配置的VPN用户全名不一致，则检查用户名中的域名是否与配置的域名一致，如果一致，则为VPN用户。

  3. 如果均不匹配，则判定该用户不是VPN用户。

使用实例

# 根据域名huawei.com来判断VPN用户，对应的总部L2TP接入服务器的IP地址为202.38.168.1。

<Quidway> system-view
[Quidway] l2tp-group 1
[Quidway-l2tp1] start l2tp ip 202.38.168.1 domain huawei.com
