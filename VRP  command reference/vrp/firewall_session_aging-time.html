firewall session aging-time

命令功能

firewall session aging-time命令用来设置建立的会话表项的有效时间。

缺省情况下，tcp有效时间为240秒，udp为40秒，icmp为20秒，fragment为5秒，fin-rst为10秒，syn为5秒，h225和h245为10800秒，h323-rtcp 和h323-rtp为120秒，h323-t120为10800秒，netbios-name、netbios-session和netbios-data为120秒，ftp为600秒，ftp数据为240秒，hwcc为120秒，ras为600秒，ils 为600秒，http为600秒，smtp为600秒，rtsp为600秒，rtcp为120秒，rtp为120秒，telnet为600秒，dns为120秒，pptp为600秒，qq为120秒，msn为240秒。

命令格式

firewall session aging-time { default | { dns | fin-rst | fragment | ftp | ftp-data | h225 | h245 | h323-rtcp | h323-rtp | h323-t120 | http | hwcc | icmp | ils | msn | netbios-name | netbios-data | netbios-session | pptp | qq | ras | rtcp | rtp | rtsp | smtp | syn | tcp | telnet | udp } time-value }

参数说明

time-value：为一个时间值，单位是秒，取值范围是1～65535。

tcp：TCP连接的超时时间。

udp：UDP连接的超时时间。

icmp：ICMP协议的超时时间。

fragment：分片报文的超时时间。

fin-rst：tcp协议的fin 和 rst报文的超时时间。

syn：tcp的syn报文的超时时间。

h225：h225协议的超时时间。

h245：h224协议的超时时间。

h323-rtcp：h323协议的rtcp报文的超时时间。

h323-rtp：h323协议的rtp报文的超时时间。

h323-t120：h323协议的t120报文的超时时间。

netbios-name：netbios协议的name报文的超时时间。

netbios-session：netbios协议的session报文的超时时间。

netbios-data：netbios协议的data报文的超时时间。

ftp：ftp协议的超时时间。

hwcc：hwcc协议的超时时间。

ras：ras协议的超时时间。

ils：ils协议的超时时间。

http：http协议的超时时间。

smtp：smtp协议的超时时间。

rtsp：rtsp协议的超时时间。

rtcp：rtcp协议的超时时间。

rtp：rtp协议的超时时间。

telnet：telnet协议的超时时间。

dns：dns协议的超时时间。

pptp：pptp协议的超时时间。

qq：qq协议的超时时间。

msn：msn协议的超时时间。

default：恢复所有的时间设定值为系统的缺省默认值。

使用指南

firewall session aging-time命令设置会话表项连接的有效时间，针对不同的协议类型设定不同的时间。单位是秒。

由于目前的会话表项最多支持同时存在500000个连接，因此所建立的会话表项不能永久存在，该命令支持用户可为TCP、UDP、ICMP（Internet Control Message Protocol）等协议分别设置会话表项的有效时间，若在设定的时间内未使用该会话表项，该会话将失效。比如，某个IP地址为10.110.10.10的用户利用端口2000进行了一次对外TCP连接，但是若在一定时间内它一直未再使用这个TCP连接，系统将把这个连接删除。

使用实例

# 设定TCP协议的连接有效时间为600秒。

<Quidway> system-view
[Quidway] firewall session aging-time tcp 600
