snmp-agent target-host inform

命令功能

snmp-agent target-host inform命令用来设置接收Inform消息的目的地。

undo snmp-agent target-host命令用来取消当前设置。

命令格式

snmp-agent target-host inform address udp-domain ip-address [ udp-port port-number ] [ vpn-instance vpn-instance-name ] params securityname security-string [ v1 | v2c | v3 [ authentication | privacy ] ]

undo snmp-agent target-host ip-address securityname security-string [ vpn-instance vpn-instance-name ]

参数说明

address：指定SNMP消息传输的目标主机地址。address指定的IP地址和securityname指定的安全名字符串组成一个二元组，用来标识一个主机。

udp-domain：指定目标主机的传输域基于UDP。

ip-address：目标主机的IP地址。

udp-port port-number：指定接收Inform报文的端口号。取值范围0～65535。缺省情况下为162。

vpn-instance vpn-instance-name：VPN实例名，字符串形式，不支持空格。长度范围是1～31。vpn-instance可选，如果在VPN中使用，需要vpn-instance指定的VPN实例、IP地址和安全名字符串组成一个三元组来标识一个主机。

params：指定产生SNMP消息的目标主机信息。

securityname security-string：指定在网管侧显示的用户安全名，字符串形式，不支持空格。长度范围1～32。

v1 | v2c | v3：v1、v2c和v3分别代表SNMPv1、SNMPv2c和SNMPv3版本。

authentication：对报文进行认证但不加密。仅SNMPv3版本支持该参数。

privacy：对报文进行认证和加密。仅SNMPv3版本支持该参数。

使用指南

SNMPv1版本不支持Inform告警。当使用Inform方式配置了v1版本时，系统将提示：

Error: The inform notification does not support the version V1.

SNMPv1版本、SNMPv2c版本对报文不认证和不加密。SNMPv3版本通过authentication和privacy配置对报文认证和加密信息。SNMPv3版本默认对报文不认证和不加密。

snmp-agent target-host inform命令和snmp-agent trap enable命令结合在一起使用。snmp-agent trap enable命令用于使能SNMP Trap。若要一台路由器可以发送Inform通知，至少需要在该路由器上配置一条snmp-agent target-host inform命令和snmp-agent trap enable命令。

使用实例

# 配置告警发送方式为Inform方式、主机的安全名为123、协议版本是v3、对报文进行认证和加密，发送BGP模块的告警到告警主机192.168.0.1。

<Quidway> system-view
[Quidway] snmp-agent trap enable bgp
[Quidway] snmp-agent target-host inform address udp-domain 192.168.0.1 params securityname 123 v3 privacy
