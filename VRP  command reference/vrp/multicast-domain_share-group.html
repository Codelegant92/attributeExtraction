multicast-domain share-group

命令功能

multicast-domain share-group命令用来指定Share-Group组以及要绑定本VPN实例的MTI（Multicast Tunnel Interface）接口。

undo multicast-domain share-group命令用来恢复缺省配置。

缺省情况下，未指定Share-MDT的组地址。

命令格式

multicast-domain share-group group-address binding mtunnel number

undo multicast-domain share-group

参数说明

group-address表示组播组地址。点分十进制形式，取值范围是224.0.1.0～239.255.255.255。

number：要创建的MTI接口编号。整数形式，取值范围是0～1023。

使用指南

在同一台PE上，不同VPN实例不能配置相同的组地址，group-address也不能与所有实例的Switch-Group地址相同，number也不能与已创建的组播隧道接口号重复。

在相同VPN实例视图下，不允许重复配置。只有取消原配置后，才能配置新的地址和组播隧道接口。

组播隧道接口下可以配置IP地址和MTU。MTU是整数形式，取值范围是46～1500，单位是字节。

执行undo multicast-domain share-group命令后，将同时删除组播隧道接口、组播隧道接口下的接口命令以及如下命令的对应配置：multicast-domain switch-group-pool、multicast-domain log switch-group-reuse、multicast-domain switch-delay。

只有VPN实例中使能了组播路由功能，才允许配置该命令。

使用实例

# 指定VPN实例mytest的Share-Group组为224.1.1.1，绑定组播隧道接口MTI0。

<Quidway> system-view
[Quidway] ip vpn-instance mytest
[Quidway-vpn-instance-mytest] multicast-domain share-group 224.1.1.1 binding mtunnel 0
