fr congestion-threshold

命令功能

fr congestion-threshold命令用来使能帧中继接口的拥塞管理功能。

undo fr congestion-threshold命令用来禁止该功能。

缺省情况下，禁止帧中继接口的拥塞管理功能。

命令格式

fr congestion-threshold { de | ecn } queue-percentage

undo fr congestion-threshold { de | ecn }

参数说明

de：发生拥塞时，丢弃DE标志位为1的帧中继报文。

ecn：发生拥塞时，对帧中继报文的BECN和FECN标志位进行处理。

queue-percentage：网络拥塞阈值，为接口队列的使用率。它等于接口队列当前长度占队列总长度的百分比，取值范围是1～100。缺省值为100。

缺省级别

2：配置级

使用指南

本命令功能类似于命令congestion-threshold，不同之处在于：本命令所应用的范围是帧中继接口，而命令congestion-threshold所应用的范围是帧中继虚电路。

通常情况下，帧中继拥塞管理被应用于帧中继网络的DCE接口或NNI接口。

使用实例

# 设置接口队列长度超过总长度80％时，开始丢弃DE标志位为1的帧中继报文。

<Quidway> system-view
[Quidway] interface serial4/1/2
[Quidway-Serial4/1/2] fr congestion-threshold de 80
