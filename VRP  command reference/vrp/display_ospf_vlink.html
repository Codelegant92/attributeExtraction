display ospf vlink

命令功能

display ospf vlink命令用来显示OSPF的虚连接信息。

命令格式

display ospf [ process-id ] vlink [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

process-id：OSPF进程号。整数形式，取值范围是1～65535。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

此命令的输出信息有利于OSPF故障诊断。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示OSPF的虚连接信息。

<Quidway> display ospf vlink

项目

描述

Virtual-link Neighbor-id

通过虚连接相连的邻居路由器ID

Neighbor-State

状态，如Down/Init/2-Way/ExStart/Exchange/Loading/Full

Interface

开始列举区域内的接口信息，显示主接口IP地址和名称。如果为串口，则显示“Unknown”。

Cost

开销

State

接口状态

Type

接口类型

Transit Area

如果当前接口为虚连接，则显示运输区域ID

Timers

与下列项相关的定时器，如Hello间隔、失效时间、巡回检测间隔（NBMA）、重传间隔和接口传输延迟

GR State

GR状态，如GR/Helper/Normal
