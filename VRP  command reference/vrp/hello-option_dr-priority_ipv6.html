hello-option dr-priority（IPv6）

命令功能

hello-option dr-priority命令用来配置路由器竞选成为DR的优先级。

undo hello-option dr-priority命令用来恢复配置参数的缺省值。

缺省情况下，路由器竞选成为DR的优先级是1。

命令格式

hello-option dr-priority priority

undo hello-option dr-priority

参数说明

priority：表示路由器竞选成为DR（Designated Router）的优先级，优先级数值越大，优先级越高。整数形式，取值范围是0～4294967295。

视图

IPv6 PIM视图

缺省级别

2：配置级

使用指南

DR负责把本地组播源的数据封装到IPv6 PIM-SM注册报文中，并单播给RP。PIM Hello消息包含用于在共享网络上竞选DR的优先级字段。所有由系统发起的Hello消息都包含这样一个优先级。优先级相同时，IPv6地址最大的路由器当选。

如果网络中至少存在一个路由器不支持在Hello报文中携带优先级参数，则DR的选择依赖于每台设备的IPv6地址，IPv6地址最大的路由器成为DR。

此命令与接口视图下的pim ipv6 hello-option dr-priority含义是相同的，只是视图不同。缺省情况下，如果接口上没有配置，取IPv6 PIM视图下配置的值；否则取接口视图下配置的值。

使用实例

# 在公网实例中，配置路由器竞选成为DR的优先级为3。

<Quidway> system-view
[Quidway] pim-ipv6
[Quidway-pim6] hello-option dr-priority 3
