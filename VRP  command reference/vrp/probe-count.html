probe-count

命令功能

probe-count命令用来配置NQA测试例的测试探针数目。

undo probe-count命令用来恢复缺省配置。

缺省情况下，测试探针数目是3。

命令格式

probe-count number

undo probe-count

参数说明

number：一次测试中的探针数目。取值范围是1～15。Traceroute测试的探针数目不能大于10。

视图

NQA视图

缺省级别

2：配置级

使用指南

对于Jitter测试例、ICMP Jitter测试例、Path Jitter测试例、LSP Jitter测试例、使能接口板发包的Jitter和使能接口板发包的ICMP Jitter测试例，测试例发送的报文数是probe-count和jitter-packetnum的乘积。

对于Jitter测试例、LSP Jitter测试例、ICMP Jitter测试例、Path Jitter测试例，测试例发送的报文数是probe-count和jitter-packetnum的乘积。

在Jitter测试中，probe-count命令设置的测试次数与jitter-packetnum命令设置的测试包个数的乘积不能超过3000。

使用实例

# 设置测试例user test的探测次数为15。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] probe-count 15
