display msdp brief

命令功能

display msdp brief命令用来查看MSDP对等体状态的简要信息。

命令格式

display msdp [ vpn-instance vpn-instance-name | all-instance ] brief [ state { connect | down | listen | shutdown | up } ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

connect：处于Connect状态的MSDP对等体。

down：处于Down状态的MSDP对等体。

listen：处于Listen状态的MSDP对等体。

shutdown：处于Shutdown状态的MSDP对等体。

up：处于Up状态的MSDP对等体。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

MSDP对等体之间需要建立TCP连接，使用display msdp brief命令可以查看所有远端MSDP peer的简要信息地址及TCP连接状态。

若不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

MSDP规定IP地址高的peer作为Server，在639端口监听；IP地址低的peer作为Client，发起连接。如果连接失败会在一定时间后重新发起连接。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

# 查看公网实例的MSDP对等体状态的简要信息。

项目

描述

MSDP Peer Brief Information of VPN-Instance

MSDP对端简要信息所属的VPN实例。

Configured

共配置的MSDP Peer数目。

Up

Up状态的MSDP Peer数目。

Listen

Listen状态的MSDP Peer数目。

Connect

Connect状态的MSDP Peer数目。

Shutdown

Shutdown状态的MSDP Peer数目。

Down

Down状态的MSDP Peer数目。

Peer's Address

对等体地址。

State
MSDP会话的状态。

  • Up：连接建立，处于会话状态

  • Listen：本地作为Server，处于监听状态，连接建立

  • Connect：本地作为Client，处于连接状态，连接未建立

  • Shutdown：配置了MSDP对等体shutdown

  • Down：连接失败

Up/Down time
连接建立/失败发生的时间，时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

AS

MSDP peer的自治号，“?”表示无法获得自治域号。

SA Count

缓存中的（S, G）的数量。

Reset Count

重置次数，包括接收到Notification或者Holdtimer超时导致的重置。
