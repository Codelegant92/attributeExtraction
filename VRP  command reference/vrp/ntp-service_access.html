ntp-service access

命令功能

ntp-service access命令用来设置对本地设备NTP服务的访问控制权限。

undo ntp-service access命令用来取消设置的访问控制权限。

缺省情况下，没有访问限制。

命令格式

ntp-service access { peer | query | server | synchronization } acl-number

undo ntp-service access { peer | query | server | synchronization }

参数说明

peer：设置完全访问权限。可以对本地NTP服务进行时间请求和控制查询，本地时钟也可以同步到远程服务器。

query：设置最大访问限制，只允许对本地NTP服务进行控制查询。

server：允许服务器访问与查询。可以对本地NTP服务进行时间请求和控制查询，但本地时钟不会同步到远程服务器。

synchronization：只允许服务器访问。只允许对本地NTP服务进行时间请求。

acl-number：基本访问控制列表编号。取值范围是2000～2999。

使用指南

相对于NTP验证，ntp-service access命令实现一种简单的安全措施。当有一个访问请求时，按照最小访问限制到最大访问限制依次匹配，以第一个匹配的为准，匹配顺序为peer、server、synchronization、query。

根据所要实现的访问限制，需要配置该命令的设备也不相同，具体信息参见下表。

使用实例

# 设置允许第2000号访问列表中的peer可以对本地设备进行时间请求、查询控制、时间同步。

[Quidway] ntp-service access peer 2000

# 只允许匹配ACL 2002的peer对本地NTP设备进行时间请求、查询控制。

[Quidway] ntp-service access synchronization 2002
