multicast forwarding-table route-limit

命令功能

multicast forwarding-table route-limit命令用来限制组播转发表容量，即转发表中可以维护的最大表项数。

undo multicast forwarding-table route-limit命令用来恢复容量的缺省值。

缺省情况下，组播转发表中可以维护的最大表项数是8192。

命令格式

multicast forwarding-table route-limit limit

undo multicast forwarding-table route-limit

参数说明

limit：表示组播转发表的最大表项数。整数形式，取值范围是0～8192。

使用指南

如果新配置的最大表项数小于目前已存在的表项数量，超出限制的表项不会被立即删除。该数量限制在已有表项老化后重新向组播转发表中添加表项时才会生效，或者在执行reset multicast forwarding-table all命令后生效。

注意：

如果在路由器开始运行组播业务之后设置组播转发表的最大表项数，则必须保证设置的数量大于等于当前转发表项数，否则会导致组播业务出现故障。

因此，推荐用户在路由器开始运行组播业务之前，根据具体的网络环境，限定路由器的组播转发表的容量。

使用实例

# 配置公网实例组播转发表的最大表项数为1024。

<Quidway> system-view
[Quidway] multicast forwarding-table route-limit 1024

# 配置VPN实例组播转发表的最大表项数为64。

<Quidway> system-view
[Quidway] ip vpn-instance vpn1
[Quidway-vpn-instance-vpn1] multicast forwarding-table route-limit 64
