dialer route

命令功能

dialer route命令用来从一个DCC接口呼叫一个或多个目的地址，或接收多个对端的呼叫。

undo dialer route命令用来删除一条dialer route。

系统缺省没有定义dialer route。

命令格式

dialer route ip next-hop-address [ [ user hostname | speed { 56 | 64 } | broadcast ]* dial-string [ autodial ] [ interface-type interface-number ] ]

undo dialer route protocol next-hop-address [ dial-string ]

参数说明

protocol：网络协议关键字，支持IP等协议。

next-hop-address：对端网络地址。

user hostname：对端用户名，用于接收呼叫时的验证，可选项。

speed { 56 | 64 }：拨号速率，取值为56kbit/s和64kbit/s。

broadcast：表示广播报文可以从这条链路发送，可选项。

dial-string：对端的拨号串。

autodial：若一条dialer map配置了本参数，则路由器每隔一定时间会自动尝试用本dialer route拨号，该时间的间隔由命令dialer timer autodial设置，缺省为300秒。

interface-type interface-number：指定的拨出接口。

使用指南

配置该命令前，必须先执行命令dialer enable-circular使能轮询DCC。该命令不能在共享DCC下配置，即，该命令不能和dialer user在同一接口上同时配置。

若要发出呼叫，则需使用dial-number参数。若使用user关键字，则必须配置PPP验证。

一个拨号端口可以配置多条dialer route，一个目的地址也可配置多条dialer route。

说明：

若没有配置dialer-group命令，则DCC不会拨号。

使用实例

# 在接口Serial1/0/0上设置呼叫对端。

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] dialer route ip 131.108.2.5 user user1 14155553434
