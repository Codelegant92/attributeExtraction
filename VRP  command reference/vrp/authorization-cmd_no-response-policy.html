authorization-cmd no-response-policy

命令功能

authorization-cmd no-response-policy命令用来配置命令行授权失败时的策略。

undo authorization-cmd no-response-policy命令用来恢复命令行授权失败时的策略的默认值。

缺省情况下，命令行授权失败时允许用户继续在线。

命令格式

authorization-cmd no-response-policy { online | offline [ max-times no-response-max-time ] }

undo authorization-cmd no-response-policy

参数说明

online：按命令行授权失败后允许用户在线。

offline：按命令行授权失败后允许用户下线。

no-response-max-time：命令行授权失败的次数，取值范围是1～10。

授权方案视图

缺省级别

3：管理级

使用指南

这里的失败策略仅仅是使用于因HWTACACS服务器不可用或本地未配置用户而导致的按命令行授权失败情况。下面的两种情况不能触发命令行授权失败时的策略：

  • 服务器正常时，所执行的命令行未能通过在HWTACACS服务器端的授权；

  • 服务器不可用后，按命令行授权转入本地授权后，因执行的命令级别高于本地配置的级别而授权失败。

使用实例

# 用户按命令行授权失败3次后，用户下线。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authorization-scheme scheme1
[Quidway-aaa-author-scheme1] authorization-cmd no-response-policy offline max-times 3
