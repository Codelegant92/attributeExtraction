ospf authentication-mode

命令功能

ospf authentication-mode命令用来设置相邻路由器之间的验证模式及验证字。

ospf authentication-mode null命令用来在接口上配置null验证模式。

undo ospf authentication-mode命令用来删除接口下已设置的验证模式。

缺省情况下，接口不对OSPF报文进行验证。

命令格式

简单验证模式：

ospf authentication-mode simple [ [ plain ] plain-text | cipher cipher-text ]

undo ospf authentication-mode

MD5/HMAC-MD5验证模式：

ospf authentication-mode { md5 | hmac-md5 } [ key-id { plain plain-text | [ cipher ] cipher-text } ]

undo ospf authentication-mode

无验证模式：

ospf authentication-mode null

undo ospf authentication-mode

参数说明

simple：简单验证模式。

md5：MD5验证模式。

hmac-md5：HMAC-MD5验证模式。

null：null验证模式。

plain：明文口令类型。只能键入明文口令，在查看配置文件时以明文方式显示口令。对于简单验证模式，当此参数缺省时，默认为plain类型。

plain-text：指定明文验证字。simple模式下是1～8字节字符串，不支持空格；md5、hmac-md5模式是1～16字节字符串。

cipher：密文口令类型。对于MD5/HMAC-MD5验证模式，当此参数缺省时，默认为cipher类型。

cipher-text：指定密文验证字。simple模式下，1～8字节字符串对应明文，24字节对应密文；md5、hmac-md5模式下1～16字节字符串对应明文，24字节对应密文。

key-id：验证字标识符。整数形式，取值范围是1～255。

使用指南

配置时需要注意以下几点：

  • 同一网段的路由器接口下配置的验证模式和口令必须相同

  • 同时配置了接口验证和区域验证时，优先使用接口验证建立OSPF邻居

  • OSPF协议不支持NULL接口的配置

  • 验证字中不允许包含空格

使用实例

# 路由器Quidway1的接口Pos1/0/0与路由器Quidway2的接口Pos2/0/0在同一网段131.119.0.0上，配置接口支持简单验证模式，口令为abc。

# 在Quidway1上配置。

<Quidway> system-view 
[Quidway1] interface pos 1/0/0
[Quidway1-Pos1/0/0] ospf authentication-mode simple abc

# 在Quidway2上配置。

<Quidway> system-view 
[Quidway2] interface pos 2/0/0
[Quidway2-Pos2/0/0] ospf authentication-mode simple abc
