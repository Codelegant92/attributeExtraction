mode

命令功能

mode命令用来配置或更改Eth-Trunk的工作模式。

undo mode命令用来恢复Eth-trunk的工作模式为缺省模式。

缺省情况下，Eth-Trunk的工作模式为手工负载分担模式。

命令格式

mode { lacp-static | manual [ load-balance | backup ] }

undo mode

参数说明

lacp-static：指定为静态LACP模式。

manual：手工负载分担模式。

load-balance：手工负载分担模式。

backup：手工1:1主备模式。

缺省级别

2：配置级

使用指南

目前VRP所支持的工作模式分类如下。

  • 手工负载分担模式。该模式为链路聚合组的创建和接口的加入都需要手工配置的一种模式。在该模式下手工配置多个接口加入Eth-trunk进行分担负载。

  • 手工1:1主备模式。配置主、备两个接口建立Eth-trunk，该模式下只允许两个成员接口加入到聚合组。

  • 静态LACP模式。该模式通过LACP协议协商Eth-trunk参数后，选择活动接口转发数据。

配置手工1:1主备模式，Eth-Trunk必须是二层接口。

更改Eth-trunk的工作模式需要在确保Eth-trunk中不包含任何成员接口的情况下才可以。

本命令为覆盖式命令，即当多次执行mode命令后以最后执行的mode命令设定的模式为最终Eth-Trunk工作模式。

使用实例

# 创建一个Eth-Trunk编号为1，设置它的工作模式为手工负载分担模式。

<Quidway> system-view
[Quidway] interface eth-trunk 1
[Quidway-Eth-Trunk1] mode manual load-balance
