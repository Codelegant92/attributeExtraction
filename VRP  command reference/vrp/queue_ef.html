queue ef

命令功能

queue ef命令用来配置加速转发（Expedited-forwarding），报文进入绝对优先队列，并配置最大带宽。

undo queue ef命令用来取消配置。

命令格式

queue ef bandwidth { bandwidth [ cbs cbs ] | pct percentage }

undo queue ef

参数说明

bandwidth：EF队列允许的最大带宽，取值范围8～1000000，单位Kbps。

percentage：可用带宽占总带宽的百分比，取值范围1～100。

cbs：指定允许突发量，单位是字节，取值范围32～2000000字节，当bandwidth小于或者等于60Kbps时，cbs的缺省值为1500字节，当bandwidth大于60Kbps并且小于80000Kbps时，缺省值为bandwidth参数的数值的25倍，单位是字节，当bandwidth大于或者等于80000Kbps时，cbs的缺省值为2000000字节。

使用指南

该命令在行为视图下不能与queue af，queue-length，wred同时使用。

在策略下，缺省类default-class不能与queue ef所属behavior关联。

同一个策略下为确保转发（queue af）和加速转发（queue ef）的类指定的带宽之和必须不大于该策略所应用接口的可用带宽；

同一个策略下为确保转发（queue af）和加速转发（queue ef）的类指定的带宽百分比之和必须不大于100；

同一个策略下确保转发（queue af）和加速转发（queue ef）的类的带宽的配置必须都采用相同的值的类型，比如都采用绝对值形式，或者都采用百分比形式。

使用实例

# 配置报文进入优先队列，最大带宽为200Kbit/s，burst缺省计算为5000byte。

<Quidway> system-view
[Quidway] traffic behavior database
[Quidway-behavior-database] queue ef bandwidth 200 cbs 5000
