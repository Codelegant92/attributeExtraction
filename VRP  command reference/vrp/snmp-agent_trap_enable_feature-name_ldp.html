snmp-agent trap enable feature-name ldp

命令功能

snmp-agent trap enable feature-name ldp命令用来打开MPLS LDP模块的告警开关。

undo snmp-agent trap enable feature-name ldp命令用来关闭MPLS LDP模块的告警开关。

缺省情况下，MPLS LDP模块的告警开关处于开启状态。

命令格式

snmp-agent trap enable feature-name ldp { non-excessive all | trap-name { lsp | session-down | session-down-mib | session-pvl | session-reason | session-retry | session-up | session-up-mib } }

undo snmp-agent trap enable feature-name ldp { non-excessive all | trap-name { lsp | session-down | session-down-mib | session-pvl | session-reason | session-retry | session-up | session-up-mib } }

参数说明

non-excessive all：打开MPLS LDP模块的所有非海量告警开关。

trap-name：打开指定类型事件的告警开关。

lsp：打开LDP LSP变为Down事件的告警开关。

session-down：打开LDP会话变为Down事件的告警开关。

session-down-mib：打开LDP会话变为Down事件的MIB信息的告警开关。

session-pvl：打开LDP会话PathVectorLimitMismatch事件，即对端发送的路径向量最大值与本地配置的值不匹配事件的告警开关。

session-reason：打开LDP会话变为Down的原因的告警开关。

session-retry：打开LDP会话重新建立连接事件的告警开关。

session-up：打开LDP会话变为Up事件的告警开关。

session-up-mib：打开LDP会话变为Up事件的MIB信息的告警开关。

使用指南

目前定义的MPLS LDP模块的所有告警均为非海量告警。

如果要打开MPLS LDP模块所有事件的非海量告警开关，则选择non-excessive all批量处理；如果只打开某个或几个事件的告警开关时，则选择trap-name。

使用实例

# 打开LDP会话变为Down事件的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name ldp trap-name session-down
