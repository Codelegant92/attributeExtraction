transfer url

命令功能

transfer命令用来设置批量统计文件的上传方式。

undo transfer命令用来删除批量统计文件的上传方式。

命令格式

transfer { primary | secondary } url url

undo transfer { primary | secondary }url

参数说明

primary：指定主用文件上传方式。

secondary：指定备用文件上传方式。

url：上传方式。以URL字符串方式提供，取值范围是1～127。

URL具有FTP和TFTP两种格式，正确的格式如下：

  • FTP格式

    • 格式为—ftp://用户:密码@主机名或者IP地址/相对路径

    • 举例：ftp://user:password@host/folder/bulkstat1

    • 必须输入用户名、密码、IP地址/主机名。

  • TFTP格式

    • 格式为—tftp://主机名或者IP地址/相对路径

    • 举例：tftp://10.1.0.1/tftpboot/user/bulkstat1

    • 必须输入用户名、IP地址/主机名。

在FTP和TFTP这2种格式中，用户名长度不允许超过64个字符，密码不允许超过16个字符；上述格式中相对路径可以不进行指定，上传的文件将放在服务器默认的目录；用户名、密码、相对路径的正确性由用户保证，VRP不做检查。

使用指南

批量统计文件的上传支持主备方式，如果以主用方式上传不成功，则自动使用备用方式上传。

必须要配置主URL，才能启动统计文件；备URL可选。

文件启动后可以修改主/备URL，不能删除主URL ，但可以删除备URL，也可以添加备URL。

使用实例

# 设置批量统计文件“iftable”的主用上传方式。

<Quidway> system-view
[Quidway] bulk-file iftable
[Quidway-bulk-file-iftable] transfer primary url ftp://user:password@host/folder/bulkstat1

# 设置批量统计文件“iftable”的备用上传方式。

[Quidway-bulk-file-iftable] transfer secondary url tftp://10.1.0.1/tftpboot/user/bulkstat1

# 撤销批量统计文件“iftable”的主用上传方式。

[Quidway-bulk-file-iftable] undo transfer primary url
