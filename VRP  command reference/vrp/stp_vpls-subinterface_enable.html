stp vpls-subinterface enable

命令功能

stp vpls-subinterface enable命令用来使能主接口在收到TC报文后通告其绑定了VSI的子接口的功能。

undo stp vpls-subinterface enable命令用来禁止主接口在收到TC报文后通告其绑定了VSI的子接口的功能。

缺省情况下，禁止主接口在收到TC报文后通告其绑定了VSI的子接口。

命令格式

stp vpls-subinterface enable

undo stp vpls-subinterface enable

参数说明

无

视图

Ethernet接口视图、GE接口视图、Eth-Trunk接口视图、VE接口视图

缺省级别

2：配置级

使用指南

如果配置了MSTP功能的接口存在子接口，并且在子接口上配置了其他业务，此时，可以使用stp vpls-subinterface enable命令，当主接口在收到TC报文后，能够通告其子接口及时刷新MAC表项和ARP表项，从而保证用户业务不中断。此外，使用stp vpls-subinterface enable命令后，当主接口是discarding状态时，可以使得绑定了VSI的子接口变为flowdown状态。从而防止CE双归接入PE时，VPLS网络行成环路。

使用实例

# 使能接口GE1/0/1在收到TC报文后通告其绑定了VSI的子接口的功能。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/1
[Quidway-GigabitEthernet1/0/1] portswitch
[Quidway-GigabitEthernet1/0/1] stp vpls-subinterface enable
