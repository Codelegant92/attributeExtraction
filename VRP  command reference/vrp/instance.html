instance

命令功能

instance命令用来将指定VLAN映射到指定的生成树实例上。

undo instance命令用来删除指定VLAN和指定生成树实例的映射关系。

缺省情况下，所有VLAN均映射到CIST，即实例0上。

命令格式

instance instance-id vlan { vlan-id [ to vlan-id ] } &<1-10>

undo instance instance-id [ vlan { vlan-id [ to vlan-id ] } &<1-10> ]

参数说明

instance instance-id：生成树实例的编号。取值范围是0～48，取值为0表示CIST。

vlan-id：VLAN编号。取值范围是1～4094。

视图

MST域视图、MST进程域视图

缺省级别

2：配置级

使用指南

不能将同一个VLAN映射到多个不同的生成树实例上。如果将一个已经和生成树实例建立映射关系的VLAN又映射到另一个生成树实例上，原来的映射关系将被取消。

使用undo instance命令删除指定VLAN和指定生成树实例的映射关系后，这些VLNAN重新映射到CIST（即实例0）上。

在使用undo instance命令时，如果没有指定VLAN，则将和该生成树实例有映射关系的所有VLAN映射到CIST上。

配置instance命令后，必须执行active region-configuration命令才能激活instance命令配置的数据。

使用实例

# 将VLAN 2映射到生成树实例1上。

<Quidway> system-view
[Quidway] stp region-configuration
[Quidway-mst-region] instance 1 vlan 2

# 在MSTP进程10中，将VLAN 2映射到生成树实例1上。

<Quidway> system-view
[Quidway] stp process 10
[Quidway-mst-process-10] stp region-configuration
[Quidway-mst-process-10-region] instance 1 vlan 2
