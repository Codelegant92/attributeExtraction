mac-tunnel vlan-pool

命令功能

mac-tunnel vlan-pool命令用来配置设备上划分给MAC Tunnel使用的骨干VLAN（Backbone VLAN）资源池。

undo mac-tunnel vlan-pool命令用来删除MAC Tunnel骨干VLAN资源池中的VLAN。

缺省情况下，设备上没有配置骨干VLAN资源池。

命令格式

mac-tunnel vlan-pool { vlan-id1 [ to vlan-id2 ] }

undo mac-tunnel vlan-pool { vlan-id1 [ to vlan-id2 ] }

参数说明

vlan-id1：指定MAC Tunnel的骨干VLAN的ID。整数形式，取值范围是1～4094。

to vlan-id2：指定最后一个VLAN的编号。整数形式，取值范围是1～4094。vlan-id2必须大于vlan-id1，它和vlan-id1共同确定一个范围。

使用指南

MAC Tunnel由骨干VLAN和MAC Tunne对端设备的虚拟MAC地址唯一标识。在创建MAC Tunnel时，必须从骨干VLAN资源池中选择VLAN，未加入骨干VLAN资源池的VLAN不能用来创建MAC Tunnel。

在同一条MAC Tunnel两端的设备上，必须配置相同的骨干VLAN，并将其加入骨干VLAN资源池。

满足如下条件的VLAN可以加入到MAC Tunnel的骨干VLAN资源池：

  • 该VLAN已经被创建。

  • 该VLAN不可以是任何接口的缺省VLAN，或不可以有接口以Trunk方式加入该VLAN。

  • 该VLAN不能作为全局VLAN和业务实例绑定。

  • 该VLAN没有配置VLANIF。

  • 该VLAN没有被RRPP作为控制VLAN。

对于加入到骨干VLAN资源池中的VLAN，其部分属性发生了变化。具体包括：该VLAN自动禁止了MAC地址学习，自动丢弃未知单播、未知组播和广播报文，也不支持对上述配置的更改。

如果资源池内某一骨干VLAN已被MAC Tunnel使用，则不允许从VLAN资源池删除。

使用实例

# 配置MAC Tunnel的骨干VLAN ID为2。

<Quidway> system-view
[Quidway] mac-tunnel vlan-pool 2
