display multicast routing-table static

命令功能

display multicast routing-table static命令用来查看组播静态路由信息。

命令格式

display multicast routing-table [ vpn-instance vpn-instance-name ] static [ config ] [ source-address { mask-length | mask } ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

config：显示组播静态路由的配置信息。

source-address：表示组播源地址。点分十进制形式。

mask：表示组播源地址的掩码。点分十进制形式。

mask-length：表示组播源地址的掩码长度。整数形式，取值范围是0～32。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display multicast routing-table static命令：

  • 如果不指定vpn-instance，则显示公网实例的组播静态路由表。

  • 如果指定config，则显示所有组播静态路由的配置信息，包括激活的和未激活的。

  • 如果指定source-address，则仅显示指定组播源的组播静态路由。

  • 如果本地路由器上未配置组播静态路由则无显示信息。使用ip rpf-route-static命令配置组播静态路由。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示公网实例的全部组播静态路由。

<Quidway> display multicast routing-table static
Multicast Routing Table
Routes : 1

 Mroute 11.1.0.0/24
          Interafce = GigabitEthernet6/0/0       RPF Neighbor = 11.1.2.2
          Matched routing protocol = ospf, process-id = 100, Route-policy = none
          Preference = 1, Order = 1
 Running Configuration = ip rpf-route-static 11.1.0.0 24 ospf 100 11.1.2.2 order 1

# 显示公网实例中到达组播源10.10.0.0/16的全部组播静态路由。

<Quidway> display multicast routing-tale static 10.10.0.0 255.255.0.0
Multicast Routing Table
Routes : 1

 Mroute 11.1.0.0/24
          Interafce = GigabitEthernet6/0/0       RPF Neighbor = 11.1.2.2
          Matched routing protocol = ospf, process-id = 100, Route-policy = none
          Preference = 1, Order = 1
 Running Configuration = ip rpf-route-static 11.1.0.0 24 ospf 100 11.1.2.2 order 1

项目

描述

Multicast Routing Table

组播路由表。

Routes

路由条数。

Mroute

组播路由源地址/掩码长度。

Interface

可达组播源的出接口。

RPF-neighbor

可达组播源的RPF邻居IP地址。

Matched routing protocol

匹配的单播路由类型。

process-id

路由协议的进程号。

Route-policy

路由策略。路由的组播源地址应当匹配路由策略。

Preference

路由优先级。

Order

路由序号。

Running Configuration

配置该静态路由的命令行。
