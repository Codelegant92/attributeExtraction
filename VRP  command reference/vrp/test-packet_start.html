test-packet start

命令功能

test-packet start命令用来发送测试报文。

命令格式

test-packet start [ mac mac-address ] [ vlan vlan-id ] { interface interface-type interface-number | service-instance instance-name } [ -c count | -p speed | -s size ] *

参数说明

mac mac-address：指定测试报文的目的MAC地址。mac-address表示MAC地址，格式为H-H-H，其中H为1至4位的十六进制数。

vlan vlan-id：指定报文所属VLAN。vlan-id表示VLAN ID，整数形式，取值范围是1～4094。

interface interface-type interface-number：指定发送测试报文的出接口。interface-type表示接口类型，取值包括：ethernet和gigabitethernet。interface-number表示接口编号。

service-instance instance-name：指定发送测试报文的MAC tunnel业务实例。instance-name表示业务实例的名称，字符串形式，不支持空格，长度范围是1～31字节。该业务实例必须处于Up状态，且必须是点到点类型的。

-c count：指定发送测试报文的数量。count表示发送测试报文的数量，整数形式，取值范围是1～65535，缺省值是5。

-p speed：指定发送测试报文的速率。speed表示发送测试报文的速率，整数形式，取值包括1、5，单位是Mbit/s，缺省值是1Mbit/s。

-s size：指定测试报文的长度。size表示测试报文的长度，取值范围是64～1518，单位是字节，缺省值是64字节。

使用指南

使用本命令发送的测试报文是为了测试链路的丢包率构造的一种以太类型的报文。本命令可和EFM OAM远端环回功能配合使用，来测试链路的丢包率。

当指定通过MAC tunnel的业务实例发送测试报文时，如果该业务实例封装映射模式是：

  • transparent类型，可以不指定参数vlan vlan-id；

  • s-tagged类型，必须指定参数vlan vlan-id，且与对应业务实例的Port+VLAN中的VLAN相同，否则测试报文在出MAC tunnel遂道时会被丢弃。

在发送测试报文进行过程中，不能修改发送测试报文的参数。

如需终止发送测试报文，可按<Ctrl+C>快捷键。

使用本命令发送测试报文后，可使用display test-packet result命令查看测试结果。

使用实例

# 发送测试报文，目的MAC地址为0000-0300-0204，出接口为GigabitEthernet4/0/0。

<Quidway> system-view
[Quidway] test-packet start mac 0000-0300-0204 interface gigabitethernet4/0/0
