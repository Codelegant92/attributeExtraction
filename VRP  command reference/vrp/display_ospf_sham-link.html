display ospf sham-link

命令功能

display ospf sham-link命令用来查看OSPF区域的伪连接信息。

如果不指定进程号和区域号，将显示配置的所有伪连接信息。

命令格式

display ospf [ process-id ] sham-link [ area area-id ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

process-id：OSPF进程号。整数形式，取值范围是1～65535。

area area-id：指定OSPF区域ID，可以是整数形式，取值范围是0～4294967295；也可以是IPv4地址形式。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看所有OSPF伪连接信息。

<Quidway> display ospf sham-link
           OSPF Process 100 with Router ID 100.1.1.2
 Sham Link:
 Area         NeighborId        Source-IP       Destination-IP  State  Cost
 0.0.0.1       100.1.1.2              3.3.3.3          5.5.5.5   P-2-P 10

# 查看Area 1的OSPF伪连接信息。

<Quidway> display ospf sham-link area 1
          OSPF Process 100 with Router ID 100.1.1.2
  Sham-Link: 3.3.3.3 --> 5.5.5.5
  Neighbor ID: 100.1.1.2,    State: Full
  Area: 0.0.0.1
  Cost: 10  State: P-2-P, Type: Sham
  Timers: Hello 10 , Dead 40 , Retransmit 5 , Transmit Delay 1

项目

描述

Area

伪连接所属的OSPF区域

NeighborId

路由器的NeighborID

Source-IP

伪连接的源IP地址

Destination-IP

伪连接的目的IP地址

State

伪连接的接口状态，“P-2-P”表示是点到点链路

Cost

伪连接的开销

Type

连接类型

Timers

与下列项相关的定时器，如Hello间隔、失效时间、重传间隔和接口传输延迟
