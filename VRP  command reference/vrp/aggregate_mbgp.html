aggregate（MBGP）

命令功能

aggregate命令用来在MBGP路由表中创建一条组播聚合记录。这条聚合路由在本地的出接口为NULL0，当其他路由器收到聚合路由后会自动加上出接口。

undo aggregate命令用来关闭该功能。

缺省情况下，不进行路由聚合。

命令格式

aggregate ip-address { mask | mask-length } [ as-set | attribute-policy route-policy-name1 | detail-suppressed | origin-policy route-policy-name2 | suppress-policy route-policy-name3 ] *

undo aggregate ip-address { mask | mask-length } [ as-set | attribute-policy route-policy-name1 | detail-suppressed | origin-policy route-policy-name2 | suppress-policy route-policy-name3 ] *

参数说明

ip-address：聚合路由的IP地址。点分十进制形式。

mask：聚合路由的网络掩码。点分十进制形式。

mask-length：表示IPv4网络地址的掩码长度。

as-set：生成具有AS-SET的路由。若需聚合较多AS路径时，请慎用此参数。

attribute-policy route-policy-name1：设置聚合后路由的属性。route-policy-name1是路由策略名称。

detail-suppressed：仅通告聚合路由。

origin-policy route-policy-name2：设置允许生成聚合路由的策略名称。route-policy-name2是路由策略名称。

suppress-policy route-policy-name3：设置抑制指定路由通告的策略名称。route-policy-name3是路由策略名称。

使用实例

# 在MBGP路由表中创建一条组播聚合记录，并设置该聚合路由的地址为192.213.0.0。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] ipv4-family multicast
[Quidway-bgp-af-multicast] aggregate 192.213.0.0 255.255.0.0
