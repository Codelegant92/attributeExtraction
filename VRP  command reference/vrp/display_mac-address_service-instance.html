display mac-address service-instance

命令功能

display mac-address命令用来查看指定业务实例下MAC地址表中的静态、黑洞或具体MAC地址对应的表项信息。

命令格式

display mac-address { blackhole | static | mac-address } service-instance service-instance-name [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

static：静态表项，不被老化，系统复位或接口板热插拔后不丢失。

blackhole：黑洞表项，不被老化，系统复位或接口板热插拔后不丢失。

mac-address：指定待查看的MAC地址。格式为H-H-H。其中H为4位的十六进制数，可以输入1～4位，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。

service-instance service-instance-name：指定业务实例名字。该业务实例必须为多点到多点的业务实例。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

include：只输出包含匹配字符串的行。

exclude：只输出不包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

由于多点到多点业务实例支持基于静态MAC地址的转发功能，也可以配置MAC地址表项为黑洞表项。使用display mac-address命令可以查看MAC地址表中的静态、黑洞表项。

使用实例

# 查看业务实例的静态转发表信息。

<Quidway> display mac-address static service-instance si1
MAC Address       VLAN    PEVLAN   CEVLAN    Port   Type    Lsp/      
                  VSI/SI                                    Mac-tunnel
----------------------------------------------------------------------
0000-0000-0003    si1     -        -         -      static  tu1       
Total matching items displayed = 1

项目

描述

Mac-address

用户MAC地址

VLAN/VSI/SI

出接口所在的VLAN或者所属的VSI。SI表示服务实例名。

CEVLAN

用户内层VLAN的Tag值

Port

接口名称和编号

Type

MAC地址表项类型，包括静态表项、动态表项和黑洞表项

Mac-tunnnel

Mac Tunnel名称
