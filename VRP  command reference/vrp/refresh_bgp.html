refresh bgp

命令功能

refresh bgp命令用来手工对BGP连接进行软复位。BGP软复位可以在不中断BGP连接的情况下重新刷新BGP路由表，并应用新的策略。

命令格式

refresh bgp [ vpn-instance vpn-instance-name | vpnv4 ] { all | ipv4-address | group group-name | external | internal } { export | import }

refresh bgp ipv6 { all | ipv4-address | ipv6-address | group group-name | external | internal } { export | import }

refresh bgp vpn6-instance vpn6-instance-name { all | ipv6-address | group group-name | external | internal } { export | import }

refresh bgp vpnv6 { all | ipv4-address | ipv6-address | group group-name | external | internal } { export | import }

参数说明

vpn-instance vpn-instance-name：软复位与VPN实例的相关BGP连接。

vpnv4：软复位与VPNv4的相关BGP连接。

all：软复位所有BGP的IPv4连接。

ipv4-address：指定BGP对等体的IPv4地址。

ipv6-address：指定BGP对等体的IPv6地址。

group group-name：指定BGP对等体组。

external：软复位EBGP连接。

internal：软复位IBGP连接。

export：触发出方向的软复位。

import：触发入方向的软复位。

ipv6：对BGP4+连接进行软复位。

vpn6-instance vpn-instance-name：软复位与IPv6 VPN实例的相关BGP连接。

vpnv6：软复位与VPNv6的相关BGP连接。

使用指南

BGP软复位可以在不中断BGP连接的情况下重新刷新BGP路由表，并应用新的策略。

BGP软复位要求网络中的所有BGP路由器支持Route-refresh能力。如果有的路由器不支持此特性，可以执行reset bgp命令，重新与对等体建立会话，刷新路由表。如果不希望与对等体中断会话，可以通过配置peer keep-all-routes命令，保留对等体的所有原始路由信息，在刷新BGP路由表时可以不中断邻居会话。

使用实例

# 对所有BGP连接在入方向上进行软复位，使新的配置生效。

<Quidway> refresh bgp all import

# 在出方向上软复位所有对等体BGP4+连接。

<Quidway> refresh bgp ipv6 all export

# 对与对等体1.1.1.1的BGP VPNv6连接在出方向上进行软复位，使新的配置生效。

<Quidway> refresh bgp vpnv6 1.1.1.1 export
