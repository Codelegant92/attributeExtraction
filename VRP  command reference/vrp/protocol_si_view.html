protocol（业务实例视图）

命令功能

protocol命令用来配置业务实例对各种二层协议报文的处理方式。

缺省情况下，业务实例转发所有的二层协议报文。

命令格式

protocol { all | 8021ag | stp } { discard | forward }

参数说明

all：指定对所有二层协议报文进行相同的处理。

stp：指定对STP协议报文进行处理。

8021ag：指定对802.1ag协议报文进行处理。

discard：指定业务实例丢弃指定的协议报文。

forward：指定业务实例转发指定的协议报文。

缺省级别

2：配置级

使用指南

通过配置业务实例对指定的二层协议报文的处理方式，可以实现业务实例丢弃或者转发指定的协议报文。

只有用户业务与指定的业务实例映射后，protocol命令才可见。

业务实例与用户业务映射关系不同，可处理的协议报文也不同。

  • 基于接口的映射，可以对上述所有报文进行处理。

  • 基于接口＋VLAN、接口＋VLAN组、全局VLAN的映射，只能配置stp、802.1ag和all三个参数。其中all只包括stp和802.1ag两种协议报文。

  • 基于接口＋VLAN＋COS的映射，无法处理上述所有的协议报文，该命令无效。

说明：

如果设备上全局使能了STP，则需要在mp2mp业务实例的用户接口和隧道出接口下去使能STP，即执行stp disable命令，该mp2mp业务实例才能透传BPDU报文。

使用实例

# 配置业务实例转发STP协议报文。

<Quidway> system-view
[Quidway] service-instance instance-name mysi
[Quidway-service-instance-mysi] protocol stp forward
