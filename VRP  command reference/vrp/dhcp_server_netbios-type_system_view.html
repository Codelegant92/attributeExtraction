dhcp server netbios-type（系统视图）

命令功能

dhcp server netbios-type命令用来配置当前接口的DHCP客户端NetBIOS节点类型。

undo dhcp server netbios-type命令用来恢复缺省设置。

缺省情况下，不指定客户端的节点类型。

命令格式

dhcp server netbios-type { b-node | h-node | m-node | p-node } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

undo dhcp server netbios-type { b-node | h-node | m-node | p-node } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

参数说明

b-node：b类节点，“b”代表广播（broadcast），即，此类节点采用广播的方式获取映射关系。

p-node：p类节点，“p”代表端到端（peer-to-peer），即，此类节点采用与NetBIOS服务器通信的方式获取映射关系。

m-node：m类节点，“m”代表混合（mixed），是具有部分广播特性的p类节点。

h-node：h类节点，“h”代表混合（hybrid），是具备“端对端”通信机制的b类节点。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：指定所有接口的NetBIOS节点类型。

使用指南

DHCP客户端在广域网上使用NetBIOS协议时，需要在主机名和IP地址之间建立映射关系。

该命令配置后，不能通过display current-configuration命令查看配置情况，它的执行是通过循环调用dhcp server netbios-type（接口视图）命令实现的。

使用实例

# 配置GE 2/0/0.1到GE 2/0/0.5之间的接口地址池为其客户端分配的NetBIOS节点类型为p-node。

<Quidway> system-view
[Quidway] dhcp server netbios-type p-node interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
