display service-instance

命令功能

display service-instance命令用来查看指定的业务实例信息。

命令格式

display service-instance [ instance-name service-instance-name ] [ verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

instance-name service-instance-name：指定业务实例的名字。长度范围是1～31个字符。

verbose：查看业务实例的详细信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

include：只输出包含匹配字符串的行。

exclude：只输出不包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

根据选择参数的不同，可以查看到对应当业务实例信息：

  • 如果不输入业务实例名字，则显示所有的业务实例。

  • 如果不输入verbose参数，则只显示指定业务实例的名字、业务类型和service-ID值。

  • 如果输入verbose参数，需要区分业务类型进行显示：

    • 如果业务实例为mp2mp类型时，显示业务实例的名称、业务类型、接口类型、业务实例ID、优先级、隔离标记、二层控制报文处理方法、映射的用户信息、绑定的MAC Tunnel信息、Up/Down状态、MAC地址学习限制、MAC地址学习是否关闭、MAC地址学习到达上限后的处理方式、未知单播报文处理方式、未知组播报文处理方式、广播报文处理方式。

    • 如果业务实例为p2p类型时，显示业务实例的名称、业务类型、接口类型、I-TAG、优先级、二层控制报文处理方法、映射的用户信息、绑定的MAC Tunnel信息、Up/Down状态、MAC地址学习是否关闭、未知单播报文处理方式、未知组播报文处理方式、广播报文处理方式。

    • 其中，业务实例Up/Down状态的显示，如果业务实例配置了shutdown命令，则显示内容为“Administratively DOWN”；如果业务实例没有配置shutdown命令，则根据实际的业务实例状态显示为Up或Down状态。

表1 display service-instance命令的输出信息描述表

项目

描述

Service-instance name

业务实例的名字。

Service-type

业务类型。

Interface-type

业务接口类型。

Total number of service-instances

业务实例的总数。

Number of service-instances with status being Up

状态为Up的业务实例数。

Service-ID

业务实例ID。

Priority

业务实例优先级，分为0～7共8个级别。缺省情况下为信任用户业务8021p的优先级。

TrustTunnelPriority egress
业务实例宿端是否信任隧道优先级：

  • yes：宿端信任隧道优先级。

  • no：宿端不信任隧道优先级。

Status
业务实例状态：

  • UP：激活

  • DOWN：关闭

  • Administratively DOWN：业务实例配置shutdown命令

Mac-limit maximum

MAC地址学习的最大限制数量，如果无限制，则显示为No limit（只有多点到多点业务实例存在该显示信息）。

Mac-limit action
业务实例的MAC地址学习达到最大数量限制后对报文的处理方法（只有多点到多点业务实例存在该显示信息）：

  • Forward：转发

  • Discard：丢弃

Mac-learning
MAC地址学习功能的状态（只有多点到多点业务实例存在该显示信息）：

  • Enable：使能MAC地址学习功能

  • Disable：关闭MAC地址学习功能

Unknown-unicast
业务实例对未知单播报文的处理方法（只有多点到多点业务实例存在该显示信息）：

  • Forward：转发未知单播报文

  • Discard:：丢弃未知单播报文

Unknown-multicast
业务实例对未知组播报文的处理方法（只有多点到多点业务实例存在该显示信息）：

  • Forward：转发未知组播报文

  • Discard:：丢弃未知组播报文

Broadcast
广播报文的处理状态（只有多点到多点业务实例存在该显示信息）：

  • Forward：转发广播报文

  • Discard:：丢弃广播报文

Control-protocol
协议报文的操作：

  • Forward：转发协议报文

  • Discard：丢弃协议报文

Mapping-user

映射的用户。

IsolateFlag
用户隔离标记。

  • yes：该用户配置了隔离。

  • no：该用户没有配置隔离。

Mac-tunnel

绑定的MAC Tunnel名称。
