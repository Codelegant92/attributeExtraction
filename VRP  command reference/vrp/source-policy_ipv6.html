source-policy（IPv6）

命令功能

source-policy命令用来配置路由器对接收的组播数据报文根据源（组）地址进行过滤。

undo source-policy命令用来删除配置。

缺省情况下，未设置源过滤策略。

命令格式

source-policy acl6-number

undo source-policy

参数说明

acl6-number：表示基本或高级访问控制列表序号。整数形式，取值范围是2000～3999。

视图

IPv6 PIM视图

缺省级别

2：配置级

使用指南

如果建立了源地址过滤和基本ACL之间的关联，所有的组播数据报文需要与指定的源地址相匹配，不匹配的报文则被丢弃。

如果建立了源地址过滤和高级ACL之间的关联，所有的组播数据报文需要与指定的源地址和组播地址相匹配，不匹配的报文则被丢弃。

此命令不仅对组播数据进行过滤，也对注册报文中封装的组播数据进行过滤。

如果重复执行此命令，最新的配置会覆盖原来的配置。

请参见IPv4 PIM配置命令中的source-policy（IPv4）。

source-policy命令与acl命令配合使用。

  • 在基本ACL6视图下，通过rule命令的source参数指定发送组播数据报文的源地址。

  • 在高级ACL6视图下，通过rule命令的source参数指定发送组播数据报文的源地址，destination参数指定IPv6组播组地址。

使用实例

# 设置接受源地址为3121::1的组播数据报文，丢弃源地址是3121::2的组播数据报文。

<Quidway> system-view
[Quidway] acl ipv6 number 2001
[Quidway-acl6-basic-2001] rule permit source 3121::1 128
[Quidway-acl6-basic-2001] rule deny source 3121::2 128
[Quidway-acl6-basic-2001] quit
[Quidway] pim-ipv6
[Quidway-pim6] source-policy 2001
