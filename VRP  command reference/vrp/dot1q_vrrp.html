dot1q vrrp

命令功能

dot1q vrrp命令用来配置指定Tag值的Dot1q数据报文来维护VRRP状态。

undo dot1q vrrp命令用来取消用Dot1q数据报文来维护VRRP状态。

缺省情况，在终结子接口下没有指定任何值的Dot1q数据报文来维护VRRP状态。

命令格式

dot1q vrrp vid vid

undo dot1q vrrp

参数说明

vid vid：VLAN的ID，十进制整数形式，取值范围是1～4094。

视图

Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图

缺省级别

2：配置级

使用指南

在以太网子接口上使用dot1q vrrp命令前，应依次先在该子接口下配置control-vid vid dot1q-termination命令和dot1q termination vid命令，否则无法配置dot1q vrrp命令。

dot1q vrrp命令用来使Dot1q终结子接口能够支持VRRP协议。

当在Dot1q终结子接口、QinQ终结子接口或者VLANIF接口上同时配置VRRP和静态ARP时，需要注意：

  • 不能将与这些接口相关的静态ARP表项对应的映射IP地址作为VRRP的虚拟地址。

  • 不能将这些接口所在VRRP备份组的虚拟地址作为静态ARP表项的映射IP地址。

否则会生成错误的主机路由，影响设备之间的正常转发。

使用实例

# 配置子接口GE1/0/0.1使用VLAN 100来维护VRRP状态。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 dot1q-termination
[Quidway-GigabitEthernet1/0/0.1] dot1q termination vid 100
[Quidway-GigabitEthernet1/0/0.1] dot1q vrrp vid 100
