ip ipv6-prefix

命令功能

ip ipv6-prefix命令用来配置地址前缀列表或增加其中一个表项。

undo ip ipv6-prefix命令用来删除地址前缀列表或其中一个表项。

缺省情况下，不指定地址前缀列表。

命令格式

ip ipv6-prefix ipv6-prefix-name [ index index-number ] { deny | permit } ipv6-address prefix-length [ greater-equal greater-equal-value ] [ less-equal less-equal-value ]

undo ip ipv6-prefix ipv6-prefix-name [ index index-number ]

参数说明

ipv6-prefix-name：地址前缀列表的名称，该名称必须唯一。字符串形式，不支持空格，取值范围是1～19。

index index-number：指定地址前缀列表的表项序号。缺省情况下，该序号值按照配置先后顺序依次递增，每次加10，且第一个序号值为10。

permit：指定前缀列表表项的匹配模式为允许。在这种情况下，如果IPV6地址在指定的前缀范围内，该IPV6地址将通过过滤而不进入下一表项的测试；否则，该IPv6地址将进入下一表项的测试。

deny：指定前缀列表表项的匹配模式为拒绝。在这种情况下，如果IPv6地址在指定的前缀范围内，该IPV6地址将不能通过过滤从而不进入下一表项的测试；否则，该IPV6地址将进入下一表项的测试。

ipv6-address：用IPv6地址形式来指定IPv6地址前缀范围。如果指定::，则表示匹配全零地址。

prefix-length：用掩码长度来指定IPv6地址前缀范围。取值范围为0～128。如果使用:: 0 less-equal 128，则表示匹配所有IPv6地址。

greater-equal | less-equal：如果IPv6地址和前缀长度都已匹配，则使用该参数来指定地址前缀范围。greater-equal的意思是“大于等于”；less-equal的意思是“小于等于”。前缀长度范围可以表达为prefix-length <= greater-equal-value <= less-equal-value <= 128。如果只指定了greater-equal，则前缀范围为[greater-equal-value，128]；如果只指定了less-equal，则前缀范围为[prefix-length，less-equal-value]。

使用指南

IPv6地址前缀列表用于过滤IPv6地址。同一个地址前缀列表可包含多个表项，每个表项指定一个地址前缀范围。此时，各表项之间是“或”的关系，即只要通过其中一个表项就认为已通过该地址前缀列表的过滤，所有表项都没有通过则意味着没有通过该地址前缀列表的过滤。

配置了地址前缀列表后，缺省情况下，RM会立即通知各个协议应用该列表进行过滤。如果需要延迟列表的生效时间，需使用命令route-policy-change notify-delay配置延迟生效时间。

使用实例

# 允许掩码长度在32位到64位之间的地址通过。

<Quidway> system-view
[Quidway] ip ipv6-prefix abc permit :: 0 greater-equal 32 less-equal 64

# 拒绝前缀为3FFE:D00::/32，且前缀长度大于32位的地址通过。

<Quidway> system-view
[Quidway] ip ipv6-prefix abc deny 3FEE:D00:: 32 less-equal 128
