display pim ipv6 routing-table

命令功能

display pim ipv6 routing-table命令用来查看IPv6 PIM组播路由表。

命令格式

display pim ipv6 routing-table [ ipv6-source-address [ mask mask-length ] | ipv6-group-address [ mask mask-length ] | flags { act | del | exprune | ext | loc | niif | nonbr | none | rpt | sg_rcvr | sgjoin | spt | swt | wc | upchg } | fsm | incoming-interface [ interface-type interface-number | register ] | mode { dm | sm | ssm } | outgoing-interface { exclude | include | match } [ interface-type interface-number | none | register ] ] * [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

ipv6-source-address：指定组播源的IPv6地址。

ipv6-group-address：指定组播组的地址。

mask mask-length：指定组播源或组的掩码长度。整数形式，其中组播源的掩码长度的取值范围是0～128；组播组的掩码长度的取值范围是8～128。

flags：指定路由项的分类。

act：显示正在转发组播数据的路由项。

del：显示要删除的路由项。

exprune：显示状态为ex-pruned的路由项，即RPT上该表项被剪枝，并且没有接收者对该组播源感兴趣。

ext：显示只配置了MLD的接口加入组播组相关的路由项。

loc：显示与组播源直连的路由器上的组播路由项。

niff：显示找不到上游接口的路由项。该标识仅用于（*，G），当路由器找不到RP，或没有RP路由，或朝向RP的上游接口没有使能PIM SM的情况下，组播路由项被标志为niff。

nonbr：显示上游邻居映射失败的路由项。

none：显示没有标志的PIM路由项。

rpt：显示RPT树上已经被剪枝的路由项。

sgjoin：显示有（S, G）加入者、且下游接口未使能PIM（仅使能MLD）的路由项。

sg_rcvr：显示有（S, G）加入者、且下游接口使能了PIM的路由项。

spt：显示SPT树上的路由项。

swt：显示正在向SPT树切换的路由项。

wc：显示（*, G）形式的路由项。

upchg：显示上游接口发生了变化的路由表项。

fsm：显示有限状态机的详细信息。

incoming-interface：显示入接口为指定接口的路由项。

interface-type interface-number：指定接口类型和编号。

register：显示注册接口。

mode：指定IPv6 PIM的模式。

dm：显示密集模式路由项。

sm：显示稀疏模式的路由项。

ssm：显示指定源模式的路由项。

outgoing-interface：显示出接口为指定接口的路由项。

exclude：显示下游不包含指定接口的（S, G）路由项。如果包含此关键字，必须指定接口类型和编号。

include：显示下游包含指定接口的（S, G）路由项。如果包含此关键字，必须指定接口类型和编号。

match：显示下游匹配指定接口的（S, G）路由项。如果不指定接口类型和编号，显示下游为空的(S, G)路由项。

none：显示没有下游的路由项。

register：显示注册接口。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

请根据需要选择Flag值查看IPv6 PIM组播路由表。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
