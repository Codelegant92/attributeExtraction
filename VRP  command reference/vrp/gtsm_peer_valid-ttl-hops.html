gtsm peer valid-ttl-hops

命令功能

gtsm peer valid-ttl-hops命令用来在指定的LDP对等体上配置GTSM功能。

undo gtsm { all | peer ip-address }命令用来删除所有或指定的LDP对等体上配置的GTSM功能。

缺省情况下，没有在任何LDP对等体上配置GTSM功能。

命令格式

gtsm peer ip-address valid-ttl-hops hops

undo gtsm { all | peer ip-address }

参数说明

peer ip-address：指定LDP对等体的LSR ID。

all：所有LDP对等体。

valid-ttl-hops hops：设置GTSM功能允许的最大有效跳数。整数形式，取值范围是1～255。

视图

MPLS-LDP视图

缺省级别

2：配置级

使用指南

如果将hops设置为GTSM功能允许的最大有效跳数，当LDP对等体发来报文的TTL值在[255–hops+1, 255]范围内，则接收该报文，否则丢弃该报文。

说明：

建议在LDP对等体两端同时配置GTSM功能。

使用实例

# 在LSR上配置LSR ID为2.2.2.9的对等体发来的LDP报文的有效TTL范围是254～255。

<Quidway> system-view
[Quidway] mpls ldp
[Quidway-mpls-ldp] gtsm peer 2.2.2.9 valid-ttl-hops 2
