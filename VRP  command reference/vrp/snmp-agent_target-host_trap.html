snmp-agent target-host trap

命令功能

snmp-agent target-host trap命令用来设置接收Trap消息的目的地。

undo snmp-agent target-host命令用来取消当前设置。

命令格式

snmp-agent target-host trap address udp-domain ip-address [ udp-port port-number ] [ vpn-instance vpn-instance-name ] params securityname security-string [ v1 | v2c | v3 [ authentication | privacy ] ]

undo snmp-agent target-host ip-address securityname security-string [ vpn-instance vpn-instance-name ]

参数说明

address：指定SNMP消息传输的目标主机地址。

udp-domain：指定目标主机的传输域基于UDP。

ip-address：目标主机的IP地址。

udp-port port-number：指定接收Trap报文的端口号，取值范围0～65535，缺省情况下为162。

vpn-instance vpn-instance-name：VPN实例名，字符串形式，不支持空格，长度范围是1～31。

params：指定产生SNMP消息的日志主机信息。

securityname security-string：指定在网管侧显示的用户名，字符串形式，不支持空格，长度范围1～32。

v1 | v2c | v3：指定Trap消息的版本。缺省情况下为v1。

authentication：指明对报文进行认证但不加密。

privacy：指明对报文进行认证和加密。

使用指南

使用snmp-agent target-host命令可以指定接受trap信息的目的主机。

  • 使用snmp-agent trap enable来设置允许设备发送Trap报文。若要一台主机可以发送通知消息，至少需要在该主机上配置一条snmp-agent target-host命令和snmp-agent trap enable命令。

  • 在SNMP Trap已经使能的情况下，要使SNMPv3正常发送Trap消息，securityname中的用户名是已存在的组当中的用户，并且该组中已经设置了通知视图。

使用实例

# 允许向10.1.1.1发送SNMP Trap报文。

<Quidway> system-view
[Quidway] snmp-agent trap enable standard
[Quidway] snmp-agent target-host trap address udp-domain 10.1.1.1 params securityname comaccess
