ipv6 preference

命令功能

ipv6 preference命令用来配置IS-IS协议生成IPv6路由的优先级。

undo ipv6 preference命令用来恢复IS-IS协议IPv6路由的缺省优先级。

缺省情况下，IS-IS协议的优先级为15。

命令格式

ipv6 preference { route-policy route-policy-name | preference } *

undo ipv6 preference

参数说明

route-policy route-policy-name：指定路由策略名称。

preference：协议优先级。整数类型，取值范围是1～255。数值越低，优先级越高。

使用指南

应用路由策略，可以只对过滤后的路由设置优先级。如果设置了优先级，则只有通过route-policy过滤的路由应用本命令指定的优先级，否则使用缺省值。

在一台路由器上可能同时运行多种动态路由协议，因此存在各个路由协议之间路由信息共享和选择的问题。系统为每一种路由协议设置一个优先级，当不同协议都发现了到同一目的地的路由时，优先级高的协议将起决定作用。

使用此命令前，必须先使能IS-IS进程的IPv6能力。配置过程请参见isis ipv6 enable命令。

使用实例

# 设置IS-IS协议的优先级为20。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] ipv6 preference 20
