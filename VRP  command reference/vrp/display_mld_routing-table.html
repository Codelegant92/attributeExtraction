display mld routing-table

命令功能

display mld routing-table命令用来查看MLD路由表信息。

命令格式

display mld routing-table [ ipv6-source-address [ ipv6-source-mask-length ] | ipv6-group-address [ ipv6-group-mask-length ] | static ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

ipv6-source-address：指定组播源的IPv6地址。

ipv6-source-mask-length：指定组播源IPv6地址的掩码长度。整数形式，取值范围是0～128。

ipv6-group-address：指定组播组地址，显示相应的路由表信息。十六进制形式，取值范围是FF00::～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。

ipv6-group-mask-length：指定IPv6组播组的掩码长度。整数形式，取值范围是8～128。

static：显示接口静态加入组播组的MLD路由表信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

MLD路由表中的(S,G)或(*,G)转发项，即（组播源，组播组）作为表中的独立项，每一项都有一个出接口列表，表示哪些接口需要进行组播数据转发。

使用实例

# 显示MLD路由表中组播组的路由项信息。

<Quidway> display mld routing-table
Routing table
 Total 2 entries
 00001. (2001:AA::123, FF44::)
       List of 1 downstream interface in include mode
        GigabitEthernet4/2/0 (FE80::EE:53),
                   Protocol: STATIC
 00002. (*, FF55::)
       List of 1 downstream interface
        GigabitEthernet4/2/0 (FE80::EE:53),
                   Protocol: MLD
