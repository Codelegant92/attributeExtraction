te-class

命令功能

te-class命令用来配置一个TE-Class。

undo te-class命令用来删除一个TE-Class。

命令格式

te-class0 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class1 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class2 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class3 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class4 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class5 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class6 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

te-class7 { class-type { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } priority priority [ description description-info ] | description description-info }

undo { te-class0 | te-class1 | te-class2 | te-class3 | te-class4 | te-class5 | te-class6 | te-class7 } [ description description-info ]

参数说明

class-type：业务类型。IETF模式的DS-TE支持8种业务类型，分别为CT0～CT7。

ct0～ct7：指定业务类型值。

priority priority：优先级。这里的优先级是指CR-LSP的抢占优先级，而不是MPLS首部的EXP值。抢占优先级的取值范围是0～7。数值越小，优先级越高。
说明：
只有当一条CR-LSP的CT和建立优先级的组合<CT，setup-priority>以及CT和保持优先级的组合<CT，hold-priority>同时存在于TE-Class映射表中，该CR-LSP才能建立成功。例如，假设某节点的TE-Class映射表中仅有TE-Class[0]＝<CT0，6>和TE-Class[1]＝<CT0，7>，则只有以下三类CR-LSP能建立成功（CR-LSP的建立优先级不能高于保持优先级，所以没有setup-priority＝6，hold-priority＝7的组合）：
  • Class-Type＝CT0，setup-priority＝6，hold-priority＝6；
  • Class-Type＝CT0，setup-priority＝7，hold-priority＝6；
  • Class-Type＝CT0，setup-priority＝7，hold-priority＝7。

description-info：对该TE-Class的描述信息。字符串形式，长度范围是1～31。

使用指南

配置该命令前，需要在系统视图下使用te-class-mapping命令创建TE-Class映射表。

配置该命令需了解CT、优先级和LSP抢占的关系：

  • MAM和Extended MAM模型中，高优先级的LSP只能抢占属于相同CT的低优先级LSP的带宽；不同CT的LSP间不发生抢占。
  • RDM模型中，各CT的抢占关系是：如果0≤m<n≤7，0≤i<j≤7，则优先级为m的CTi可以抢占优先级为n的CTi的带宽和优先级为n的CTj的带宽。但是CTi的所有LSP带宽总和<=BCi的带宽值。

当在系统视图下配置undo mpls命令或在MPLS视图下配置undo mpls te命令后，所有配置的TE-Class都会被删除。在系统视图下执行undo te-class-mapping命令删除配置的TE-Class映射表时，也会删除映射表的所有TE-Class的配置。

使用实例

# 配置TE-Class映射表，并配置三个CT，分别为CT0、CT1和CT2；对应的优先级分别为0、1和2。

<Quidway> system-view
[Quidway] te-class-mapping
[Quidway-te-class-mapping] te-class0 class-type ct0 priority 0 description For-EF
[Quidway-te-class-mapping] te-class1 class-type ct1 priority 1 description For-AF
[Quidway-te-class-mapping] te-class2 class-type ct2 priority 2 description For-BE
