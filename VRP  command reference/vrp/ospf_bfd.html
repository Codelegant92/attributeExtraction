ospf bfd

命令功能

ospf bfd enable命令用来在使能OSPF的特定接口下配置BFD特性。建立起BFD会话后，使用默认参数值。

ospf bfd { min-rx-interval receive-interval | min-tx-interval transmit-interval | detect-multiplier multiplier-value } *命令用来配置BFD会话的参数值。

undo ospf bfd enable命令用来取消接口下的BFD特性。

undo ospf bfd { min-rx-interval | min-tx-interval | detect-multiplier } *命令用来恢复BFD会话参数的默认值。

命令格式

ospf bfd enable

ospf bfd { min-rx-interval receive-interval | min-tx-interval transmit-interval | detect-multiplier multiplier-value } *

undo ospf bfd enable

undo ospf bfd { min-rx-interval | min-tx-interval | detect-multiplier } *

参数说明

enable：使能BFD功能。

min-rx-interval receive-interval：指定期望从对端接收BFD报文的最小接收间隔。receive-interval取值范围是100～1000，单位是毫秒。缺省值是100毫秒。

min-tx-interval transmit-interval：指定向对端发送BFD报文的最小发送间隔。transmit-interval取值范围是100～1000，单位是毫秒。缺省值是100毫秒。

detect-multiplier multiplier-value：指定本地检测倍数。multiplier-value取值范围是3～50。缺省值是3。

说明：

  • 使能BFD特性后，OSPF只和邻居状态达到Full的邻居建立起BFD会话。

  • 不同产品，上述最大最小值可能不同。

使用指南

本地的min-rx-interval值与对端的min-tx-interval值进行协商得到receive-interval，如果在receive-interval × multiplier-value时间间隔内没有收到对方发送的BFD报文，就宣告邻居Down掉。

如果没有配置全局BFD，接口上的BFD特性可以配置，但不会创建BFD会话。同样，如果只配置BFD的会话参数，但没有执行ospf bfd enable命令，也不会创建接口的BFD会话。

接口配置的BFD特性优先级高于进程配置的BFD特性优先级。如果打开了接口的BFD开关，则按照接口上BFD参数建立BFD会话。

ospf bfd enable命令和ospf bfd block命令是两条互斥命令，不能同时使能。

说明：

通过undo ospf bfd enable取消接口下的BFD特性后，该接口上关于BFD会话的参数配置依然存在。

使用实例

# 在Pos1/0/0接口上使能BFD特性，并指定最小接收间隔为400ms，本地检测倍数为4。

<Quidway> system-view
[Quidway] interface pos1/0/0
[Quidway-Pos1/0/0] ospf bfd enable
[Quidway-Pos1/0/0] ospf bfd min-rx-interval 400 detect-multiplier 4

# 取消Pos1/0/0接口上的BFD特性。

<Quidway> system-view
[Quidway] interface pos1/0/0
[Quidway-Pos1/0/0] undo ospf bfd enable
