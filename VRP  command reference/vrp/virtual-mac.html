virtual-mac

命令功能

virtual-mac命令用来配置设备的虚拟MAC地址。

undo virtual-mac命令用来删除设备上配置的虚拟MAC地址。

缺省情况下，虚拟MAC地址不存在（默认为全0，无效值）。

命令格式

virtual-mac mac-address

undo virtual-mac

参数说明

mac-address：指定设备的虚拟MAC地址。格式为H-H-H，其中H为1至4位的十六进制数。

使用指南

虚拟MAC地址用于创建MAC Tunnel，与骨干VLAN唯一表述一条隧道。每台VRP设备上只允许创建一个虚拟MAC地址，而且该虚拟MAC必须是单播地址。

如果虚拟MAC地址已经存在，执行命令virtual-mac将修改当前的虚拟MAC地址。

虚拟MAC地址不能与MAC Tunnel上中间设备的桥MAC（即物理MAC）地址相同。

只有配置了虚拟MAC地址后，才能创建MAC Tunnel。要想修改虚拟MAC地址，则必须先删除设备上的所有相关联的MAC Tunnel。

如果遇到提示GTL资源不足时，用户需要释放已经使用的GTL资源或者到公司买新的资源。

使用实例

# 配置设备的虚拟MAC地址为0001-0001-0001。

<Quidway> system-view
[Quidway] virtual-mac 0001-0001-0001
