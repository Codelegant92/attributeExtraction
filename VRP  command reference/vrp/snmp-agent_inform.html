snmp-agent inform

命令功能

snmp-agent inform用于配置全局的Inform方式的相关参数，包括告警确认的超时时间、告警重复发送的次数和待确认告警的最大条数。

undo snmp-agent inform命令用于恢复缺省值。

命令格式

snmp-agent inform { timeout seconds | resend-times times | pending number } *

undo snmp-agent inform { timeout | resend-times | pending } *

参数说明

timeout seconds：Inform方式发送的告警等待网管确认的超时时间。取值范围为1～1800，单位是秒。缺省值为15秒。在不同产品上，本参数的取值范围和缺省值可能存在差异。

resend-times times：Inform方式发送的告警因没有收到网管确认消息而重复发送的次数。取值范围为0～10。缺省值为3次。在不同产品上，本参数的取值范围和缺省值可能存在差异。

pending number：Inform方式的待确认告警最大条数。取值范围为1～2048。缺省值为39条。在不同产品上，本参数的取值范围和缺省值可能存在差异。

使用指南

告警确认的超时时间、告警重复发送的次数、待确认告警的最大条数三个参数相互依赖。例如，告警确认的超时时间或者告警重复发送的次数由小变大，待确认告警的最大条数不变，会导致等待确认的告警增多，待确认告警队列更容易被填充满。

一旦待确认队列满，每添加一条待确认的告警就会删除最早的一条待确认告警。已删除的告警不能再重发。这时，可以增加待确认告警的最大条数，防止待确认告警被删除。

使用实例

# 配置Inform方式告警重复发送次数为5次，待确认告警的最大条数为100条。

<Quidway> system-view
[Quidway] snmp-agent inform resend-times 5 pending 100
