license active

命令功能

license active命令用来激活主备板的License文件。

undo license active命令用来去激活主备板的License文件。

缺省情况下，License文件没有被激活。

命令格式

license active file-name

undo license active

参数说明

file-name：要激活的License文件名。字符串形式，不支持空格，长度范围是1～135。

使用指南

首次激活License文件和升级License文件时，新的License文件必须在主板和备板上同时存在。

license active命令用来激活License文件，有以下两种情况。

  • 首次激活License文件

    当系统中没有激活过License文件时，输入激活License文件的命令，有两种结果：主板和备板验证License文件的结果相同，主板和备板分别进行激活；主板和备板验证License文件的结果不相同，提示用户主板和备板的License文件不一致，不进行License文件的激活。

  • 升级License文件

    系统中存在已经激活的License文件时，要激活一个新的License文件。

    • 主板和备板验证新的License文件结果相同时：

      将新的License文件的配置与当前的配置进行比较。如果低于当前配置则提示用户确认是否继续升级，用户选择否，系统回退到之前的License；用户选择是，主板和备板分别激活该文件，系统升级到新的License文件。

    • 主板和备板验证新的License文件结果不相同时：

      提示用户主板和备板的License文件不一致，回退到之前的License。

注意：

如果激活的License文件中允许配置的VSI实例数小于已配置的VSI实例数，系统会提示用户删除多余的VSI实例。如果用户进行了处理，不再提示；否则，以后每5分钟提示一次用户，直到48小时后系统重启。系统重启后按照配置文件中的顺序恢复VSI实例，超出License文件允许个数的VSI实例被删除。

使用实例

# 激活主备板的License文件xx.dat。

<Quidway> license active xx.dat
Info: Please wait for a while...
Info: Succeeded in activating the License file on the master board.
