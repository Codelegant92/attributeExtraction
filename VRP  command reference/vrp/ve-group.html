ve-group

命令功能

ve-group命令用来配置虚拟以太网接口为多业务接入模式，并将虚拟以太网接口绑定到指定的VE-group。

undo ve-group命令用来配置虚拟以太网接口为普通模式。

默认情况下，虚拟以太网接口工作在普通模式下。

命令格式

ve-group ve-group-id { l2-terminate | l3-access }

参数说明

ve-group ve-group-id：VE-group组号。整数形式，取值范围是1～8192。

l2-terminate：指定虚拟以太网接口为用来终结VLL的二层虚拟以太网接口（L2VE）。

l3-access：指定虚拟以太网接口为用来接入MPLS L3VPN的三层虚拟以太网接口（L3VE）。

视图

虚拟以太网接口视图

缺省级别

2：配置级

使用指南

在使用命令ve-group将VE接口配置成多业务接入模式前，如果这个VE接口已经与承载IPoEoA的PVC（Permanent Virtual Channel）建立联系，则必须先解除绑定关系；如果接口下已经存在其它配置，则必须先删除其它配置，否则系统会提示用户配置错误。

在VRP中，每块单板上只能建立2048个VE-group，系统中一共能够建立8192个。

在虚拟以太网接口下，配置ve-group ve-group-id l2-terminate命令则此接口变为L2VE。

同一VE-Group中的两个VE接口不能跨板。

使用实例

# 配置接口VE1/0/0为L2VE。

<Quidway> system-view
[Quidway] interface virtual-ethernet 1/0/0
[Quidway-Virtual-Ethernet1/0/0] ve-group 1 l2-terminate
