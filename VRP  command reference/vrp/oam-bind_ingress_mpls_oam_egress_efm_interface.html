oam-bind ingress mpls oam egress efm interface

命令功能

oam-bind ingress mpls oam egress efm interface命令配置MPLS OAM向EFM OAM传递故障。

undo oam-bind ingress mpls oam egress efm interface命令取消MPLS OAM向EFM OAM传递故障。

命令格式

oam-bind ingress mpls oam { lsp-name lsp-name | lsr-id ingress-lsr-id tunnel-id tunnel-id } egress efm interface interface-type interface-number

undo oam-bind ingress mpls oam { lsp-name lsp-name | lsr-id ingress-lsr-id tunnel-id tunnel-id } egress efm interface interface-type interface-number

参数说明

lsp-name：LSP名称，字符串形式，长度范围是1～15。

lsr-id：绑定被检测LSP的隧道出节点的IP地址。

tunnel-id：绑定的被检测LSP的隧道ID，取值范围是1～65534。

interface-type interface-number：接口类型和编号，取值包括：ethernet和gigabitethernet。该接口必须已经使能EFM OAM功能。

使用指南

配置LSP出节点与EFM OAM绑定，OAM管理模块通过接口向EFM OAM传递故障。

配置该命令时，相关的MPLS隧道必需已经创建，指定的接口也已使能EFM功能。

使用实例

#配置MPLS OAM向EFM OAM传递故障。

<Quidway> system-view
[Quidway] oam-mgr
[Quidway-oam-mgr] oam-bind ingress mpls oam lsp-name atob egress efm interface gigabitethernet 3/0/1
