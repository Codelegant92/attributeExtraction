mapping-type

命令功能

mapping-type命令用来配置业务实例对用户业务的封装映射模式。

undo mapping-type命令用来恢复业务实例与用户业务的封装映射模式为缺省模式。

缺省情况下，映射模式为s-tagged bundling。

命令格式

mapping-type { s-tagged { bundling | one-to-one } | transparent }

undo mapping-type

参数说明

s-tagged：指定映射模式为s-tagged模式。s-tagged模式下，业务实例区分用户业务的VLAN Tag，业务实例可以与一个业务VLAN或多个业务VLAN绑定。

s-tagged one-to-one：指定在s-tagged模式下，一个业务实例只能与一个业务VLAN绑定。

s-tagged bundling：指定在s-tagged模式下，一个业务实例可以与多个业务VLAN绑定。

transparent：指定映射模式为透明传输模式，即整个入接口作为一个用户业务映射到业务实例，不区分用户报文的VLAN Tag。

使用指南

业务实例与用户业务映射模式可以针对不同的业务传输需求灵活配置。

  • 在s-tagged one-to-one模式下，VRP设备会自动把用户报文中的S-TAG剥离，然后再封装I-TAG字段。出隧道的时候将根据封装的I-TAG还原S-TAG（包括优先级）。

  • 在s-tagged bundling模式下，由于多个业务VLAN会映射到同一个业务实例，因此S-TAG不会被剥离，否则无法恢复。该模式下，要求MAC Tunnel两端设备配置的VLAN一致。

  • 在transparent模式下，业务实例只根据接口来选择用户，不判断和处理用户报文中的VLAN信息。接口的所有用户业务将经过MAC Tunnel透传到远端的目的设备。

多点到多点业务实例与用户业务的映射类型不允许配置为transparent模式。

如果业务实例已经与用户业务绑定，则不允许再修改业务实例的映射模式。

使用实例

# 配置业务实例的封装映射模式为s-tagged bundling模式。

<Quidway> system-view
[Quidway] service-instance instance-name mysi
[Quidway-service-instance-mysi] mapping-type s-tagged bundling
