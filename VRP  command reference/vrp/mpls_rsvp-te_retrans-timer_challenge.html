mpls rsvp-te retrans-timer challenge

命令功能

mpls rsvp-te retrans-timer challenge命令用来配置challenge消息重传间隔。

undo mpls rsvp-te retrans-timer challenge命令用来将challenge消息重传时间间隔恢复为缺省值。

缺省情况下，challenge消息重传间隔为1000毫秒。

命令格式

mpls rsvp-te retrans-timer challenge retransmission-interval

undo mpls rsvp-te retrans-timer challenge

参数说明

retransmission-interval：challenge消息重传间隔，整数形式，单位是毫秒，取值范围是500～10000，缺省值为1000。

使用指南

假设节点A和邻居节点B之间需要进行RSVP认证，则A对B的认证过程如下：

  • A发送带INTEGRITY对象的RSVP消息给B；
  • B收到该消息时，如果发现本地的对A认证的KeyID与INTEGRITY中的KeyID不同，则向A发出Challenge消息，同时开启Challenge消息超时计时器，定期重传Challenge消息并记录发送次数；
  • A收到B的Challenge消息，直接复制其中的CHALLENGE对象，并加上自己的INTEGRITY对象，生成Response消息发给B；
  • B收到A的Response消息，对比消息中的CHALLENGE对象与本地是否一致：
    • 如果一致，则停止Challenge消息超时计时器并将计数清零，认证成功；
    • 如果计数达到设置的最大值时B仍未收到Response消息，则认证失败。

mpls rsvp-te challenge-lost命令可用来修改上述Challenge消息超时计时器的最大计数值。Challenge消息的重传时间间隔则可通过mpls rsvp-te retrans-timer challenge命令修改。

使用实例

# 配置challenge消息重传间隔为800ms。

<Quidway> system-view
[Quidway] mpls
[Quidway-mpls] mpls rsvp-te retrans-timer challenge 800
