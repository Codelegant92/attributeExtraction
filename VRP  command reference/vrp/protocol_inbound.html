protocol inbound

命令功能

protocol inbound命令用来指定所在的用户界面所支持的协议。

缺省情况下，系统支持协议Telnet。

命令格式

protocol inbound { all | ssh | telnet }

参数说明

all：支持所有的协议，包括Telnet和SSH。

ssh：只支持SSH协议。

telnet：只支持Telnet协议。

视图

用户界面视图

缺省级别

3：管理级

使用指南

如果使用该命令配置某个或者某几个用户界面支持的协议为all或者ssh，而其他用户界面未配置（默认采用Telnet协议）。此时，如果有用户登录，将优先采用Telnet协议的用户界面，并采用Telnet默认的password验证方式，如果没有配置密码，那么用户将无法登陆路由器。因此，为确保登录成功，需要为采用Telnet协议的用户界面配置验证方式（和登录密码）。

如果使用该命令配置某用户界面支持的协议是SSH，为确保登录成功，则必须配置相应的验证方式为authentication-mode aaa；若配置验证方式为authentication-mode password和authentication-mode none，则protocol inbound ssh配置结果将失败。

说明：

当使用该命令指定所在用户界面支持的协议为SSH时，如果SSH使能但本机RSA密钥没有配置，则SSH仍然不可用。配置结果待下次登录请求时生效。

使用实例

# 配置VTY0到VTY4的用户界面仅支持SSH协议。

<Quidway> system-view
[Quidway] user-interface vty 0 4
[Quidway-ui-vty0-4] protocol inbound ssh
