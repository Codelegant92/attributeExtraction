peer default-route-advertise (BGP)

命令功能

peer default-route-advertise命令用来设置给对等体/对等体组发布缺省路由。

undo peer default-route-advertise命令用来取消已有的设置。

缺省情况下，不向对等体/对等体组发布缺省路由。

命令格式

peer { group-name | ipv4-address | ipv6-address } default-route-advertise [ route-policy route-policy-name ] [ conditional-route-match-all { ipv4-address1 { mask1 | mask-length1 } } &<1-4> | conditional-route-match-any { ipv4-address2 { mask2 | mask-length2 } } &<1-4> ]

undo peer { group-name | ipv4-address | ipv6-address } default-route-advertise

参数说明

group-name：对等体组的名称。

ipv4-address：对等体的IPv4地址。

Ipv6-address：对等体的IPv6地址。

route-policy route-policy-name：指定Route-Policy名称。

conditional-route-match-all ipv4-address1 { mask1 | mask-length1 }：指定条件路由的IPv4地址，以及掩码/掩码长度。匹配所有条件路由则发送缺省路由。

conditional-route-match-any ipv4-address2 { mask2 | mask-length2 }：指定条件路由的IPv4地址，以及掩码/掩码长度。匹配任一条件路由则发送缺省路由。

说明：

  • ipv4-address参数仅在BGP视图、BGP-IPv4单播地址族视图、BGP-IPv6单播地址族视图和BGP-VPN实例视图下生效。

  • ipv6-address参数仅在BGP-IPv6单播地址族视图和BGP-VPN6实例视图下生效。

  • conditional-route-match-all和conditional-route-match-any关键字仅在BGP视图、BGP-IPv4单播地址族视图和BGP-VPN实例视图下生效。

缺省级别

2：配置级

使用指南

该命令不需要在路由表中存在缺省路由，而是无条件地向对等体发送一个下一跳为自身的缺省路由。

如果需要配置6PE，请在IPv6单播地址族视图下指定Peer为IPv4单播地址。

使用实例

# 设置给对等体发布缺省路由。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] peer 1.1.1.2 as-number 200
[Quidway-bgp] ipv4-family unicast
[Quidway-bgp-af-ipv4] peer 1.1.1.2 default-route-advertise

# 设置给对等体发布缺省路由。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] ipv6-family unicast
[Quidway-bgp] peer 1:2::3:4 as-number 200
[Quidway-bgp-af-ipv6] peer 1:2::3:4 default-route-advertise
