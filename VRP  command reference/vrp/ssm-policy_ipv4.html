ssm-policy（IPv4）

命令功能

ssm-policy命令用来配置SSM组播组地址范围。

undo ssm-policy命令用来恢复缺省配置。

缺省情况下，SSM组地址范围是232.0.0.0/8。

命令格式

ssm-policy basic-acl-number

undo ssm-policy

参数说明

basic-acl-number：表示基本访问控制列表号，定义组播组地址范围。整数形式，取值范围是2000～2999。

视图

公网实例PIM视图、VPN实例PIM视图

缺省级别

2：配置级

使用指南

利用ssm-policy命令可以指定PIM SSM组地址的地址范围，所有使能PIM-SM协议的接口将会认为属于该范围内的组播组采用了PIM SSM模式。

SSM组地址范围可以超出232.0.0.0/8。SSM组地址范围之外的组播组一律采用ASM模式。

启用SSM模式的条件如下：

  • 组播组地址属于SSM组地址范围，用户主机网段运行IGMPv3，Report消息中指定了组播源地址，则启动SSM模式。

  • 组播组地址属于SSM组地址范围，用户主机网段运行IGMPv1或IGMPv2，且路由器上配置了SSM Mapping，则启动SSM模式。

ssm-policy命令与acl命令配合使用。在ACL视图下，通过rule命令的source参数定义SSM组播组地址范围。

使用实例

# 在公网实例中，配置PIM SSM组播地址范围是232.1.0.0/16。

<Quidway> system-view
[Quidway] acl number 2000
[Quidway-acl-basic-2000] rule permit source 232.1.0.0 0.0.255.255
[Quidway-acl-basic-2000] quit
[Quidway] pim
[Quidway-pim] ssm-policy 2000
