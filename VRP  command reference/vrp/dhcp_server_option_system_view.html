dhcp server option（系统视图）

命令功能

dhcp server option命令用来配置指定范围内的接口的DHCP自定义选项。

undo dhcp server option命令用来删除配置。

命令格式

dhcp server option code { ascii ascii-string | hex hex-string | ip-address ip-address &<1-8> } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

undo dhcp server option code { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

参数说明

code：指定用户定义的选项Option的数值。整数形式，取值范围是2～254。

ascii ascii-string：ASCII字符串。ascii-string是字符串形式，长度范围是1～63。

hex hex-string：2位或4位长度的16进制数字串（如hh或hhhh）。

ip-address ip-address &<1-8>：执行指定option功能的IP地址，可以输入1～8个。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：指定所有接口地址池的自定义选项。

使用指南

dhcp server option命令实现配置多个接口DHCP自定义选项。

该命令配置后，不能通过display current-configuration命令查看配置情况，它的执行是通过循环调用dhcp server option（接口视图）命令实现的。

说明：

DHCP自定义选项是可选的，只在需要时才配置。

使用实例

# 为GE 2/0/0.1到GE 2/0/0.5之间的接口DHCP地址池定义code为100的16进制数字为0x11和0x22。

<Quidway> system-view
[Quidway] dhcp server option 100 hex 11 22 interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
