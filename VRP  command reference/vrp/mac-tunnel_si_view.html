mac-tunnel（业务实例视图）

命令功能

mac-tunnel命令用来配置业务实例与MAC Tunnel的绑定。

undo mac-tunnel命令用来解除业务实例与MAC Tunnel的绑定关系。

缺省情况下，业务实例没有与MAC Tunnel绑定。

命令格式

mac-tunnel tunnel-name &<1-6>

undo mac-tunnel tunnel-name &<1-6>

参数说明

tunnel-name：指定MAC Tunnel名称。长度范围是1～31个字符。

使用指南

MAC Tunnel是业务实例传输数据的载体，业务实例与具体的用户业务映射后，需要与MAC Tunnel绑定后才能传输用户业务。

在业务实例与隧道绑定前，必须已经配置业务实例的ID，否则无法与隧道绑定。而且要绑定的目标隧道也必须配置完整。

对于点到点的业务实例，只能与一条隧道绑定，但是可以与其他多个业务实例共享相同的隧道；如果业务实例已经与隧道绑定，则新的隧道绑定配置将覆盖老的配置。

多点到多点的业务实例可以与多条隧道进行绑定，也可以与其他多个业务实例共享相同的隧道，但是要求多条隧道的远端虚拟MAC地址不能相同。

如果隧道配置了保护组，则业务实例绑定到工作隧道之后，保护隧道也自动绑定了该业务实例。不允许直接将保护隧道绑定到业务实例。

对于已经和隧道绑定的业务实例，如果再次绑定隧道，则有两种情况：

  • p2p类型的业务实例，直接覆盖上一条隧道。

  • mp2mp类型的业务实例，增加绑定一条隧道。

使用实例

# 配置业务实例mysi与名为Mytunnel的MAC Tunnel绑定。

<Quidway> system-view
[Quidway] service-instance instance-name mysi
[Quidway-service-instance-mysi] mac-tunnel Mytunnel
