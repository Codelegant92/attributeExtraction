dhcp server expired（系统视图）

命令功能

dhcp server expired命令用来配置指定范围的接口DHCP地址池中IP地址的租用有效期限。

undo dhcp server expired命令用来恢复缺省设置。

缺省情况下，租用有效期限为1天。

命令格式

dhcp server expired { day day [ hour hour [ minute minute ] ] | unlimited } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

undo dhcp server expired { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

参数说明

day day：指定租用有效期的天数。整数形式，取值范围是0～365。缺省值是1。

hour hour：指定租用有效期的小时数。整数形式，取值范围是0～23。缺省值是0。

minute minute：指定租用有效期的分钟数。整数形式，取值范围是0～59。缺省值是0。

unlimited：有效期限为无限长。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：所有接口。

使用指南

根据不同的主机地址使用的不同，可以使用不同的租期分配方式，实现一次配置给多接口地址池分配租期。本命令配置后，不能使用display current-configuration命令查看，实际执行时，通过循环调用dhcp server expired（接口视图）命令实现批量配置。

使用实例

# 配置GE2/0/0.1到GE2/0/0.5之间的接口地址池的IP地址租用有效期无限长。

<Quidway> system-view
[Quidway] dhcp server expired unlimited interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
