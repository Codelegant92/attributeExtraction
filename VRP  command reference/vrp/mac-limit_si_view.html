mac-limit（业务实例视图）

命令功能

mac-limit命令用来配置当前业务实例上的MAC地址学习限制规则。

undo mac-limit命令用来关闭当前业务实例的MAC地址学习限制。

缺省情况下，业务实例的MAC地址学习限制功能处于关闭状态。

命令格式

mac-limit { action { discard | forward } | maximum max-number } *

undo mac-limit

参数说明

action：指定MAC地址表项数目达到限制值后，系统对报文采取的动作。

discard：指定MAC地址表项数目达到限制值后，如果报文的目的MAC地址在MAC地址表中没有对应表项，则该报文将被丢弃。

forward：指定MAC地址表项数目达到限制值后，即使报文的目的MAC地址在MAC地址表中没有对应表项，系统将在所有相连的隧道内广播该报文。

maximum max-number：指定当前业务实例可以学习的MAC地址表项数量的最大值。整数形式，取值范围是0～131072。缺省情况下，MAC地址表项的最大值为0，即不限制表项数量。

使用指南

基于业务实例的MAC地址学习功能是多点到多点业务实例选择MAC Tunnel路径的基础。MAC地址学习表项的数量是有限的，因此需要配置MAC地址学习限制规则。

基于业务实例的MAC地址学习限制功能只对多点到多点的业务实例有效，对点到点的业务实例无效。

如果该业务实例已经学习了一些MAC地址，请使用undo mac-address dynamic命令清除已经学习的MAC地址，否则会使MAC地址学习限制功能的数量限制不准确。

只有配置了当前服务实例上最多可以学习的MAC数量后，采取动作的配置才会生效。

使用实例

# 配置业务实例的MAC学习限制，当MAC地址表数目达到限制后，直接丢弃带有新MAC地址的报文。

<Quidway> system-view
[Quidway] service-instance instance-name mysi
[Quidway-service-instance-mysi] mac-limit action discard
