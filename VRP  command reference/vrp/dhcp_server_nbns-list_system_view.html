dhcp server nbns-list（系统视图）

命令功能

dhcp server nbns-list命令用来配置指定范围的接口的地址池的DHCP客户端的NetBIOS服务器地址。

undo dhcp server nbns-list命令用来删除配置。

缺省情况下，未配置NetBIOS地址。

命令格式

dhcp server nbns-list ip-address &<1-8> { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

undo dhcp server nbns-list { ip-address | all } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

参数说明

ip-address：指定NetBIOS服务器的IP地址，一条命令中最多可以配置8个IP地址，用空格分隔。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：undo命令中的前一个all表示所有NetBIOS服务器的IP地址；后一个all表示所有接口。

使用指南

使用dhcp server nbns-list命令可以一次给多个接口DHCP客户端指定NetBIOS服务器地址。每个DHCP地址池最多可以配置8个NetBIOS服务器IP地址。

本命令配置后，不能通过display current-configuration查看配置，该命令的执行是通过循环调用dhcp server nbns-list（接口视图）实现的。

使用实例

# 配置GE 2/0/0.1到GE 2/0/0.5之间的接口地址池为其客户端分配的NetBIOS服务器地址为10.12.1.99。

<Quidway> system-view
[Quidway] dhcp server nbns-list 10.12.1.99 interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
