mpls oam egress

命令功能

mpls oam egress命令用来在LSP的出节点配置MPLS OAM参数。

undo mpls oam egress命令用来在LSP出节点取消指定的MPLS OAM参数。

缺省情况下，出节点使用OAM自动协议扩展功能，反向通道采用共享模式，反向BDI的发送频率是OAM检测报文的发送频率。

命令格式

mpls oam egress { lsp-name lsp-name | lsr-id ingress-lsr-id tunnel-id tunnel-id } [ auto-protocol [ overtime over-time ] | type { cv | ffd frequency ffd-fre } ] [ backward-lsp tunnel tunnel-number [ private | share ] [ bdi-frequency { detect-freq | per-second } ] ]

undo mpls oam egress { all | lsp-name lsp-name | lsr-id ingress-lsr-id tunnel-id tunnel-id }

参数说明

lsp-name lsp-name：静态LSP的名称。字符串形式，不支持空格，长度范围是1～19。

lsr-id ingress-lsr-id：绑定被检测LSP的隧道入节点的LSR ID。

tunnel-id tunnel-id：绑定的被检测LSP的隧道ID，整数形式，取值范围取决于产品的许可证文件。

auto-protocol：使能OAM自动协议扩展功能。

overtime over-time：首包超时等待时间，是指当配置支持自动协议时，OAM配置生效后等待首个检测报文的时间。如果超过这个时间没有接收到检测报文，认为发生缺陷，触发BDI报文向入节点告警。首包超时等待时间单位为30秒，取值范围是0～255，缺省值为10，即5分钟。

type：设置OAM检测报文的类型。

cv：设置OAM检测报文的类型是CV报文。

ffd frequency ffd-freffd-fre：设置OAM检测报文的类型是FFD报文，并设置FFD报文的发送频率。发送频率的可选值为100、200、500，单位是毫秒。其值根据产品而略有不同。

backward-lsp tunnel tunnel-number：绑定的反向LSP的隧道接口编号。

private：独占反向通道。用户一旦指定反向LSP为“独占”方式后，该LSP就不能再被其他LSP绑定为反向通道。

share：共享反向通道，多条前向LSP可以共用一条反向通道。

bdi-frequency：反向通告报文BDI的发送频率。

detect-freq：反向BDI的发送频率按OAM检测报文的发送频率。

per-second：反向BDI的发送频率为每秒一发。

all：删除所有出节点OAM配置。

使用指南

在配制MPLS OAM参数前，必须先执行mpls oam命令全局使能MPLS OAM功能。

如果执行static-lsp egress或static-cr-lsp egress命令创建前向LSP时，指定了参数lsrid ingress-lsr-id和tunnel-id tunnel-id，则在此步骤中可以选用参数lsr-id ingress-lsr-id tunnel-id tunnel-id，否则只能使用参数lsp-name lsp-name。

如果用户配置出节点的MPLS OAM检测报文类型，则自动协议功能失效。

如果反向通道是静态LSP或者静态CR-LSP，则反向通道必须采用独占方式。

只有自动协议扩展可以自动使能，其他OAM参数配置后均需要手工启动。

使用实例

# 某LSP为RSVP-TE建立的LSP，LSP入节点的LSR ID为1.1.1.1，Tunnel ID为100，在出节点配置MPLS OAM协议参数：启用自动协议，反向LSP为独占方式，BDI报文发送频率继承检测报文发送频率。

<Quidway> system-view
[Quidway] mpls oam egress lsr-id 1.1.1.1 tunnel-id 100 auto-protocol backward-lsp tunnel 3/0/0 private bdi-frequency detect-freq

# 修改该LSP的MPLS OAM检测报文类型为CV，修改后，自动协议功能失效。

[Quidway] mpls oam egress lsr-id 1.1.1.1 tunnel-id 100 type cv
