vpn-target（VSI-BGP视图）

命令功能

vpn-target命令用来将当前VSI实例与一个或多个VPN-Target进行关联。

undo vpn-target命令用于删除当前VSI实例关联的VPN-Target。

缺省情况下，VSI实例没有与任何VPN-Target进行关联，必须在创建VSI实例时配置。

命令格式

vpn-target vpn-target &<1-16> [ both | export-extcommunity | import-extcommunity ]

undo vpn-target { all | vpn-target &<1-16> } [ both | export-extcommunity | import-extcommunity ]

参数说明

vpn-target：添加VPN-Target扩展团体属性到VSI实例。用户可以使用以下两种格式之一来表示VPN-Target值。

  • 16位自治系统号:32位用户自定义数，例如：1:3。自治系统号的取值范围是0～65535；用户自定义数的取值范围是0～4294967295。其中，自治系统号和用户自定义数不能同时为0，即VPN Target的值不能是0:0。

  • 32位IP地址:16位用户自定义数，例如：192.168.122.15:1。IP地址的取值范围是0.0.0.0～255.255.255.255；用户自定义数的取值范围是0～65535。

export-extcommunity：定义出方向路由信息携带的团体属性值。

import-extcommunity：定义可以接收带有指定扩展团体属性值的路由信息。

both：从目的VPN扩展团体来的入方向路由信息和到目的VPN扩展团体的出方向路由信息。

all：删除所有的VPN-Target属性。

缺省级别

2：配置级

使用指南

当PE根据VSI实例向其他PE发送路由时，它将向发送的路由附加Export VPN-Target。当PE从其他PE接收路由时，它根据Import VPN-Target决定这些路由是否可以被添加到相应的VSI实例中。因此，通过VPN-Target可以控制节点之间的路由发布。

使用实例

# 将当前VSI实例与VPN-Target关联。

<Quidway> system-view
[Quidway] vsi company2
[Quidway-vsi-company2] pwsignal bgp
[Quidway-vsi-company2-bgp] vpn-target 3:3 export-extcommunity
[Quidway-vsi-company2-bgp] vpn-target 4:4 import-extcommunity
[Quidway-vsi-company2-bgp] vpn-target 5:5 both
