ripng max-packet-length

命令功能

ripng max-packet-length命令用来设置接口接收或发送RIPng报文的最大长度。

undo ripng max-packet-length命令用来恢复RIPng报文最大长度为缺省值。

缺省情况下，接口接收和发送RIPng报文的最大长度为512字节。

命令格式

ripng max-packet-length { mtu | value }

undo ripng max-packet-length

参数说明

mtu：指定RIPng报文最大长度为接口的MTU值。

value：设置RIPng报文的最大长度。整数形式，取值范围为72~65535，单位是字节。缺省值为512字节。

使用指南

配置本命令之前，需要在全局和接口下都使能IPv6。

配置本命令时，需要配置两端的最大报文长度值相等，且不允许配置大于接口MTU的值。

说明：

本命令可增加RIPng报文的长度，所以请谨慎使用，建议只有在对端能够接收大于512字节的RIPng报文时配置本命令。

使用实例

# 设置RIPng报文长度为接口的MTU值。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ripng max-packet-length mtu

# 恢复RIPng报文长度为缺省值。

[Quidway-GigabitEthernet1/0/0] undo ripng max-packet-length
