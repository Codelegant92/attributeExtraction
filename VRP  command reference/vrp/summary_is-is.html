summary (IS-IS)

命令功能

summary命令用来设置IS-IS生成聚合路由。

undo summary命令用来取消聚合。

缺省情况下，系统不做路由聚合。

命令格式

summary ip-address mask [ avoid-feedback | generate_null0_route | tag tag | [ level-1 | level-1-2 | level-2 ] ] *

undo summary ip-address mask [ level-1 | level-1-2 | level-2 ]

参数说明

ip-address：需要生成聚合路由的地址范围，点分十进制形式。

mask：掩码，点分十进制形式。

avoid-feedback：避免通过路由计算学习到聚合路由。

generate_null0_route：为防止路由循环而生成Null 0路由。

tag tag：管理标签号。整数形式，取值范围是1～4294967295。

level-1：只对引入到Level-1区域的路由进行聚合。

level-1-2：对向Level-1区域以及骨干网络引入的路由都进行聚合。

level-2：只对引入骨干网络的路由进行聚合。

说明：

如果命令中没有指定Level，缺省为Level-2。

视图

IS-IS视图

缺省级别

2：配置级

使用指南

可以将有相同前缀的路由聚合为一条路由，这样一方面减小路由表规模，另外可以减少本路由器生成的LSP报文大小和LSDB的规模。其中，被聚合的路由可以是IS-IS协议发现的路由，也可以是被引入的路由。另外，聚合后路由的开销值取所有被聚合路由中学到的路由的最小开销值。

使用实例

# 设置一条202.0.0.0/8的聚合路由。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] summary 202.0.0.0 255.0.0.0
