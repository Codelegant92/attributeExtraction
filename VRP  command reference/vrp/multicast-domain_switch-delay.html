multicast-domain switch-delay

命令功能

multicast-domain switch-delay命令用来指定VPN实例从Share-MDT向Switch-MDT切换的延迟时间。

undo multicast-domain switch-delay命令用来恢复参数缺省值。

缺省值是5秒，即源PE发出切换信令后延迟5秒向Switch-MDT切换。

命令格式

multicast-domain switch-delay switch-delay

undo multicast-domain switch-delay

参数说明

switch-delay：VPN实例向Switch-MDT切换的延迟时间。整数形式，取值范围是3～60，单位是秒。

使用指南

如果该VPN中的Share-MDT组没有配置，不允许配置本命令。

在相同VPN实例视图下重复配置该命令，新配置覆盖旧配置。

有下游PE加入切换组时，公网切换组转发树的建立需要时间，所以配置该切换延迟。上游PE先发送切换消息，延迟时间过后系统才使用该切换组封装数据转发，即把数据由Share-MDT切换到Switch-MDT。在switch-delay定时器超时前，系统将继续检测数据转发速率。如果速率始终高于切换阈值则切换至Switch-MDT，否则继续使用Share-MDT进行报文转发。

在实际的切换中，从流量变化到设备感知存在1～30秒的额外时间滞后（即：从数据转发速率发生变化，到系统启动switch-delay计时器，存在1～30秒的延时），而这种时间滞后是不计入switch-delay中的。因此，从数据转发速率发生变化，到系统完成切换，总的延迟时间将较switch-delay长1～30秒。

使用实例

# 配置VPN实例mytest向Switch-MDT切换的延迟时间为20秒。

<Quidway> system-view
[Quidway] ip vpn-instance mytest
[Quidway-vpn-instance-mytest] multicast-domain switch-delay 20
