display multicast ipv6 forwarding-table

命令功能

display multicast ipv6 forwarding-table命令用来查看IPv6组播转发表信息。

命令格式

display multicast ipv6 forwarding-table [ ipv6-source-address [ ipv6-source-mask-length ] | ipv6-group-address [ ipv6-group-mask-length ] | incoming-interface { interface-type interface-number | register } | outgoing-interface { { exclude | include | match } { interface-type interface-number | register | none } } | slot slot-number | statistics ] *

参数说明

ipv6-source-address：指定组播源的IPv6地址。

ipv6-source-mask-length：指定组播源地址的掩码长度。取值范围是0～128。

ipv6-group-address：指定组播组的IPv6地址。取值范围是FF00::0～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。

ipv6-group-mask-length：指定组播组地址掩码长度。取值范围是8～128。

interface-type interface-number：指定接口的类型和编号。

incoming-interface：指定IPv6组播转发项的入接口。

register：表示PIM-SM的注册接口。

outgoing-interface：指定IPv6组播转发项的出接口。

exclude：显示下游列表中不包含指定接口的路由转发项。

include：显示下游列表中包含指定接口的路由转发项。

match：显示下游列表匹配指定接口的路由转发项。下游接口列表中应只有一项，且该项符合匹配条件。配置了match参数后，若不指定出接口就显示下游接口列表为空的路由转发项。

none：显示没有下游的（S, G）转发项。

slot slot-number：指定接口板所在的槽位号。此参数的支持与否依赖于各产品上接口板的具体实现。缺省显示的是主控板上的信息。

statistics：显示组播转发表中的统计信息。

使用指南

如果不指定槽位号，则显示主控板的信息。

显示信息包括（S, G）转发项的数目、不同源地址的数目、不同组地址的数目、下游注册的（S, G）转发项数目、组播转发表中出接口列表为空的（S, G）转发项的数目。

项目

描述

IPv6 Multicast Forwarding Table

IPv6组播转发表。

Total 2 entries, 2 matched

转发项总数和符合查询条件的转发表数量。

00001

(S,G)项的序号。

(2001:AA::123, FF33::)

组播转发表的(S,G)项。

MID

组播转发项在MFIB表中的唯一标识，用于快速检索组播转发表。

Flags

（S，G）转发表项的状态标志。

Uptime

（S，G）项已存在时间。

Timeout in

（S，G）项还有多少时间超时。

Incoming interface

（S，G）项的入接口。

List of 1 outgoing interfaces: 1: LoopBack0

出接口列表，包括出接口序号和出接口名。

Matched 0 packets(0 bytes)

与该转发项匹配的组播报文个数和字节数。

Wrong If 0 packets

入接口错误的报文个数。

Forwarded 0 packets(0 bytes)

根据该转发项转发的报文个数和字节数。
