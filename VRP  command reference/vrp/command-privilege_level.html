command-privilege level

命令功能

command-privilege level命令用来设置指定视图内的命令的级别。

undo command-privilege level命令用来取消当前设置。

缺省情况下，ping、tracert、telnet等为参观级（0级）；display为监控级（1级）；大部分的配置命令为配置级；用户密钥设置、FTP、XModem、TFTP以及文件系统操作的命令为管理级。

命令格式

command-privilege level level view view-name command-key

undo command-privilege view view-name command-key

参数说明

level level：指定命令的优先级。整数形式，取值范围是0～15。

view view-name：指定视图名称。可在终端界面上所列举的视图名称中选择。

command-key：需要设置的命令。字符串形式，可以同时指定多个命令。

使用指南

在早期版本中，命令级别共分为参观、监控、配置、管理4个级别，标识为0～3。其中，ping、tracert、telnet等为参观级（0级）；display为监控级（1级）；大部分的配置命令为配置级；用户密钥设置、FTP、XModem、TFTP以及文件系统操作的命令为管理级。在使用command-privilege level rearrange命令批量提升命令级别后，原2级命令提升为10级，原3级命令提升为15级。

但这种级别的粒度还比较粗，如果需要实现权限的精细管理，使得某级别的用户只能访问同等级别以及同等级别以下的命令，则执行该命令重新配置指定命令的级别。

重新配置指定命令的级别后，管理员可以根据用户需要为其分配权限。用户登录时，可以根据用户名对应的权限设置（通过命令user-privilege配置）对命令进行操作；也可以根据用户界面（user-interface）自身的权限设置（通过命令local-user level配置）进行操作。如果两者权限发生冲突时，取用户名对应的权限。

注意：

不要随意改动命令的级别。

使用实例

# 设置“interface”命令的优先级为0。

<Quidway> system-view
[Quidway] command-privilege level 0 view system interface
