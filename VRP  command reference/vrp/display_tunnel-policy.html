display tunnel-policy

命令功能

display tunnel-policy命令用来查看隧道策略的配置情况。

命令格式

display tunnel-policy [ tunnel-policy-name ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

tunnel-policy-name：隧道策略名。如果指定tunnel-policy-name，则显示指定名称的隧道策略信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

配置完隧道策略后，可以用display tunnel-policy命令确认配置结果是否正确。在应用隧道策略前，可以用display tunnel-policy命令查看该隧道策略的相关信息。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示所有的隧道策略信息。

<Quidway> display tunnel-policy
Total   tunnel policy num:              2
Sel-Seq tunnel policy num:              1
Binding tunnel policy num:              1
Invalid tunnel policy num:              0

Tunnel Policy Name  Select-Seq                        Load balance No
---------------------------------------------------------------------
po2                  CR-LSP LSP                       2

Tunnel Policy Name  Destination          Tunnel Intf        Down switch
------------------------------------------------------------------------
po1                 1.1.1.1              Tunnel1/0/2        Disable
表1 display tunnel-policy命令输出信息描述

项目

描述

Total tunnel policy num

隧道策略总个数。

Select-Seq tunnel policy num

隧道优先级策略个数。

Binding tunnel policy num

绑定策略个数。

Invalid tunnel policy num

无效策略个数。

Tunnel Policy Name

隧道策略的名称。

Select-Seq

隧道类型的优先级顺序，放在前面的类型的隧道的优先级比放在后面的高。

Load balance No

允许进行负载分担的隧道条数，缺省值为1。

Destination

被绑定的隧道的目的地址，即对端实际接收报文的接口地址。

Tunnel Intf

被绑定的隧道的本端Tunnel接口。

Down Switch
隧道切换功能状态。
  • Enable表示使能该功能。
  • Disable表示未使能该功能。
