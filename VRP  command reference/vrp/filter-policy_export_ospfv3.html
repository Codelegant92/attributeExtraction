filter-policy export (OSPFv3)

命令功能

filter-policy export命令用来对引入的路由信息进行过滤。

undo filter-policy export命令用来取消此设置。

缺省情况下，不对引入的路由信息过滤。

命令格式

filter-policy { acl6-number | ipv6-prefix ipv6-prefix-name } export [ protocol [ process-id ] ]

undo filter-policy export [ protocol [ process-id ] ]

参数说明

acl6-number：ACL6的序号。整数形式，取值范围是2000～2999。

ipv6-prefix ipv6-prefix-name：指定IPv6前缀列表名称。字符串形式，长度范围是1～169个字符。

protocol：如果引入了其他路由协议，可以使用此参数来对特定的路由信息进行过滤，其取值可以是bgp、direct、static、ripng、isis或ospfv3。此参数缺省时将对所有引入的路由信息进行过滤。

process-id：当路由协议为ripng、isis或ospfv3时可以指定进程号。

使用指南

filter-policy export命令只对本机使用import-route (OSPFv3)引入的路由起作用。

如果没有配置import-route (OSPFv3)命令来引入其它外部路由（包括不同进程的OSPFv3路由），则filter-policy export命令失效。

使用实例

# 根据序号为2002的ACL6对所有引入的路由信息进行过滤。

<Quidway> system-view
[Quidway] ospfv3 1
[Quidway-ospfv3-1] filter-policy 2002 export
