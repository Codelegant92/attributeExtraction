ppp authentication-mode

命令功能

ppp authentication-mode命令用来设置本端PPP协议对远端路由器的验证方式。

undo ppp authentication-mode命令用来取消配置的验证方式，即不进行验证。

缺省情况下，不验证。

命令格式

ppp authentication-mode { chap [ pap ] | pap } [ call-in ]

undo ppp authentication-mode

参数说明

chap：选择CHAP验证方式。

pap：选择PAP验证方式。

call-in：表示只在远端用户呼入时才验证对方。

缺省级别

使用指南

ppp authentication-mode chap pap命令用来配置在LCP协商时优先进行chap协商，若对方不支持，再进行pap协商。如果被验证方两种方式都不支持，协商将无法通过。但不是在一次PPP协商过程同时协商两种验证方式。

PPP有两种验证方式：

  • PAP为两次握手验证，口令为明文。

  • CHAP为三次握手验证，口令为密文。

另外可以采用已经定义的AAA验证方式进行验证。

无论是CHAP或PAP，只是一种验证过程，最终能否通过验证，还需要AAA来作决定，AAA可以利用本地验证数据库验证或由AAA服务器进行验证。

使用实例

# 在接口POS1/0/0上，采用PAP方法验证对端路由器。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] ppp authentication-mode pap
