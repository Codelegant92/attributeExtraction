protected-vlan

命令功能

protected-vlan命令用来配置RRPP域的保护VLAN列表。

undo protected-vlan命令用来删除RRPP域的保护VLAN列表。

缺省情况下，RRPP保护VLAN引用实例0映射的VLAN列表，VLAN范围是1～4094。即，保护VLAN是1～4094。

命令格式

protected-vlan reference-instance { instance-id1 [ to instance-id2 ] } &<1-10>

undo protected-vlan reference-instance { instance-id1 [ to instance-id2 ] } &<1-10>

参数说明

instance-id：实例ID号，整数形式，取值范围是0～48。

缺省级别

使用指南

RRPP通过引用MSTP实例的方式来创建RRPP域的保护VLAN列表。

一条protected-vlan命令中，最多可以使用10次to形式输入10个实例范围。

配置RRPP保护VLAN列表需要注意以下事项：

  • 在配置RRPP环之前必须配置保护VLAN。

  • 在配置RRPP环之前可以删除、修改已经配置的保护VLAN。在配置环之后不允许更改保护VLAN。

  • 在同一物理拓扑中，本域的控制VLAN不能做为其他域的保护VLAN。

  • 控制VLAN必须在保护VLAN范围内，否则不允许配置环。

  • 控制VLAN在创建环前允许加入别的实例，在创建环后不允许更改，除非删除环。

  • 删除RRPP域时，会自动删除该域中的保护VLAN。

  • 实例和VLAN映射关系发生变化时，RRPP域的保护VLAN随之变化。

  • RRPP端口允许通过的所有VLAN都应该被RRPP域保护。

使用实例

# 配置RRPP域1的保护VLAN为实例2、3映射的VLAN。

<Quidway> system-view
[Quidway] rrpp domain 1
[Quidway-rrpp-domain1] control-vlan 100
[Quidway-rrpp-domain1] protected-vlan reference-instance 2 to 3
