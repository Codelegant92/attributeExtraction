display vll ccc

命令功能

display vll ccc命令用来查看CCC连接信息。

命令格式

display vll ccc [ ccc-name | type { local | remote } ]

参数说明

ccc-name：用于显示的CCC连接的名称。

type：指定要查看的CCC连接类型。

local：本地CCC连接。

remote：远程CCC连接。

缺省级别

1：监控级

使用指南

配置完CCC连接后，可执行display vll ccc命令查看配置是否成功。执行此命令后，如果不指定连接名或连接类型，将显示所有已配置的CCC连接信息。

使用实例

# 查看指定名称的CCC连接。

<Quidway> display vll ccc c1 
name: CE1-CE2, type: remote, state: up
intf: GigabitEthernet1/0/1 (up ethernet), in-label: 100 ,
out-label: 200 , nexthop: 10.1.1.2
no control word
VC last up time : 2009/03/23 11:16:07
VC total up time: 0 days, 0 hours, 1 minutes, 37 seconds

表1 display vll ccc命令输出信息描述

项目

描述

total ccc vc

表示本端的CCC本地连接和远程连接的总和。

local ccc vc : 0, 0 up

表示本端的CCC本地连接的总数为0，有0个状态为up。

remote ccc vc : 1, 0 up

表示本端的CCC远程连接的总数为1，有0个状态为up。

name

表示CCC连接的名称，字符串形式。

type
CCC连接的类型：

  • local：本地连接，即同一个PE的两个CE之间建立的连接。

  • remote：远程连接，即两个不同PE上的CE之间建立的连接。

state
CCC连接的状态：

  • down：表示CCC连接建立不成功，可能是配置不正确或关联的接口为down导致的。

  • up：表示CCC连接建立成功。

intf

表示CCC连接关联的接口。

in-label

该CCC连接的入标签，手工指定的，与对端出标签一致。

out-label

该CCC连接的出标签，也是手工指定的，与对端入标签一致。

nexthop

该CCC连接的公网下一跳地址。

# 显示所有本地CCC连接信息。

<Quidway> display vll ccc type local
total local  ccc vc : 1,  1 up
name: c2, type: local, state: up,
intf1: GigabitEthernet1/0/0 (up),   intf2: GigabitEthernet2/0/0 (up)

项目

描述

total local ccc vc :

表示CCC本地连接的总数和状态为Up的数量。

name

表示CCC连接的名称，字符串形式。

type: local,

表示显示的是CCC本地连接的信息。local表示本地连接，即同一个PE的两个直连CE之间建立的连接。

state
CCC连接的状态：

  • down：表示CCC连接建立不成功，可能是关联的接口为down或配置不正确导致的。

  • up：表示CCC连接建立成功。

intf1: GigabitEthernet1/0/0 (up), intf2: GigabitEthernet2/0/0 (up)

表示本端建立连接的两个CE的接入接口。其中一个为GE1/0/0（接口状态为up），另一个为GE2/0/0接口状态为up）。
