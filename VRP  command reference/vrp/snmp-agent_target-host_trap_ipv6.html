snmp-agent target-host trap ipv6

命令功能

snmp-agent target-host trap ipv6 命令用来设置接收Trap消息的目的地。

undo snmp-agent target-host ipv6命令用来取消当前设置。

命令格式

snmp-agent target-host trap ipv6 address udp-domain ip-address [ udp-port port-number ] params securityname security-string [ v1 | v2c | v3 [ authentication | privacy ] ]

undo snmp-agent target-host ipv6 ip-address securityname security-string [

参数说明

address：指定SNMP消息传输的目标主机地址。

udp-domain：指定目标主机的传输域基于UDP。

ip-address：目标主机IPv6的IP地址。

udp-port port-number：指定接收Trap报文的端口号，取值范围为0～65535，缺省情况下为162。

params：指定产生SNMP消息的日志主机信息。

securityname security-string：指定在网管侧显示的用户名，字符串形式，不支持空格，长度范围为1～32。

v1 | v2c | v3：指定Trap消息的版本。缺省情况下为v1。

authentication：指明对报文进行认证但不加密。

privacy：指明对报文进行认证和加密。

使用指南

此命令用于配置IPv6网管主机，其作用是在产生Trap的时候，使Trap基于IPv6协议发送。

目前IPv6协议下，VRP不支持向VPN接口发送告警。

使用实例

# 配置一个IPv6网管，使用SNMP v3协议，安全名为Huawei，加密方式为鉴权且加密。

<Quidway> system-view
[Quidway] snmp-agent trap enable standard
[Quidway] snmp-agent target-host trap ipv6 address udp-domain 1000::2 params securityname Huawei v3 authentication privacy
