dialer threshold

命令功能

dialer threshold命令用来设定DCC接口上链路的负载阈值，当DCC接口的所有链路的流量与可用带宽的比例超过设定的百分比时，启动另一条链路呼叫同一个目的地址。

undo dialer threshold命令用来恢复缺省值。

缺省不启动流量控制。

命令格式

dialer threshold traffic-percentage [ in | in-out | out ]

undo dialer threshold

参数说明

traffic-percentage：链路实际流量与带宽的百分比，取值范围是1～99。

in：计算实际负载时只计算向内的流量。

in-out：计算实际负载时只计算向内和向外流量中较大的一个。

out：计算实际负载时只计算向外的流量。

使用指南

在配置该命令前，必须先执行命令dialer user设置对端用户名，或执行命令dialer enable-circular使能轮询DCC。

当DCC接口的一条链路的流量与带宽的比例超过设定的负载阈值后，启动第二条链路与原有链路进行MP捆绑；当两条链路的流量与带宽的比例超过设定的负载阈值后，启动第三条与原有链路进行MP捆绑；……依此类推。

反过来，当所有N条（N为大于2的整数）链路的流量与N-1条链路的带宽的比例小于设定的负载阈值时，关闭一条链路……

在轮询DCC下，本命令用来与Dialer circular-group对应的接口（包括ISDN的BRI/PRI接口以及Dialer接口）上；在共享DCC下，本命令只能用于Dialer接口。另外，本命令须与ppp mp命令结合使用。

使用实例

# 设置接口Dialer1的流量门限为80%。

<Quidway> system-view
[Quidway] interface dialer 1
[Quidway-Dialer1] dialer threshold 80
