t1 channel-set（CT3接口视图）

命令功能

t1 channel-set命令用来对T1通道的时隙进行捆绑，创建一个或多个速率为N×56kbit/s或N×64kbit/s的同步串口。

undo t1 channel-set命令用来取消对时隙的捆绑。

命令格式

t1 t1-number channel-set set-number timeslot-list slot-list [ speed { 56k | 64k } ]

undo t1 t1-number channel-set set-number

参数说明

t1-number：T1通道号。整数形式，取值范围是1～28。

set-number：时隙捆绑形成的channel-set索引号。整数形式，取值范围是0～23。

slot-list：时隙编号。整数形式，取值范围是0～23。使用“,”分隔多个时隙，使用“-”表示时隙范围。

speed：配置时隙捆绑的方式。选用参数56k时，捆绑方式为N×56kbit/s；选用参数64k时，捆绑方式为N×64kbit/s。如果不指定速率，缺省采用56k bit/s。

使用指南

只有当CT3接口和其T1通道均工作在通道化模式时，才可以对T1通道进行时隙捆绑。

使用实例

# 将CT3接口1/1/0的1号T1通道的24个时隙捆绑为一个channel-set，索引号为0。

<Quidway> system-view
[Quidway] controller t3 1/1/0
[Quidway-T3 1/1/0] t1 1 channel-set 0 timeslot-list 0-23

表1 display interface serial命令输出信息描述

项目

描述

Serial3/1/0/1:0 current state :
串口当前的物理状态：

  • UP：物理状态正常启动。

  • DOWN：物理连接不正常。

  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Line protocol current state :

显示该接口的链路协议状态：

  • UP：接口的链路协议处于正常的启动状态。

  • DOWN：接口的链路协议不正常。

The Maximum Transmit Unit is

接口的最大传输单元（MTU）。例如串口缺省值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

Hold timer is

报文的生命周期。报文如果在生命周期内没有被送出去，则将丢弃。

Derived from

形成串口的T3接口号和T1通道号。

Timeslot(s) Used:

形成串口的时隙号。

baudrate is

串口的带宽。

Internet protocol processing : disabled

接口没有配置IP地址。

Link layer protocol is

串口的数据链路层协议。

LCP initial

LCP初始化。

Physical layer is

物理层链路信息。

QoS max-bandwidth :

QoS最大带宽。

Output queue : (Urgent queue : Size/Length/Discards) Output queue : (Protocol queue : Size/Length/Discards) Output queue : (FIFO queuing : Size/Length/Discards)

接口输出队列的当前状态，通常包括三类输出队列（Output queue）：

  • 紧急队列（Urgent queue），链路层的协议报文，比如PPP的协商报文和Keepalive报文等将进入这个队列。

  • 协议队列（Protocol queue），IP优先级为6的报文将进入协议队列。

  • 普通的报文队列（FIFO queue），根据接口上应用的队列类型，可能是FIFO（First In First Out Queue）、PQ（Priority Queue）、CQ（Custom Queue）或CBQ（Class-based Queue）等。

当发生拥塞时，接口首先发送紧急队列中的报文，然后是协议队列中的报文，最后发送普通队列中的报文。对于各种发送队列，显示为Size/Length/Discards形式的数字，这里各字段含义如下：

  • Size ：队列中的分组数。

  • Length：用分组表示的最大队列的长度。

  • Discards：由于队列已满而丢失的分组数。

通过检查Discards在一段时间内与Size和Length的关系，可以考察该接口的性能是否满足要求。例如：当Discards的值较大时可能表示路由器正忙于进行其他工作，而无法及时处理进入的分组，如果这种情况持续很长的时间，则通常表示需要一个更强大的路由器以满足工作的要求。

Statistics last cleared:

上次清除本接口统计信息的时间。

Last 300 seconds input rate

Last seconds output rate

这一字段显示了在前300秒内通过接口发送和接收的字节速率和报文速率。

Input:

接口收到的总报文数和总字节数。

errors

在物理层检测时发现的错误报文数目。

runts

接口接收到小于规定的最小报文长度报文数。例如对于以太网报文的最小长度为64字节。接受到的小于64字节的报文将被丢弃。

giants

接收到长度大于规定长度的报文数目。此接口接收报文的最大长度为1500字节。

CRC

收到了长度正常但CRC校验错误的报文数目。

overruns

接收的报文速度大于转发处理能力导致无法处理的报文。

aborts

接收报文的异常错误。

ONR

内存ownership错误。

Output:

接口发送的报文数和总字节数。

underruns

因为接口读取内存的速度小于转发的速度而无法发送报文数目。
