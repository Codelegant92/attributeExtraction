isis authentication-mode

命令功能

isis authentication-mode命令用来设置IS-IS接口以指定的方式和密码验证Hello报文。

undo isis authentication-mode命令用来取消验证，同时删除密码。

缺省情况下，不设置密码，也不做认证。

命令格式

isis authentication-mode { simple password | md5 password-key } [ level-1 | level-2 ] [ ip | osi ] [ send-only ]

undo isis authentication-mode [ level-1 | level-2 ]

参数说明

simple：认证密码采用明文形式发送。

password：明文模式下的认证密码。字符串类型，不支持空格，长度为1～16，可以为字母或数字。

md5：认证密码通过MD5加密后发送。

password-key：md5模式下的认证密码。字符串类型，可以为字母或数字，不支持空格。其中，认证密码可以是长度为1～16的明文；也可以是长度为24的密文。

level-1：设置Level-1级别的认证。

level-2：设置Level-2级别的认证。

ip：IP认证密码。

osi：OSI认证密码。

send-only：对发送的Hello报文加载认证信息。

说明：

参数level-1和level-2仅在以太网接口上是可见的，而且必须先在接口上使能IS-IS。

使用指南

当IS-IS接口链路类型为Level-1-2时，如果不选择参数level-1或level-2，则为Level-1和Level-2的Hello报文都配置认证模式和密码。

如果设置了密码，没有指定参数ip和osi，则默认为osi。

认证密码选用ip或osi不受实际的网络环境影响。

说明：

如果通过isis circuit-type命令将广播接口模拟为P2P接口或者通过undo isis circuit-type命令将该接口恢复为广播接口，则IS-IS接口认证恢复为缺省配置。

使用实例

# 为接口POS1/0/0设置明文认证密码huawei。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] isis authentication-mode simple huawei
