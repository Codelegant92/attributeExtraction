display mld group

命令功能

display mld group命令用来查看组播组信息。

命令格式

display mld group [ ipv6-group-address | interface interface-type interface-number ] * [ static | verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

ipv6-group-address：指定IPv6组播组地址。十六进制形式，可输入的地址格式是FFxA:xxxx:xxxx::xxxx，其中x的取值范围是0～F，A的取值范围是0和3～F

interface interface-type interface-number：指定要查询的接口类型和编号。指定此参数，将显示此接口上的MLD组播组信息。

static：显示静态加入的组播组信息。

verbose：显示运行MLD的接口动态加入的组播组的详细信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

只有指定static，才会显示静态加入的组信息摘要。否则，将显示所有运行MLD的接口上所有其他组播组信息摘要。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 显示静态加入的组播组信息。

<Quidway> display mld group static
Static join group information
 Total 2 entries
  (*, FF56::1)
  Interface:GE6/0/2
  Expires:Never

  (1111::1, FF56::1)
  Interface:GE6/0/2
  Expires:Never

项目

描述

Total entry on this router

路由器上MLD动态加入的组播组个数。

Interface group report information of VPN-Instance

接口收到的组报告信息所属的VPN实例。

GigabitEthernet6/0/2(FE80::2E0:49FF:FE29:1103)

接口名称和IP地址。

Total entry on this interface

接口上MLD动态加入的组播组个数

Total 1 MLD Group reported

接口上收到1个MLD报告消息。

Group

IPv6组播组地址

Uptime
发现组播组的运行时间。时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

Expires

从MLD组列表删除记录的预期时间间隔。

Last Reporter

报告组成员关系的最后主机的链路本地地址。

Last-listener-query-counter

发送最后侦听者查询消息的次数。

Last-listener-query-timer-expiry

发送最后侦听者查询消息的时间间隔。

Version1-host-present-timer-expiry

MLDv1主机的超时时间。
