rpr bind ring-if

命令功能

rpr bind ring-if命令用来将一个RPR接口绑定到逻辑接口的东向或者西向。

undo rpr bind命令用来删除绑定关系，不需要指定方向。

命令格式

rpr bind ring-if interface-number { west | east }

undo rpr bind

参数说明

west：本地节点的西向连接。

east：本地节点的东向连接。

interface-number：逻辑接口号。整数形式，取值范围是0～31。

使用指南

成功执行该命令，需要注意以下三点：

  • 待绑定的RPR物理接口存在；

  • RPR物理接口还没有绑定到任何一个RPR逻辑口；

  • 同时指定RPR逻辑口的相应方向也没有绑定。

RPR物理接口只能存在于一个RPR环。

如果只绑定一个物理接口，且逻辑口未执行shutdown命令进行关闭，节点会进入边状态。所以，建议先执行shutdown命令关闭逻辑口，再进行物理接口的绑定，最后执行undo shutdown命令启动该逻辑接口。

如果一个RPR接口已经执行rpr bind ring-if命令并成功，就不能再通过rpr bind ring-if命令来修改绑定关系。必须先执行undo rpr bind命令，再通过rpr bind ring-if命令来修改绑定关系。

另外，用户要保证绑定的物理口与实际Mate电缆连接情况一致。因为硬件无法检测到Mate电缆连接是否和配置一致，所以需要现场检测。

使用实例

# 绑定物理接口到逻辑接口。

<Quidway> system-view
[Quidway] interface rpr 4/0/0
[Quidway-rpr4/0/0] shutdown
[Quidway-rpr4/0/0] rpr bind ring-if 4 west
[Quidway-rpr4/0/0] undo shutdown
