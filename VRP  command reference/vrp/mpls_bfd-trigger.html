mpls bfd-trigger

命令功能

mpls bfd-trigger命令用来配置LDP BFD触发策略。

undo mpls bfd-trigger命令用来删除LDP BFD触发策略。

命令格式

mpls bfd-trigger [ host [ nexthop next-hop-address | outgoing-interface interface-type interface-number ] | fec-list list-name ]

undo mpls bfd-trigger

参数说明

host：指定LDP BFD以所有主机方式触发。

nexthop next-hop-address：指定LSP的下一跳地址。

outgoing-interface interface-type interface-number：指定LSP的出接口类型和接口号。

fec-list list-name：指定LDP BFD以FEC列表方式触发，并指定FEC列表的名称。

缺省级别

2：配置级

使用指南

LDP BFD的触发策略有两种：主机触发和FEC列表触发。

如果需要所有的主机地址都能触发BFD会话的建立，则采用主机触发方式。还可以通过指定nexthop和outgoing-interface来约束哪些LSP可以建立BFD会话。

如果只需要其中的一部分主机能够触发BFD会话的建立，可以采用FEC列表触发方式来指定相应的主机地址。指定FEC列表触发方式前，必须先执行fec-list和fec-node命令配置FEC列表。

说明：

在执行此命令之前，必须在MPLS视图下使能BFD功能。

使用实例

# 在MPLS视图下配置主机触发策略。

<Quidway> system-view
[Quidway] mpls
[Quidway-mpls] mpls bfd enable
[Quidway-mpls] mpls bfd-trigger host
