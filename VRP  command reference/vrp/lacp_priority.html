lacp priority

命令功能

lacp priority命令用来设置系统LACP优先级或接口LACP优先级。

undo lacp priority命令用来恢复系统LACP优先级或接口LACP优先级为缺省值。

命令格式

lacp priority priority

undo lacp priority

参数说明

priority：LACP优先级值，取值范围是0～65535。缺省值是32768。取值越小LACP优先级越高。

视图

系统视图、Ethernet接口视图、GE接口视图

缺省级别

2：配置级

使用指南

执行该命令前需首先保证Eth-Trunk工作模式为静态LACP模式。

配置系统优先级是为了区别本端设备与对端设备优先级的高低，系统优先级高的将被选作链路聚合组的主动端，按照主动端设备的接口来选择活动接口。

配置接口优先级是为了区别同一设备不同接口的优先级，接口优先级高的将被选作活动接口。

如果Eth-Trunk两端设备都没有配置系统优先级，两端系统优先级都是缺省值32768，此时需要按照系统ID来选择主动端，系统ID小的优先。

如果在静态LACP模式下执行max active-linknumber命令配置了活动接口数目上限阈值，当手工加入链路组的接口数超过了该阈值的限制，就需要选择哪些接口为活动接口，而设置接口LACP优先级即可保证在静态LACP模式下高优先级的接口成为活动接口。

没有执行max active-linknumber命令，活动接口上限阈值为最大值16。只要手工加入的成员接口数值小于16，就不需要选择活动接口，所有的接口都处于活动状态。

使用实例

# 设置LACP系统优先级为1。

<Quidway> system-view
[Quidway] lacp priority 1

# 设置GigabitEthernet 1/0/1接口的LACP接口优先级为1。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/1
[Quidway-GigabitEthernet1/0/1] lacp priority 1
