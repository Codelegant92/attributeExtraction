display mpls oam egress

命令功能

display mpls oam egress命令用来在OAM宿端查看配置了OAM功能的LSP当前状态与配置信息。

命令格式

display mpls oam egress { all | lsp-name lsp-name | lsr-id ingress-lsr-id tunnel-id tunnel-id } [ slot slot-id | verbose ] [ | { begin | include | exclude } regular-expression ]

参数说明

all：查看宿端所有启用OAM功能的LSP的报文信息与缺陷状态。

lsp-name lsp-name：被检测的LSP名称，字符串形式，不支持空格，长度范围是1～19。

lsr-id ingress-lsr-id：隧道入节点的LSR ID。

tunnel-id tunnel-id：Tunnel的ID号，整数形式，取值范围取决于产品的许可证文件。

slot slot-id：槽位号。显示指定槽位接口板的全部OAM信息。使用此参数可以看到OAM实例的报文统计信息。

verbose：显示全部信息。

|：使用正则表达式对输出信息进行过滤。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

如果执行此命令时不指定接口板槽号，将显示主控板的OAM信息。主控板的OAM信息不包括OAM实例的报文统计信息。

使用实例

# 在出节点查询LSR ID为1.1.1.1、Tunnel ID为100的LSP的OAM信息。

<Quidway> display mpls oam egress lsr-id 1.1.1.1 tunnel-id 100
--------------------------------------------------------------------------
No. Lsp-name          Ttsi                  Type    Frequency  Status
--------------------------------------------------------------------------
1   --                1.1.1.1 : 100         CV        1 s      Start/No-defect

项目

描述.

Lsp name

被检测的静态LSP名称。如果被检测的是动态LSP，此处显示“--”

Ttsi

TTSI，包括LSR ID和Tunnel ID两部分。冒号左侧是LSR ID，冒号右侧是Tunnel ID

Type

OAM报文检测类型

Frequency

OAM检测频率

Status
检查状态和缺陷状态，包括：

  • Start/No-defect：检测到没有故障

  • Start/In-defect：检测到处于故障状态
