ppp ipcp dns

命令功能

ppp ipcp dns命令用来使能路由器为对端提供DNS服务器地址。

undo ppp ipcp dns命令用来禁止路由器为对端提供DNS服务器地址。

缺省情况下，路由器不为对端提供DNS服务器地址。

命令格式

ppp ipcp dns { primary-dns-address [ secondary-dns-address ] | admit-any }

undo ppp ipcp dns { primary-dns-address [ secondary-dns-address ] | admit-any }

参数说明

primary-dns-address：主DNS服务器地址。

secondary-dns-address：从DNS服务器地址。

admit-any：接受对端提出的任何DNS地址。

使用指南

当其他设备与路由器通过PPP协议相连时（通常为PC机拨号连接路由器），通过协商，路由器可以为对端设备分配DNS服务器地址。这样，对端设备就可以通过域名直接访问网络。

如果用户通过PC机连接路由器，用户可以在PC机上使用命令winipcfg（windows98）或ipconfig /all（windows 2000/XP）来查看路由器为其提供的DNS服务器地址。

VRP可以为对端设备提供主DNS服务器和从DNS服务器两个服务器地址。

使用实例

# 设置本地路由器的Primary DNS Server地址为100.1.1.1，Secondary DNS Server地址为100.1.1.2。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] ppp ipcp dns 100.1.1.1 100.1.1.2
