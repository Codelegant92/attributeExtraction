mpls te bandwidth（Tunnel接口视图）

命令功能

mpls te bandwidth命令用来配置MPLS TE隧道所需的带宽。

undo mpls te bandwidth命令用来恢复缺省设置。

缺省情况下，没有配置隧道带宽。

命令格式

mpls te bandwidth { ct0 ct0-bw-value | ct1 ct1-bw-value } [ flow-queue flow-queue ]

mpls te bandwidth { { ct0 ct0-bw-value | ct1 ct1-bw-value | ct2 ct2-bw-value | ct3 ct3-bw-value | ct4 ct4-bw-value | ct5 ct5-bw-value | ct6 ct6-bw-value | ct7 ct7-bw-value } *

undo mpls te bandwidth { all | { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } * }

undo mpls te bandwidth { all | { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 } * }

参数说明

ct0ct0-bw-value：为CT0预留带宽。ct0-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct1ct1-bw-value：为CT1预留带宽。ct1-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

flow-queue flow-queue：指定流量队列模板。只有配置单CT0和单CT1时才能配置该参数。flow-queue是流量模板名字，字符串形式，区分大小写，长度范围是1～31个字符。

ct2ct2-bw-value：为CT2预留带宽。ct2-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct3ct3-bw-value：为CT3预留带宽。ct3-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct4ct4-bw-value：为CT4预留带宽。ct4-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct5ct5-bw-value：为CT5预留带宽。ct5-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct6ct6-bw-value：为CT6预留带宽。ct6-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

ct7ct7-bw-value：为CT7预留带宽。ct7-bw-value为整数形式，取值范围是1～32000000，单位是Kbit/s。

使用指南

在Non-IETF模式和IETF模式下都可以配置CT带宽。Non-IETF模式只可以配置CT0和CT1，而IETF模式下可以配置一个或多个CT，且同时需要配置优先级。配置CT和优先级组合需要先配置相应的TE-CLASS。

静态TE隧道不支持多CT。

同一个TE隧道接口下，多CT带宽配置与如下特性互斥：

  • 自动带宽调整
  • CSPF仲裁（Tie-Breaking）
  • 资源隔离VPN
  • 旁路隧道属性

TE的单CT配置没有以上限制。

undo mpls te bandwidth命令用来恢复缺省设置。分三种情况：

  • undo mpls te bandwidth all表示将所有带宽配置全部清除。
  • undo mpls te bandwidth { ct0 | ct1 | ct2 | ct3 | ct4 | ct5 | ct6 | ct7 }*表示删除当前TE隧道已经配置的一个或多个CT的带宽。

当系统从VRPV500R005及之前的版本升级到VRPV500R006版本后，系统当前配置信息中，Tunnel接口下的原有带宽配置命令mpls te bandwidth bc0 bw-value和mpls te bandwidth bc1 bw-value将分别被替换为mpls te bandwidth ct0 bw-value和mpls te bandwidth ct1 bw-value，命令的作用不变。

使用实例

# 在Tunnel1/0/0接口下配置对应TE隧道所需的带宽。CT0带宽为2Mbit/s；CT1带宽为2Mbit/s；CT2带宽为1Mbit/s。

<Quidway> system-view
[Quidway] mpls lsr-id 1.1.1.1
[Quidway] mpls
[Quidway-mpls] mpls te
[Quidway-mpls] quit
[Quidway] interface tunnel1/0/0
[Quidway-Tunnel1/0/0] tunnel-protocol mpls te
[Quidway-Tunnel1/0/0] destination 2.2.2.2
[Quidway-Tunnel1/0/0] mpls te tunnel-id 100
[Quidway-Tunnel1/0/0] mpls te bandwidth ct0 2000 ct1 2000 ct2 1000
[Quidway-Tunnel1/0/0] mpls te commit
