ping mac

命令功能

ping mac命令用来检测当前设备和目的设备之间的连通性，这种检测称为通用MAC Ping。

命令格式

ping mac mac-address { vlan vlan-id | vsi vsi-name } [ interface interface-type interface-number | -c count | -s packetsize | -t timeout ] *

参数说明

mac-address：指定目的设备的桥MAC地址。格式为H-H-H，其中H为1至4位的十六进制数。不能为广播MAC地址或组播MAC地址。

vlan vlan-id：指定VLAN。vlan-id表示VLAN ID，整数形式，取值范围是1～4094。

vsi-name：VSI实例对应的名称。字符串形式，长度为1～31个字符。

interface interface-type interface-number：指定本设备发送ping报文的出接口。interface-type表示接口类型，取值包括：ethernet、eth-trunk和gigabitethernet。interface-number表示接口编号。如果：

  • 指定本参数，且接口在指定VLAN或VSI内则直接从该接口发出Ping报文。

  • 不指定本参数，设备首先根据指定的目的MAC地址、VLAN或VSI查询MAC地址转发表，如果：

  • 查询到对应的转发表项，按该表项发送Ping报文；

  • 未查询到对应的转发表项，在该VLAN内广播Ping报文。

-c count：指定Ping的次数。count表示Ping的次数，整数形式，取值范围是1～4294967295，缺省值是5。

-s packetsize：指定发送的Ping报文的大小。packetsize表示Ping报文的大小，整数形式，取值范围是95～1500，单位是字节，缺省值是95。该大小是指去掉二层报文头后的报文大小。

-t timeout：指定等待应答报文的超时时间。timeout表示等待应答报文的超时时间，整数形式，取值范围是1～65535，单位是ms，缺省值是2000ms。

使用指南

使用本命令时，通用MAC Ping功能必须处于使能状态，否则需首先使用ping mac enable命令使能通用MAC Ping功能。

通用MAC Ping不需要由MEP发起，目的节点也不需要是MEP或MIP，即在源设备、中间设备和目的设备上均不需要配置MD、MA和MEP就可以进行通用MAC Ping。但是需要指定目的节点所属的VLAN或VSI。

使用实例

# Ping某个设备两次。Ping报文大小为112字节。该设备属于VLAN10。该设备的MAC地址是0001-0300-0204。

<Quidway> system-view
[Quidway] ping mac enable
[Quidway] ping mac 0001-0300-0204 vlan 10 -c 2 -s 112
