snmp-agent trap enable feature-name mrm

命令功能

snmp-agent trap enable feature-name mrm命令用来将打开组播路由管理相关的告警开关。

undo snmp-agent trap enable feature-name mrm命令用来关闭组播路由管理相关的告警开关。

命令格式

snmp-agent trap enable feature-name mrm { non-excessive all | trap-name { cacglbchn | cacglbchnexceed | cacglbtotal | cacglbtotalexceed | cacoifchn | cacoifchnexceed | cacoiftotal | cacoiftotalexceed } }

undo snmp-agent trap enable feature-name mrm { [ non-excessive | excessive ] all | trap-name { cacglbchn | cacglbchnexceed | cacglbtotal | cacglbtotalexceed | cacoifchn | cacoifchnexceed | cacoiftotal | cacoiftotalexceed } }

参数说明

non-excessive all：打开组播路由管理模块所有非海量事件的告警开关。

trap-name：打开指定类型事件的告警开关。

cacglbchn：打开如下事件的告警开关，节目组的表项数量超出基于节目组的CAC限制值导致PIM表项加入失败。

cacglbchnexceed：打开如下事件的告警开关，节目组的已有表项数量超出限制值导致配置基于节目组的CAC限制失败。

cacglbtotal：打开如下事件的告警开关，实例的所有表项数量超出基于实例的CAC总体限制值导致PIM表项加入失败。

cacglbtotalexceed：打开如下事件的告警开关，实例的已有表项数量超出限制值导致配置基于实例的CAC总体限制失败。

cacoifchn：打开如下事件的告警开关，以当前接口作为出接口的PIM表项数量或带宽超出在出接口上配置的基于节目组的CAC限制值。

cacoifchnexceed：打开如下事件的告警开关，已有的以当前接口作为出接口的PIM表项数量或带宽超出在出接口上配置的基于节目组的CAC限制值。

cacoiftotal：打开如下事件的告警开关，以当前接口作为出接口的PIM表项数量或带宽超出在出接口上配置的CAC总体限制值。

cacoiftotalexceed：打开如下事件的告警开关，已有的以当前接口作为出接口的PIM表项数量或带宽超出在出接口上配置的CAC总体限制值。

non-excessive：关闭组播路由管理模块所有非海量事件的告警开关。

excessive：关闭组播路由管理模块所有海量事件的告警开关。

all：关闭组播路由管理模块所有事件的告警开关。

使用指南

目前定义的组播路由管理模块告警均为海量事件。打开某个或多个事件的告警开关时，需要指定trap-name。

使用实例

# 在系统视图下，打开节目组的表项数量超出基于节目组的CAC限制值的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name mrm trap-name cacglbchn

# 在系统视图下，关闭节目组的表项数量超出基于节目组的CAC限制值的告警开关。

<Quidway> system-view
[Quidway] undo snmp-agent trap enable feature-name mrm trap-name cacglbchn
