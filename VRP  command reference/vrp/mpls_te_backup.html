mpls te backup

命令功能

mpls te backup命令用来配置当前隧道使用的备份模式。

undo mpls te backup hot-standby命令用来删除热备份功能。

undo mpls te backup ordinary命令用来删除普通备份功能，或删除热备份功能中的逃生路径。

缺省情况下，隧道不进行备份。

命令格式

mpls te backup { hot-standby [ wtr interval ] | ordinary [ best-effort ] }

undo mpls te backup { hot-standby | ordinary }

参数说明

hot-standby：热备份，主LSP和备份LSP同时存在，主LSP失效时，直接将业务切换至备份LSP。

wtr interval：配置热备份回切的时间。interval的范围为0～60s，默认时间为10s。

ordinary：普通备份，主LSP失效后才创建备份LSP。

best-effort：表示配置逃生路径。配置逃生路径后，当主LSP和备份LSP都失效时，系统将触发建立逃生路径。

使用指南

使能隧道备份功能后，记录路由也自动使能，而不论用户是否配置了命令mpls te record-route。

隧道备份功能与以下命令互斥：
  • mpls te reoptimization（Tunnel接口视图）
  • mpls te auto-bandwidth
  • mpls te resv-style

命令mpls te backup ordinary与mpls te backup ordinary best-effort互斥。

使用实例

# 使能当前隧道使用热备份。

<Quidway> system-view
[Quidway] interface tunnel 1/0/0
[Quidway-Tunnel1/0/0] tunnel-protocol mpls te
[Quidway-Tunnel1/0/0] destination 2.2.2.2
[Quidway-Tunnel1/0/0] mpls te tunnel-id 100
[Quidway-Tunnel1/0/0] mpls te backup hot-standby
[Quidway-Tunnel1/0/0] mpls te commit
