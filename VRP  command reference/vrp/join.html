join

命令功能

join命令用来将当前测试例加入指定的NQA测试组。

undo join命令用来将测试例从测试组中退出。

缺省情况下，测试例不属于任何测试组。

命令格式

join group nqa admin-name test-name

undo join group nqa admin-name test-name

参数说明

admin-name：指定NQA测试组的管理者。字符串形式，长度范围是1～32。

test-name：指定NQA测试组的测试例名。字符串形式，长度范围是1～32。

缺省级别

2：配置级

使用指南

只有和测试组的测试类型相同的测试例，才能加入测试组中。并且，一个测试例只能加入一个测试组中。

一旦加入测试组后：

  • 不能更改测试例的测试类型，只能更改测试组的测试类型。

  • 测试例加入组前，如果配置的报文大小是非缺省值，则加入组后此值保持不变，如果报文的大小是缺省值，则采用测试组的配置。

  • 测试例加入测试组前，如果配置了VPN实例，则加入测试组后，VPN实例保持不变；如果没有配置VPN实例，则加入测试组后，采用测试组配置的VPN实例。

  • 组员测试例只能配置目的地址、目的端口号、源地址、源端口号。

  • 对于测试例不支持的配置，在加入组后，采用测试组的配置。

组员测试例退出测试组后，其源地址、源端口、目的地址、目的端口保持在组中的配置，其他的参数都恢复为缺省值。

使用实例

# 设置测试例user test加入已创建的测试组user group1。

<Quidway> system-view
[Quidway] nqa test-instance user group1
[Quidway-nqa-user-group1] switch-to group
[Quidway-nqa-user-group1] quit
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] join group nqa user group1
