site

命令功能

site命令用于配置VSI实例的Site ID。

缺省情况下，系统没有配置VSI实例的Site ID。

命令格式

site site-id [ range site-range ] [ default-offset { 0 | 1 } ]

undo site site-id

参数说明

site-id：用于标识一个VSI实例的Site ID。十进制整数形式，取值范围是0～65534。

range site-range：VSI实例中的Site的个数范围。十进制整数形式，取值范围是1～65534，缺省值为10。

default-offset：缺省的初始Site ID的偏差值。十进制整数形式，取值为0或1。缺省值为0。

视图

VSI-BGP视图

缺省级别

2：配置级

使用指南

配置VSI实例的Site ID时，需保证已配置该实例的route-distinguisher（VSI-BGP视图）。

各个PE上同一VSI的Site ID不能设为相同。本端的Site ID值要小于对端的site-range与default-offset之和。但是本端的Site ID值要大于对端的default-offset。

一台设备为其所有Kompella方式L2 VPN实例和VPLS VSI实例的Range分配标签时共用一个标签块，因此Kompella方式L2 VPN实例和VPLS VSI实例的Range总和不能超过该标签块的大小，否则系统会提示分配的标签数量超过了系统允许的最大值，无法获得标签，分配VSI实例的Site ID或创建CE失败。系统允许使用的最大标签块大小与产品相关。

使用实例

# 配置当前VSI实例的Site ID为1，可连接的Site的个数为100。

<Quidway> system-view
[Quidway] vsi company2
[Quidway-vsi-company2] pwsignal bgp
[Quidway-vsi-company2-bgp] site 1 range 100
