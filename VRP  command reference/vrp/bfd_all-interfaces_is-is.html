bfd all-interfaces(IS-IS)

命令功能

bfd all-interfaces 命令用来配置BFD会话的参数值。

undo bfd all-interfaces 命令用来恢复BFD会话参数的默认值。

命令格式

bfd all-interfaces { min-rx-interval receive-interval | min-tx-interval transmit-interval | detect-multiplier multiplier-value } *

undo bfd all-interfaces { min-rx-interval | min-tx-interval | detect-multiplier } *

参数说明

min-rx-interval receive-interval：指定期望从对端接收BFD报文的最小接收间隔。receive-interval取值范围是100～1000，单位是毫秒。缺省值是100毫秒。

min-tx-interval transmit-interval：指定向对端发送BFD报文的最小发送间隔。transmit-interval取值范围是100～1000，单位是毫秒。缺省值是100毫秒。

detect-multiplier multiplier-value：指定本地检测倍数。multiplier-value取值范围是3～50，缺省值是3。

说明：

不同产品，上述最大、最小值可能不同。

缺省级别

2：配置级

使用指南

本地的min-rx-interval值与对端的min-tx-interval值进行协商得到receive-interval，如果在receive-interval×multiplier-value时间间隔内没有收到对方发送的BFD报文，就宣告邻居失效。

如果只配置BFD的会话参数，但没有执行bfd all-interfaces enable命令，则不会创建BFD会话。

说明：

接口配置的BFD特性优先级高于进程配置的BFD特性优先级。如果打开了接口的BFD开关，则按照接口上BFD参数建立BFD会话。

使用实例

# 配置IS-IS进程的BFD特性，设置BFD报文的最小发送间隔为300ms。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] bfd all-interfaces enable
[Quidway-isis-1] bfd all-interface min-tx-interval 300
