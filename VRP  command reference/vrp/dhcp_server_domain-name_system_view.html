dhcp server domain-name（系统视图）

命令功能

dhcp server domain-name命令用来配置指定范围的接口的地址池分配给DHCP客户端的域名。

undo dhcp server domain-name命令用来删除分配给DHCP客户端的域名。

缺省情况下，未配置DHCP客户端的域名。

命令格式

dhcp server domain-name domain-name { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan vlan-id1 [ to vlan-id2 ] & <1-10> }

undo dhcp server domain-name { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan vlan-id1 [ to vlan-id2 ] & <1-10> }

参数说明

domain-name：指定DHCP服务器分配给客户端的域名，字符串形式，长度是3～50。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：表示所有接口。

缺省级别

使用指南

实现一次配置给多接口分配DHCP客户端的域名，使用dhcp server domain-name命令配置指定范围的接口的地址池分配给DHCP客户端的域名。

本命令配置后不能通过display current-configuration查看，它是通过循环执行dhcp server domain-name命令实现批量配置。

使用实例

# 将GE2/0/0.1到GE2/0/0.5之间的接口DHCP地址池的域名设为ge2_1_5.com.cn。

<Quidway> system-view
[Quidway] dhcp server domain-name ge2_1_5.com.cn interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
