display current-configuration

命令功能

display current-configuration命令用来显示路由器当前生效的配置参数。

对于某些正在生效的配置参数，如果与缺省工作参数相同，则不显示。

命令格式

display current-configuration [ all | unavailable-card | configuration [ configuration-type ] | controller | interface interface-type [ interface-number ] ] [ | { begin | exclude | include } regular-expression | feature feature-name [ filter filter-expression ] | filter filter-expression ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

|：使用正则表达式过滤输出的显示信息。

all：所有板卡的配置信息，包括不在位的板卡的配置信息。

begin：显示以指定正则表达式开头的配置信息。

exclude：显示不含指定正则表达式的配置信息。

include：显示包含指定正则表达式的配置信息。

regular-expression：正则表达式字符串。

configuration：显示指定的配置信息。

configuration-type：配置类型，依赖于系统当前已有的配置，可以显示AAA配置、系统配置、用户界面配置及配置输出。

controller：显示控制器接口的配置信息，如E1控制器接口。

feature：显示特性的配置信息。

feature-name：指定特性名称。

filter：显示过滤后的配置信息。

filter-expression：表示过滤条件的第一个匹配字段。

interface：显示指定接口的配置。

interface-type：接口类型。

unavailable-card：不在位的板卡的配置信息。

使用指南

当用户完成一组配置之后，需要验证是否配置正确，则可以执行display current-configuration命令来查看当前生效的参数，对于某些参数，虽然用户已经配置，但如果这些参数所在的功能如果没有生效，则不予显示，如：某接口链路层封装X.25，这时用户可以在该接口上配置PPP参数，但执行display current-configuration时看不到该接口上的PPP配置信息。

当配置信息比较多时，可以使用正则表达式对输出信息进行过滤。正则表达式的规则请参考《VRP 配置指南 基础配置》的命令行介绍部分。

当用户需要查看当前所有板卡的配置信息，包括不在位不生效的板卡的配置信息时，可以执行display current-configuration all命令。

当用户需要查看当前不在位的板卡的配置信息时，可以执行display current-configuration unavailable-card命令。

使用实例

# 显示路由器当前生效的配置参数。

<Quidway> display current-configuration
#
 sysname Quidway
#
 super password level 3 simple 123456
#
interface GigabitEthernet1/0/0
#
interface Pos2/0/0
 link-protocol ppp
#
user-interface con 0
 set authentication password simple 123456
 history-command max-size 30
user-interface aux 0
user-interface vty 0 4
#
return

# 查看以字符串“user”开始的所有配置信息。

<Quidway> display current-configuration | begin user
user-interface con 0
user-interface aux 0
user-interface vty 0 4
 authentication-mode none
 user privilege level 3
 idle-timeout 0 0
#
return

# 查看包含字符串“user”的所有配置信息。

<Quidway> display current-configuration | include user
user-interface con 0
user-interface aux 0
user-interface vty 0 4
user privilege level 3
