mpls l2vpn pw bfd

命令功能

mpls l2vpn pw bfd命令用来设置AC接口上支持动态BFD检测PW功能及调整BFD检测参数。

undo mpls l2vpn pw bfd命令用来删除AC接口上的动态BFD检测PW功能。

undo mpls l2vpn pw bfd [ detect-multiplier | min-rx-interval | min-tx-interval ] *命令用来将动态BFD检测PW的BFD参数恢复为默认值。

缺省情况下，接口上没有配置动态BFD检测PW功能。

命令格式

mpls l2vpn pw bfd [ detect-multiplier multiplier | min-rx-interval rx-interval| min-tx-interval tx-interval ] * [ remote-vcid vc-id ] [ secondary ]

undo mpls l2vpn pw bfd [ secondary ]

undo mpls l2vpn pw bfd [ detect-multiplier | min-rx-interval | min-tx-interval ] * [ secondary ]

参数说明

detect-multiplier multiplier：BFD本地检测时间倍数。取值范围是3～50，缺省值是3。

min-rx-interval rx-interval：BFD期望的最小接收时间间隔。单位是毫秒。取值范围是100～1000，缺省值是1000ms。

min-tx-interval tx-interval：BFD期望的最小发送时间间隔。单位是毫秒。取值范围是100～1000，缺省值是1000ms。

说明：

BFD检测参数的取值范围在不同产品上会有差异，请以实际显示为准。

remote-vcid vc-id：对端的VCID。被检测的PW是多跳PW时必须配置该参数，应该是PW对端的VC ID。取值范围是1～4294967295。

secondary：设置备用PW上支持BFD检测及配置检测参数。缺省情况下是针对主PW的配置。

使用指南

为降低对系统资源的占用，一旦检测到BFD会话Down，系统自动将本端的接收间隔和发送间隔调整为1000ms～3000ms之间的一个随机值，当BFD会话的状态重新变为Up后，再恢复成用户配置的时间间隔。

使用实例

# 在AC接口Pos1/0/0上设置PW支持动态BFD检测功能，并且配置BFD检测时间。

<Quidway> system-view
[Quidway] interface pos1/0/0
[Quidway-Pos1/0/0] mpls l2vpn pw bfd min-rx-interval 100 min-tx-interval 100
