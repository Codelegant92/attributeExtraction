display interface

命令功能

display interface命令用来查看接口当前运行状态和接口统计信息。

命令格式

display interface [ interface-type [ interface-number ] | slot slot-number ] [ | { begin | exclude | include } regular-expression ]

参数说明

interface-type interface-number：接口类型和接口编号。如果不指定接口类型，将显示所有接口的运行状态和统计信息。如果不指定接口编号，将显示所有该类型接口的运行状态和统计信息。

slot slot-number：槽位编号。显示指定接口板上所有接口的运行状态和统计信息。

|：使用正则表达式过滤输出的显示信息。

begin：显示以包含指定正则表达式的行开始的配置信息。

exclude：显示不包含指定正则表达式的行的配置信息。

include：显示包含指定正则表达式的行的配置信息。

regular-expression：正则表达式。字符串形式，支持空格，区分大小写。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

表1 display interface命令输出信息描述

项目

描述

GigabitEthernet6/2/0 current state :
显示接口GigabitEthernet6/2/0的物理状态。

  • UP：接口物理层处于正常启动的状态。

  • DOWN：接口物理层出现故障。

  • Administratively down：如果网络管理员在该接口执行shutdown（接口视图）命令，将显示状态为Administratively down。

  • Flow Down：接口的流量状态为Down。该状态与绑定的管理VRRP状态一致。如果绑定的管理VRRP状态为Backup或者Initialize，则业务接口的流量状态为Down。

Line protocol current state :

显示该接口的链路协议状态。

  • UP：接口的链路协议处于正常的启动状态。

  • UP（BFD status down）：与该接口绑定的BFD会话状态变为down。

  • UP（Main BFD status down）：与主接口联动的BFD会话状态down，并且联动子接口状态。只能在子接口显示该状态。

  • DOWN：接口的链路协议层出现故障，或者没有在此接口配置IP地址。

  • UP (spoofing)：表示该接口的链路协议状态spoofing特性，也就是该接口的链路协议状态永远是Up。

Last line protocol up time:

接口的协议状态最近一次变为Up的时间。

Description :

对接口的说明，支持空格，区分大小写，最多可输入242个字符。该说明信息便于用户了解该接口的作用。

The Maximum Transmit Unit is

接口的最大传输单元（MTU）。例如以太网接口或串口的缺省值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

Hold timer is

报文的生命周期。报文如果在生命周期内没有被发送出去，则将丢弃。

Internet Address is

接口的IP地址和子网掩码。

IP Sending Frames' Format is

接口发送的Ethernet帧的格式。Ethernet_2为缺省的帧格式。Ethernet在接收帧时，可以识别如下几种帧格式：

  • Ethernet_2

  • Ethernet_SNAP

  • 802.2

  • 802.3

Hardware address is

接口的MAC地址。

QoS max-bandwidth :

QoS最大带宽。

Output queue : (Urgent queue : Size/Length/Discards) Output queue : (Protocol queue : Size/Length/Discards)Output queue : (FIFO queuing : Size/Length/Discards)
接口输出队列的当前状态，通常包括三类输出队列（Output queue）：

  • 紧急队列（Urgent queue），链路层的协议报文，比如PPP的协商报文和Keepalive报文等将进入这个队列。

  • 协议队列（Protocol queue），IP优先级为6的报文将进入协议队列。

  • 普通的报文队列（FIFO queue），根据接口上应用的队列类型，可能是FIFO（First In First Out Queue）、PQ（Priority Queue）、CQ（Custom Queue）或CBQ（Class-based Queue）等。

当发生拥塞时，接口首先发送紧急队列中的报文，然后是协议队列中的报文，最后发送普通队列中的报文。对于各种发送队列，显示为Size/Length/Discards形式的数字，这里各字段含义如下。

  • Size ：队列中的分组数。

  • Length：用分组表示的最大队列的长度。

  • Discards：由于队列已满而丢失的分组数。

通过检查Discards在一段时间内与Size和Length的关系，可以考察该接口的性能是否满足要求。例如：当Discards的值较大时可能表示路由器正忙于进行其他工作，而无法及时处理进入的分组，如果这种情况持续很长的时间，则通常表示需要一个更强大的路由器以满足工作的要求。

Last physical up time :

接口上次物理Up的时刻，“-”表示接口的此物理状态没有发生过变化。

Last physical down time :

接口上次物理Down的时刻，“-”表示接口的此物理状态没有发生过变化。
