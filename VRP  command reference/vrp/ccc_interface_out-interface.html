ccc interface out-interface

命令功能

ccc interface out-interface命令用于在连接相同PE的两个CE间创建一条本地CCC连接。

undo ccc命令用来删除创建的CCC连接。

缺省情况下，系统没有创建任何CCC本地连接。

命令格式

ccc [ ip-interworking ] ccc-connection-name interface interface-type1 interface-number1 out-interface interface-type2 interface-number2

undo ccc ccc-connection-name

参数说明

ip-interworking：使能CCC本地连接的IP异种介质互通功能。

ccc-connection-name：CCC连接名，唯一标识PE上的一个CCC连接，字符串形式，不支持空格，区分大小写，长度范围是1～20。

interface interface-type1 interface-number1：与第一个CE相连的接口类型和编号。

out-interface interface-type2 interface-number2：与第二个CE相连的接口类型和编号。

使用指南

同一个接口不能既作为L2VPN的AC（Attachment Circuit）接口又作为L3VPN的AC接口。当某个接口绑定L2VPN后，该接口上配置的IP地址、路由协议等三层特性会全部变为无效。如果一个接口同时绑定L2VPN和L3VPN，只有L2VPN可用。删除该L2VPN后，绑定的L3VPN会重新变得可用。

说明：

  • 对于NE40E、NE80E、NE5000E产品，数据流量的入接口和出接口都不支持VLANIF接口。

  • 当子接口配置了与VLL的绑定关系时，必须先删除子接口和VLL的绑定关系，才能删除子接口。

  • 当子接口配置了与VLL的绑定关系时，不允许改变主接口的封装协议类型。

CCC本地连接是双向的，因此只需要创建一条连接。不要配置名称为type的CCC连接，否则无法通过display vll ccc ccc-name命令，查看该CCC连接的信息。

使用实例

# 在两个CE间创建一条CCC本地连接，这两个CE连接到同一PE。

<Quidway> system-view
[Quidway] ccc ccc-connect-1 interface pos 1/0/0 out-interface pos 1/0/1
