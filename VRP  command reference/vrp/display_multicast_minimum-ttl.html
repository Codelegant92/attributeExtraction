display multicast minimum-ttl

命令功能

display multicast minimum-ttl命令用来查看接口转发组播数据报文的最小TTL值。

命令格式

display multicast [ vpn-instance vpn-instance-name | all-instance ] minimum-ttl [ interface-type interface-number ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

interface-type interface-number：表示接口类型和接口编号。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

接口转发组播报文的最小TTL值是通过multicast minimum-ttl命令配置的。

使用display multicast minimum-ttl命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 如果指定interface-type interface-number，则显示指定接口的最小TTL值，否则显示所有接口的最小TTL值。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
