display interface { ethernet | gigabitethernet }

命令功能

display interface { ethernet | gigabitethernet }命令用来显示以太网接口各种配置参数以及当前运行状态等信息。

命令格式

display interface { ethernet | gigabitethernet } [ interface-number ]

参数说明

ethernet：查看快速以太网接口的状态信息。

gigabitethernet：查看千兆以太网接口的状态信息。

interface-number：接口编号。如果不指定接口编号，将显示所有接口的配置及状态信息。

使用指南

在监控以太网接口的状态或检查以太网接口的故障原因时，可执行此命令获取以太网接口的状态信息和报文统计信息等。

表1 display interface ethernet命令输出信息描述

项目

描述

current state :
显示接口的物理状态。

  • UP：接口处于正常启动的状态

  • DOWN：接口在物理上出现故障

  • Administratively down：如果网络管理员在该接口执行shutdown（接口视图）命令，将显示状态为Administratively down。

Line protocol current state :

显示该接口的链路协议状态。

  • UP：接口的链路协议处于正常的启动状态。

  • DOWN：接口的链路协议不正常，或者没有在此接口配置IP地址。

Last line protocol up time:

接口的链路协议上次Up的时刻，“-”表示接口的链路协议状态没有发生过变化。

The Maximum Transmit Unit is

接口的最大传输单元（MTU）。例如串口默认值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

Hold timer is

报文的生命周期。报文如果在生命周期内没有被送出去，则将丢弃。

Internet Address is

在接口上配置的IP地址和掩码长度。

IP Sending Frames' Format is

接口发送的Ethernet帧的格式。Ethernet_II为默认的帧格式。Ethernet在接收帧时，可以识别如下几种帧格式。

  • Ethernet_II

  • Ethernet_SNAP

  • 802.2

  • 802.3

Hardware address is

接口的MAC地址。

Encapsulation

子接口的封装方式。该字段只有在子接口时才显示。

The number of Vlan is

子接口的VLAN数量。该字段只有在子接口时才显示。

Vlan ID

子接口上封装的最小VLAN ID。该字段只有在子接口时才显示。

Media type:

接口介质的类型。包括介质的类型、波长和传输距离。波长和传输距离会因接口介质类型的不同而不同。介质类型一般有双绞线、单模光纤和多模光纤等。

Loopback not set

没有设置接口环回功能。

promiscuous mode not set

该接口不处于混杂模式。

100Mbps-speed mode

工作速率是100Mbps。对于10/100M双速率的接口，可以使用命令speed（Ethernet接口视图）更改配置。

full-duplex mode

工作模式是全双工。对于10/100M电接口，可以使用命令duplex更改配置。

link type is autonegotiation

链路连接类型是自动协商。此外还有强制连接等。

The maximum frame length is

帧的最大长度。

QoS max-bandwidth :

QoS最大带宽。

Output queue : (Urgent queue : Size/Length/Discards) Output queue : (Protocol queue : Size/Length/Discards)Output queue : (FIFO queuing : Size/Length/Discards)
接口输出队列的当前状态，通常包括三类输出队列（Output queue）：

  • 紧急队列（Urgent queue），链路层的协议报文，比如PPP的协商报文和Keepalive报文等将进入这个队列。

  • 协议队列（Protocol queue），IP优先级为6的报文将进入协议队列。

  • 普通的报文队列（FIFO queue），根据接口上应用的队列类型，可能是FIFO（First In First Out Queue）、PQ（Priority Queue）、CQ（Custom Queue）或CBQ（Class-based Queue）等。

当发生拥塞时，接口首先发送紧急队列中的报文，然后是协议队列中的报文，最后发送普通队列中的报文。队列状态中包含的三个字段Size、Length和Discards分别表示队列的当前报文数、最大可容纳报文数和丢弃的报文数。通过检查Discards在一段时间内与Size和Length的关系，可以考察该接口的性能是否满足要求。例如：当Discards的值较大时可能表示路由器正忙于进行其他工作，而无法及时处理进入的分组，如果这种情况持续很长的时间，则通常表示需要一个更强大的路由器以满足工作的要求

Last physical up time :

接口上次物理Up的时刻，“-”表示接口的此物理状态没有发生过变化。

Last physical down time :

接口上次物理Down的时刻，“-”表示接口的此物理状态没有发生过变化。

Last 5 minutes input Last 5 minutes output

这一字段显示了在前5分钟内通过接口发送和接收的字节速率和报文速率如果该接口处于混杂模式下，即它读取LAN上所有的分组，这样就提供了一种测试在网络中流动的数据的方法如果该接口不处于混杂模式下，即仅对它发送和接收的流量记录，那么这里的数据是该接口发送和接收的流量的测定

Input: 84668 packets, 6279745 bytes

接口收到的总报文数和总字节数

broadcasts

接收或发送的广播报文的数目

multicasts

接收或发送的组播报文的数目

errors

在物理层检测时发现的错误报文数目

runts

接口接收到小于规定的最小报文长度报文数。例如对于以太网报文的最小长度为64字节。接收到的小于64字节的报文将被丢弃

giants

接收到长度大于规定长度的报文数目。此接口接收报文的最大长度为1500字节

CRC

收到了长度正常但CRC校验错误的报文数目

collisions

接收或发送的报文时，检测到冲突的报文数目

Align error

排列错误

Other error

其他错误

Output:23991 packets, 1331395 bytes

接口发送的报文数和总字节数

underruns

因为接口读取内存的速度小于转发的速度而无法发送报文数目

packets had been deferred

因为延时或超时无法发送报文数目
