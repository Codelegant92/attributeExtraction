mpls rsvp-te authentication lifetime

命令功能

mpls rsvp-te authentication lifetime命令用来配置RSVP-TE认证生存时间。

undo mpls rsvp-te authentication lifetime命令用来恢复RSVP-TE认证生存时间的缺省配置。

缺省情况下，RSVP-TE认证生存时间为30分钟。

命令格式

mpls rsvp-te authentication lifetime lifetime

undo mpls rsvp-te authentication lifetime

参数说明

lifetime：认证生存时间，HH：MM：SS格式，取值范围是00：00：01～23：59：59，缺省值为00：30：00。

使用指南

执行该命令前，应先在接口启动验证功能。

RSVP认证时间的功能：

  • 当RSVP邻居之间不存在CR-LSP时可以保持RSVP邻居关系，直到RSVP认证生存时间超时。RSVP认证时间不影响已存在的CR-LSP的状态。

  • 可以防止RSVP认证无法终止。例如，设备RTA和RTB建立好了RSVP认证关系后，如果RTA发给RTB的RSVP报文因为受篡改而导致密钥被破坏，RTB收到报文后发现密钥不正确会将报文丢弃，这样就导致RTA不断地给RTB发送被损坏的RSVP消息而该消息不断的被RTB丢弃。但邻居之间的认证关系无法拆除。这种情况下，需要配置认证生存时间，如果邻居之间在生存时间内收到合法的RSVP报文，则重置RSVP认证生存时间，否则认证时间超时后删除RSVP-TE邻居的认证关系。

使用实例

# 配置认证生存时间为40分钟。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] mpls rsvp-te authentication cipher 123456789
[Quidway-GigabitEthernet1/0/0] mpls rsvp-te authentication lifetime 00:40:00
