protection mode

命令功能

protection mode命令用来配置保护组的保护模式，此命令只对APS保护组有效。

undo protection mode命令用来恢复保护组的保护模式为缺省值。

缺省情况下，保护模式为one-to-one，即1∶1保护模式。

命令格式

protection mode { one-plus-one { bidirectional | unidirectional } | one-to-one }

undo protection mode

参数说明

one-plus-one：1＋1保护模式。该保护模式下，保护倒换有单向倒换与双向倒换两种方式。

bidirectional：双向倒换。

unidirectional：单向倒换。

one-to-one：1∶1保护模式。

使用指南

APS的保护倒换有1∶1保护倒换和1＋1保护倒换两种模式：

  • 1∶1保护倒换：在MAC Tunnel保护组中，正常情况下，只有工作隧道传输业务。如果发生保护倒换，业务将切换到保护隧道进行传输，工作隧道内没有业务流。该模式下，保护倒换是由隧道的发端来执行的。

  • 1＋1保护倒换：在MAC Tunnel保护组中，在隧道的入端，业务将被复制，工作隧道和保护隧道内传输完全相同的业务。正常情况下，隧道的收端只选择从工作隧道接收业务，来自保护隧道的业务将被丢弃。在该模式下，保护倒换是由隧道的收端来执行的。

  • 在1＋1模式下，保护倒换分为单向倒换和双向倒换两种方式：

  • 单向倒换：正反MAC Tunnel分别配置一条保护隧道，当某一方向的MAC Tunnel发生故障时，只需要将该方向的MAC Tunnel工作隧道倒换到保护隧道上，另一个方向的MAC Tunnel不发生倒换

  • 双向倒换：某一方向的MAC Tunnel工作隧道发生故障时，正反两个方向的MAC Tunnel需要同时倒换到保护隧道进行数据传输。

本命令只有在MAC Tunnel保护组配置完成后在工作隧道上可见，而且只能在工作隧道上配置。

执行本命令前，必须先关闭APS协议。

在同一视图下重复执行protection mode命令，新配置覆盖原有配置。

使用实例

# 在工作MAC Tunnel视图下配置保护组的APS协议保护模式为1＋1双向倒换。

<Quidway> system-view
[Quidway] mac-tunnel tunnel-name mytunnel
[Quidway-mac-tunnel-mytunnel] protection mac-tunnel ProTunnel protocol aps
[Quidway-mac-tunnel-mytunnel] protection mode one-plus-one bidirectional
