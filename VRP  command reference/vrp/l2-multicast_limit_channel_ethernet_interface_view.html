l2-multicast limit channel (Ethernet接口视图)

命令功能

l2-multicast limit channel命令用来配置端口和端口+VLAN内节目组内组播组的数量和带宽限制。可以使用不同的VLAN段同时配置多个VLAN。

undo l2-multicast limit channel命令用来取消端口和端口+VLAN内节目组内组播组的数量和带宽限制。

缺省情况下，不对任何端口和端口+VLAN内节目组内组播组的进行数量和带宽限制。

命令格式

l2-multicast limit channel channel-name { max-entry count | bandwidth bandwidth } [ vlan { { vlan-id [ to vlan-id ] } &<1-10> } ]

undo l2-multicast limit channel channel-name { max-entry | bandwidth } [ vlan { { vlan-id [ to vlan-id ] } &<1-10> } ]

参数说明

channel channel-name：节目组名称。字符串形式，大小写敏感，取值范围是1～31。

max-entry count：组播组限制数量。整数形式，取值范围是1～8000。

bandwidth bandwidth：组播组带宽。整数形式，取值范围是1～4194304>，单位是千比特/秒。

vlan vlan-id：节目组内组播组的所在的VLAN编号。整数形式，取值范围是1～4094。

视图

Ethernet接口视图、GigabitEthernet接口视图、Eth-Trunk接口视图

缺省级别

2 ：配置级

使用指南

此命令为覆盖式命令。

使用实例

# 将VLAN 10到20，VLAN 30的基于端口＋VLAN节目组channel1中的组播组数量限制配置为1000。

<Quidway> system-view
[Quidway] interface GigabitEthernet1/0/0
[Quidway-GigabitEthernet1/0/0] l2-multicast limit channel channel1 max-entry 1000 vlan 10 to 20 30

# 将VLAN 10到20，VLAN 30的基于VLAN节目组channel1中的组播组带宽限制配置为1000千比特/秒。

<Quidway> system-view
[Quidway] interface GigabitEthernet1/0/0
[Quidway-GigabitEthernet1/0/0] l2-multicast limit channel channel1 bandwidth 1000 vlan 10 to 20 30
