ttl-mode

命令功能

ttl-mode命令用来配置MPLS对TTL的处理模式。

undo ttl-mode命令用来恢复缺省MPLS对TTL的处理模式。

缺省情况下，MPLS对TTL的处理模式为Pipe。

命令格式

ttl-mode { pipe | uniform }

undo ttl-mode

参数说明

pipe：指定MPLS对TTL的处理模式为Pipe。

uniform：指定MPLS对TTL的处理模式Uniform。

缺省级别

2：配置级

使用指南

配置MPLS对TTL的处理模式与配置MPLS的IP TTL复制功能是一致的。

  • 如果配置为Uniform模式，与执行ttl propagate vpn命令使能IP TTL复制的功能相同。报文在MPLS域中每经过一跳，IP TTL减1。Traceroute的结果将反映报文实际经过的路径。

  • 如果配置为Pipe模式，与没有使能IP TTL复制的功能相同。则报文经过MPLS域时，IP TTL只在Ingress和Egress分别减1。Traceroute的结果不包括MPLS骨干网络中每一跳，就好像Ingress与Egress是直连的。

使用实例

# 配置MPLS对TTL的处理模式为Uniform。

<Quidway> system-view
[Quidway] ip vpn-instance vpna
[Quidway-vpn-instance-vpna] ttl-mode uniform
