static-rp（IPv6）

命令功能

static-rp命令用来配置静态RP。

undo static-rp命令用来恢复缺省配置。

缺省情况下，未配置静态RP。

命令格式

static-rp rp-address [ basic-acl6-number ] [ preferred ]

undo static-rp rp-address

参数说明

rp-address：表示静态RP地址。该地址必须是合法的IPv6单播地址。

basic-acl6-number：表示基本访问控制列表，用来控制静态RP所服务的组播组范围。整数形式，取值范围是2000～2999。

preferred：表示静态RP优先。如果未指定此参数，则优先使用由BSR机制选举出的动态RP。

使用指南

注意：

通过重复执行该命令可以配置最多50个静态RP，但同一个ACL不能对应到多个静态RP。如果不引用ACL，则只能配置一个静态RP。

当网络内仅有一个动态RP时，可以手工配置静态RP而不使用动态RP，这样能够避免C-RP和BSR之间频繁的信息交互占用带宽。为了让静态RP功能正常发挥作用，PIM-SM域内所有路由器都必须配置完全相同的static-rp命令。

如果配置的静态RP地址是本机某个状态为UP的接口地址，本机就作为静态RP。作为静态RP的接口不必使能PIM协议。

如果没有指定ACL访问控制列表，则配置的静态RP为所有组播组服务；如果指定了访问控制列表，但访问控制列表没有配置规则，则所配置的静态RP为所有组服务;否则配置的静态RP只为能够通过该列表过滤的组播组服务。

如果配置静态RP的命令中未携带preferred关键字，则路由器优先选择BSR机制选出的动态RP。如果网络中未配置动态RP或动态RP失效，静态RP才能生效。如果配置静态RP的命令中携带preferred关键字，则路由器优先选择静态RP。

如果存在多个静态RP为某个组服务的情况，则选择IP地址最大的RP为该组服务。

当静态RP引用的ACL规则发生变化时，需要重新为所有组选择静态RP。对于具有相同rp-address地址的配置，新配置将覆盖旧配置。

static-rp命令与acl命令配合使用。在ACL6视图下，通过rule命令的source参数定义静态RP所服务的IPv6组播组范围。

使用实例

# 在公网实例中，配置地址为1111::1111的路由器为静态RP，为ACL2001定义的组提供服务，并且启用静态RP优先。

<Quidway> system-view
[Quidway] pim-ipv6
[Quidway-pim6] static-rp 1111::1111 2001 preferred
