display pim control-message counters

命令功能

display pim control-message counters命令用来查看发送、接收和无效的PIM控制消息数目。

命令格式

display pim control-message counters [ message-type { assert | bsr | crp | graft | graft-ack | hello | join-prune | probe | register | register-stop | state-refresh } ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display pim control-message counters interface interface-type interface-number [ message-type { assert | graft | graft-ack | hello | join-prune | state-refresh | bsr } ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

interface interface-type interface-number：表示路由器的接口类型和接口号。

message-type message-type：指定PIM控制消息类型，包括Assert、BSR、CRP、Graft、Graft-Ack、Hello、Probe、Register、Register-Stop、Join/Prune和State-Refresh消息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display pim control-message counters命令时：

  • 若不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 若指定了interface interface-type interface-number，则仅统计该接口下的各种PIM控制消息的收/发数目。

  • 若指定了message-type message-type，则在路由器所有接口上统计该类型控制消息的收/发数目。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看公网实例下GE1/0/0接口上发送、接收和无效的各种PIM控制消息数目。
