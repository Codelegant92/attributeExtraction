display ipv6 routing-table vpn6-instance

命令功能

display ipv6 routing-table vpn6-instance命令用来查看IPv6 VPN实例路由表的信息。

命令格式

display ipv6 routing-table vpn6-instance vpn6-instance-name [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table vpn6-instance vpn6-instance-name acl acl6-number [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table vpn6-instance vpn6-instance-name ipv6-address [ prefix-length ] [ longer-match ] [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table vpn6-instance vpn6-instance-name ipv6-address1 [ prefix-length1 ] ipv6-address2 [ prefix-length2 ] [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table vpn6-instance vpn6-instance-name ipv6-prefix ipv6-prefix-name [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ip routing-table vpn6-instance vpn6-instance-name statistics [ | count ] [ | { begin | exclude | include } regular-expression ]

display ip routing-table vpn6-instance vpn6-instance-name protocol protocol [ inactive | verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn6-instance-name：VPNv6实例的名称。

verbose：显示当前IPv6 VPN实例路由表中所有路由的详细信息，包括激活和未激活的路由信息。

acl：使用ACL6对输出的信息进行过滤。如果指定的ACL6不存在，将显示所有激活路由的信息。

acl6-number：基本ACL6编号。

longer-match：仅显示与特定网络/掩码匹配的IPv6 VPN路由。

ipv6-address：IPv6目的地址。

prefix-length：前缀长度。

ipv6-address1/ipv6-address2：IPv6地址。ipv6-address1和ipv6-address2共同决定一个地址范围，只有在此范围内的IPv6 VPN路由才会被显示。

prefix-length1/prefix-length2：前缀长度。

ipv6-prefix-name：指定IPv6前缀列表的名称。

statistics：显示IPv6 VPN实例路由表的综合路由统计信息。

protocol：显示指定协议的路由信息。

inactive：只显示未激活路由的概要信息。

verbose：显示激活和未激活路由的详细信息。

protocol：指定显示路由信息的协议。该参数可为下列关键字：

  • direct：显示直连路由信息。

  • static：显示静态路由信息。

  • bgp：显示BGP4+路由信息。

  • isis：显示IS-IS路由信息。

  • ospfv3：显示OSPFv3路由信息。

  • ripng：显示RIPng路由信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

显示内容包括目的地址、前缀长度、协议类型、优先级、度量值、下一跳和输出接口。

该命令（不含verbose参数）只显示当前被优选的路由。

在使用display ipv6 routing-table vpn6-instance vpn6-instance-name ipv6-address prefix-length [ longer-match ] [ verbose ]显示路由信息时，可通过调整参数组合显示所需信息：

  • 指定参数ipv6-address prefix-length：显示与目的地址精确匹配的IPv6 VPN路由。

  • 指定参数ipv6-address prefix-length longer-match：显示与目的地址在输入的前缀范围内匹配的IPv6路由。前缀长度为0时，显示IPv6 VPN实例路由表中所有路由。

例如，名称为vpna的IPv6 VPN实例的路由表中有4条路由，2000::20/128、2000::/100、2000::/64和1000::/64。

  • 如果使用带前缀长度的精确匹配display ipv6 routing-table vpn6-instance vpna 2000:: 64，只有路由2000::/64会显示。

  • 如果使用不带前缀长度的精确匹配display ipv6 routing-table vpn6-instance vpna 2000::，只有路由2000::/100会显示。

  • 如果使用前缀长度非零的longer-match匹配display ipv6 routing-table vpn6-instance vpna 2000:: 127 longer-match，只有路由2000::/100和2000::/64会显示。

  • 如果使用前缀长度为零的longer-match匹配display ipv6 routing-table vpn6-instance vpna 2000:: 0 longer-match，则4条路由都会显示。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

Destination

目的网络/主机的地址

PrefixLength

前缀长度

NextHop

为了将报文发送到目的地址所要经过的紧邻的下一跳IPv6地址

Preference

优先级

Interface

下一跳可达的出接口

Protocol

路由协议名称

State

该路由的当前状态

Cost

路由开销

Tunnel ID

隧道ID，值为0x0表示没有使用隧道或隧道建立不成功

Label

路由携带的标签值

Age

路由生成时间
