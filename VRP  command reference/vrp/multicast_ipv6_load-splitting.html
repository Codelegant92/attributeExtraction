multicast ipv6 load-splitting

命令功能

multicast ipv6 load-splitting命令用来使能IPv6组播路由的负载分担。

undo multicast ipv6 load-splitting命令用来恢复缺省配置。

缺省情况下，未使能IPv6组播路由的负载分担。

命令格式

multicast ipv6 load-splitting { balance-preferred | group | source | source-group }

undo multicast ipv6 load-splitting

参数说明

balance-preferred：表示基于组播均衡优先进行负载分担。该方式解决了根据组播组、组播源或源/组进行负载分担时对于连续组地址分担不均的问题。

group：表示基于组播组进行负载分担。

source：表示基于组播源进行负载分担。

source-group：表示同时基于组播源、组播组进行负载分担。

使用指南

路由器从单播、MBGP、组播静态路由表中选择一条最优路由作为RPF路由，创建组播路由表项，转发组播数据流。存在多条最优路由时，使用multicast ipv6 load-splitting命令，可以实现IPv6组播路由的负载分担。

使用实例

# 设置根据源来进行组播负载分担。

<Quidway> system-view
[Quidway] multicast ipv6 load-splitting source
