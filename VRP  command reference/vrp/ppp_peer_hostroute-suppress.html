ppp peer hostroute-suppress

命令功能

ppp peer hostroute-suppress命令用来使能抑制对端主机路由添加到本端直连路由表的功能。

undo ppp peer hostroute-suppress命令用来去使能抑制对端主机路由添加到本端直连路由表的功能。

缺省情况下，本端的直连路由表中包含对端主机路由。

命令格式

ppp peer hostroute-suppress

undo ppp peer hostroute-suppress

参数说明

无

使用指南

PPP链路不严格限制对端路由和本端路由在同一网段。当PPP链路两端不在同一网段时也可以互通，并且自动将对端不在同一个网段的主机路由加到直连路由表中。

但是当有一端配错了IP地址，另一端也会自动把错误的对端主机路由加到本端直连路由表中，造成在网络中发布错误的路由信息。

可以使用该命令来控制是否将对端IP地址的主机路由添加到本端的直连路由表中。

执行ppp peer hostroute-suppress命令后，本端的直连路由表中将不再包含对端主机路由。

当使能或去使能抑制直连对端主机的路由加入本端的直连路由表功能后，必须重新启动接口，保证配置生效。

目前MP接口不支持此命令。

使用实例

# 使能抑制对端主机路由加入到本端直连路由表中的功能，并重启接口。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] ppp peer hostroute-suppress
[Quidway-Pos1/0/0] shutdown
[Quidway-Pos1/0/0] undo shutdown
