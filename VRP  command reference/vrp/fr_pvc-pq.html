fr pvc-pq

命令功能

fr pvc-pq命令用来将帧中继接口的队列类型设置为PVC PQ（PVC Priority Queueing，虚电路优先级队列），并可以为各队列设置长度，即队列最多能容纳的报文个数。

undo fr pvc-pq命令用来将接口的队列类型恢复为FIFO。

缺省情况下，帧中继接口的队列类型为FIFO。

命令格式

fr pvc-pq [ top-limit middle-limit normal-limit bottom-limit ]

undo fr pvc-pq

参数说明

top-limit：高优先队列的队列长度，取值范围是1～1024，缺省值为20。

middle-limit：中优先队列的队列长度，取值范围是1～1024，缺省值为40。

normal-limit：正常优先队列的队列长度，取值范围是1～1024，缺省值为60。

bottom-limit：低优先队列的队列长度，取值范围是1～1024，缺省值为80。

缺省级别

2：配置级

使用指南

在执行fr pvc-pq命令前必须先执行fr traffic-shaping命令使能接口的帧中继流量整形功能。

当接口使能了帧中继流量整形之后，接口队列类型只能为FIFO或者PVC PQ。

PVC PQ是FRTS新增的一种队列机制，它类似于PQ队列，也有四种队列类型：top、middle、normal、bottom，队列优先级依次降低。在帧中继class中配置DLCI进入PVC PQ的哪个队列。当接口发生拥塞时，不同DLCI入不同的PVC PQ队列。发送时，按照队列优先级，在优先级高的队列发送完毕后，再发送低优先级队列中的报文。

使用实例

# 设置接口Serial 2/0/0的队列类型为PVC PQ。

<Quidway> system-view
[Quidway] interface serial2/0/0
[Quidway-Serial2/0/0] fr pvc-pq
