mtrace-response-address

命令功能

mtrace-response-address命令配置NQA mtrace的回应地址，即IGMP-Tracert-Response报文的目的地址。

undo mtrace-response-address命令用来恢复缺省情况。

缺省情况下，NQA mtrace的回应地址为224.0.1.32，TTL为64。

命令格式

mtrace-response-address ipv4 ip-address [ ttl value ]

undo mtrace-response-address

参数说明

ipv4 ip-address：设置NQA mtrace的回应地址，即Response报文的目的地址。可以为单播IPv4地址，也可以是组播IPv4地址。

ttl value：设置Response报文的TTL，该配置只对组播回应地址有效。取值范围是1～255。

视图

缺省级别

2：配置级

使用指南

IGMP-Tracert-Response报文可以使用的目的地址：

  • 查询者本地接口地址，属于IPv4单播地址。

  • 被追踪的组播组地址，仅适用于查询者位于组播分发树上的情况。

使用实例

# 设置NQA mtrace测试例user test的回应地址为本地接口地址11.1.1.1。

[Quidway-nqa-user-test] mtrace-response-address ipv4 11.1.1.1

# 设置NQA mtrace测试例user test的回应地址为被追踪的组地址225.1.1.1。

[Quidway-nqa-user-test] mtrace-response-address ipv4 225.1.1.1 ttl 10

# 设置NQA mtrace测试例user test使用缺省回应地址，设置TTL为10。

[Quidway-nqa-user-test] mtrace-response-address ipv4 224.0.1.32 ttl 10

# 恢复NQA mtrace测试例user test的回应地址为224.0.1.32。

[Quidway-nqa-user-test] undo mtrace-response-address
