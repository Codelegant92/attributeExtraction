ip as-path-filter

命令功能

ip as-path-filter命令用来增加一个AS路径过滤器表项。

undo ip as-path-filter命令用来删除指定的AS路径过滤器。

命令格式

ip as-path-filter as-path-filter-number { deny | permit } regular-expression

undo ip as-path-filter as-path-filter-number [ { deny | permit } regular-expression ]

参数说明

as-path-filter-number：指定的AS路径过滤器号。整数形式，取值范围1～256。

deny：指定AS路径过滤器的匹配模式为拒绝。

permit：指定AS路径过滤器的匹配模式为允许。

regular-expression：AS路径正则表达式。

使用指南

配置了AS路径过滤器后，缺省情况下，RM会立即通知各个协议应用该过滤器进行过滤。如果需要延迟过滤器的生效时间，需使用命令route-policy-change notify-delay配置延迟生效时间。

使用实例

# 创建序号为1的AS路径过滤器，允许AS路径中以10开始的路由通过。

<Quidway> system-view
[Quidway] ip as-path-filter 1 permit ^10

# 创建序号为2的AS路径过滤器，允许AS路径中包含20的路由通过。

<Quidway> system-view
[Quidway] ip as-path-filter 2 permit [ 20 ]

# 创建序号为3的AS路径过滤器，不允许AS路径中包含30的路由通过。

<Quidway> system-view
[Quidway] ip as-path-filter 3 deny [ 30 ]
[Quidway] ip as-path-filter 3 permit .*
