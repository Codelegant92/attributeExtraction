authentication-mode（认证方案视图）

命令功能

authentication-mode命令用来配置当前认证方案使用的认证模式。

缺省情况下，认证模式为local，即本地认证。

命令格式

authentication-mode { hwtacacs | radius | local }* [ none ]

authentication-mode none

参数说明

hwtacacs：采用HWTACACS认证模式。

radius：采用RADIUS认证模式。

local：采用本地认证模式。

none：不进行认证，直接让用户通过。

视图

认证方案视图

缺省级别

3：管理级

使用指南

如果在一个认证方案中采用多种认证模式，将按照配置的顺序进行认证。

  • 当配置的认证方式是先本地认证后远端认证时

    如果登录的帐号在本地没有创建，但是在远端服务器上是存在的，认证模式将由本地认证转入远端认证。

  • 当配置的认证方式是先远端认证后本地认证时

    如果登录的帐号在远端服务器上没有创建，但是在本地是存在的，经过远端认证时，将被认为认证失败，不再转入本地认证。

    只有在远端认证服务器Down时，才会转入本地认证。

如果选用了不认证（none），它必须做为最后一种认证模式。

使用实例

# 配置名为scheme1的认证方案采用本地认证。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authentication-scheme scheme1
[Quidway-aaa-authen-scheme1] authentication-mode local

# 配置名为scheme2的认证方案首次采用RADIUS认证，二次认证为本地认证，然后是HWTACACS认证，若这三种认证都没有响应则让用户通过。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authentication-scheme scheme2
[Quidway-aaa-authen-scheme2] authentication-mode radius local hwtacacs none
