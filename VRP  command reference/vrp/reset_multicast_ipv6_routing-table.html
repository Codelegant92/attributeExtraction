reset multicast ipv6 routing-table

命令功能

reset multicast ipv6 routing-table命令用来重置组播路由表中的路由表项，转发表中相应的转发项将同时被删除。

命令格式

reset multicast ipv6 routing-table all

reset multicast ipv6 routing-table { ipv6-group-address [ ipv6-group-mask-length ] | ipv6-source-address [ ipv6-source-mask-length ] | incoming-interface { interface-type interface-number | register } } *

参数说明

all：重置组播核心路由表上的所有路由项。

ipv6-group-address：指定组播组的IPv6地址。

ipv6-group-mask-length：组播组地址的掩码长度。取值范围是8～128。

ipv6-source-address：指定组播源的IPv6地址。

ipv6-source-mask-length：指定组播源地址的掩码长度。取值范围是0～128。

incoming-interface：指定路由项的入接口。

interface-type interface-number：指定接口类型和接口编号。

register：表示IPv6 PIM的注册接口。

使用指南

使用reset multicast ipv6 routing-table命令：

  • 如果指定ipv6-group-address，则仅删除该组地址对应的路由表项。

  • 如果指定ipv6-group-address ipv6-group-mask-length，则删除该组地址范围对应的路由表项。

  • 如果指定ipv6-source-address，则仅删除该源地址对应的路由表项。

  • 如果指定ipv6-source-address ipv6-source-mask-length，则删除该源地址范围对应的路由表项。

  • 参数ipv6-group-address和ipv6-source-address的次序可以颠倒。必须保证ipv6-group-address和ipv6-source-address合法，否则系统会提示输入错误。

  • 如果指定incoming-interface interface-type interface-number，则仅删除上游接口为指定接口的路由表项。

  • 如果指定incoming-interface register，则仅删除上游接口为注册接口的路由表项。

使用实例

# 从组播核心路由表中重置组播组地址为FF02::101的路由项。

<Quidway> reset multicast ipv6 routing-table FF02::101
