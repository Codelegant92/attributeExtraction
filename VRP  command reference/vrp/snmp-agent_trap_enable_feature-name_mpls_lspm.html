snmp-agent trap enable feature-name mpls_lspm

命令功能

snmp-agent trap enable feature-name mpls_lspm命令用来打开MPLS LSPM模块的告警开关。

undo snmp-agent trap enable feature-name mpls_lspm命令用来关闭MPLS LSPM模块的告警开关。

缺省情况下，MPLS LSPM模块的告警开关处于开启状态。

命令格式

snmp-agent trap enable feature-name mpls_lspm { non-excessive all | trap-name { hwmplsstaticcrlspdown | hwmplsstaticcrlspup | hwmplsstaticlspdown | hwmplsstaticlspup | hwmplstefrrprotaval | hwmplstefrrprotnotaval | hwmplstefrrresume | hwmplstefrrswitch | hwmplstunnelchangebw | hwmplstunneldown | hwmplstunnelhsbresume | hwmplstunnelhsbswitch | hwmplstunnelobresume | hwmplstunnelobswitch | hwmplstunnelup | mplstunneldown | mplstunnelreoptimized | mplstunnelrerouted | mplstunnelup | mplsxcdown | mplsxcup } }

undo snmp-agent trap enable feature-name mpls_lspm { non-excessive all | trap-name { hwmplsstaticcrlspdown | hwmplsstaticcrlspup | hwmplsstaticlspdown | hwmplsstaticlspup | hwmplstefrrprotaval | hwmplstefrrprotnotaval | hwmplstefrrresume | hwmplstefrrswitch | hwmplstunnelchangebw | hwmplstunneldown | hwmplstunnelhsbresume | hwmplstunnelhsbswitch | hwmplstunnelobresume | hwmplstunnelobswitch | hwmplstunnelup | mplstunneldown | mplstunnelreoptimized | mplstunnelrerouted | mplstunnelup | mplsxcdown | mplsxcup } }

参数说明

non-excessive all：打开MPLS LSPM模块的所有非海量告警开关。

trap-name：打开指定类型事件的告警开关。

hwmplsstaticcrlspdown：打开静态CR-LSP变为Down事件的私有告警开关。

hwmplsstaticcrlspup：打开静态CR-LSP变为Up事件的私有告警开关。

hwmplsstaticlspdown：打开静态LSP变为Down事件的私有告警开关。

hwmplsstaticlspup ：打开静态LSP变为Up事件的私有告警开关。

hwmplstefrrprotaval：打开TE FRR旁路隧道绑定事件的私有告警开关。

hwmplstefrrprotnotaval：打开TE FRR旁路隧道去绑定事件的私有告警开关。

hwmplstefrrresume：打开TE FRR故障恢复事件的私有告警开关。

hwmplstefrrswitch：打开TE FRR保护切换事件的私有告警开关。

hwmplstunnelchangebw：打开隧道带宽发生变化事件的私有告警开关。

hwmplstunneldown：打开隧道变为Down事件的私有告警开关。

hwmplstunnelhsbresume：打开CR-LSP热备份方式下从备份LSP切换到主LSP事件的私有告警开关。

hwmplstunnelhsbswitch：打开CR-LSP热备份方式下从主LSP切换到备份LSP事件的私有告警开关。

hwmplstunnelobresume：打开CR-LSP普通备份方式下从备份LSP切换到主LSP事件的私有告警开关。

hwmplstunnelobswitch：打开CR-LSP普通备份方式下从主LSP切换到备份LSP事件的私有告警开关。

hwmplstunnelup：打开隧道变为Up事件的私有告警开关。

mplstunneldown：打开隧道变为Down事件的告警开关。

mplstunnelreoptimized：打开隧道重优化事件的告警开关。

mplstunnelrerouted：打开隧道重路由事件的告警开关。

mplstunnelup：打开隧道变为Up事件的告警开关。

mplsxcdown：打开LSP变为Down事件的告警开关。

mplsxcup：打开LSP变为Up事件的告警开关。

使用指南

目前定义的MPLS LSPM模块的海量告警包括：mplsxcup和mplsxcdown，其余都是非海量告警。

如果要打开MPLS LSPM模块所有事件的非海量告警开关，则选择non-excessive all批量处理；如果只打开某个或几个事件的告警开关时，则选择trap-name。

使用实例

# 打开LSP变为Down事件的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name mpls_lspm trap-name mplsxcdown
