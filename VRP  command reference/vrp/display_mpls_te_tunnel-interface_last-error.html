display mpls te tunnel-interface last-error

命令功能

display mpls te tunnel-interface last-error命令用来查看本地节点的隧道接口最近发生的错误。

命令格式

display mpls te tunnel-interface last-error [ tunnel-name ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

tunnel-name：只显示指定隧道接口的最新错误信息。tunnel-name为字符串形式，不支持空格，区分大小写，不支持缩写形式。以Tunnel1/0/0为例，输入格式必须是Tunnel1/0/0，不能是Tunnel 1/0/0或者tunnel1/0/0，也不能是T1/0/0或tun 1/0/0。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

display mpls te tunnel-interface last-error命令可用来在Ingress节点查看本节点发生的错误或者下游节点发送的PathErr消息携带的错误，包括：
  • CSPF计算失败；

  • RSVP超时；

  • RSVP GR平滑过程中删除LSP；
  • 老化过程中删除LSP；
  • 触发RSVP GR出错。

该命令针对每条TE隧道记录最近5次错误的信息，并细分出错的是主CR-LSP、Modified LSP还是备份CR-LSP。

若是下游PathErr消息携带的错误，显示信息的“Error Info”字段会提示下游发生错误的节点的IP地址并给出错误原因，若是Ingress发生错误则只记录错误原因。

使用实例

表1 display mpls te tunnel-interface last-err命令输出信息描述

项目

描述

Tunnel Name

隧道的名称

Error Num

错误的编号

LSP Type

LSP的类型：

  • Modified LSP：处于Modified状态的LSP
  • Primary LSP：主LSP
  • Backup LSP：备份LSP

LSP ID

LSP的ID，格式为Ingress-LSR-ID/Tunnel-ID/Local LSP ID

Error Code

错误码，十六进制整数，用于唯一标识一类错误

Occur Time

错误产生的时间

Error Info

错误的描述信息

error node

发生错误的节点

Routing Problem

表示从收到的RSVP PathErr消息得知的错误，常见的有：

  • No route available toward destination：表示没有到隧道目的地址的路由
  • Bad EXPLICT_ROUTE object：表示收到无法识别的EXPLICT_ROUTE对象
  • bad strict node：找不到严格显示路径中指定的某一节点

Traffic Control Error

流量控制参数错误，常见的有：

  • Service conflict：业务冲突，即试图合并两个不兼容的资源请求
  • Bad Tspec value：不合理或无法识别的资源请求

Cspf failed to calculate a path for Tunnel.

CSPF计算失败
