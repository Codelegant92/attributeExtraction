reset igmp group

命令功能

reset igmp group命令用来删除公网实例或VPN实例或所有实例的接口上动态加入的IGMP组记录。

命令格式

reset igmp [ vpn-instance vpn-instance-name | all-instance ] group { all | interface interface-type interface-number { all | group-address [ mask { group-mask | group-mask-length } ] [ source-address [ mask { source-mask | source-mask-length } ] ] } }

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称，字符串形式，长度范围是1～31。

all-instance：指定所有实例。

all：表示所有IGMP组。

interface interface-type interface-numbe：表示路由器的接口类型和接口号，用来指定具体的接口。

group-address：表示组播组地址，用来指定一个组播组。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

mask：指定组播组或源的地址掩码。

group-mask：表示组播组地址掩码。点分十进制形式。

group-mask-length：表示组播组地址掩码长度。整数形式，取值范围是4～32。

source-address：表示组播源地址，用来指定一个组播源。点分十进制形式。

source-mask：表示组播源地址掩码。点分十进制形式。

source-mask-length：表示组播源地址掩码长度。整数形式，取值范围是0～32。

使用指南

本命令仅对动态加入的IGMP组有效，对静态IGMP组无效。

使用reset igmp group命令时：

  • 若不指定vpn-instance或all-instance，则只删除公网实例的所有已动态加入的IGMP组记录。

  • 若使用reset igmp group all命令，则删除路由器所有接口上的所有已动态加入的IGMP组记录。

  • 若指定interface interface-type interface-number all，则删除特定接口上所有已动态加入的IGMP组记录。

  • 若指定group-address参数，则删除指定组地址对应的IGMP组记录。

  • 若指定source-address参数，则删除指定源地址对应的IGMP组记录。

  • 删除IGMP组记录后，不影响该组的再次加入。

使用实例

# 删除公网实例所有接口上的所有IGMP组记录。

<Quidway> reset igmp group all

# 删除公网实例接口GE1/0/0上的所有IGMP组记录。

<Quidway> reset igmp group interface gigabitethernet1/0/0 all

# 删除公网实例接口GE1/0/0上的组地址225.0.0.1对应的IGMP组记录。

<Quidway> reset igmp group interface gigabitethernet1/0/0 225.0.0.1

# 删除公网实例接口GE1/0/0上组地址范围在225.1.1.0到225.1.1.255网段之间的IGMP组记录。

<Quidway> reset igmp group interface gigabitethernet1/0/0 225.1.1.0 mask 255.255.255.0
