x25 xot pvc

命令功能

x25 xot pvc命令用来增加一条XOT的PVC路由。

undo x25 pvc命令用来删除指定XOT的PVC路由。

缺省情况下，系统没有PVC路由。

命令格式

x25 xot pvc pvc-number1 ip-address interface serial interface-number pvc pvc-number2 [ xot-option ]

undo x25 pvc pvc-number1

参数说明

pvc-number1：本端接口上的永久虚电路号，取值范围是1～4095，并且应在实际永久虚电路范围内，永久虚电路的范围是1～（Lic－1）。

ip-address：XOT连接的对端目的IP地址。

interface-number：接口编号。

pvc-number2：对端接口上的永久虚电路号，取值范围是1～4095，并且应在实际永久虚电路范围内，永久虚电路的范围是1～（Lic－1）。

xot-option：XOT通道参数选项，具体配置请参照以下参数及下表。

packet-size input-packet output-packet：指定该虚电路的最大接收分组长度和最大发送分组长度，以字节为单位，这两个值必须在16到4096范围内（包括16和4096），并且必须是2的整数次幂。

window-size input-window-size output-window-size：指定该虚电路的接收窗口和发送窗口的尺寸，当X.25的窗口模数为8时，input-window-size和output-window-size的取值范围是1～7；当X.25的窗口模数为128时，取值范围是1～127。缺省情况下，接收窗口和发送窗口的值都为2。

表1 XOT通道参数选项

选项

意义

timer seconds

XOT连接的keepalive定时器的延时，定时器超时发送keepalive报文，以探测连接的可用性，取值范围是1～3600秒。

retry times

发送keepalive的最大失败次数，失败次数超过times时，断开XOT连接。取值范围是3～3600次。

source interface-type interface-number

发起XOT连接的接口。

缺省级别

2：配置级

使用指南

用户配置X25 PVC的XOT交换命令后，可以从本端的X.25网跨越IP网实现和远端X.25网的互联。配置keepalive属性后，可支持XOT的链路检测。

使用实例

# 在X.25虚电路范围配置好的情况下，将路由器RTA接口Serial1/0/0（10.1.1.1）的PVC1与对端路由器RTB接口Serial1/0/0（10.1.1.2）的PVC2通过XOT隧道互连，进行分组交换。

# 在路由器RTA上进行配置

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] ip address 10.1.1.1 255.255.255.0
[Quidway-Serial1/0/0]link-protocol x25 dte ietf
[Quidway-Serial1/0/0]x25 xot pvc 1 10.1.1.2 interface serial1/0/0 pvc 2

# 在路由器RTB上进行配置

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] ip address 10.1.1.2 255.255.255.0
[Quidway-Serial1/0/0]link-protocol x25 dce ietf
[Quidway-Serial1/0/0]x25 xot pvc 2 10.1.1.1 interface serial1/0/0 pvc 1
