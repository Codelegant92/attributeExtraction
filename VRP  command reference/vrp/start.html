start

命令功能

start命令用来配置NQA测试例的启动方式和结束方式。

缺省情况下，测试报文发送完毕后，测试自动结束。

命令格式

start at [ yyyy/mm/dd ] hh:mm:ss [ end { at [ yyyy/mm/dd ] hh:mm:ss | delay { seconds second | hh:mm:ss } | lifetime { seconds second | hh:mm:ss } } ]

start delay { seconds second | hh:mm:ss } [ end { at [ yyyy/mm/dd ] hh:mm:ss | delay { seconds second | hh:mm:ss } | lifetime { seconds second | hh:mm:ss } } ]

start now [ end { at [ yyyy/mm/dd ] hh:mm:ss | delay { seconds second | hh:mm:ss } | lifetime { seconds second | hh:mm:ss } } ]

参数说明

start at：在指定的时刻开始测试例的执行。

start delay：延迟启动测试例的执行。

start now：立即启动当前测试例的执行。

end at：在指定的时刻结束当前测试例的执行。

end delay：延迟结束测试例的执行。

yyyy/mm/dd：测试例启动或停止的日期，yyyy指定年，mm指定月，dd指定天。年的取值范围是2000～2099。

hh:mm:ss：测试例启动或停止的时刻。hh指定小时，mm指定分钟，ss指定秒。

seconds second：延迟指定的时间后启动或停止。延迟时间的单位是秒，取值范围是1～86399。

hh:mm:ss：延迟指定的时间后启动或停止。hh范围是0～23，mm和ss范围是0～59。以这种方式指定延迟时间后，系统会自动转换成以秒表示的形式。

lifetime：配置测试例的持续时间。

seconds second：以秒为单位设置测试例的生命周期。取值范围是1～86399。

hh:mm:ss：生命周期的时刻。hh指定小时，mm指定分钟，ss指定秒。

使用指南

如果当前正在执行的测试例总数已经达到系统允许的最大数目，start命令将失败。

start命令一旦被执行，即测试例一旦执行结束，start命令就会从配置文件中删除。

对于同一个测试例，执行start now命令需要在上一次测试执行完成后，才能再次执行。

使用实例

# 立即开始执行当前的测试例。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] start now

# 使测试例在当前时间的10小时后启动。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] start delay 10:00:00
