dhcp snooping check dhcp-rate alarm threshold

命令功能

dhcp snooping check dhcp-rate alarm threshold命令用来配置DHCP报文上送到DHCP协议栈的速率的告警阈值，丢弃的报文数超过此阈值发送告警。

undo dhcp snooping check dhcp-rate alarm threshold命令用来配置DHCP报文上送到DHCP协议栈的速率的告警阈值为缺省值。

缺省情况下，三层接口和VLAN的告警阈值为全局告警阈值，即100个/秒。可以使用dhcp snooping alarm threshold threshold命令修改缺省值。

命令格式

dhcp snooping check dhcp-rate alarm threshold threshold

undo dhcp snooping check dhcp-rate alarm threshold

参数说明

threshold：DHCP速率限制的告警阈值，使能DHCP速率检查后，丢弃的报文达到此告警阈值，将产生告警信息。取值范围1~1000，单位个/秒，全局告警阈值的缺省值是100个/秒。

使用指南

执行dhcp snooping check dhcp-rate alarm threshold命令后，在一定的时间内只允许规定数目的报文上送协议栈处理，超过速率规定的报文将被丢弃。

配置了告警功能后，如果丢弃的报文达到此命令设置的告警阈值，设备将向网管产生告警。

使用实例

# 设置检查DHCP报文速率的告警阈值为50个/秒。

<Quidway> system-view
[Quidway] dhcp snooping check dhcp-rate alarm threshold 50
