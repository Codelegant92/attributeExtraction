link-protocol fr

命令功能

link-protocol fr命令用来封装接口链路层协议为帧中继。

缺省情况下，接口的链路层协议封装为PPP。当封装帧中继协议时，缺省情况下，帧的封装格式为IETF。

命令格式

link-protocol fr [ ietf | nonstandard ]

参数说明

ietf：IETF标准封装，按照RFC1490规定的格式进行封装，为缺省封装格式。

nonstandard：非标准兼容的封装格式。

视图

帧中继接口视图、MFR接口视图

缺省级别

2：配置级

使用指南

如果要配置帧中继网络，必须先把接口的协议类型封装为帧中继。

说明：
  • 改变接口的帧中继封装格式时，如果该接口下有子接口，需要先删除子接口。
  • 改变接口的帧中继封装格式后，系统会自动删除该接口下帧中继的所有配置。此时需要重新进行帧中继的相关配置。

使用实例

# 在接口Serial1/0/0:0上封装帧中继，并选择非标准兼容封装格式。

<Quidway> system-view
[Quidway] interface serial 1/0/0:0
[Quidway-Serial1/0/0:0] link-protocol fr nonstandard
