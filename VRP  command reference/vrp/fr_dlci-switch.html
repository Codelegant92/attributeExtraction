fr dlci-switch

命令功能

fr dlci-switch命令用来配置帧中继PVC交换的静态路由。

undo fr dlci-switch命令用来删除帧中继PVC交换静态路由。

缺省情况下，没有配置帧中继PVC交换的静态路由。

命令格式

fr dlci-switch in-dlci interface interface-type interface-number dlci out-dlci

undo fr dlci-switch in-dlci

参数说明

in-dlci：接口上收到报文的DLCI。取值范围是16～1007。

interface-type：出接口的接口类型。

interface-number：出接口的接口编号，三维形式：槽号/卡号/接口号。

out-dlci：路由器转发报文的指定接口的DLCI。取值范围是16～1007。

缺省级别

2：配置级

使用指南

用于帧中继交换功能的接口，必须使用命令fr interface-type将接口类型配置为NNI或DCE，否则帧中继交换功能将不起作用。

在配置帧中继PVC的静态路由之前或之后，必需使用fr switching命令才能使能帧中继PVC交换静态路由。

转发报文的接口类型可以是帧中继接口或MFR接口，如果指定tunnel接口作为转发接口，可以实现在IP网络上承载帧中继报文。

使用实例

# 配置一条静态路由，允许在Serial1/0/0上DLCI为100的链路上的报文通过Serial2/0/0上DLCI为200的链路转发。

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] fr dlci-switch 100 interface serial2/0/0 dlci 200

# 配置一条静态路由，允许在Serial4/1/2上DLCI为200的链路上的报文通过Tunnel接口4/0/0上DLCI为300的链路转发。

[Quidway-Serial4/1/2] fr dlci-switch 200 interface tunnel4/0/0 dlci 300
