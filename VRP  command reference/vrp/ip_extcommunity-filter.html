ip extcommunity-filter

命令功能

ip extcommunity-filter命令用来增加一个扩展团体属性过滤器。

undo ip extcommunity-filter命令用来删除特定的扩展团体属性过滤器。

命令格式

ip extcommunity-filter extcomm-filter-number { deny | permit } { rt { as-number : nn | ipv4-address : nn } } &<1-16>

undo ip extcommunity-filter extcomm-filter-number [ { deny | permit } { rt { as-number : nn | ipv4-address : nn } } &<1-16> ]

参数说明

extcomm-filter-number：扩展团体属性过滤器号。

rt：指定Route Target（路由目标）扩展团体属性。

as-number：自治系统号。

ipv4-address：IPv4地址。

nn：一个整数。对于as-number，其取值范围为0～4294967295；对于ipv4-address，其取值范围为0～65535。

permit：指定扩展团体属性过滤器的匹配模式为允许。

deny：指定扩展团体属性过滤器的匹配模式为拒绝。

使用指南

扩展团体属性过滤器可以作为Route-Policy中的匹配条件。

配置了RD属性过滤器后，缺省情况下，RM会立即通知各个协议应用该过滤器进行过滤。如果需要延迟过滤器的生效时间，需使用命令route-policy-change notify-delay配置延迟生效时间。

undo ip extcommunity-filter命令可以删除单个节点的扩展团体属性过滤器。

使用实例

# 配置序号为1的RT扩展团体属性过滤器。

<Quidway> system-view
[Quidway] ip extcommunity-filter 1 deny rt 200:200
