nqa event

命令功能

nqa event entry-entry命令用来配置NQA告警关联的事件。

undo nqa event entry-entry命令用来取消NQA告警关联的事件。

缺省情况下，无NQA告警的关联事件。

命令格式

nqa event entry-entry { log | trap | log-trap | | none } [ description ]

undo nqa event entry-entry

参数说明

entry-entry事件表号。整数形式，取值范围是1～65535。

log：对事件的处理方式是发送日志。

trap：对事件的处理方式是向网管站发trap消息。

log-trap：对事件的处理方式是发送日志，并同时向网管站发trap消息。

none：对事件的处理方式是标记为没有事件发生。

description：事件的描述。字符串形式，长度范围是1～127。

使用指南

执行undo nqa event命令，将删除了一条相应的告警事件。不能删除正在被告警绑定的事件。
说明：
用户在配置时要注意，看是否事件行已经存在，如果输入已存在的行号，而且原事件没有被告警关联，会将该行号对应的原来的告警事件的配置覆盖。

使用实例

# 创建事件表号10的事件，触发此事件会发送日志。

<Quidway> system-view
[Quidway] nqa event 10 log
