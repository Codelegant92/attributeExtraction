display ospf bfd session

命令功能

display ospf bfd session命令用来查看使能BFD特性邻居的信息。

命令格式

display ospf [ process-id ] bfd session interface-type interface-number [ router-id ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ospf [ process-id ] bfd session { router-id | all } [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

interface-type interface-number：接口类型及接口号。

router-id：对端的Router ID。

process-id：OSPF进程号，取值范围是1～65535。

all：指定该OSPF进程下所有使能了OSPF的接口。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看邻居BFD信息。

<Quidway> display ospf bfd session all
          OSPF Process 1 with Router ID 1.1.1.1

项目

描述

NeighborId

邻居Router ID

AreaId

所在区域ID

Interface

与邻居建立BFD会话的接口

BFDState

BFD状态

Rx

经过协商后的BFD报文的最小接收间隔

Tx

经过协商后的BFD报文的最小发送间隔

Multiplier

远端的检测倍数

BFD Local Dis

BFD动态分配的本地标示符

LocalIpAdd

本地IP地址

RemoteIpAdd

远端IP地址

Diagnostic Info
诊断信息：

  • Init：BFD会话正处于初始状态

  • Admin down：本端BFD会话执行了shutdown操作

  • BFD global disable：BFD未全局使能

  • BFD session number exceed：BFD会话数量超过规格限制

  • Detect down：本端链路进入Down状态

  • Receive admin down：对端链路进入Down状态

  • BFD is in rearranging：建立BFD会话的单板变更，BFD会话数据转移到其它单板上
