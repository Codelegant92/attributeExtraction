mpls l2vpn oam-mapping

命令功能

mpls l2vpn oam-mapping命令用来使能AC侧状态和PSN(Packet Switching Network)侧状态的映射功能。

undo mpls l2vpn oam-mapping命令用来禁用AC侧状态和PSN侧状态的映射功能。

缺省情况下，系统没有使能AC侧状态和PSN侧状态的映射功能。

命令格式

mpls l2vpn oam-mapping [ auto ]

mpls l2vpn oam-mapping { 1ag md md-name ma ma-name | 3ah }

undo mpls l2vpn oam-mapping

参数说明

auto：自动使能故障检测和故障通告功能，此参数只有在PPP、FR、HDLC类型的接口下才可以配置。

1ag：使能通过IEEE802.1ag协议进行的故障检测和故障通告的功能，此参数只有在Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图下才可以配置。

md md-name：维护域名称。字符串形式，不支持空格，区分大小写，长度范围是1～43。

ma ma-name：维护联盟名称。字符串形式，不支持空格，区分大小写，长度范围是1～43。

3ah：使能通过IEEE802.3ah协议进行的故障检测和故障通告的功能，此参数只有在Ethernet接口视图、GE接口视图下才可以配置。

使用指南

配置mpls l2vpn oam-mapping命令后，系统建立AC状态和PSN侧状态的映射表，以实现AC侧故障通告给远端PE，及PSN故障通告给本地CE。

  • 只有在PPP、FR或HDLC类型的接口上配置了VLL或者PWE3，才能在该接口上配置mpls l2vpn oam-mapping [ auto ]命令。

  • 只有以太类型的接口上配置了VLL或者PWE3并且使能了以太OAM，才能在该接口下配置mpls l2vpn oam-mapping { 1ag md md-name ma ma-name | 3ah }命令。

使能了AC侧状态和PSN侧状态的映射后，当本地AC侧发生故障时：

  • 如果使能了BFD for PW，则通过BFD通告AC故障给对端PE。

  • 如果没有使能BFD for PW，则通过信令通告AC故障给对端PE。

如果配置了auto参数后，无需使用oam命令配置接口的OAM故障检测和OAM故障通告。系统根据AC链路的类型自动配置OAM故障检测和OAM故障通告。

使用实例

# 在HDLC类型的接口上，使能故障检测和故障通告功能。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] link-protocol hdlc
[Quidway-Pos1/0/0] mpls l2vc 1.1.1.1 100
[Quidway-Pos1/0/0] mpls l2vpn oam-mapping auto

# 在以太类型的接口上，使能通过IEEE802.3ah协议进行的故障检测和故障通告的功能。

<Quidway> system-view
[Quidway] efm enable
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] efm enable
[Quidway-GigabitEthernet1/0/0] mpls l2vc 1.1.1.1 101
[Quidway-GgiabitEthernet1/0/0] mpls l2vpn oam-mapping 3ah
