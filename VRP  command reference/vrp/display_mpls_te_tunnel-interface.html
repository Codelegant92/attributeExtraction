display mpls te tunnel-interface

命令功能

display mpls te tunnel-interface命令用来查看本地节点的隧道接口信息。

命令格式

display mpls te tunnel-interface [ tunnel tunnel-number | auto-bypass-tunnel tunnel-name ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

tunnel-number：本节点存在的隧道接口编号。

auto-bypass-tunnel：显示某一条指定的自动生成的Bypass tunnel信息。

tunnel-name：隧道名称，字符串形式，不支持空格，长度范围是1～63。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

项目

描述

Tunnel Name

隧道的名称

Tunnel State Desc
隧道状态的描述：
  • CR-LSP is Up：隧道正在使用
  • CR-LSP is Down：隧道未建立成功
  • Modifying CR-LSP is waiting for deleting：隧道正在等待删除
  • Modifying CR-LSP is waiting for switching：隧道正在等待切换
  • Primary CR-LSP Up and HotBackup CR-LSP Up：配置了CR-LSP热备份，且主、备CR-LSP都建立成功
  • Primary CR-LSP UP and HotBackup CR-LSP setting Up：配置了CR-LSP热备份，且主CR-LSP的状态为Up，正在创建热备份CR-LSP
  • Primary CR-LSP Down and Backup CR-LSP setting Up：配置了CR-LSP普通备份或逃生路径，且主CR-LSP故障，正在试图建立普通备份CR-LSP或逃生路径。或者配置了CR-LSP热备份和逃生路径，且主CR-LSP和热备份CR-LSP都故障时，正在试图建立逃生路径。
  • Backup CR-LSP In use and Primary CR-LSP setting Up：配置了CR-LSP热备份或普通备份或逃生路径，且主CR-LSP故障，备份CR-LSP或逃生路径建立成功，使用备份CR-LSP或逃生路径传输数据
  • CR-LSP is setting Up：正在试图建立CR-LSP
  • CR-LSP setup Failed：CR-LSP建立不成功
  • Primary CR-LSP UP and Ordinary CR-LSP setting Up：配置了CR-LSP普通备份，且主CR-LSP处于Frr-in-use状态时，正在试图建立普通备份CR-LSP
