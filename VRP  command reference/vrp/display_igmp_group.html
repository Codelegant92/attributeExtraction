display igmp group

命令功能

display igmp group命令用来查看IGMP组播组信息，显示信息包括通过成员报告动态加入的组播组和通过命令行静态加入的组播组。

命令格式

display igmp [ vpn-instance vpn-instance-name | all-instance ] group [ group-address | interface interface-type interface-number ] [ static | verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vpn-instance vpn-instance-name：显示指定VPN实例的IGMP组播组信息。vpn-instance-name指定VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：显示所有实例的IGMP组播组信息。

group-address：组播组地址，用来指定一个组播组。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

interface interface-type interface-number：显示指定接口的IGMP组播组信息。interface-type interface-number表示路由器的接口类型和接口号，用来指定具体的接口。

static：显示静态加入的组播组信息。

verbose：显示动态加入的组播组的详细信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display igmp group命令时：

  • 如果不指定vpn-instance或all-instance参数则只显示公网实例的相关信息。
  • 若指定了static参数，则仅显示静态组播组信息，查看静态组信息必须指定static。
  • 若指定了verbose参数，则显示组播组运行IGMP的各种状态参数。
  • 若指定了interface参数，则仅显示该接口下的组播组信息。
  • 若指定了group-address参数，则仅显示和该组相关的IGMP信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看公网实例动态加入的IGMP组播组信息。

<Quidway> display igmp group
Interface group report information of VPN-Instance: public net
 GigabitEthernet6/0/1(10.1.6.2):
  Total 1 IGMP Group reported
   Group Address   Last Reporter   Uptime      Expires
   225.1.1.2       10.1.6.10       00:02:04    00:01:17
表1 display igmp group命令输出信息描述

项目

描述

Interface group report information of VPN-Instance

接口上IGMP组信息所属的VPN实例。

GigabitEthernet6/0/1(10.1.6.2)

接口编号（IP地址）。

Total 1 IGMP Group reported

接口上动态加入的组播组个数。

Group Address

组播组地址。

Last Reporter

最后发送Report消息的主机。

Uptime
发现组播组的时间，时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”

Expires
记录将从IGMP组表中删除的预定时间，时间格式是：

  • 小于等于24小时：“时：分：秒”

  • 大于24小时但小于等于一周：“天：时”

  • 大于一周：“周：天”
