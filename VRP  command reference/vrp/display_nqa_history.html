display nqa history

命令功能

display nqa history test-instance命令用来查看NQA测试的历史统计。

命令格式

display nqa history test-instance[ admin-name test-name ]

参数说明

test-instance：指定NQA测试例。

admin-name：NQA测试例的管理者。

test-name：NQA测试例的测试例名。

所有视图

缺省级别

1：监控级

使用指南

如果不指定可选参数，将显示所有NQA测试例的历史统计。

说明：

对于Jitter类型的测试、ICMPJitter测试例、PathJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例，如果测试失败，将不会记录在历史统计中。

项目

描述

NQA entry(admin, dns) history
NQA测试项，包括：

  • admin：NQA测试例的管理者标识。

  • dns：NQA测试例的名称。

Index

测试纪录的索引值

T/H/P

  • T：Times，表示这是该测试例第几次测试

  • H：Hop，表示这是第几跳

  • P：Probe，表示这是本次测试中第几次探测

Response

从发出探测报文到收到响应报文所经过的时间

Status
本次探测的状态：

  • success：探测成功结束

  • timeout：探测超时，没有收到响应报文

  • busy：探测正在进行

  • drop：探测报文被丢弃

Address

进行测试的目的IP地址

Time

探测的开始时间
