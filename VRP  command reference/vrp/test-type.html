test-type

命令功能

test-type命令用来配置NQA测试例的测试类型。

缺省情况下，不配置任何测试类型。

命令格式

test-type { dhcp | dns | ftp | http | icmp | icmpjitter | jitter | lspjitter | lspping | lsptrace | macping | mactnlping | mping | mtrace | pathjitter | pathmtu | pwe3ping | pwe3trace | snmp | tcp | trace | udp }

参数说明

dhcp：DHCP测试。

dns：DNS测试。

ftp：FTP服务测试。

http：HTTP服务测试。

icmp：ICMP测试。

icmpjitter ：ICMP Jitter测试，通过ICMP报文测试网络的抖动情况。

jitter：Jitter测试，测试对UDP报文处理的抖动情况。

lspjitter：LSP抖动测试。

lspping：LSP Ping测试。

lsptrace：LSP Trace route测试。

macping：MAC Ping测试。

mactnlping：MAC Tunnel Ping测试。

mping：MPing类型测试。

mtrace：MTtrace类型测试。

pathjitter：path Jitter测试，通过ICMP报文测试报文传送过程中逐跳抖动情况。

pathmtu：path MTU测试，找到网络中最小的MTU值。

pwe3ping：PWE3Ping测试。

pwe3ping：PWE3Trace测试。

snmp：SNMP测试。

tcp：TCP测试。

trace：Trace route测试。

udp：UCP测试。

缺省级别

2：配置级

使用指南

不能修改正在运行的测试例的类型。

使用实例

# 配置NQA测试例的测试类型为TCP测试。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] test-type tcp

# 配置NQA测试例的测试类型为ICMP Jitter测试。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] test-type icmpjitter
