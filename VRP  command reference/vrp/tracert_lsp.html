tracert lsp

命令功能

tracert lsp命令用来在LSP上使用tracert功能。

命令格式

tracert lsp [ -a source-ip | -exp exp-value | -h ttl-value | -r reply-mode | -t time-out ] * { ip destination-address mask-length [ ip-address ] | te tunnel tunnel-number }

参数说明

-a source-ip：发送echo请求报文的源IP地址。

-exp exp-value：发送的echo请求报文的EXP值，取值范围是0～7。

-h ttl-value：发送的echo请求报文的TTL值，取值范围是1～255。

-r reply-mode：应答模式。整数形式，缺省值为2。

-t time-out：等待echo应答的超时时间，单位为毫秒，取值范围是0～65535。

ip destination-address mask-length：LDP LSP的目的IPv4地址和掩码长度。

ip-address：MPLS Ping报文IP头中的目的地址，前缀为127.0.0.0/8。

tunnel-number：TE Tunnel的接口编号。

使用指南

该命令与普通tracert类似，区别仅在于此命令是在LSP上进行。

使用实例

# 对目的地址4.4.4.9/32进行MPLS Tracert。

<Quidway> tracert lsp ip 4.4.4.9 32
  LSP Trace Route FEC: LDP IPV4 PREFIX 4.4.4.9/32 , press CTRL_C to break.
  TTL   Replier            Time    Type      Downstream
  0                                Ingress   10.1.2.2/[1028 ]
  1     10.1.2.2           94 ms   Transit   10.1.5.2/[3 ]
  2     10.1.5.2           94 ms   Egress
# 对TE隧道进行MPLS Tracert。
<Quidway> tracert lsp te tunnel 1/0/0
  LSP Trace Route FEC: TE TUNNEL IPV4 SESSION QUERY Tunnel1/0/0 , press CTRL_C to break.
  TTL   Replier            Time    Type      Downstream
  0                                Ingress   10.1.2.2/[13312 ]
  1     10.1.2.2           63 ms   Transit
  2     6.6.6.6            93 ms   Egress
