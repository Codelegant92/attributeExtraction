snmp-agent inform address

命令功能

snmp-agent inform address命令用于配置针对目标主机的Inform方式的相关参数，包括告警确认的超时时间和告警重复发送的次数。

undo snmp-agent inform address命令用于恢复缺省值。

命令格式

snmp-agent inform { timeout seconds | resend-times times } * address udp-domain ip-address [ vpn-instance vpn-instance-name ] params securityname security-string

undo snmp-agent inform { timeout | resend-times } * address udp-domain ip-address [ vpn-instance vpn-instance-name ] params securityname security-string

参数说明

timeout seconds：Inform方式发送的告警等待网管确认的超时时间，取值范围为1～1800，单位是秒。缺省值与配置的全局的timeout seconds值相等，默认为15秒。本参数的缺省值和取值范围以各产品实现的规格为准。

resend-times times：Inform方式发送的告警因没有收到网管确认消息而重复发送的次数，取值范围为0～10。缺省值与配置的全局的resend-times times值相等，默认为3次。本参数的缺省值和取值范围以各产品实现的规格为准。

address：指定SNMP消息传输的目标主机地址。address指定的IP地址和securityname指定的安全名字符串组成一个二元组，用来标识一个主机。

udp-domain：指定目标主机的传输域基于UDP。

ip-address：目标主机的IP地址。

vpn-instance vpn-instance-name：VPN实例名，字符串形式，不支持空格，长度范围是1～31。vpn-instance可选，如果在VPN中使用，需要vpn-instance指定的VPN实例、IP地址和安全名字符串组成一个三元组来标识一个主机。

params：指定产生SNMP消息的目标主机信息。

securityname security-string：指定在网管侧显示的用户名，字符串形式，不支持空格，长度范围1～32。

使用指南

如果已经配置了全局Inform方式参数，再配置本命令，对于指定的目标主机来说，本命令配置的参数生效。

使用实例

# 配置目标主机（IP:1.1.1.1, securityname: ABC）inform方式告警重复发送次数为10次。

<Quidway> system-view
[Quidway] snmp-agent inform resend-times 10 address udp-domain 1.1.1.1  params securityname ABC
