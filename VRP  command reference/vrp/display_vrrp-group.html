display vrrp-group

命令功能

display vrrp-group命令用来显示VRRP管理组信息。

命令格式

display vrrp-group [ vrrp-group-id | statistics | verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vrrp-group-id：VRRP管理组ID，取值范围1～16。

statistics：显示VRRP管理组的统计信息。

verbose：显示VRRP管理组的详细信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

项目

描述

Total number of vrrp-groups

当前VRRP管理组的总数

vrrp-group 1

当前的VRRP管理组ID

state
VRRP管理组的状态，有三种状态：

  • Master

  • Slave

  • Initialize

enable

表示VRRP管理组已启用

description

对VRRP管理组的描述

priority config

配置的VRRP管理组的优先级，取值范围是1~254

using vrrp priority

将加入VRRP管理组的VRRP备份组优先级的平均值作为该管理组的优先级。只有当使用vrrp-group priority using-vrrp-priority命令配置时，才显示为“yes”，否则显示“no”

increase vrrp priority

加入VRRP管理组的VRRP备份组的优先级增加值

priority run

当前工作的VRRP管理组的实际优先级

preempt

抢占模式，可配置为抢占和非抢占模式

delay time

抢占延迟时间，配置为抢占模式时才有效

timer

VGMP Hello报文定时间隔，默认值1000，单位毫秒，取值范围200~60000

group-send

VGMP消息群发功能

peer status
对等路由器的联机状况，有两种状况：

  • OnLine

  • OffLine

vrrp number

VRRP管理组所管理的VRRP备份组总数

interface

VRRP备份组所在接口

vrrp id

VRRP备份组ID

up
VRRP备份组的工作状态，有两种取值：

  • Up

  • Down

data
VRRP备份组加入VRRP管理组有三种方式：

  • 数据通道data

  • 传输通道transfer-only

  • 普通业务通道

create time

VRRP管理组已创建时间当已创建的时间小于一天时，显示时间的格式是“HH：mm：ss”；当创建的时间大于一天时，显示时间的格式是“X Day(s)：Y Hour(s)”。

last change time
VRRP管理组最后修改时间，是VRRP管理组最后一次修改的时刻距离当前时刻的差值。

  • 当最后修改时间距离当前时刻的差值小于一天时，显示时间的格式是“HH：mm：ss”

  • 当最后修改时间距离当前时刻的差值大于一天时，显示时间的格式是“X Day(s)：Y Hour(s)”

track bfd

VRRP管理组监视的BFD的ID

preempt mode

依靠BFD进行抢占该抢占模式能起作用的前提是，VRRP管理组本身要使能抢占模式，否则此处配置抢占无效

bfd-session state

BFD的工作状态
