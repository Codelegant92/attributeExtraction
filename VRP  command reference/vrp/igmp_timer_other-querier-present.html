igmp timer other-querier-present

命令功能

igmp timer other-querier-present命令用来在接口上设置其他IGMP查询器的存活时间。

undo igmp timer other-querier-present命令用来恢复该配置的缺省情况。

缺省情况下，其他IGMP查询器的存活时间的计算公式是：其他IGMP查询器的存活时间 ＝ 健壮系数 × IGMP普遍查询消息发送间隔 +（1/2）× 最大查询响应时间。当健壮系数、IGMP普遍查询消息发送间隔和最大查询响应时间都取缺省值时，其他IGMP查询器的存活时间的值为125秒。

命令格式

igmp timer other-querier-present interval

undo igmp timer other-querier-present

参数说明

interval：表示其他IGMP查询器存活时间。整数形式，取值范围是60～300，单位为秒。

使用指南

本命令仅适用于IGMPv2和v3。

当用户主机网段上连接多台组播路由器时，从中选出一台作为查询路由器（简称查询器），负责向该网段周期发送查询消息。如果非查询器在“其他IGMP查询器的存活时间”内，收不到查询消息，就认为查询器失效，自动发起查询器选举。

在IGMPv1版本中，查询器的选择由PIM协议决定；在IGMPv2和v3版本中，共享网段上IP地址最小的路由器充当查询器。

注意：

如果配置的“其他IGMP查询器的存活时间”小于“IGMP普遍组查询消息发送间隔”，将导致网络上查询器的重复改变。

该命令和timer other-querier-present（IGMP视图）命令功能完全相同，差别只是作用范围不同。IGMP视图下的配置具有全局性，对所有接口有效；接口视图下的配置只对当前接口有效。系统优先采用接口视图下的配置。当接口视图下没有进行配置，才采用IGMP视图下的配置。

使用实例

# 在GE1/0/0口上配置其他查询器存活时间为200秒。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp timer other-querier-present 200
