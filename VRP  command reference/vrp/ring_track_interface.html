ring track interface

命令功能

ring track interface命令用来配置RRPP环上的监视接口。

undo ring track interface命令用来删除RRPP环上的监视接口。

缺省情况下，RRPP环上没有配置监视接口。

命令格式

ring ring-id track interface interface-type interface-number

undo ring ring-id track interface interface-type interface-number

undo ring ring-id track interface all

参数说明

ring-id：RRPP环ID。整数形式，取值范围是1～64。

interface-type interface-number：接口类型和接口编号。接口可以是Ethernet接口、GigabitEthernet接口、Eth-Trunk接口、Ethernet子接口、GigabitEthernet子接口、Eth-Trunk子接口或VLANIF接口。接口编号采用槽位编号/子卡号/接口编号的格式。

使用指南

监视接口可在RRPP主环或子环的任意节点上配置，监视接口不能是RRPP环上的端口。

一个RRPP环上可以配置多个监视接口，最多可以配置8个。

不同的RRPP环可以使用同一个监视接口。

配置Eth-Trunk接口为监视接口时，需注意：

  • 如果Eth-Trunk接口已经配置为监视接口，不能再配置该Eth-Trunk接口下的成员接口为监视接口。
  • 如果Eth-Trunk接口的成员接口已经配置为监视接口，不能再配置该Eth-Trunk接口为监视接口。

当监视接口的状态或接口上的BFD（Bidirectional Forwarding Detection）状态变化时，监视接口所在节点清除动态MAC地址表项，同时发送COMMON-FLUSH-FDB报文通知RRPP环上其他节点清除动态MAC地址表项。

使用实例

# 配置RRPP域中某节点上的GigabitEthernet1/0/0为监视接口。

<Quidway> system-view
[Quidway] rrpp domain 1
[Quidway-rrpp-domain-region1] ring 1 track interface GigabitEthernet 1/0/0
