display controller t1

命令功能

display controller t1命令用来查看CT1的配置和状态信息。

命令格式

display controller t1 [ controller-number ]

参数说明

controller-number：CT1接口的编号，包括槽号、接口卡号、端口号三部分。如果不指定编号，将显示所有CT1接口的配置和状态信息。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和报文统计信息等。

使用实例

# 查看CT1接口3/0/7的配置情况和状态信息。

<Quidway> display controller t1 3/0/7
T1 3/0/7 current state : DOWN
  Frame-format ESF, code B8ZS, clock slave, loopback not set
  Front card ID: 8TPA
  Front card PCB version: RTC18TPA REV.0
  Back card ID: 8TKA
  Back card PCB version: RTC18TKA REV.0
  CPLD software version number: 1
  No cable
  Alarm: LOS LOF
  Error: 0 FERR, 0 CERR, 0 LCV, 0 FEBE

项目

描述

T1 3/0/7 current state :
T1接口当前的物理状态：

  • UP：物理状态正常启动。

  • DOWN：物理连接不正常。

  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Frame-format

CT1接口的帧格式：请参见frame-format（CT1接口视图）。

code

CT1接口的线路编解码格式：请参见code（CT1接口视图）。

clock

时钟模式：请参见clock（CT1接口视图）。

loopback not set

没有使能CT1接口环回功能。只有在进行某些特殊功能测试时，才将CT1接口设为环回。

Front card ID:

前卡ID。如对于NE16E/08E/05，前卡就是PMC前卡，安装在VIU上的扣卡。

Front card PCB version:

前卡PCB版本信息。

Back card ID:

后卡ID。如对于NE16E/08E/05，后卡是安装在VIEU上的扣卡。前卡和后卡要配合使用，也就是说前卡和后卡必须匹配。

Back card PCB version:

后卡PCB版本信息。

CPLD software version number:

CPLD版本号。

No cable

没有线缆。

Alarm:Error:
告警类型和错误类型。可能出现的错误类型有以下几类。

  • LOS：Loss of Signal，帧同步丢失。

  • LOF：Loss of Frame，帧丢失。

  • CERR：CRC Error，循环冗余校验错。

  • LCV：Line Code Violation，线路码错误。
