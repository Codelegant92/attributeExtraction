snmp-agent trap enable feature-name ospf

命令功能

snmp-agent trap enable feature-name ospf命令用来打开OSPFP模块的告警开关。

undo snmp-agent trap enable feature-name ospf命令关闭OSPFP模块的告警开关。

缺省情况下，OSPF模块的告警开关处于关闭状态。

命令格式

snmp-agent trap enable feature-name ospf { non-excessive all | trap-name { ospfifauthfailure | ospfifconfigerror | ospfifrxbadpacket | ospfifstatechange | ospflsdbapproachingoverflow | ospflsdboverflow | ospfmaxagelsa | ospfnbrrestarthelperstatuschange | ospfnbrstatechange | ospfnssatranslatorstatuschange | ospforiginatelsa | ospfrestartstatuschange | ospftxretransmit | ospfvirtifauthfailure | ospfvirtifconfigerror | ospfvirtifrxbadpacket | ospfvirtifstatechange | ospfvirtiftxretransmit | ospfvirtnbrrestarthelperstatuschange | ospfvirtnbrstatechange } }

undo snmp-agent trap enable feature-name ospf { non-excessive all | trap-name { ospfifauthfailure | ospfifconfigerror | ospfifrxbadpacket | ospfifstatechange | ospflsdbapproachingoverflow | ospflsdboverflow | ospfmaxagelsa | ospfnbrrestarthelperstatuschange | ospfnbrstatechange| ospfnssatranslatorstatuschange | ospforiginatelsa | ospfrestartstatuschange | ospftxretransmit | ospfvirtifauthfailure | ospfvirtifconfigerror | ospfvirtifrxbadpacket | ospfvirtifstatechange | ospfvirtiftxretransmit | ospfvirtnbrrestarthelperstatuschange | ospfvirtnbrstatechange } }

参数说明

non-excessive all：打开OSPF的所有非海量告警开关。

trap-name：打开指定类型事件的告警开关。

ospfifauthfailure：接口认证失败信息。

ospfifconfigerror：接口配置错误信息。

ospfifrxbadpacket：接收的错误报文信息。

ospfifstatechange：接口状态变化信息。

ospflsdbapproachingoverflow：LSDB接近溢出信息。

ospflsdboverflow：LSDB溢出信息。

ospfmaxagelsa：LSA的max age信息。

ospfnbrrestarthelperstatuschange：邻居状态变化信息。

ospfnbrstatechange：GR过程中Helper路由器状态的变化信息。

ospfnssatranslatorstatuschange：NSSA区域中负责将Type7 LSA转换为Type5 LSA的路由器的状态变化。

ospforiginatelsa：本地生成的LSA信息。

ospfrestartstatuschange：GR过程中重启路由器状态的变化信息。

ospftxretransmit：跟踪接口上的报文接收/发送信息。

ospfvirtifauthfailure：虚接口认证失败信息。

ospfvirtifconfigerror：虚接口配置错误信息。

ospfvirtifrxbadpacket：虚接口接收的错误报文信息。

ospfvirtifstatechange：虚接口状态变化信息。

ospfvirtiftxretransmit：跟踪虚接口上的报文接收/发送信息。

ospfvirtnbrrestarthelperstatuschange：GR过程中虚连接Helper邻居状态的变化信息。

ospfvirtnbrstatechange：虚接口邻居状态变化信息。

使用指南

如果要打开OSPF所有事件的非海量告警开关，请选择non-excessive all批量处理；如果只打开某个或几个事件的告警开关时，请选择trap-name。

说明：
  • OSPF模块的非海量告警包括：
    • ospfIfStateChange
    • ospfVirtIfStateChange
    • ospfNbrStateChange
    • ospfVirtNbrStateChange
    • ospfIfAuthFailure
    • ospfVirtIfAuthFailure
    • ospfIfRxBadPacket
    • ospfVirtIfRxBadPacket
    • ospfLsdbOverflow
    • ospfLsdbApproachingOverflow
    • ospfRestartStatusChange
    • ospfNbrRestartHelperStatusChange
    • ospfVirtNbrRestartHelperStatusChange
    • ospfNssaTranslatorStatusChange
  • OSPF模块的海量告警包括：
    • ospfIfConfigError
    • ospfVirtIfConfigError
    • ospfTxRetransmit
    • ospfVirtIfTxRetransmit
    • ospfOriginateLsa
    • ospfMaxAgeLsa

使用实例

# 设置发送OSPF的所有非海量告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name ospf non-excessive all

# 设置发送OSPF虚链接的邻居状态变化的告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name ospf trap-name ospfvirtnbrstatechange
