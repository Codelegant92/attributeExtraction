ipv6 fibcache

命令功能

ipv6 fibcache命令用来使能路由设备的FIB缓存能力。

undo ipv6 fibcache命令用来禁止路由设备的FIB缓存能力。

默认情况下，不使能路由设备的FIB缓存能力。

命令格式

ipv6 fibcache { slot-id | all } [ vpn6-instance vpn6-instance-name ]

undo ipv6 fibcache {slot-id | all } [ vpn6-instance vpn6-instance-name ]

参数说明

slot-id：使能指定接口板的FIB缓存，仅在分布式环境中可见。

all：表示在所有接口板和主控板上使能FIB缓存，仅在分布式环境中可见。

vpn6-instance vpn6-instance-name：IPv6 VPN实例的名称，用来使能指定IPv6 VPN实例的FIB缓存能力。字符串形式，区分大小写，长度范围是1～31。

使用指南

在分布式环境中，此命令可以用来使能指定接口板或主控板的FIB的缓存能力。

说明：

FIB缓存不支持逐包方式的负载分担。

使用实例

# 使能路由设备的FIB缓存能力。

<Quidway> system-view
[Quidway] ipv6 fibcache all

# 使能指定IPv6 VPN实例的FIB缓存能力。

<Quidway> system-view
[Quidway] ipv6 vpn6-instance vpn6
[Quidway-vpn6-instance-vpn6] quit
[Quidway] ipv6 fibcache 6 vpn6-instance vpn6
