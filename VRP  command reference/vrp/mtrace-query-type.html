mtrace-query-type

命令功能

mtrace-query-type命令配置MTrace的发起方式，即IGMP-Tracert-Query报文的发送方式。

undo mtrace-query-type命令恢复配置的缺省情况。

缺省情况下，使用multicast-tree方式发起MTrace。

命令格式

mtrace-query-type { all-router | last-hop | destination | multicast-tree }

undo mtrace-query-type

参数说明

all-router：限定使用all-router发送方式。

last-hop：限定使用last-hop发送方式。

destination：限定使用destination发送方式。

multicast-tree：限定使用multicast-tree发送方式。

使用指南

各种mtrace发起方式的适用环境如下：

  • all-router发送方式

    查询者与目的主机直接相连。

  • last-hop发送方式

    已知最后一跳路由器地址，且单播报文可达。

    如果选择last-hop方式，则必须使用mtrace-last-hop-address命令，配置最后一跳路由器地址。

    查询者将IGMP-Tracert-Query消息封装在组播报文中，目的地址为224.0.0.2，发送到本网段，由最后一跳路由器识别并接收。如果选择last-hop方式，则必须使用mtrace-last-hop-address命令，配置最后一跳路由器地址。

  • destination发送方式

    目的主机网段单播报文可达。查询者将IGMP-Tracert-Query消息封装在普通的单播报文中，报文发送到最后一跳，判断最后一跳是否与接收者直连。

  • multicast-tree发送方式

    查询者位于组播源到目的主机的组播路径上。查询者将IGMP-Tracert-Query消息封装在（S，G）数据报文中，沿组播分发树发往最后一跳路由器。

使用实例

# 限定mtrace测试例user test的发起方式。

[Quidway-nqa-user-test] mtrace-query-type last-hop

# 恢复mtrace测试例user test的发起方式为multicast-tree。

[Quidway-nqa-user-test] undo mtrace-query-type
