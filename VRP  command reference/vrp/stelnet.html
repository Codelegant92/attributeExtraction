stelnet

命令功能

stelnet命令用来连接SSH服务器，连接成功后可对服务器执行相应的操作。

缺省情况，该命令只指定IPv4或IPv6主机名，不指定其他可选项。

命令格式

stelnet { [ -a source-address ] host-ipv4 [ port ] [ prefer_kex { dh_group1 | dh_exchange_group } ] [ prefer_ctos_cipher { des | 3des | aes128 } ] [ prefer_stoc_cipher { des | 3des | aes128 } ] [ prefer_ctos_hmac { sha1 | sha1_96 | md5 | md5_96 } ] [ prefer_stoc_hmac { sha1 | sha1_96 | md5 | md5_96 } ] [ -vpn-instance vpn-instance-name ]

stelnet ipv6 [ -a source-address ] host-ipv6 [ -i interface-type interface-number ] } [ port ] [ prefer_kex { dh_group1 | dh_exchange_group } ] [ prefer_ctos_cipher { des | 3des | aes128 } ] [ prefer_stoc_cipher { des | 3des | aes128 } ] [ prefer_ctos_hmac { sha1 | sha1_96 | md5 | md5_96 } ] [ prefer_stoc_hmac { sha1 | sha1_96 | md5 | md5_96 }]

参数说明

source-address：表示指定STelnet的源地址。

host-ipv4：远程系统的IP地址或主机名，字符串形式，长度范围为1～20。

host-ipv6：远程系统的IPv6地址或主机名，字符串形式，长度范围为1～46。

interface-type interface-number：只有host-ipv6是链路本地型的IPv6地址时，必须指定该链路本地地址所对应的接口名，其他类型的IPv6地址无需指定出接口名。

port：SSH服务器正在监听的端口号，整数形式，取值范围是22或1025～55535。默认值是协议标准端口号22。

prefer_kex：密钥交换首选算法，目前支持dh-group1和dh-exchange-group两种算法，缺省算法为dh-group1。

prefer_ctos_cipher：客户端到服务器端的首选加密算法，目前支持des、3des和aes128三种加密算法，缺省算法为aes128。

prefer_stoc_cipher：服务器端到客户端的首选加密算法，目前支持des、3des和aes128三种加密算法，缺省算法为aes128。

prefer_ctos_hmac：客户端到服务器端的首选HMAC算法，目前支持sha1、sha1-96、md5、md5-96四种HMAC算法，缺省算法为sha1-96。

prefer_stoc_hmac：服务器端到客户端的首选HMAC算法，目前支持sha1、sha1-96、md5、md5-96四种HMAC算法，缺省算法为sha1-96。

-vpn-instance vpn-instance-name：VPN实例名。字符串形式，长度范围是1～31个字符。

使用指南

使用stelnet命令连接SSH服务器之前，SSH服务器端的STelnet服务必须使能。

只有当服务器正在监听的端口号是22时，SSH客户端登录时可以不指定端口号，否则如果是其他监听端口号，SSH客户端登录时必须指定端口号。

使用实例

# SSH服务器名是IPv6的链路本地型地址，必须指定出接口，且STelnet客户端选择加密算法、HMAC算法及交换算法。

<Quidway> system-view
[Quidway] stelnet ipv6 fe80::100 -i Ethernet 2/2/0 perfer_ctos_cipher 3des perfer_stoc_cipher des perfer_ctos_hmac md5_96 perfer_stoc_hmac md5_96 perfer_kex dh_exchange_group
