mpls l2vc

命令功能

mpls l2vc命令用来创建动态VLL连接。

undo mpls l2vc命令用来删除CE接口上动态VLL连接。

缺省情况下，系统没有创建动态VLL连接。只有使用ATM和Frame Relay封装形式的VLL才使能控制字特性。

命令格式

mpls l2vc { ip-address | pw-template pw-template-name } * vc-id [ [ [ group-id group-id ] ] | [ tunnel-policy policy-name ] | [ { control-word | no-control-word } ] | [ ip-interworking | ip-layer2 | { raw | tagged } ] | [ secondary ] ] *

undo mpls l2vc [ secondary ]

参数说明

ip-address：PW对端设备的LSR-ID。

pw-template pw-template-name：指定的PW模板名称。字符串形式，不支持空格，区分大小写，长度范围是1～19。

vc-id：L2VC ID，十进制整数形式，取值范围是1～4294967295。

group-id group-id：组号，十进制整数形式，取值范围是1～4294967295。组号用于表示对一组VC进行相同的操作，以便减少PE之间报文交互的数量。只有属性相同的VC才能配置相同的Group-ID，否则可能引起误拆除PW。

tunnel-policy policy-name：指定的隧道策略名。字符串形式，不支持空格，区分大小写，长度范围是1～19。

control-word：使能控制字（Control Word）特性。

no-control-word：禁止控制字（Control Word）特性。

raw：指定封装类型为不带VLAN tag。

tagged：指定封装类型为带VLAN tag。

ip-interworking：使能LDP-PW方式的IP异种介质互通功能。如果是和华为设备进行异种介质互通，则选择参数ip-interworking。

ip-layer2：如果是和其他厂商设备进行异种互通，需要选择参数ip-layer2。

secondary：配置该VC为备份VC。如果不指定该关键字，新创建的VC为主VC。只有本地存在主VC才能配置备份VC；如果本地没有主VC则不能配置备份VC。

使用指南

同一个接口不能既作为L2VPN的AC（Attachment Circuit）接口又作为L3VPN的AC接口。当某个接口绑定L2VPN后，该接口上配置的IP地址、路由协议等三层特性会全部变为无效。如果一个接口同时绑定L2VPN和L3VPN，只有L2VPN可用。删除该L2VPN后，绑定的L3VPN会重新变得可用。

说明：

  • 当子接口配置了与VLL的绑定关系时，必须先删除子接口和VLL的绑定关系，才能删除子接口。

  • 当子接口配置了与VLL的绑定关系时，不允许改变主接口的封装协议类型。

PW两端的PE设备均需创建动态VC连接。其中，目的地址是对端PE设备的IP地址。

可以通过在PW模板上配置模板属性，如远端Peer、隧道策略、控制字和VCCV能力。配置LDP-PW时可以直接引用该PW模板，不用显式指定PW的属性信息。PW模板属性信息指定后可以随时更新，通过命令reset pw才能使其生效。

如果在mpls l2vc命令行中指定了PW属性，则PW模板中的相应PW属性不起作用。

如果未指定隧道策略名，采用缺省的隧道策略。缺省策略指定优先选择LSP隧道，且负载分担个数为1。如果隧道策略名已指定，但未配置策略，仍采用缺省策略。

使能控制字特性通常用在ATM和Frame Relay封装形式的VLL上，此外还可以用来进行VC的连接性检测，具体使用方法请参见vccv命令。

只有在以太网类型的链路上，该命令中才会出现参数raw和tagged。

配置备份PW之前必须先配置对应的主PW。删除主PW必须先删除备份PW。

使用实例

# 创建Martini方式的连接。主用VLL连接的对端地址为2.2.2.9，VC ID为100；备用VLL连接的对端地址为3.3.3.9，VC ID为200。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] mpls l2vc 2.2.2.9 100
[Quidway-Pos1/0/0] mpls l2vc 3.3.3.9 200 secondary
