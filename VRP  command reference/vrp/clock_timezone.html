clock timezone

命令功能

clock timezone命令用来对本地时区信息进行设置。

undo clock timezone命令用来将本地时区恢复为缺省的UTC（Universal Time Coordinated）时区。

如果没有指定时区名称，则系统采用默认值“Default Zone Name”。

命令格式

clock timezone time-zone-name { add | minus } offset

undo clock timezone

参数说明

time-zone-name：指定时区名称。字符串形式，取值范围是1～32。

add：与通用协调时间UTC相比，time-zone-name增加的时间偏移量。反过来说，就是在系统默认的UTC时区的基础上，减去offset，就可以得到time-zone-name所标识的时区时间。

minus：与UTC时间相比，time-zone-name减少的时间偏移量。反过来说，就是在系统默认的UTC时区的基础上，加上offset，就可以得到time-zone-name所标识的时区时间。

offset：指定与UTC的时间差。形式是：HH:MM:SS，HH表示小时，取值范围是0～23，MM和SS分别表示分和秒，取值范围是0～59。

使用指南

在输入设定的时间参数时，只支持24小时制。其中，可以不输入MMSS，表示0分0秒。但至少需要输入一位数的HH的值，例如输入0，则表示0小时0分0秒。

设置生效后，可以用display clock命令进行查看。另外，日志、诊断等信息时间采用经过时区和夏令时调整过的本地时间。

使用实例

# 在中国北京，设置本地时区名称为BJ。

因北京处于+8时区，那么在配置时，需要在UTC时间基础上，再加上8小时，才能得到预期中的BJ时区。

<Quidway> clock timezone BJ minus 08:00:00
