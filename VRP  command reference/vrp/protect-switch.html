protect-switch

命令功能

protect-switch命令用来切换1:1主备模式下的活动和非活动接口。

命令格式

protect-switch

参数说明

无

视图

Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

正常情况下，1:1主备模式中的主接口处于活动状态，可以转发数据，备份接口处于非活动状态，不能转发数据。如果需要用备份链路转发数据有两种方法：

  • 将主备接口重新定义角色，把原备份接口配置为主接口。

  • 不改变主备接口角色，使用protect-switch命令切换活动接口和非活动接口。

如果使用前一种方法，在重新配置主备接口时会产生短暂的数据中断，而第二种方法可以在不中断数据传输的情况下切换。

每执行一次protect-switch命令，活动接口和非活动接口进行一次切换。

使用实例

# 切换Eth-Trunk 1的备份接口为活动接口。

<Quidway> system-view
[Quidway] interface eht-trunk 1
[Quidway-Eth-Trunk1] portswitch
[Quidway-Eth-Trunk1] protect-switch
