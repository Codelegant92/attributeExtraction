pvc

命令功能

pvc命令用来创建一条指定VPI/VCI的PVC，并进入PVC视图。

undo pvc命令用来删除指定的PVC。

缺省情况下，不创建任何PVC。

命令格式

pvc { pvc-name [ vpi/vci ] | vpi/vci }

undo pvc { pvc-name [ vpi/vci ] | vpi/vci }

参数说明

pvc-name：PVC名，最长为16个字符，在ATM接口上保持唯一，不区分大小写，并且不能是合法的VPI/VCI值对，例如不允许“1/20”作为PVC名。

说明：

串口子接口和IMA子接口下创建的PVC不可配置pvc-name参数。

vpi/vci：vpi为ATM网络虚路径标识（Virtual Path Identifier），在ATM接口上取值范围是0～255；vci为ATM网络虚通道标识（Virtual Channel Identifier），取值范围是0～2047。通常，vci取值0到31保留用于特定用途，建议用户不要使用。

说明：

参数vpi与vci不能同时为0。

缺省级别

2：配置级

使用指南

如果使用pvc-name创建PVC，必须同时指定vpi/vci。一旦为PVC指定了名称，就可以通过pvc pvc-name重新进入PVC视图；删除该PVC时，既可以通过指定pvc-name删除，也可以通过指定vpi/vci删除。

每条PVC的VPI/VCI值对在一个ATM接口（包括主接口和子接口）范围内唯一。

实际可以创建PVC的数量由pvc max-number命令决定。

使用实例

# 创建一条名为“huawei”、VPI/VCI为1/101的PVC。

<Quidway> system-view
[Quidway] interface atm 1/0/0
[Quidway-Atm1/0/0] pvc huawei 1/101
