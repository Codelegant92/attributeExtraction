using（CE1接口视图）

命令功能

using命令用来切换CE1接口的工作模式，创建不同带宽的同步串口。

undo using命令用来恢复缺省设置。

缺省情况下，CE1接口工作在通道化模式，需要进行时隙捆绑，形成传输速率为N×64kbit/s的同步串口。

命令格式

using { e1 | ce1 }

undo using

参数说明

e1：工作模式为净通道模式。

ce1：工作模式为通道化模式。

使用指南

CE1接口有两种工作方式：

  • 当工作在通道化工作模式下时，物理上分为32个时隙，可以对除时隙0以外的31个时隙进行捆绑。使用该接口时，可以任意地将全部时隙分成若干组，每组时隙捆绑以后作为一个接口使用，其逻辑特性与同步串口相同。使用interface serial命令进入捆绑后形成的接口的视图。

  • 当工作在净通道工作模式下时，整个接口不再时隙化，不支持channel-set和frame-format命令，而是作为一个2.048M的接口。

两种方式下接口都支持PPP、帧中继、LAPB和X.25等链路层协议。

切换CE1接口的工作模式时：

  • 如果要将接口从通道化模式切换到净通道模式，需要执行相应的undo命令删除在通道化模式下的配置，再执行using e1命令。
  • 如果要将接口从净通道模式切换到通道化模式，需要删除净通道模式及同步串口下的所有配置，不需删除同步串口，执行using ce1命令或undo using命令。

使用实例

# 设置把CE1接口的工作模式切换到净通道模式。

<Quidway> system-view
[Quidway] controller e1 1/0/0
[Quidway-E1 1/0/0] using e1
