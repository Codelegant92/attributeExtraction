apply behavior

命令功能

apply behavior 命令用来为路由设置QoS流行为。

undo apply behavior命令用来取消配置。

缺省情况下，没有设置QoS流行为。

命令格式

apply behavior behavior-name

undo apply behavior

参数说明

behavior-name：QoS流行为名称。字符串形式，长度范围为1～31。

视图

Route-Policy视图

缺省级别

2：配置级

使用指南

QoS可以定义流行为，而Route-Policy可以为路由关联特定的流行为。这是QPPB可以设置的三种QoS参数中的一种。

apply behavior与apply qos-local-id和apply ip-precedence（Route-Policy视图）命令互斥。

必须先在系统视图下配置behavior-name，才可以配置apply behavior behavior-name。

说明：

产品NE16E/08E/05不支持apply behavior命令。

使用实例

# 在系统视图下配置名为example的behavior，然后在Route-Policy视图下应用该QoS流行为。

<Quidway> system-view
[Quidway] traffic behavior example
[Quidway-behaviror-example] quit
[Quidway] route-policy test permit node 10
[Quidway-route-policy] apply behavior example
