source-policy（IPv4）

命令功能

source-policy命令用来配置路由器对接收的组播数据报文根据源或源/组进行过滤。

undo source-policy命令用来删除配置。

缺省情况下，路由器不根据源或源/组过滤组播数据报文。

命令格式

source-policy acl-number

undo source-policy

参数说明

acl-number：表示基本或高级访问控制列表序号。整数形式，取值范围是2000～3999。

视图

缺省级别

2：配置级

使用指南

如果配置了source-policy，并且使用了基本访问控制列表，则对接收的所有组播数据报文匹配源地址，未通过该过滤规则的报文将被丢弃。

如果配置了source-policy，并且使用了高级访问控制列表，则对接收的所有组播数据报文匹配源地址和组地址，未通过该过滤规则的报文将被丢弃。

该命令还过滤注册报文中封装的组播数据。

重复执行此命令，新的配置会覆盖上一次的配置。

source-policy命令与acl命令配合使用。

  • 在基本ACL视图下，通过rule命令的source参数指定发送组播数据报文的源地址。

  • 在高级ACL视图下，通过rule命令的source参数指定发送组播数据报文的源地址，destination参数指定组播组地址。

使用实例

# 在公网实例中，配置接受源为10.10.1.2的组播数据包，丢弃源为10.10.1.1的组播数据包。

<Quidway> system-view
[Quidway] acl number 2001
[Quidway-acl-basic-2001] rule permit source 10.10.1.2 0
[Quidway-acl-basic-2001] rule deny source 10.10.1.1 0
[Quidway-acl-basic-2001] quit
[Quidway] pim
[Quidway-pim] source-policy 2001
