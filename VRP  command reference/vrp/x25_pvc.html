x25 pvc

命令功能

x25 pvc命令用来配置一条数据报封装的PVC路由。

undo x25 pvc命令用来删除PVC数据报封装的路由。

命令格式

x25 pvc pvc-number { compressedtcp protocol-address | ip ip-address }* x121-address x.121-address [ option ]

undo x25 pvc pvc-number

参数说明

pvc-number：虚电路号，范围必须在1到4095之间（包括1和4095），而且必须在永久虚电路信道区间内。

compressedtcp：使用TCP头压缩。

protocol-address：该虚电路对端的网络协议地址。

ip：使用IP协议。

x.121-address：该虚电路对端的X.121地址，字符串形式，长度范围是1～15 ，只有0～9数字是有效的。

option：该虚电路的属性。

使用指南

系统缺省不创建用于数据报封装的虚电路。当创建一条这样的永久虚电路时，如果不为该永久虚电路设置相应的属性，则它的流量控制参数与它所在的X.25接口的流量控制参数相同（一个X.25接口的流量控制参数是通过x25 packet-size、x25 window-size等命令设置的）。

因为在创建永久虚电路的同时就已经隐含地创建了相应的地址映射，所以在创建永久虚电路之前不需要（事实上也不能）首先创建地址映射。

在创建永久虚电路之前，首先应该开启永久虚电路信道区间。永久虚电路信道区间是指从1到最近的未被禁止的信道区间的下限减1之间（包括1和下限减1）的这个信道区间。当然，如果下限本身就是1，那么永久虚电路信道区间自然而然地被禁止。下表列举了几种典型配置下永久虚电路信道区间。

虚电路的属性选项及其意义详细解释如下：

broadcast：将广播报文向虚电路的对端转发。

compress：指定报文压缩功能。

encapsulation-type：指定封装方式，可选择为nonstandard、ietf、multi-protocol、snap。

packet-size input-packet output-packet：指定该虚电路的最大接收分组长度和最大发送分组长度，以字节为单位，这两个值必须在16到4096范围内（包括16和4096），并且必须是2的整数次幂。

window-size input-window-size output-window-size：指定该虚电路的接收窗口和发送窗口的尺寸，在1到该永久虚电路所在的X.25接口的模数－1之间（包括1和模数－1）。

使用实例

# 配置接口Serial1/0/0的链路层协议为X.25，并启用永久虚电路信道区间，设置两条虚电路。

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] link-protocol x25
[Quidway-Serial1/0/0] x25 vc-range bi-channel 8 1024
[Quidway-Serial1/0/0] shutdown
[Quidway-Serial1/0/0] undo shutdown
[Quidway-Serial1/0/0] x25 pvc 2 ip 202.38.168.1 x121-address 20112451 broadcast packet-size 512 512
[Quidway-Serial1/0/0] x25 pvc 6 ip 202.38.168.3 x121-address 20112453 broadcast window-size 5 5
