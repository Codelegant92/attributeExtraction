display msdp sa-cache

命令功能

display msdp sa-cache命令用来查看SA缓存中的（S，G）信息。

命令格式

display msdp [ vpn-instance vpn-instance-name | all-instance ] sa-cache [ group-address | source-address | as-number ] * [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

group-address：表示（S，G）项中的组播组地址，显示SA缓存中该组对应的（S，G）信息。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

source-address：表示（S，G）项中的源地址，显示SA缓存中该源对应的（S，G）信息。点分十进制形式。

as-number：表示自治域号。整数形式，取值范围是1～65535。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

只有使能了SA缓存的MSDP对等体才能显示缓存信息（缺省情况下，SA缓存是使能的）。

使用display msdp sa-cache命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 如果指定了group-address，则显示指定组对应的（S，G）表项。

  • 如果指定了source-address，则显示指定源对应的（S，G）表项。

  • 如果指定了as-number，则显示Origin RP属于指定AS的（S，G）表项。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
