display ospf brief

命令功能

display ospf brief命令用来查看OSPF的概要信息。如果不指定OSPF进程号，该命令将显示所有OSPF进程的概要信息。

命令格式

display ospf [ process-id ] brief [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

process-id：OSPF进程号。整数形式，取值范围是1～65535。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

该命令除了用来查看OSPF概要信息，还会输出下列信息：

  • 路由器ID。

  • 配置的VPN域标识符。

在检查OSPF的故障原因时，可执行此命令获取OSPF相关概要信息。用户可以根据这些信息进行OSPF的故障诊断等。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

表1 display ospf brief命令输出信息描述

项目

描述

RouterID

当前的OSPF路由器ID

Border Router
边界路由器

  • AS：自治系统边界路由器ASBR

  • AREA：区域边界路由器ABR

  • NSSA：NSSA区域边界路由器

Domain–Id

路由器所属的域标识符

Route Tag

与外部路由相关联的标记

PE Router, Multi-VPN-Instance is enabled

只有当前进程是支持多VPN实例的PE路由器时才显示此项内容

Applications Supported: MPLS Traffic-Engineering

目前OSPF支持TE功能

Opaque Capable

使能Opaque能力显示

Graceful-restart capability
是否使能Graceful Restart能力：

  • disabled：没有使能Graceful Restart能力

  • planned only：支持Planned-GR方式

  • un-planned：支持Unplanned-GR方式

  • totally：支持Totally GR方式

Helper support capability
是否使能Helper模式：

  • enabled：已使能Helper模式

  • not configured：未使能Helper模式

filter capability
是否使能Helper模式的过滤规则：
  • ip-prefix：使能Helper模式的ip-prefix过滤器
  • acl-number：使能Helper模式的基本ACL号过滤器
  • disabled：未使能Helper模式的过滤器

policy capability
是否使能Helper模式的支持策略：
  • strict lsa check：严格检查LSA的支持策略
  • planned and un-planned：planned和unplanned的支持策略

Local-mt capable

是否使能本地MT功能

OSPF protocol Hot Standby Capable

使能OSPF协议热备份时显示

OSPF stub router State

使能Stub路由器时显示

OSPF stub router State Reason: Startup Synchronize

重启路由器时，正在使能BGP-OSPF联动功能

IGP-shortcut capable

使能IGP快捷时显示

Forwarding Adjacency Capable

使能邻接转发时显示

Spf-schedule-interval

进行SPF计算的时间间隔

Route Preference

缺省路由优先级

Default ASE parameters: Metric

外部LSA的度量值缺省值

Default ASE parameters: Tag

外部LSA标记的缺省值

Default ASE parameters: Type

外部LSA类型的缺省值

SPF Computation count

进行SPF计算的次数

RFC1583 Compatible

是否使能RFC1583兼容

Retransmission limitation is enabled

使能重传限制功能

Mib-binding process

当前进程绑定MIB时显示

Trap-binding process

当前进程绑定Trap时显示

OSPF is in LSDB overflow status

进程为Overflow状态时显示

OSPF is in Graceful Restart status

进程为Graceful Restart状态时显示

OSPF is in protocol hot standby state

使能热备份时显示

Import routes limitation is enabled, the maximal limitation value:

使能引入外部路由限制功能，显示最大限制值

Area Count

当前进程中的区域数

Nssa Area Count

当前进程中的NSSA区域数

ExChange/Loading Neighbors

处于ExChange/Loading状态的Neighbor的数量

Area

开始列举当前进程中各区域的信息。显示当前区域ID，格式为XXX.XXX.XXX.XXX

MPLS Traffic-Engineering enabled

当前区域使能流量工程

Authtype:

区域验证类型，包括无验证/简单验证/MD5验证/HMAC-MD5验证

Area flag

说明区域属性：Transit/ Vlink/ Stub/ Nssa

SPF scheduled Count

说明上一次的SPF计算类型：All/ Router/ Net/ Intra/ SumNet/ SumASB/ ASE/ NSSA

Area summary filter（import）

只有定义了入方向的过滤器时此项才有显示

Area summary filter（export）

只有定义了出方向的过滤器时此项才有显示

Interface

显示区域内的接口信息。

Cost

显示运行OSPF协议的接口的开销值

State

接口的状态有：Down、Waiting、Loopback、P-2-P、DR、BDR DROTHER。其中，DR、BDR DROTHER在Broadcast和NBMA网络中才存在；P-2-P在P2P P2MP Vlink上才存在。

Type

接口的类型，可以是PTP、Broadcast、NBMA、PTMP

MTU

接口的MTU值

Transit Area

显示虚连接所在的传输区域的区域ID

Priority

接口优先级

Designated Router

如果当前接口不是DR则不显示（可选）

Backup Designated Router

如果当前接口不是BDR则不显示（可选）

Timers

定时器的间隔信息

Hello

Hello定时器的时间间隔

Dead

Dead定时器的时间间隔

Poll

Poll定时器的时间间隔

Retransmit

Retransmit定时器的时间间隔

Transmit Delay

接口传送LSA的时间延迟（可选）

Passive interface, No hellos

接口为被动接口时显示（可选）

MPLS Traffic-Engineering Link

接口使能MPLS-TE

IGP-shortcut Link

链路使能IGP快捷时显示

Forwarding-adjacency Link

链路使能邻接转发时显示

Global DS-TE Mode
当前全局配置下的DS-TE模式信息：
  • Non-standard IETF Mode：不支持IETF模式
  • Standard IETF Mode：支持IETF模式

注：不同的配置情况下，该命令的显示信息有所不同。一个举例无法给出所有的显示信息，因此上述举例的显示内容可能以表格中所列出的内容不太一致。有些内容必须在特定的配置使能之后才能显示。
