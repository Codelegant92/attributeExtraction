display ospf ldp-sync

命令功能

display ospf ldp-sync命令用来查看接口LDP和OSPF同步状态的信息。

命令格式

display ospf ldp-sync interface { all | interface-type interface-number } [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

all：查看所有接口LDP和OSPF同步状态的信息。

interface-type interface-number：查看指定接口LDP和OSPF同步状态的信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

所有使能了LDP和OSPF同步功能的接口，都可以通过该命令查看LDP和OSPF的同步状态信息。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看指定接口LDP和OSPF同步状态的信息。

<Quidway> display ospf ldp-sync interface gigabitethernet 1/0/0
 Interface gigabitethernet1/0/0
 HoldDown Timer: 9       HoldMaxCost Timer: 50
 LDP State: Up             OSPF Sync State: Sync-Achieved

项目

描述

Interface

与邻居相连的接口。

HoldDown Timer

接口不建立OSPF邻居而等待LDP会话建立的定时器。缺省值是10秒。

HoldMaxCost Timer

OSPF在本地设备的LSA中通告最大metric值的定时器。缺省值是10秒。

LDP State
LDP会话的状态，包括以下几种：

  • Up

  • Down。

OSPF Sync State
LDP和OSPF的同步状态，包括以下几种：

  • Sync-Achieved：已同步

  • HoldDown：表示接口不建立OSPF邻居而等待LDP会话建立的状态

  • HoldMaxCost：表示OSPF在本地设备的LSA或LSP中通告最大metric值的状态

  • Init：初始状态。
