snmp-agent trap enable feature-name ospfv3

命令功能

snmp-agent trap enable feature-name ospfv3命令用来打开OSPF模块的告警开关。

snmp-agent trap enable feature-name ospfv3命令用来关闭OSPF模块的告警开关。

缺省情况下，OSPF模块的告警开关处于开启状态。

命令格式

snmp-agent trap enable feature-name ospfv3 { non-excessive all | trap-name { ospfv3ifconfigerror | ospfv3ifrxbadpacket | ospfv3ifstatechange | ospfv3nbrrestarthelperstatuschange | ospfv3nbrstatechange | ospfv3restartstatuschange | ospfv3virtifconfigerror | ospfv3virtifrxbadpacket | ospfv3virtifstatechange | ospfv3virtnbrrestarthelperstatuschange | ospfv3virtnbrstatechange } }

参数说明

non-excessive all：打开OSPFv3的所有非海量告警开关。

trap-name：打开指定类型事件的告警开关。

ospfv3ifconfigerror：接口配置错误信息。

ospfv3ifrxbadpacket：接收的错误报文信息。

ospfv3ifstatechange：接口状态变化信息。

ospfv3nbrrestarthelperstatuschange：GR过程中Helper路由器状态的变化信息。

ospfv3nbrstatechange：邻居状态变化信息。

ospfv3restartstatuschange：GR过程中重启路由器状态的变化信息。

ospfv3virtifconfigerror：虚接口配置错误信息。

ospfv3virtifrxbadpacket：虚接口接收的错误报文信息。

ospfv3virtifstatechange：虚接口状态变化信息。

ospfv3virtnbrrestarthelperstatuschange：GR过程中虚连接Helper邻居状态的变化信息。

ospfv3virtnbrstatechange：虚连接邻居状态变化信息。

使用指南

如果要打开OSPFv3所有事件的非海量告警开关，请选择non-excessive all批量处理；如果只打开某个或几个事件的告警开关时，请选择trap-name。

说明：
  • OSPFv3模块的非海量告警包括：

    • ospfv3VirtIfStateChange

    • ospfv3NbrStateChange

    • ospfv3VirtNbrStateChange

    • ospfv3IfConfigError

    • ospfv3VirtIfConfigError

    • ospfv3IfRxBadPacket

    • ospfv3VirtIfRxBadPacket

    • ospfv3IfStateChange

    • ospfv3RestartStatusChange

    • ospfv3NbrRestartHelperStatusChange

    • ospfv3VirtNbrRestartHelperStatusChange

  • OSPFv3模块没有海量告警开关。

使用实例

# 设置发送OSPFv3的所有非海量告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable ospfv3 non-excessive all

# 设置发送OSPFv3接口配置错误的告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name ospfv3 trap-name ospfv3ifconfigerror
