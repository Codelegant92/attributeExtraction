transmission-alarm threshold

命令功能

transmission-alarm threshold命令用来配置b3tca、sdbere和sfbere的告警门限值。

undo transmission-alarm threshold命令用来恢复b3tca、sdbere和sfbere的告警门限值为缺省值。

命令格式

transmission-alarm threshold { b3tca b3tca | sdbere sdbere | sfbere sfbere } *

undo transmission-alarm threshold

参数说明

b3tca b3tca：B3超越门限值告警指示的门限值。该门限值采用指数形式10-n表示，其中指数n通过b3tca指定。b3tca是整数形式，取值范围是3～9，缺省值是6。因此，b3tca告警门限值的缺省值是10-6。

sdbere sdbere：信号损耗门限告警指示的门限值。该门限值采用指数形式10-n表示，其中指数n通过sdbere指定。sdbere是整数形式，取值范围是3～9，缺省值是6。因此，sdbere告警门限值的缺省值是10-6。sdbere的值不能小于sfbere的值，也就是sdbere的告警门限值不能大于sfbere的告警门限值。

sfbere sfbere：信号失效门限告警指示的门限值。该门限值采用指数形式10-n表示，其中指数n通过sfbere指定。sfbere是整数形式，取值范围是3～9，缺省值是3。因此，sfbere告警门限值的缺省值是10-3。sdbere的值不能小于sfbere的值，也就是sdbere的告警门限值不能大于sfbere的告警门限值。

使用指南

支持该命令的接口类型有ATM、POS、CPOS、E1/CE1、T1/CT1、E3/CE3、T3/CT3、10GE-WAN、基于SDH和SONET的RPR接口。设备上支持的具体接口类型由产品决定。

该命令用来配置b3tca、sdbere和sfbere三类告警可以由传输设备上报VRP时的告警门限值。

使用实例

# 在POS1/0/0接口上设置b3tca、sdbere和sfbere三类告警的门限值分别为10-7、10-9和10-8。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] transmission-alarm threshold b3tca 7 sdbere 9 sfbere 8
