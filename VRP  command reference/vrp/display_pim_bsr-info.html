display pim bsr-info

命令功能

display pim bsr-info命令用来查看PIM-SM域中BSR自举路由器的信息。

命令格式

display pim [ vpn-instance vpn-instance-name | all-instance ] bsr-info [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display pim bsr-info命令时，如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

该命令适用于使用BSR动态选举RP的PIM-SM域。域内任何一台路由器上都可以使用该命令查看BSR信息。

  • 如果当前路由器上配置了C-BSR，则显示信息中包括Elected BSR和本地配置的C-BSR信息。

  • 如果当前路由器上未配置C-BSR，则显示信息中仅包括Elected BSR。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

使用实例

# 查看当前公网实例PIM-SM域中的BSR信息。该路由器上没有配置C-BSR，因此只显示Elected BSR的信息。

<Quidway> display pim bsr-info
 VPN-Instance: public net
 Elected AdminScoped BSR Count: 0
 Elected BSR Address: 10.1.2.2
     Priority: 0
     Hash mask length: 30
     State: Accept Preferred
     Scope: Not scoped
     Uptime: 00:01:46
     Expires: 00:02:02
     C-RP Count: 1

<Quidway> display pim bsr-info

     Hash mask length: 30

     Next BSR message scheduled at: 00:00:31

项目

描述

VPN-Instance

BSR信息所属的VPN实例。

Elected AdminScoped BSR Count

当选管理域BSR的个数。

Elected BSR Address

当选BSR的地址。

Priority

BSR优先级，缺省值为0。

Hash mask lenth

进行RP Hash计算时的掩码长度。

State

BSR的状态。

Scope

该BSR是管理域BSR时，管理域中的组播地址范围。

Not scoped表示该BSR不是管理域BSR。

Uptime

BSR已经存在的时间。

Expires

BSR存在的剩余时间。

C-RP Count

通过此BSR学到的RP的数量。

Next BSR message scheduled at

下一个BSR消息将在多少时间后发送。只有当选BSR上维护该定时器，定时器超时则发送BSR消息。

Candidate AdminScoped BSR Count

候选管理域BSR的个数。

Cancidate BSR Address

候选BSR的地址。

Wait to be BSR
当前C-BSR是否有效标识，有以下两种取值：
  • 0：表示当前C-BSR有效，即参与竞选BSR；
  • 1：表示当前C-BSR无效，即不参与竞选BSR。

当路由器上配置的C-BSR超过最大规格限制时，如果继续配置C-BSR，该标识位将取1。
