transport trace vlan

命令功能

transport trace vlan命令用来配置对LTM报文进行透传。

undo transport trace vlan命令用来取消对LTM报文透传的配置。

缺省情况下，不透传LTM报文。

命令格式

transport trace vlan { vlan-id1 [ to vlan-id2 ] } &<1-10>

undo transport trace vlan { vlan-id1[ to vlan-id2 ] } &<1-10>

参数说明

vlan-id1：起始VLAN ID，取值范围是1～4094。

vlan-id2：结束VLAN ID，取值范围是1～4094，该值必须大于vlan-id1。

视图

Ethernet接口视图、GE接口视图、Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

在配置了Vlan stacking或PBB-TE或VLL接入VPLS隧道的PE设备上，配置这些隧道的端口如果在LUPF-20单板上，收到LTM报文是不能直接透传转发的，需要通过此命令行设置LTM报文透传。

对于配置了cfm mac-tunnel mip trace enable命令使能Trace回复功能的节点，接口下配置透传后，收到的LTM报文将直接转发，不会回复LTR报文。

使用实例

# 配置GE1/0/1接口对VLAN 2-10的LTM报文进行透传。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/1
[Quidway-GigabitEthernet] transport trace vlan 2 to 10
