igmp-snooping max-response-time

命令功能

igmp-snooping max-response-time命令用来配置IGMP查询报文的最大响应时间。

undo igmp-snooping max-response-time命令用来恢复该配置为缺省值。

缺省情况下，IGMP查询报文的最大响应时间是10秒。

命令格式

igmp-snooping max-response-time max-response-time

undo igmp-snooping max-response-time

参数说明

max-response-time：最大响应时间，取值范围是1～25，单位为秒。

视图

VLAN视图、VSI视图

缺省级别

2：配置级

使用指南

通过设置最大响应时间，可以控制主机发送组成员关系报告的最后期限。合理的设置最大响应时间，可以使主机快速响应查询报文，同时避免大量主机同时发送响应报文而造成的流量拥塞。

端口收到report后，将组超时定时器设置为GMI(Group Membership Interval)，GMI = ((Robustness Variable)×( Query Interval)) + (Query Response Interval)。

本命令用来设置上述公式中的Query Response Interval。

在VLAN视图下配置时，以下情况会导致配置不成功：

  • VLAN已经绑定了VSI；

  • VLAN内包含VE接口。

使用实例

# 配置VLAN 2内的最大响应时间为20秒。

<Quidway> system-view
[Quidway] vlan 2
[Quidway-vlan2] igmp-snooping max-response-time 20
