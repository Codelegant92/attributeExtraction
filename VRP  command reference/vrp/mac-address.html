mac-address

命令功能

mac-address命令用来配置基于业务实例的静态MAC表项和黑洞MAC表项。表项内容包括报文的目的MAC地址、用于转发该报文的MAC Tunnel名称和报文所属业务实例的名称。

undo mac-address命令用来删除基于业务实例的MAC地址表项。

命令格式

mac-address static mac-address mac-tunnel tunnel-name service-instance service-instance-name

mac-address blackhole mac-address service-instance service-instance-name

undo mac-address mac-address service-instance service-instance-name

参数说明

blackhole：指定表项的类型为黑洞表项。黑洞表项不被老化，系统复位或接口板热插拔后不丢失。

static：指定表项的类型为静态表项。静态表项不老化，系统复位或接口板热插拔后不丢失。

mac-address：指定报文的目的MAC地址。格式为H-H-H。其中H为1～4位的十六进制数，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。该MAC地址不可以是组播或广播地址，也不能是设备的虚MAC地址。

mac-tunnel tunnel-name：指定用于转发该报文的MAC Tunnel的名字。长度范围是1～31个字符。

service-instance service-instance-name：指定报文所属业务实例的名字。长度范围是1～31个字符。

使用指南

在用户业务与多点到多点业务实例映射后，可以通过配置基于业务实例的静态MAC地址表项，使业务实例静态选择用于传输业务的MAC Tunnel。静态MAC地址表项配置后即生效，不需要经过MAC地址学习过程，能更快速的实现业务传输。

该命令只对多点到多点的业务实例有效。

静态MAC地址表项中的业务实例必须已经与MAC Tunnel绑定。

使用实例

# 配置带有目的MAC地址0-0-1的业务实例mysi，被名为mytunnel的MAC Tunnel静态转发。

<Quidway> system-view
[Quidway] mac-address static 0-0-1 mac-tunnel mytunnel service-instance mysi
