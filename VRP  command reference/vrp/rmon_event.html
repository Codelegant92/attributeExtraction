rmon event

命令功能

rmon event命令用来在事件表中添加一行。

undo rmon event命令用来在事件表中删除一行。

命令格式

rmon event entry-number [ description string ] { log | trap object | log-trap object | none } [ owner owner-name ]

undo rmon event entry-number

参数说明

entry-number：添加或删除行的行号。整数形式，取值范围是1～65535。

description string：事件的描述。字符串形式，长度范围是1～127。

log：对事件的处理方式是发送日志。

trap：对事件的处理方式是向网管站发trap消息。

log-trap：对事件的处理方式是发送日志，并同时向网管站发trap消息。

object：trap消息发往的团体名。字符串形式，长度范围是1～127。

none：对事件的处理方式是标记为没有事件发生。

owner owner-name：创建者名称。字符串形式，长度范围是1～127。

使用指南

RMON的事件管理定义事件号及事件处理方式，以便对报警事件作出相应处理。事件组中定义的行号对应于告警组中的指定事件号。

使用实例

# 在事件表中添加第10行，标记为trap事件。

<Quidway> system-view
[Quidway] rmon event 10 trap public
