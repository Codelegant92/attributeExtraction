ntp-service unicast-peer

命令功能

ntp-service unicast-peer命令用来配置NTP对等体模式。

undo ntp-service unicast-peer命令用来取消NTP对等体模式。

命令格式

ntp-service unicast-peer ip-address [ version number | authentication-keyid key-id | source-interface interface-type interface-number | vpn-instance vpn-instance-name | preference ] *

undo ntp-service unicast-peer ip-address [ vpn-instance vpn-instance-name ]

参数说明

ip-address：远程服务器IP地址。主机地址，不能为广播、组播地址或参考时钟的IP地址。

version number：NTP版本号。取值范围是1～3。缺省值是3。

authentication-keyid：定义身份验证密钥。

key-id：向该远程服务器发送消息使用的密钥ID号，范围是1～4294967295。

source-interface interface-type interface-number：主动对等体向被动对等体发送NTP消息时使用的源接口，消息包中的源IP地址从该接口获取。

vpn-instance vpn-instance-name：VPN实例名称。字符串形式，不支持空格，长度范围是1～31。

preference：指定该服务器为优先选择的远程服务器。

使用指南

设置以ip-address所指定的远程服务器作为本地的对等体，本地运行在 symmetric active模式。在这种配置下，本地设备能同步到远程服务器，远程服务器也能同步到本地服务器。

使用实例

# 本地被配置成由peer 10.10.1.1提供同步时间，本地对等体也能为peer提供同步时间，版本号为3。NTP消息包的IP地址从GE1/0/0获得。

[Quidway] ntp-service unicast-peer 10.10.1.1 version 3 source-interface GigabitEthernet 1/0/0
