igmp-snooping send-query enable

命令功能

igmp-snooping send-query enable命令用来配置允许路由器主动向非路由端口发送源IP地址为非0.0.0.0的igmp query报文。

undo igmp-snooping send-query enable命令用来配置禁止路由器主动向非路由端口发送源IP地址为非0.0.0.0的igmp query报文。

缺省情况下，禁止路由器主动向非路由端口发送源IP地址为非0.0.0.0的igmp-query报文。

命令格式

igmp-snooping send-query enable

undo igmp-snooping send-query enable

参数说明

无

使用指南

当链路拓扑发生变化时，路由器会收到link change事件。发送源地址非0.0.0.0的IGMP Query 消息，可以更新网络中路由器上的出端口信息，将组播数据流迅速切换到新的转发路径上，从而保证组播业务不中断。

  • 更新路由端口

    • 对于mstp link change事件，IGMP Query消息发往该VLAN内的所有非路由端口。

    • 对于rrpp/rpr link change事件，IGMP Query消息只发往接收到该事件的端口。

  • 更新组成员端口

    • 当IGMP Query消息到达用户主机时，组成员主机回应IGMP Report消息。IGMP Report消息上送，更新组成员端口。

使用实例

# 配置允许路由器向非路由端口发送源地址是0.0.0.0的IGMP Query报文。

<Quidway> system-view
[Quidway] igmp-snooping send-query enable

# 配置禁止路由器向非路由端口发送源地址是0.0.0.0的IGMP Query报文。

<Quidway> system-view
[Quidway] undo igmp-snooping send-query enable
