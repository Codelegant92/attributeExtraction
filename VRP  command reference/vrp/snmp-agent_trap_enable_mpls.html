snmp-agent trap enable（MPLS）

命令功能

snmp-agent trap enable命令用来使能SNMP告警相关参数。

undo snmp-agent trap enable命令用来去使能SNMP告警相关参数。

命令格式

snmp-agent trap enable { static-lsp | ldp | lsp { mplsxcup | mplsxcdown } | tunnel-ps | te { tunnel-reop | te-frr [ private ] | hot-standby | ordinary | state-change-private | bandwidth-change } }

undo snmp-agent trap enable { static-lsp | ldp | lsp { mplsxcup | mplsxcdown } | tunnel-ps | te { tunnel-reop | te-frr [ private ] | hot-standby | ordinary | state-change-private | bandwidth-change } }

参数说明

static-lsp：使能静态LSP告警。

ldp：使能LDP告警。

lsp mplsxcup：打开mplsxcup海量告警。

lsp mplsxcdown：打开mplsxcdown海量告警。

tunnel-ps：使能TE保护切换告警。

te tunnel-reop：使能TE重优化路由告警。

te te-frr：使能TE快速重路由公有告警。

te-frr private：使能TE快速重路由私有告警。

te hot-standby：使能TE热备份隧道告警。

te ordinary：使能TE 普通备份隧道告警。

state-change-private：使能系统在隧道状态改变时发送相关私有告警信息。

bandwidth-change：使能系统在隧道带宽改变时发送相关告警信息。

缺省级别

2：配置级

使用指南

缺省情况下，MPLS LSP创建过程中不使能相关告警。

当需要查看LSP状态是否为Up或Down，同时打开mplsxcup或mplsxcdown海量告警开关时，可执行snmp-agent trap enable lsp { mplsxcup | mplsxcdown }命令。

执行undo snmp-agent trap enable命令后不会显示mplsxcup或mplsxcdown海量告警信息，但并不改变该告警的开启或关闭状态。当再次执行snmp-agent trap enable命令时，自动恢复该告警原状态的显示信息。

使用实例

# 使能TE快速重路由私有告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable te te-frr private

# 打开mplsxcup海量告警。

<Quidway> system-view
[Quidway] snmp-agent trap enable lsp mplsxcup
