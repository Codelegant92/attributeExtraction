ipv6 default-route-advertise

命令功能

ipv6 default-route-advertise命令用来设置路由器发布的缺省IPv6路由。

undo ipv6 default-route-advertise命令用来取消此项功能。

缺省情况下，该功能被禁止。

命令格式

ipv6 default-route-advertise [ always | match default | route-policy route-policy-name ] [ cost cost ] [ tag tag ] [ level-1 | level-2 | level-1-2 ] [ avoid-learning ]

undo ipv6 default-route-advertise

参数说明

always：总是发布缺省IPv6路由，此为命令缺省值。

match default：如果路由表中存在其他路由协议生成的缺省IPv6路由，则在LSP中发布此缺省路由。

route-policy route-policy-name：指定路由策略名称。本地路由器只发布通过路由策略的外部路由；如果没有路由通过路由策略，则不发布缺省路由。

cost cost：指定缺省路由的开销值。

tag tag：发布的缺省路由的标记值。整数形式，取值范围是1～4294967295。

level-1：设置发布的缺省IPv6路由级别为Level-1。

level-2：设置发布的缺省IPv6路由级别为Level-2。

level-1-2：设置发布的缺省IPv6路由级别为Level-1-2。

avoid-learning：避免IS-IS进程学到缺省路由并添加到路由表。如果路由表中已存在的IPv6缺省路由为活跃状态，则将此路由置为不活跃状态。

说明：

如果不指定级别，则默认为生成Level-2级别的缺省路由。

使用指南

本命令可在Level-1或Level-2路由器上设置。

通过使用路由策略，可以强制IS-IS只在路由表中有匹配的路由项时才生成缺省路由。

  • 如果在路由策略视图中执行apply isis level-1命令，可以在L1 LSP中生成缺省路由。

  • 如果在路由策略视图中执行apply isis level-2命令，则可以在L2 LSP中生成缺省路由。

  • 如果在路由策略视图中执行apply isis level-1-2命令，则可以在L1 LSP、L2 LSP中各自生成缺省路由。

使用此命令前，必须先使能IS-IS进程的IPv6能力。配置过程请参见isis ipv6 enable命令。

使用实例

# 设置当前路由器在Level-1-2的LSP中生成缺省IPv6路由。

<Quidway> system-view
[Quidway] isis 1
[Quidway-isis-1] ipv6 default-route-advertise
