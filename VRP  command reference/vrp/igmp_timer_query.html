igmp timer query

命令功能

igmp timer query命令用来在接口上配置IGMP普遍组查询消息的发送间隔。

undo igmp timer query命令用来恢复该配置参数的缺省值。

缺省情况下，IGMP普遍组查询消息的发送间隔是60秒。

命令格式

igmp timer query interval

undo igmp timer query

参数说明

interval：表示路由器周期性发送IGMP普遍组查询消息的时间间隔。整数形式，取值范围是1～18000，单位是秒。

使用指南

在主机所在的共享网络上，可能同时连接着多台组播路由器。这些组播路由器之间自发进行选举过程，获胜者称为查询器，负责维护接口上IGMP组成员关系。组成员关系超时时间=IGMP普遍查询消息发送间隔 × 健壮系数 + 最大查询响应时间。

  • 当查询器启动时，发送“健壮系数”次的普遍组查询消息，询问该网络中哪些组播组存在成员。发送间隔是“普遍组查询消息发送间隔”的1/4。“健壮系数”可以使用igmp robust-count命令或robust-count命令设置。

  • 之后，查询器周期性的发送普遍组查询消息，维护接口上的组成员关系。“普遍组查询消息发送间隔”越小，查询器就越灵敏，但是网络带宽和路由器资源的占用也就越大。

该命令和timer query（IGMP视图）命令功能完全相同，差别只是作用范围不同。IGMP视图下的配置具有全局性，对所有接口有效；接口视图下的配置只对当前接口有效。系统优先采用接口视图下的配置。当接口视图下没有进行配置，才采用IGMP视图下的配置。

使用实例

# 设置接口GE1/0/0发送普遍组查询报文的时间间隔为50秒。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp timer query 50
