ipv6 route-static vpn6-instance

命令功能

ipv6 route-static vpn6-instance命令用来在VPN实例下配置IPv6静态路由。

undo ipv6 route-static vpn6-instance 命令用来撤销VPN实例下的单播IPv6静态路由。

缺省情况下，系统没有为VPN实例配置IPv6静态路由。

命令格式

ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address mask-length [ interface-type interface-number ] nexthop-ipv6-address [ preference value ] [ tag tag ] [ description text ]

ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address prefix-length nexthop-ipv6-address [ public ] [ preference preference ] [ tag tag ] [ description text ]

ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address prefix-length vpn6-instance vpn6-destination-name nexthop-ipv6-address [ preference preference ] [ tag tag ] [ description text ]

undo ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address prefix-length [ interface-type interface-number [ nexthop-ipv6-address ] | nexthop-ipv6-address ] [ preference preference ] [ tag tag ]

undo ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address prefix-length [ nexthop-ipv6-address ] [ public ] [ preference preference ] [ tag tag ]

undo ipv6 route-static vpn6-instance vpn6-instance-name dest-ipv6-address prefix-length [ vpn6-instance vpn6-destination-name nexthop-ipv6-addres ] [ preference preference ] [ tag tag ]

undo ipv6 route-static vpn6-instance vpn6-instance-name all

参数说明

vpn6-instance-name：IPv6 VPN实例名。每一个IPv6 VPN实例都有自己独立的单播路由表，配置的静态路由就会被加入到指定的IPv6 VPN实例的路由表中。

dest-ipv6-address：指定目的IPv6地址。

prefix-length：IPv6地址前缀的长度，即掩码中连续“1”的个数。“1”必须是连续的。

interface-type：接口类型。

interface-number：接口编号。

nexthop-ipv6-address：路由器的下一跳IPv6地址。

vpn6-destination-name：目的IPv6 VPN实例名。配置了目的IPv6 VPN实例名后，路由器可根据已设置的网关地址，在静态路由表中找出到目的IPv6 VPN实例的出接口。

public：指定网关是公网的地址。当某路由器上配置成属于某IPv6 VPN实例后，它的下一跳或下一跳网关路由器可以属于此IPv6 VPN实例，也可以属于公网。命令中有关键字public，说明指定下一跳为公网路由器。

preference：路由优先级。整数形式，取值范围为1～255。

tag tag：指定静态路由的tag属性值。整数形式，取值范围是1～4294967295。缺省值是0。配置不同的tag属性值，可对静态路由进行分类，以实现不同的路由管理策略。例如，其它协议引入静态路由时，可通过路由策略引入具有特定tag属性值的路由。

description text：对静态路由的描述信息。字符串形式，允许空格，长度范围是1～19。

all：删除为指定IPv6 VPN实例配置的所有路由。

使用指南

当IPv6 VPN网络结构比较简单，可以使用ipv6 route-static vpn6-instance命令为该VPN配置静态路由。巧妙的设置和使用静态路由可以改进网络的性能。但当网络发生故障或者拓扑发生变化后，静态路由不会自动改变，因此请谨慎配置静态路由。

目前的VRP版本中，配置VPN用户访问公网时，除了使用NAT外，还可以使用带关键字public的ipv6 route-static vpn6-instance命令来配置下一跳是公网的VPN路由。

如果将目的地址与掩码都配置为全0，则表示配置的是缺省路由。

使用实例

# 配置一条缺省路由，它的下一跳为2001::1。

<Quidway> system-view
[Quidway] ipv6 route-static vpn6-instance vpn1 :: 0 2001::1
