interval

命令功能

interval命令用来配置NQA测试例的报文间隔。

undo interval命令用来恢复缺省配置。

缺省情况下：

  • 对于对Jitter测试例、ICMPJitter测试例、PathJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例，报文间隔是20毫秒。

  • 对于其他类型的测试例，报文间隔是4秒。

命令格式

interval { milliseconds interval | seconds interval }

undo interval

参数说明

milliseconds interval：以毫秒为单位设置报文间隔。取值范围是20～60000。当配置的发包间隔的毫秒数是1000的整数倍时，系统会自动把毫秒数转换为秒的形式。

seconds interval：以秒设置报文间隔。取值范围是1～60。

缺省级别

2：配置级

使用指南

只能对Jitter、ICMPJitter、PathJitter、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例、ICMP、SNMP、LSP Ping、PWE3Ping、TCP、UDP、MPing或LSPJitter类型的测试例配置报文间隔。不能更改正在运行的测试例的报文间隔。

配置毫秒级的报文时间间隔，只能适用于Jitter、ICMPJitter、PathJitter、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter类型的测试例。

除了Jitter、ICMPJitter、PathJitter、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter类型的测试例，其他类型测试例的报文间隔时间必须大于timeout命令设置的测试超时时间。

使用实例

# 设置测试例user test的报文间隔为1000毫秒。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] interval milliseconds 1000
