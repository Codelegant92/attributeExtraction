command-privilege level rearrange

命令功能

command-privilege level rearrange用来将默认注册为2、3级的命令，分别批量提升到10、15级。

undo command-privilege level rearrange用来将批量提升到10、15级的命令重新恢复到2、3级。

命令格式

command-privilege level rearrange

undo command-privilege level rearrange

参数说明

无

视图

系统视图

缺省级别

3：管理级

使用指南

被提升的命令必须是没有被命令command-privilege level单独修改过的。被command-privilege level单独修改过的命令，维持原来级别不作提升。

命令级别恢复的操作对象只能是被批量提升的命令级别。被command-privilege level单独修改过的命令，维持原来级别不作修改。

在执行命令command-privilege level rearrange之前，需要用户首先设置相应的15级用户对应的super密码，不然非Console口的用户将无法提升自己的级别，无法访问相应级别的命令。

说明：

设置了相应的15级用户对应的super密码后，无法从系统取回。请妥善保管密码，以免遗忘或者遗失。

命令级别批量提升以后，命令undo command-privilege level rearrange的级别也被调整到了15级，所以应该确保执行此命令的用户级别是15级。

命令级别被批量提升后，会提示用户命令级别已经被批量提升，以减少对其他用户的影响。

命令被批量提升后，在没有被恢复之前，如果用户再次进行命令级别批量提升操作，此操作将会无效，不会改变当前的命令状态。

使用实例

# 将当前的命令级别批量提升。

<Quidway> system-view
[Quidway] command-privilege level rearrange
You have not set the super password corresponding to a Level 15 user.
It is recommended to quit the operation and set the password.
Are you sure to continue ?[ Y / N ] y
=== Message from Con0 to all other terminals : ===
The Command levels have been upgraded in batch !
