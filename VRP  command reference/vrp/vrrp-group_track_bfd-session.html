vrrp-group track bfd-session

命令功能

vrrp-group vrrp-group-id track bfd-session bfd-session-id { reduced priority-level | preempt }命令用来配置VRRP管理组监视BFD会话，以加快Master设备出现故障时的主备切换速度。

undo vrrp-group vrrp-group-id track bfd-session [ bfd-session-id ]命令用来删除VRRP管理组监视BFD会话。

命令格式

vrrp-group vrrp-group-id track bfd-session bfd-session-id { reduced priority-level | preempt }

undo vrrp-group vrrp-group-id track bfd-session [ bfd-session-id ]

参数说明

bfd-session-id：BFD会话ID，取值范围1～8191。

vrrp-group-id：管理组号，取值范围1～16。

reduced priority-level：当BFD会话Down时，Master优先级减小的值。取值范围1～254，缺省值是10。

preempt：抢占方式。当BFD会话Down时，Backup设备立即抢占成为Master。

使用指南

VRRP管理组监视BFD有两种情况：

  • 一种是备用路由器监视主用路由器是否可用，这时如果主用路由器Down，要求备用路由器尽快倒换到Master，而不是等超时后倒换，减少用户报文中断时间。

  • 另一种情况是监视网络侧链路是否可用，如果不可用则降低VGMP的优先级，如果重新变得可用升高优先级，必要时抢占。

使用实例

# 为VRRP管理组1配置监视本地标识符为10的BFD会话。

<Quidway> system-view
[Quidway] vrrp-group 1 track bfd-session 10 reduced 20
