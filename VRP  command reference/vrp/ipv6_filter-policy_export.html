ipv6 filter-policy export

命令功能

ipv6 filter-policy export命令用来配置IS-IS对引入的IPv6路由信息在发布时进行过滤。

undo ipv6 filter-policy export命令用来取消对引入的IPv6路由进行过滤。

缺省情况下，IS-IS将不对发布的IPv6路由信息进行过滤。

命令格式

ipv6 filter-policy { acl6-number | ipv6-prefix ipv6-prefix-name | route-policy route-policy-name } export [ protocol [ process-id ] ]

undo ipv6 filter-policy export [ protocol ]

参数说明

acl6-number：基本ACL6的编号，取值范围是2000～2999。

ipv6-prefix ipv6-prefix-name：IPv6地址前缀列表名称。

route-policy route-policy-name：基于标签和其它协议参数过滤路由的路由策略名称。

protocol：指定哪些已引入的路由信息在发布时要进行过滤，目前可包括direct、static、ripng、bgp、ospfv3以及其他isis进程的路由信息。如果省略该参数，将对所有要发布的路由进行过滤。

process-id：当protocol为ripng、ospfv3或其他isis进程时，需要指定进程号。

使用指南

ipv6 filter-policy export命令一般和ipv6 import-route命令结合使用，它只对已引入的路由在发布给其他路由器时进行过滤。如果没有指定protocol参数，将对所有协议引入的路由进行过滤；如果指定了protocol参数，则只对特定协议引入的路由进行过滤。

使用此命令前，必须先使能IS-IS进程的IPv6能力。配置过程请参见isis ipv6 enable命令。

使用实例

# 配置IS-IS使用编号为2002的ACL6对已引入的IPv6路由在发布给其他路由器时进行过滤。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] ipv6 filter-policy 2002 export
