filter-policy import (RIP)

命令功能

filter-policy import命令用来过滤接收的RIP路由信息。

undo filter-policy import命令用来取消过滤路由的操作。

命令格式

filter-policy acl-number import [ interface-type interface-number ]

filter-policy gateway ip-prefix-name import

filter-policy ip-prefix ip-prefix-name [ gateway ip-prefix-name ] import [ interface-type interface-number ]

undo filter-policy import [ interface-type interface-number ]

参数说明

acl-number：用于过滤路由信息目的地址的基本或高级ACL编号。整数形式，取值范围是2000～2999。

interface-type interface-number：接口类型和接口号，基于接收接口过滤路由。

ip-prefix：使用IP地址前缀列表过滤路由。

ip-prefix-name：用于过滤路由信息目的地址的地址前缀列表名。字符串形式，不支持空格，区分大小写，长度范围是1～169。

gateway：基于发布网关过滤路由。

使用指南

filter-policy import 命令用来过滤接收的RIP路由信息：

  • 对报文中的指定路由进行过滤

  • 不侦听来自指定路由器的路由更新报文

undo filter-policy import命令用来取消过滤路由的操作。如果已经配置了基于接口的路由策略，删除时必须指定interface-type interface-number，且一次只能删一个接口上的策略。

该命令在RIP进程下配置，如果基于接口对路由进行过滤，则一个接口只能配置一个路由策略；如果不指定接口，就认为是配置全局过滤策略，同样每次只能配置一个策略，如果重复配置，新的策略将覆盖之前的策略。

使用实例

# 设置按照地址前缀列表abc过滤从所有接口收到的RIP路由更新报文。

<Quidway> system-view
[Quidway] rip 100
[Quidway-rip-100] filter-policy ip-prefix abc import
