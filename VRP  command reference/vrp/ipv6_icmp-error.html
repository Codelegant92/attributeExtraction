ipv6 icmp-error

命令功能

ipv6 icmp-error命令用来限制ICMPv6消息的发送速率。

undo ipv6 icmp-error命令用来恢复速率限制和令牌桶容量的缺省值。

缺省情况下，令牌桶容量值为10，速率限制间隔为100毫秒。

命令格式

ipv6 icmp-error { bucket bucket-size | ratelimit interval } *

undo ipv6 icmp-error

参数说明

bucket bucket-size：指定令牌桶可容纳的令牌数。整数形式，取值范围是1～200。缺省值是10。

interval：指定速率限制间隔时间。整数形式，单位为毫秒，取值范围是0～2147483647。缺省值是100毫秒。

使用指南

使用令牌桶算法，一个令牌代表一个ICMPv6消息。每隔一定的时间向虚拟的桶中放置令牌，直到达到令牌桶允许的最大数量。一旦ICMPv6消息超出了桶的容量，报文就会被丢弃。

使用实例

# 设置ICMPv6错误报文的速率限制为100。

<Quidway> system-view
[Quidway] ipv6 icmp-error ratelimit 100

# 设置ICMPv6的令牌桶容量为50。

[Quidway] ipv6 icmp-error bucket 50

# 设置ICMPv6错误报文的速率限制为100，令牌桶容量为50。

[Quidway] ipv6 icmp-error bucket 50 ratelimit 100
