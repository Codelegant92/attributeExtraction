load-balance unequal-cost enable（接口视图）

命令功能

load-balance unequal-cost enable命令用来使能当前接口非等价负载分担功能。

undo load-balance unequal-cost enable 命令用来去使能当前接口非等价负载分担功能。

缺省情况下，不使能当前接口非等价负载分担功能。

命令格式

load-balance unequal-cost enable

undo load-balance unequal-cost enable

参数说明

无

使用指南

非等价负载分担UCMP（Unequal Cost Multiple Path），是指如果到达目的地有多条带宽不同的等价物理链路，则流量会根据带宽按比例分担到每条物理链路上。这样所有链路可根据带宽不同而分担不同比例的流量，使负载分担更合理。

说明：
  • 该命令只支持基于物理接口的配置，不支持基于逻辑接口的配置。
  • 只有当本设备中到达同一目标地址的所有接口上都使能了UCMP功能，且触发了FIB表项重新下刷，各链路才在该设备上进行非等价负载分担。如果其中任一接口没有使能UCMP功能，即使触发了FIB表项重新下刷，各链路仍进行等价负载分担。

当需要在带宽不同的几条等价链路上实现非等价负载分担时，可以在各接口上配置该功能。通过执行命令load-balance（系统视图）可以配置负载分担的方式，现在只支持通过流方式进行非等价负载分担。

通过命令行在物理接口上使能/去使能UCMP功能，并不立即触发路由重新计算或FIB重新下刷。可通过重启本设备中已经改变了非等价负载分担配置（使能/去使能）的任一接口来触发FIB表项重新下刷，从而使能UCMP。

非等价负载分担只是针对等价路由，也就是说与路由协议层面无关，不区分IGP还是BGP。

在各条非等价负载分担路径中，任一条链路的带宽不能小于总带宽的1/16。否则该链路不参与非等价负载分担。

使用实例

# 在接口GigabitEthernet1/0/0上使能非等价负载分担功能。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] load-balance unequal-cost enable
