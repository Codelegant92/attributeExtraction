display ipv6 fibcache

命令功能

display ipv6 fibcache命令用来查看FIB缓存中的路由信息。

命令格式

display ipv6 fibcache slot-id [ vpn6-instance vpn6-instance-name ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

slot-id：接口板的槽位号，仅在分布式环境中有效。

vpn6-instance vpn6-instance-name：IPv6 VPN实例的名称，用来查看指定IPv6 VPN实例的FIB缓存中的路由总数。字符串形式，区分大小写，长度范围是1～31。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

需要先使用ipv6 fibcache命令使能IPv6 VPN FIB缓存功能后，执行display ipv6 fibcache vpn6-instance命令才有显示信息。执行命令reset ipv6 fibcache可清空FIB缓存表项。

项目

描述

Destination

目的地

FIB PrefixLength

目的地网络地址前缀的长度

NextHop

转发报文的下一跳路由设备

FIB Flag
描述路由的符号，有以下几种情况：
  • B表示BlackHole
  • D表示Dynamic
  • G表示Gateway
  • H表示Host
  • S表示Static
  • U表示Up

Label

标签值。如果数据到下一跳需通过MPLS转发，应携带标签。

Tunnel ID

隧道标识符。如果报文到下一跳需通过MPLS转发，其Tunnel ID不为0。如果报文通过IP转发，其Tunnel ID为0。

Time Stamp

FIB缓存内部使用的，指的是添加路由到缓存的时间

reference

目前该字段无意义，其值始终为0。

Interface

转发报文的出接口

IP6Token

目前该字段无意义，其值始终为0。
