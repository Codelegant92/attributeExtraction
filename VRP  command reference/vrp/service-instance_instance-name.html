service-instance instance-name

命令功能

service-instance instance-name命令用来创建一个业务实例并指定其类型。如果业务实例已经存在，则直接进入该业务实例视图。

undo service-instance instance-name命令用来删除指定的业务实例。

缺省情况下，业务实例的状态为Down，类型为多点到多点（MP2MP）类型。

命令格式

service-instance instance-name service-instance-name [ service-type { mp2mp | p2p } ]

undo service-instance instance-name service-instance-name

参数说明

instance-name service-instance-name：指定业务实例名称。长度范围是1～31个字符，首位不能是空格，名称中间不能有空格、“?”和“-”。

service-type：指定业务实例的类型。

mp2mp：指定业务实例为多点到多点的类型。

p2p：指定业务实例为点到点的类型。

使用指南

业务实例是关联用户业务与MAC Tunnel的实体，业务实例在设备上体现用户的特定业务。数据业务必须首先与业务实例绑定，然后才能在MAC Tunnel中传输。

创建新的业务实例，必须保证已有的业务实例的数量没有达到最大值。业务实例创建以后，就不允许再更改其类型。

执行命令undo service-instance instance-name删除一个业务实例，则该业务实例下原有的所有配置也将全部被清除，而且与该业务实例相关的静态、动态和黑洞MAC表项也会被清除。

使用实例

# 创建业务实例，取名mysi。

<Quidway> system-view
[Quidway] service-instance instance-name mysi

# 创建点到点类型业务实例，取名myp2psi。

<Quidway> system-view
[Quidway] service-instance instance-name myp2psi service-type p2p
