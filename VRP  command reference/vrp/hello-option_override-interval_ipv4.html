hello-option override-interval（IPv4）

命令功能

hello-option override-interval命令用来配置Hello消息中携带的否决剪枝的时间间隔。

undo hello-option override-interval命令用来恢复时间间隔为缺省值。

缺省情况下，Hello消息中携带的否决剪枝的时间间隔是2500毫秒。

命令格式

hello-option override-interval interval

undo hello-option override-interval

参数说明

interval：表示Hello消息中携带的否决剪枝的时间间隔。整数形式，取值范围是1～65535，单位是毫秒。

使用指南

该命令对于PIM-SM和PIM-DM都有效。

路由器向外发送的Hello消息携带lan-delay（消息传输延迟时间）参数和override-interval（剪枝否决时间间隔）参数。override-interval表示允许下游路由器否决Prune剪枝动作的时间。当同一链路中的所有路由器上的override-interval值不同时，将进行协商从中选取最大值。

当一台路由器从上游接口接收到剪枝消息时，说明该LAN内还存在着其他的下游路由器。如果这台路由器仍然需要接收组播数据，则必须在override-interval时间内向上游发送剪枝否决消息。

lan-delay + override-interval ＝ PPT。路由器接收到下游剪枝消息后，并不立即执行剪枝操作，而是等待一段时间PPT（Prune-Pending Timer），超时则剪枝。在PPT时间内如果收到下游发来的剪枝否决消息，则取消剪枝操作。

使用实例

# 在公网实例中，配置Hello消息中携带的否决剪枝的时间间隔为2000毫秒。

<Quidway> system-view
[Quidway] pim
[Quidway-pim] hello-option override-interval 2000
