filter-policy import (OSPF)

命令功能

filter-policy import命令用来设置OSPF规则，对接收的OSPF区域内、区域间和自制系统外部的路由进行过滤。

undo filter-policy import命令用来取消对接收路由信息的过滤。

缺省情况下，不对接收到的路由信息进行过滤。

命令格式

filter-policy { acl-number | ip-prefix ip-prefix-name } import

undo filter-policy import

参数说明

acl-number：基本访问控制列表号。整数形式。

ip-prefix ip-prefix-name：指定地址前缀列表名称。字符串形式。

使用指南

filter-policy命令就可以用来设置路由信息的过滤条件，没有通过过滤的路由不会被添加进OSPF路由表，也不会对外发布出去。接收到的路由无论是否通过过滤，都不会对LSDB有影响。

说明：

由于OSPF是基于链路状态的动态路由协议，路由信息隐藏在链路状态中，所以不能使用filter-policy import命令对发布和接收的LSA进行过滤。该命令实际上是对OSPF计算出来的路由进行过滤，只有通过过滤的路由才被添加到路由表中。

使用实例

# 使用编号为2000的ACL对接收的路由信息进行过滤。

<Quidway> system-view 
[Quidway] ospf 100
[Quidway-ospf-100] filter-policy 2000 import
