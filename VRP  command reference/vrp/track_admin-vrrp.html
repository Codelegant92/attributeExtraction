track admin-vrrp

命令功能

track admin-vrrp命令用来把业务接口与指定的管理VRRP备份组进行绑定。

undo track admin-vrrp命令用来删除业务接口与管理VRRP备份组的绑定关系。

缺省情况下，没有配置业务接口与任何管理VRRP绑定。

命令格式

track admin-vrrp interface interface-type interface-number vrid virtual-router-id

undo track admin-vrrp

参数说明

interface interface-type interface-number：配置了管理VRRP的接口类型和编号。

vrid virtual-router-id：VRRP备份组号。十进制整数形式，取值范围是1～255。

视图

Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图、VE接口视图、VE子接口视图

缺省级别

2：配置级

使用指南

设备上配置了管理VRRP，为了使业务接口的状态可以跟随配置了管理VRRP的接口状态一同变化，可以使用track admin-vrrp命令，配置业务接口和管理VRRP的绑定。

业务接口与配置了管理VRRP的接口不能是同一个接口，否则业务的Flow Up、Flow Down状态与接口的Up、Down状态会相互影响。

一个管理VRRP最多可以和512个业务接口进行绑定，具体数量由PAF和License文件决定。

使用实例

# 在接口GE1/0/0上，创建VRRP备份组1，该备份组为管理VRRP。业务接口为GE1/0/0.1，配置业务接口与管理VRRP接口绑定。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ip address 10.1.1.1 24
[Quidway-GigabitEthernet1/0/0] vrrp vrid 1 virtual-ip 10.1.1.111
[Quidway-GigabitEthernet1/0/0] admin-vrrp vrid 1
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] track admin-vrrp interface gigabitethernet1/0/0 vrid 1
