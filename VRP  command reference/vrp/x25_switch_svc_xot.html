x25 switch svc xot

命令功能

x25 switch svc xot命令用来增加一条转发地址为XOT通道的X.25交换路由。

undo x25 switch svc xot命令用来删除指定的X.25交换路由。

缺省情况下，未配置X.25交换路由。

命令格式

x25 switch svc x.121-address [ sub-dest destination-address ] [ sub-source source-address ] xot ip-address &<1-6> [ xot-option ]

undo x25 switch svc x.121-address [ sub-dest destination-address ] [ sub-source source-address ] [ xot ip-address &<1-6> ]

参数说明

x.121-address：目的X.121地址。此参数为模式匹配字符串，长度为1～15个字符。关于模式匹配的具体说明请参见下面两个表。

表1 X.121模式匹配规则

通配符

匹配规则

举例

可匹配的字符串

*

匹配前面字符的0个或多个字符

fo*

fo、foo、fooo

+

匹配前面字符的0个或多个字符

fo+

fo、foo、fooo

^

匹配输入字符串的开始

^hell

hell、hello、hellaaa

$

匹配输入字符串的结尾

ar$

ar、car、hear

\char

匹配由char指定的单个字符

b\+

b+

.

匹配任意单个字符

l.st

last、lbst、lost

匹配任意0个或多个字符

fo.*

fo、foo、fot

.+

匹配任意1个或多个字符

fo.+

foo、fot、foot
表2 X.121地址模式匹配字符串的输入规则

字符

输入规则

*

不能置于字符串首部。不能置于符号^之后。不能置于符号+和*的前面和后面。

+

不能置于字符串首部。不能置于符号^之后。不能置于符号+和*的前面和后面。

\

不能置于字符串尾部。

^

不能置于符号+和*之前。

sub-dest destination-address：替换目的地址。

sub-source source-address：替换源地址。

ip-address：XOT连接的目的IP地址，最多可以同时配置六个。

xot-option：XOT通道参数选项，具体配置请参见下表。

表3 XOT通道参数选项

选项

意义

timer seconds

XOT连接的keepalive定时器的延时，定时器超时发送keepalive报文，以探测连接的可用性，取值范围是1～3600秒。

retry times

发送keepalive的最大失败次数，失败次数超过times时，断开XOT连接。取值范围是3～3600次。

source interface-type interface-number

发起XOT连接的接口。

使用指南

用户配置X.25 SVC的XOT交换命令后，可以从本端的X.25网跨越IP网实现和远端X.25网的互联。配置keepalive属性后，可支持XOT的链路检测。

使用实例

# 将虚电路1交换到IP地址为10.1.1.1的目的地址。

<Quidway>system-view
[Quidway] x25 switch svc 1 xot 10.1.1.1
