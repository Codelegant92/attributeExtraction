display controller cpos

命令功能

display controller cpos命令用来查看CPOS所有E1或T1通道的状态信息，以及再生段、复用段、和高阶通道的告警及错误信息。

命令格式

display controller cpos [ cpos-number ]

参数说明

cpos-number：CPOS的编号，包括槽号/卡号/端口号。如果不指定CPOS号，则显示所有CPOS的所有通道信息。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

表1 display controller cpos命令输出信息描述

项目

描述

Cpos4/0/0 current state :

CPOS接口当前的物理状态。

Frame-format SDH, multiplex AU-3, clock master, loopback not set

CPOS接口的物理层信息：帧格式为SDH、采用AU-3的复用路径、主时钟模式（使用内部时钟信号）、没有使能环回。

Tx: J0: 0x01, J1: "NetEngine", C2: 0x02

发送的开销字节。

Rx: J0: 0x01, J1: "NetEngine", C2: 0x02

接收的开销字节。

regenerator section:

再生段的告警和错误统计。

Multiplex section:

复用段的告警和错误统计。

Higher order path(VC-3-x):

高阶通道的告警和错误统计，VC-3-x表示第x个VC-3。采用AU-3的复用路径时，一个STM-1由三个VC-3复用而成，因此有三个高阶通道。如果采用AU-4的复用路径，则只有一个高阶通道VC-4。

Alarm:Error:

告警统计和错误统计。可能出现的错误类型有以下几类。

  • FRED：Receive Loss of Basic Frame Alignment，接收到基本帧丢失位。也可以认为是收到有红色告警错误的帧。

  • COFA：Change of Frame Alignment，帧对齐改变。

  • SEF：Severely Errored Frame，严重错帧，连续4个帧同步错误将产生一个SEF。

  • FERR：Framing Bit Error，指有Ft/Fs/FPS/FAS错误的帧。

  • CERR：CRC Error，循环冗余校验错。

  • BERR：PRBS Bit Error（随机码测试位错，只用于测试）。

  • BIP：Bit-Interleaved Parity，比特交叉奇偶校验。

  • REI：Remote Error Indication，远端错误指示。

上面前三种错误（FRED、COFA、SEF）统称为Alarm Error，简写为AERR。

Cpos4/0/0 CT1 1 is up

CPOS接口4/0/0的T1通道1当前的物理状态。

Frame-format ESF, clock master, loopback not set

T1通道的物理层信息：帧格式为ESF、主时钟模式（使用内部时钟信号）、没有使能环回。
