apply lsp vpn

命令功能

apply lsp vpn命令用来配置为满足策略路由过滤条件的VPN报文从指定的LSP转发。

undo apply lsp vpn命令用来取消配置。

缺省情况下，不对VPN报文进行策略路由。

命令格式

apply lsp vpn vpn-instance-name ce-address [ pe-address [ p-address | interface-type interface-number | secondary ] ]

undo apply lsp vpn vpn-instance-name ce-address

参数说明

vpn-instance-name：VPN对应的VPN实例名称。

ce-address：目的CE设备的IP地址。

pe-address：对端PE设备的IP地址，指定LSP经过的PE设备。

p-address：P设备的IP地址，与pe-address配合，指定LSP接下来经过的P设备。

interface-type interface-number：与pe-address配合，指定LSP接下来使用的出接口。

secondary：指定使用LDP FRR生成的备份LSP。如果不配置secondary参数，只选择主LSP转发；如果配置secondary参数，只选择备份LSP转发。

缺省级别

2：配置级

使用指南

配置了策略路由到PE、P的地址后，如果去往PE、P没有可达的LSP，则策略路由不起作用。

使用实例

# 配置一条策略路由，使满足ACL 3000过滤的VPN报文通过目的CE地址为20.1.1.1的LSP转发。

<Quidway> system-view
[Quidway] policy-based-route map1 permit node 10
[Quidway-policy-based-route-map1-10] if-match acl 3000
[Quidway-policy-based-route-map1-10] apply lsp vpn vpn1 20.1.1.1
