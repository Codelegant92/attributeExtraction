exchange-mode

命令功能

exchange-mode命令用来配置IKE的协商模式。

undo exchange-mode命令用来恢复缺省的协商模式。

缺省情况下，IKE协商采用主模式。

命令格式

exchange-mode { aggressive | main }

undo exchange-mode

参数说明

aggressive：指定IKE协商采用野蛮模式。

main：指定IKE协商采用主模式。

使用指南

在选择协商模式的时候，可以根据某些特定的网络环境需求来确定是用主模式还是用野蛮模式。

主模式将密钥交换信息与身份、验证信息相分离。这种分离保护了身份信息，从而提供了更高的安全性。

野蛮模式缺少身份认证，但可以满足某些特定的网络环境需求。如果无法预先知道发起者的地址、或者发起者的地址总在变化，而双方都希望采用预共享密钥验证方法来创建IKE SA，就可以用野蛮模式；另外，如果发起者已知响应者的策略，或者对响应者的策略有全面的了解，采用野蛮模式能够更快地创建IKE SA。

使用实例

# 配置IKE使用野蛮模式进行协商。

<Quidway> system-view
[Quidway] ike peer huawei
[Quidway-ike-peer-huawei] exchange-mode aggressive
