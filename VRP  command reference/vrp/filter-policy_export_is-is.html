filter-policy export (IS-IS)

命令功能

filter-policy export命令用来配置IS-IS对已引入的路由在向外发布时进行过滤。

undo filter-policy export命令用来取消过滤功能。

缺省情况下，不使能该过滤功能。

命令格式

filter-policy { acl-number | ip-prefix ip-prefix-name | route-policy route-policy-name } export [ protocol [ process-id ] ]

undo filter-policy export [ protocol ]

参数说明

acl-number：基本ACL的编号。

ip-prefix ip-prefix-name：IP地址前缀列表名称。

route-policy route-policy-name：基于标签和其它协议参数过滤路由的路由策略名称。

protocol：指定哪些已引入的路由信息在发布时要进行过滤，目前可包括：direct、static、rip、bgp、ospf以及其他isis进程的路由信息。如果省略该参数，将对所有发布的路由进行过滤。

process-id：当protocol为rip、ospf或其他isis进程时，需要指定进程号。

使用指南

当只需要发布满足特定条件的路由时，可以用该命令配置过滤规则，只有符合过滤条件的路由才能被发布。

filter-policy export命令一般和import-route命令结合使用，它只对已引入的路由在发布给其他路由器时进行过滤。如果没有指定protocol参数，将对所有协议引入的路由进行过滤；如果指定了protocol参数，则只对特定协议引入的路由进行过滤。

使用实例

# 配置IS-IS使用编号为2000的ACL对已引入的路由在发布给其他路由器时进行过滤。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] filter-policy 2000 export
