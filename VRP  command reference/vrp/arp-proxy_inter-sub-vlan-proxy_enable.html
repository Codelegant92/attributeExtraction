arp-proxy inter-sub-vlan-proxy enable

命令功能

arp-proxy inter-sub-vlan-proxy enable命令用来启动VLAN间的Proxy ARP功能或者使能终结子接口的ARP代理功能。

undo arp-proxy inter-sub-vlan-proxy enable命令用来关闭VLAN间的Proxy ARP功能或者终结子接口的ARP代理功能。

缺省情况下，关闭VLAN间Proxy ARP功能。

说明：

除NE5000E/NE80E/NE40E产品外，其它产品执行arp-proxy enable命令使能VLAN间Proxy ARP功能。

命令格式

arp-proxy inter-sub-vlan-proxy enable

undo arp-proxy inter-sub-vlan-proxy enable

参数说明

无

视图

VLANIF接口视图、Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图、VE子接口视图

缺省级别

2：配置级

使用指南

如果需要在不同的VLAN间的实现二层互通，必须在接口上使能VLAN间Proxy ARP功能。

在Super-VLAN内，Sub-VLAN间的主机默认是不能互访的，如果需要互访，必须在Super-VLAN的VLANIF接口上配置VLAN间Proxy ARP功能。

不同设备支持VLAN间Proxy ARP的接口不同，如下表所示。

Ethernet子接口、GE子接口、Eth-Trunk子接口、VE子接口需要配置QinQ，即封装二层VLAN后，才能支持VLAN间Proxy ARP功能。

使用实例

# 使能GE子接口上的VLAN间Proxy ARP功能。

<Quidway> system-view
[Quidway] interface gigabitethernet2/0/9.3
[Quidway-GigabitEthernet2/0/9.3] arp-proxy inter-sub-vlan-proxy enable

# 使能终结子接口GE1/0/0.1上的Proxy ARP功能。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] mode user-termination
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 dot1q-termination
[Quidway-GigabitEthernet1/0/0.1] arp-proxy inter-sub-vlan-proxy enable
