snmp-agent trap suppress feature-name lsp

命令功能

snmp-agent trap suppress feature-name lsp命令用来配置LSP海量告警的抑制时间。

undo snmp-agent trap suppress feature-name lsp命令用来删除LSP海量告警的抑制时间。

缺省情况下，LSP海量告警的抑制时间是60秒。

命令格式

snmp-agent trap suppress feature-name lsp trap-name { mplsxcup | mplsxcdown } trap-interval trap-interval [ max-trap-number max-trap-number ]

undo snmp-agent trap suppress feature-name lsp trap-name { mplsxcup | mplsxcdown } trap-interval

参数说明

mplsxcup：设置LSP状态变为Up的海量告警。

mplsxcdown：设置LSP状态变为Down的海量告警。

trap-interval trap-interval：设置LSP海量告警的抑制时间间隔。整数形式，取值范围是0～65535，单位是秒。缺省值是60秒。

max-trap-number max-trap-number：设置在抑制间隔内最多可以发送的trap数量，整数形式，取值范围是0～65535。缺省值是60秒发送60个。

使用指南

为了防止LSP状态的频繁变化导致输出大量告警，可以配置此命令来设置LSP海量告警的抑制时间间隔。每个抑制时间间隔内，指定类型的告警信息只输出一次，从而减少了告警的输出数量。

例如设置LSP状态变为Down的告警的抑制时间间隔为60秒，则每60秒输出一条LSP状态变为Down的告警，其余同类告警不再输出，而是通过日志记录60秒内被抑制的同类告警的数量。到下一抑制时间间隔，新的LSP状态变为Down的告警才能再次被输出一条。

snmp-agent trap suppress feature-name lsp和snmp-agent trap enable lsp { mplsxcup | mplsxcdown }命令相互依存，配置的时候需要注意以下几点：

  • 执行snmp-agent trap suppress feature-name lsp命令配置LSP海量告警的抑制时间前，必须执行snmp-agent trap enable lsp { mplsxcup | mplsxcdown }命令打开mplsxcup或mplsxcdown海量告警开关。且打开mplsxcup或mplsxcdown海量告警开关的同时，LSP海量告警的抑制时间自动设置为60秒。

  • 当执行undo snmp-agent trap enable lsp { mplsxcup | mplsxcdown }命令关闭mplsxcup或mplsxcdown海量告警开关时，LSP海量告警的抑制时间恢复为60秒。

  • 如果在LSP海量告警的抑制时间间隔内重新配置了snmp-agent trap enable lsp { mplsxcup | mplsxcdown }命令，则LSP海量告警的抑制时间会重置为60秒。

使用实例

# 设置LSP状态变为Up的海量告警的抑制时间间隔为80秒。

<Quidway> system-view
[Quidway] snmp-agent trap suppress feature-name lsp trap-name mplsxcup trap-interval 80
