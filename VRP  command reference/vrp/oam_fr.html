oam（FR）

命令功能

oam命令用来使能FR链路的故障检测和故障通告。

undo oam命令用来去使能FR链路的故障检测和故障通告。

命令格式

oam { detect { full-status-report | liv } | notify { full-status-report | liv } } *

undo oam { detect { full-status-report | liv } | notify { full-status-report | liv } } *

参数说明

detect：PE设备检测本端AC链路故障的方式。

full-status-report：用全状态报文进行故障检测和通告，并传递PVC的状态。

liv：只用链路完整性验证(Link Integrity Verification Only)报文进行故障检测和通告。

notify：把PSN侧故障通告给本地CE。

使用指南

PE检测到本端AC链路故障后，会把此故障发送给OAM-Manager模块。通过OAM-Manager模块把此AC链路故障通告给远端PE，从而完成端到端的FRR功能。

对于AC接口：

如果是DCE类型的FR主接口，不仅支持LIV类型的故障检测和通告，也支持全状态检测或通告。

如果是DTE类型FR主接口，只支持LIV方式的故障检测和故障通告，不支持全状态故障检测和通告。

如果是DCE类型FR子接口，只支持全状态故障通告，不支持全状态故障检测，也不支持LIV方式的故障检测和通告。

如果是DTE类型的FR子接口，支持LIV方式和全状态方式的故障检测，不支持故障通告。

如果是NNI类型的FR接口，既不支持故障检测也不支持故障通告。

使用实例

# 配置FR的DCE类型主接口采用全状态故障检测和通告。

[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] link-protocol fr
[Quidway-Pos1/0/0] fr interface-type dce
[Quidway-Pos1/0/0] oam detect full-status-report notify full-status-report
