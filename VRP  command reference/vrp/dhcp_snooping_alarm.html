dhcp snooping alarm

命令功能

dhcp snooping alarm命令用来打开被丢弃的ARP报文、IP报文、DHCP Reply报文（Untrusted端口丢弃的）和DHCP Request报文数量达到告警阈值以及用户数达到最大用户数限制的告警阈值后的告警开关。

undo dhcp snooping alarm命令用来关闭告警开关。

缺省情况下，告警开关是关闭的。

命令格式

dhcp snooping alarm { arp | ip | dhcp-request | dhcp-chaddr | dhcp-reply | user-limit } enable [ interface interface-type interface-number ]

undo dhcp snooping alarm { arp | ip | dhcp-request | dhcp-chaddr | dhcp- reply | user-limit } enable [ interface interface-type interface-number ]

参数说明

arp：丢弃ARP报文达到告警阈值后产生告警。

ip：丢弃IP报文达到告警阈值后产生告警。

dhcp-reply：untrusted接口上丢弃的DHCP Server应答报文（包括Offer、ACK和NAK报文）达到告警阈值后产生告警。

dhcp-chaddr：丢弃DHCP报文里的CHADDR字段和以太帧头的源MAC不一致的报文，达到告警阈值后产生告警。

dhcp-request：丢弃DHCP Request报文达到告警阈值后产生告警。

user-limit：用户数达到最大用户数限制的告警阈值后产生告警。

interface-type：接口类型。此参数仅在VLAN视图下才可配置。

interface-number：接口号。此参数仅在VLAN视图下才可配置。

视图

Ethernet接口视图、Ethernet子接口视图、GE接口视图、GE子接口视图、Eth-Trunk接口视图、Eth-Trunk子接口视图、QinQ终结子接口视图、Doltq终结子接口视图、Virtual-Ethernet接口视图、VLAN视图

缺省级别

2：配置级

使用指南

执行dhcp snooping alarm后，就使能了向网管告警的功能。

在接口视图或VLAN视图下，执行本命令之前必须先在系统视图下执行dhcp snooping enable命令使能全局下的DHCP Snooping功能。

说明：
如果当前VLAN是RRPP的控制VLAN，则无法使能用户数达到告警阈值后的告警功能。

使用实例

# 在Ethernet4/0/0接口上打开ARP报文丢弃告警开关。

<Quidway> system-view
[Quidway] interface ethernet 4/0/0
[Quidway-Ethernet4/0/0] dhcp snooping alarm arp enable

# 在VLAN100内打开用户数达到最大用户数限制的告警阈值的告警开关。

<Quidway> system-view
[Quidway] dhcp snooping enable
[Quidway] vlan 100
[Quidway-vlan100] dhcp snooping alarm user-limit enable

# 在接口GE1/0/2上打开用户数达到最大用户数限制的告警阈值的告警开关。

<Quidway> system-view
[Quidway] dhcp snooping enable
[Quidway] interface gigabitethernet1/0/2
[Quidway-gigabitethernet1/0/2] dhcp snooping alarm user-limit enable
