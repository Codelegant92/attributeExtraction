authorization-mode

命令功能

authorization-mode命令用来配置当前授权方案中的授权模式。

缺省情况下，采用本地授权模式。

命令格式

authorization-mode { hwtacacs | if-authenticated | local }* [ none ]

authorization-mode none

参数说明

hwtacacs：使用HWTACACS授权模式。

local：本地授权模式。

if-authenticated：使用if-authenticated授权模式。如果用户通过了认证，同时使用的认证模式不是none，则使用户授权通过，否则授权不通过。

None：直接使其授权通过。

视图

授权方案视图

缺省级别

3：管理级

使用指南

如果在一个授权方案中使用多次授权，授权模式的执行顺序按照配置的先后，只有在当前授权模式没有响应时，才会尝试下一个授权模式，如果授权失败则将不会再进行授权。

如果选用了直接授权（none），它必须做为最后一种授权模式。

使用实例

# 配置名为scheme1的授权方案采用本地授权模式。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authorization-scheme scheme1
[Quidway-aaa-author-scheme1] authorization-mode local

# 配置名为scheme2的授权方案首先使用if-authenticated的授权模式，然后是本地授权，再然后使用hwtacacs，如果前面这三种授权模式都没有响应，则直接使用户授权通过。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authorization-scheme scheme2
[Quidway-aaa-author-scheme2] authorization-mode if-authenticated local hwtacacs none
