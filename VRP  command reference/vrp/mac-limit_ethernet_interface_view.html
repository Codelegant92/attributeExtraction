mac-limit（Ethernet接口视图）

命令功能

mac-limit命令用来配置当前接口或VLAN上的MAC地址学习限制规则。

undo mac-limit命令用来删除配置的MAC地址学习限制规则。

命令格式

mac-limit { action { discard | forward } | alarm { disable | enable } | maximum max rate interval } *

undo mac-limit

参数说明

action：当MAC地址表项数目达到限制后，系统所采取的动作。

discard：当MAC地址表项数目达到限制后，使用新MAC地址的报文被丢弃。

forward：当MAC地址表项数目达到限制后，使用新MAC地址的报文被继续转发，但不加入MAC地址表。

alarm：当MAC地址表项数目达到限制后，是否发送告警。

disable：当MAC地址表项数目达到限制后，不发送告警。

enable：当MAC地址表项数目达到限制后，以syslog方式提示告警。

max：当前接口或当前VLAN上最多可以学习的MAC数量。整数形式，取值范围是0～64512。当配置值为0时，表示不对学习数量进行限制。

interval：当前接口或当前VLAN上每次进行MAC学习的时间间隔。整数形式，取值范围是0～1000，单位是毫秒。当配置值为0时，表示不对学习速度进行限制。

缺省级别

2：配置级

使用指南

对于子接口，配置MAC学习限制规则前，必须已经绑定了VSI。

如果接口或VLAN上已经学习了一些MAC地址，请使用undo mac-address dynamic命令清除已经学习的MAC地址，否则会使MAC地址学习限制功能的数量限制不准确。

使用实例

# 配置以太接口GE1/0/0的MAC地址学习限制规则为最多允许学习100个MAC地址，每次MAC学习的时间间隔为50毫秒。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] portswitch
[Quidway-GigabitEthernet1/0/0] mac-limit maximum 100 rate 50

# 配置VLAN2最多允许学习1000个MAC地址，每次学习的间隔为100毫秒，达到限制后，使用新MAC地址的报文被继续转发。

<Quidway> system-view
[Quidway] vlan 2
[Quidway-vlan2] mac-limit action forward alarm enable maximum 1000 rate 100
