qinq stacking vid

命令功能

qinq stacking vid命令用来对进入以太网子接口的用户VLAN Tag上再打上一层Tag，即VLAN Stacking。

undo qinq stacking vid命令用来取消对指定接入的用户VLAN Tag进行stacking处理。

缺省情况下，子接口没有配置Stacking功能。

命令格式

qinq stacking vid low-ce-vid [ to high-ce-vid ] [ vlan-group group-id ]

undo qinq stacking vid low-ce-vid [ to high-ce-vid ] [ vlan-group group-id ]

参数说明

low-ce-vid：CE的VLAN ID（内层Tag的值）取值范围的下限，十进制整数形式，取值范围是1～4094。

high-ce-vid：CE的VLAN ID（内层Tag的值）取值范围的上限，十进制整数形式，取值范围是1～4094。

vlan-group group-id：用户组号，十进制整数形式，取值范围是1～16。

视图

Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图

缺省级别

2：配置级

使用指南

在子接口上使用qinq stacking vid命令前，应先在该子接口对应的主接口下配置mode user-termination命令，否则系统会提示配置错误。

qinq stacking vid命令与control-vid命令不能同时存在，因此使用qinq stacking vid命令前请不要配置control-vid命令。

如果以太网子接口收到的用户报文不带tag，那么该子接口会丢弃该用户报文。

在同一主接口上，不同的以太网子接口下使用qinq stacking vid命令，配置的ce-vid的范围不能重叠。

使用实例

# 在GE1/0/0.1子接口上配置VLAN Stacking功能，ce-vid的取值范围是100～200，用户组号为1。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet] mode user-termination
[Quidway-GigabitEthernet] quit
[Quidway] interface gigabitethernet1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] vlan-group 1
[Quidway-GigabitEthernet1/0/0.1-vlan-group 1] quit
[Quidway-GigabitEthernet1/0/0.1] qinq stacking vid 100 to 200 vlan-group 1
