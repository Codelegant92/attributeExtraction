snmp-agent trap enable feature-name l3vpn

命令功能

snmp-agent trap enable feature-name l3vpn命令用来打开L3VPN相关事件的告警开关。

undo snmp-agent trap enable feature-name l3vpn命令用来关闭L3VPN相关事件的告警开关。

命令格式

snmp-agent trap enable feature-name l3vpn { non-excessive all | trap-name { l3vpn_mib_trap_mid_thresh_exceed | l3vpn_mib_trap_thresh_cleared | l3vpn_mib_trap_thresh_exceed | l3vpn_mib_trap_vrf_down | l3vpn_mib_trap_vrf_up } }

undo snmp-agent trap enable feature-name l3vpn { all | non-excessive all | trap-name { l3vpn_mib_trap_mid_thresh_exceed | l3vpn_mib_trap_thresh_cleared | l3vpn_mib_trap_thresh_exceed | l3vpn_mib_trap_vrf_down | l3vpn_mib_trap_vrf_up } }

参数说明

non-excessive all：打开L3VPN所有非海量事件的告警开关。

trap-name：打开指定类型L3VPN事件的告警开关。

l3vpn_mib_trap_mid_thresh_exceed：打开私网路由数量超过私网路由表的路由中限事件的告警开关。该路由中限值是由命令routing-table limit（VPN实例视图）或者命令prefix limit设定的。

l3vpn_mib_trap_thresh_cleared：打开私网路由数量降低到门限值事件的告警开关。该门限值是由命令routing-table limit（VPN实例视图）或者命令prefix limit设定的。

l3vpn_mib_trap_thresh_exceed：打开私网路由数量超过私网路由表的路由上限事件的告警开关。该上限是由命令routing-table limit（VPN实例视图）或者命令prefix limit设定的。

l3vpn_mib_trap_vrf_down：打开VRF Down事件的告警开关。

l3vpn_mib_trap_vrf_up：打开VRF Up事件的告警开关。

all: 关闭L3VPN模块所有的告警开关。

使用指南

目前定义的L3VPN非海量事件有：l3vpn_mib_trap_mid_thresh_exceed、l3vpn_mib_trap_thresh_cleared、l3vpn_mib_trap_thresh_exceed、l3vpn_mib_trap_vrf_down、l3vpn_mib_trap_vrf_up。

当用户希望打开L3VPN模块所有非海量事件的告警开关时，可以选定non-excessive all批量处理；当用户希望打开某个或几个事件的告警开关时，可以指定type-name。

使用实例

# 在系统视图下打开VRF Down事件的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name l3vpn trap-name l3vpn_mib_trap_vrf_down

# 在系统视图下关闭VRF Down事件的告警开关。

<Quidway> system-view
[Quidway] undo snmp-agent trap enable feature-name l3vpn trap-name l3vpn_mib_trap_vrf_down
