startup paf

命令功能

startup paf命令用来设置下次启动的PAF文件。

命令格式

startup paf { default | file-name } [ slave-board ]

参数说明

default：指定系统从下次启动的系统软件中读取PAF文件进行启动。

file-name：指定下次启动时PAF文件的存储路径和文件名。字符串形式，格式为[ dirive-name ][ file-name ]，长度范围1～48。如果未指定dirive-name，则此值为默认的设备名。

slave-board：指定下次启动后备用主控板的PAF文件名。

使用指南

PAF文件的扩展名应为.txt，必须存放在存储设备的根目录下，缺省的存储位置为CF卡。

说明：

目前此命令仅在NE40E、NE80E、NE5000E上可用。

使用实例

# 设置下一次启动的PAF文件。

<Quidway> startup paf cfcard:/paf.txt 
Info: Succeeded in setting main board resource file for system.

# 设置下一次启动时从系统软件中读取PAF文件。

<Quidway> startup paf default 
Info: Succeeded in setting main board resource file for system.
