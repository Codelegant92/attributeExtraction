filter-policy export (OSPF)

命令功能

filter-policy export命令用来设置OSPF规则，对import-route (OSPF)命令引入的路由信息向OSPF自治系统内部以5类LSA形式发布时进行过滤。

undo filter-policy export命令用来取消设置的过滤规则。

缺省情况下，不对发布的路由信息进行过滤。

命令格式

filter-policy { acl-number | ip-prefix ip-prefix-name } export [ protocol [ process-id ] ]

undo filter-policy export [ protocol [ process-id ] ]

参数说明

acl-number：基本访问控制列表号。整数形式。

ip-prefix ip-prefix-name：地址前缀列表名称。字符串形式。

protocol：发布路由信息的协议，目前的协议包括direct、rip、isis、bgp、ospf和static。

process-id：当发布的路由协议为rip、isis、ospf时，可以指定进程号，缺省值是1。

使用指南

本命令需要配置在ASBR上。

有些情况下，只有符合某些条件的路由信息才能发布。filter-policy命令就可以用来设置路由信息的过滤条件，只有通过过滤的信息才能被发布出去。

使用实例

# 使用编号为2000的ACL对OSPF发布的路由进行过滤。

<Quidway> system-view 
[Quidway] ospf 100
[Quidway-ospf-100] filter-policy 2000 export
