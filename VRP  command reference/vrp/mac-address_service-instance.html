mac-address service-instance

命令功能

mac-address service-instance命令用于配置业务在出隧道后到达目的CE设备的静态转发MAC地址表项。本命令用来配置基于业务实例的静态MAC表项和黑洞MAC表项。

表项内容包括：

  • 报文的目的MAC地址；

  • 隧道出接口编号；

  • 业务实例的名称；

  • 目标CE设备的VLAN。

undo mac-address命令用来删除基于指定业务实例的MAC地址表项。

命令格式

mac-address static mac-address interface-type interface-number service-instance service-instance-name vlan vlan-id

undo mac-address mac-address service-instance service-instance-name

参数说明

static：指定表项的类型为静态表项。静态表项不老化，系统复位或接口板热插拔后不丢失。

mac-address：指定报文的目的MAC地址。格式为H-H-H。其中H为1～4位的十六进制数，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。该MAC地址不可以是组播或广播地址，也不能是设备的虚MAC地址。

interface-type interface-number：指定需要被隔离的接口。

其中：

  • interface-type表示接口的类型，包括以太网接口和由Eth-Trunk接口。

  • interface-number表示接口的编号。

service-instance service-instance-name：指定报文所属业务实例名字。长度范围是1～31个字符，首位不能是空格。

vlan vlan-id：指定报文目标CE设备所属的外层VLAN ID，取值范围是1～4094。

使用指南

通过将业务实例与设备的接口、目的CE设备的MAC地址和CE设备所属的VLAN ID相映射，实现业务出隧道后被静态转发给目标CE设备。

配置接口+VLAN时，可以将用户报文的外层VLAN替换为MAC表项中配置的VLAN，但是必须保证映射到该服务实例的用户使用了该接口，对于外层VLAN则没有限制。

静态MAC地址表项配置后即生效，不需要经过MAC地址学习过程，能更快速的实现业务传输。

该命令只对多点到多点的业务实例有效。

静态MAC地址表项中的业务实例必须已经与MAC Tunnel绑定。

使用实例

# 配置目的MAC地址为0-0-1的报文经业务实例mysi传输后，从接口GE1/0/1静态转发到在VLAN 30内的CE设备。

[Quidway] mac-address static 0-0-1 gigabitethernet1/0/1 service-instance mysi vlan 30
