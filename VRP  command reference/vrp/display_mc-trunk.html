display mc-trunk

命令功能

display mc-trunk命令用来显示MC-Trunk的信息。

命令格式

display mc-trunk mctrunk-id

参数说明

mc-trunk mctrunk-id：MC-Trunk的编号。整数形式，取值范围为1～16。

所有视图

缺省级别

1：监控级

使用指南

配置MC-Trunk后，可以使用该命令查看MC-Trunk的相关信息。

使用实例

# 显示ID为1的MC-Trunk中的信息。

<Quidway> display mc-trunk 1

项目

描述

MC-TRUNK-ID

MC-Trunk的ID。

Priority

MC-Trunk的优先级。

System-ID

MC-Trunk的系统ID。

Peer-IP/Source-IP

MC-Trunk对端IP/源端IP。

State
MC-Trunk的主备状态。有以下取值：

  • 1：Init

  • 2：Backup

  • 3：Master

Causation
MC-Trunk主备状态的原因。有以下取值：

  • PRI(1):根据优先级确定的主备状态。

  • TIMEOUT(2):本端在定时器超时后没有收到对端协议报文，从备用转为主用。

  • BFD_DOWN(3):本端的BFD检测到本端与对端间的链路Down。

  • PEER_TIMEOUT(4):对端在定时器超时后没有收到本端协议报文，从备用转为主用。

  • PEER_BFD_DOWN(5):对端的BFD检测到对端与本端间的链路Down。

  • ALL_MEMBER_DOWN(6):MC-Trunk中的所有成员Down。

  • INIT(7):MC-Trunk正在初始化。

Send-Period

MC-Trunk发送报文周期。

Fail-Time

MC-Trunk接收超时时间。

Receive/Send

MC-Trunk接收/发送的报文。

RecDrop/SndDrop

MC-Trunk接收/发送丢弃的报文。

Peer-Priority

对端优先级。

Peer-System-ID

对端系统ID。

Peer-Fail-Time

对端超时时间。

BFD-Session

MC-Trunk绑定的BFD会话。

Type

成员类型。取值为: Eth-Trunk。

ID

成员ID。

LocalPhyState
成员链路的物理状态。有以下取值：

  • 1: Up

  • 2: Down

Work-Mode
成员工作模式。有以下取值：

  • 1: auto

  • 2: force-backup

  • 3: force-master

State
成员主备状态。有以下取值：

  • 1: Master

  • 2: Backup

Causation
成员主备状态的原因。有以下取值：

  • FORCE_BACKUP(1): 成员的工作模式为强制备用。

  • FORCE_MASTER(2): 成员的工作模式为强制主用。

  • MCTRUNK_INIT(3): 成员的工作模式为自动，但是MC-Trunk正在初始化。

  • MCTRUNK_BACKUP(4):成员的工作模式为自动,但是MC-Trunk的状态为备用。

  • MCTRUNK_MASTER(5):成员的工作模式为自动,但是MC-Trunk的状态为主用。

  • PEER_MEMBER_DOWN(6):MC-Trunk对端的成员状态为Down。

  • PEER_MEMBER_UP(7):MC-Trunk对端的成员状态为Up。
