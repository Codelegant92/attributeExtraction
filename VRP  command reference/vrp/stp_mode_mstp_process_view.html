stp mode（MSTP进程视图）

命令功能

stp mode命令用来配置当前MSTP进程的工作模式。

undo stp mode命令用来恢复当前MSTP进程的缺省工作模式。

缺省情况下，当前MSTP进程的工作模式为MSTP。

命令格式

stp mode { mstp | rstp | stp }

undo stp mode

参数说明

mstp：MSTP模式。

rstp：RSTP模式。

stp：STP兼容模式。

视图

MSTP进程视图

缺省级别

2：配置级

使用指南

MSTP进程被创建后，其缺省的工作模式是MSTP。

MSTP、RSTP和STP的协议报文各不相同。为了实现MSTP、RSTP和STP协议间的兼容，设备提供了三种工作模式：MSTP模式、RSTP模式和STP模式。

  • 执行stp mode mstp命令后，设备会使所有不和STP或者RSTP交换机直接相连的端口工作在MSTP模式下，和STP或者RSTP交换机直接相连的端口工作在STP模式或者RSTP模式下。

  • 执行stp mode stp命令后，设备会将所有端口的工作模式设置为STP模式，即向外发送STP BPDU报文。如果对接的设备只支持STP协议，那么可以选择此种模式。

  • 执行stp mode rstp命令后，设备会将所有端口的工作模式设置为RSTP模式，即向外发送RSTP BPDU报文。如果对接的设备只支持RSTP协议，那么可以选择此种模式。

使用实例

# 配置MSTP进程1的工作模式为STP模式。

<Quidway> system-view
[Quidway] stp process 1
[Quidway-stp-process-1] stp mode stp
