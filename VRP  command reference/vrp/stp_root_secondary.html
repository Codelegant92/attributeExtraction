stp root secondary

命令功能

stp root secondary命令用来配置当前交换机作为指定生成树实例的备份根交换机。

undo stp root命令用来取消当前交换机作为指定生成树实例的备份根交换机资格。

缺省情况下，交换机不作为生成树实例的备份根交换机。

命令格式

stp [ instance instance-id ] root secondary

undo stp [ instance instance-id ] root

参数说明

instance instance-id：指定生成树实例。instance-id表示生成树实例的编号，取值范围是0～48，取值为0表示的是CIST。

使用指南

注意：

如果当前交换机已配置为指定生成树实例的备份根交换机，就不能再改变优先级。否则可能会使备份根交换机失去它的地位。

在使用stp root secondary命令和undo stp root命令时，如果不输入instance instance-id参数，则所作的配置在CIST实例上生效。

可以为每个生成树实例配置一到多个备份根交换机。当根交换机出现故障或被关机时，备份根交换机可以取代根交换机而成为指定生成树实例的根交换机。如果配置了多个备份根交换机，则MAC地址最小的备份根交换机将成为指定生成树实例的根交换机。

使用实例

# 指定当前交换机为指定生成树实例4的备份根交换机。

<Quidway> system-view
[Quidway] stp instance 4 root secondary
