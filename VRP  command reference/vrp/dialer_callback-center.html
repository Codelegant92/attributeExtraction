dialer callback-center

命令功能

dialer callback-center命令用来配置PPP回呼的参照依据。

undo dialer callback-center命令用来取消路由器的回呼Server功能。

缺省情况下，未配置PPP回呼Server功能。

命令格式

dialer callback-center [ dial-number | user ] *

undo dialer callback-center

参数说明

dial-number：根据local-user callback-number命令中配置的参数telephone-number进行回呼。

user：根据dialer route命令中的参数user hostname进行回呼。

视图

接口视图、Dialer接口视图

缺省级别

2：配置级

使用指南

在配置该命令前，必须先执行命令dialer user设置对端用户名，或执行命令dialer enable-circular使能轮询DCC。

当应用PPP回呼时，在Server端必须配置本命令。

参数user表示DCC将使用配置的dialer route命令参数进行回呼，参数dial-number 表示DCC将使用local-user命令中配置的callback-number向对端回呼。

当user和dial-number两个参数同时被应用时，路由器首先尝试按照第一个参数的要求进行回呼，当无法进行回呼时，再尝试应用第二个参数进行回呼。

使用实例

# 配置对端用户名，并设置向该呼叫用户名进行回呼。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] local-user quidwayb password cipher quidwayb
[Quidway-aaa] quit
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] dialer route ip 1.1.1.2 user quidwayb 8810052
[Quidway-Serial1/0/0] dialer callback-center user
