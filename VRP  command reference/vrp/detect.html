detect

命令功能

detect命令用来指定ASPF策略阻塞的应用层协议。

undo detect命令用来取消所做的配置。

命令格式

detect {activex-blocking [ acl acl-number ] | all | ftp | h323 | http | hwcc | java-blocking [ acl acl-number ] | rtsp | smtp | tcp | udp }

undo detect {activex-blocking | all | ftp | h323 | http | hwcc | java-blocking | rtsp | smtp | tcp | udp }

参数说明

acl-number：基本访问控制列表（ACL）号，取值范围是2000～2999。

activex-blocking：ActiveX阻塞。

all：检测所有应用层协议。

ftp：FTP协议检测。

h323：H.323协议检测。

http：HTTP协议检测。

hwcc: HWCC协议检测。

java-blocking：java阻塞。

rtsp：RSTP协议检测。

smtp：SMTP协议检测。

tcp：TCP协议检测。

udp：UDP协议检测。

使用指南

当需要过滤应用层的FTP、H.323、HTTP、HWCC、MSN、QQ、RTSP、SMTP协议报文或者TCP/UDP报文时，或者实现Java阻断及ActiveX阻断时，可以配置ASPF。

使用实例

# 配置ASPF策略，策略号为1。允许Java阻断，设置访问控制列表2001，使ASPF策略能够阻塞来自目标服务器10.1.1.1上的Java Applets。

<Quidway> system-view
[Quidway] acl number 2001
[Quidway-acl-basic-2001] rule permit source 10.1.1.1 0
[Quidway] quit
[Quidway] aspf-policy 1
[Quidway-aspf-policy-1] detect java-blocking acl 2001
