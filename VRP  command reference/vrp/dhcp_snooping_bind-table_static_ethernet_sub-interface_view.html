dhcp snooping bind-table static（Ethernet子接口视图）

命令功能

dhcp snooping bind-table static命令用来使能DHCP Snooping静态表支持QinQ报文。

undo dhcp snooping bind-table命令用来取消DHCP Snooping静态表支持QinQ报文

命令格式

dhcp snooping bind-table static ip-address ip-address [ mac-address mac-address | vlan vlan-id [ ce-vlan ce-vid ] ]

undo dhcp snooping bind-table ip-address ip-address [ vlan vid [ ce-vlan ce-vid ] ]

参数说明

ip-address：用户IP地址，点分十进制格式。

mac-address：用户MAC地址，形式是H-H-H，其中H为1至4位的十六进制数。

vlan vlan-id：VLAN ID，十进制整数形式，取值范围是1～4094。它表示pe-vid，即外层Tag的值。

ce-vlan ce-vid：CE的VLAN ID（内层Tag的值），十进制整数形式，取值范围是1～4094。

视图

Ethernet子接口视图、GE子接口视图、Eth-Trunk子接口视图

缺省级别

2：配置级

使用指南

在子接口上使用dhcp snooping bind-table static命令之前，应先在该子接口下使能control-vid命令，并使能dot1q termination vid命令或者qinq termination pe-vid ce-vid命令。

执行dhcp snooping bind-table static命令后，DHCP Snooping绑定表中的动态表项不需配置，使能DHCP Snooping功能即可自动生成。但静态表项需要通过命令配置。

如果在建立绑定表时需要精确到接口信息，必须使能Option82功能。不使能Option82功能，表项中有VLAN信息没有接口信息。

使用实例

# 在子接口GE1/0/0.1上配置IP与MAC静态绑定关系。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] dhcp snooping bind-table static ip-address 10.10.10.2 mac-address 0028-0120-0327 vlan 100 ce-vlan 200
