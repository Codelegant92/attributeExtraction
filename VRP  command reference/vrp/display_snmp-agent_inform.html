display snmp-agent inform

命令功能

display snmp-agent inform命令用于显示全局Inform参数和指定目标主机的Inform参数。

命令格式

display snmp-agent inform [ address udp-domain ip-address [ vpn-instance vpn-instance-name ] params securityname security-string ]

参数说明

address：指定SNMP消息传输的目标主机地址。address指定的IP地址和securityname指定的安全名字符串组成一个二元组，用来标识一个主机。

udp-domain：指定目标主机的传输域基于UDP。

ip-address：目标主机的IP地址。

vpn-instance vpn-instance-name：VPN实例名，字符串形式，不支持空格，长度范围是1～31。vpn-instance可选，如果在VPN中使用，需要vpn-instance指定的VPN实例、IP地址和安全名字符串组成一个三元组来标识一个主机。

params：指定产生SNMP消息的目标主机信息。

securityname security-string：指定在网管侧显示的安全名，字符串形式，不支持空格，长度范围1～32。

使用指南

通过Inform方式发送告警的配置信息，包括：

  • 全局配置的重传次数
  • 超时时间
  • 待确认告警队列配置的最大告警条数
  • 待确认告警队列当前告警条数

或是显示指定目标主机的Inform配置信息和报文统计信息。包括：

  • 该目标主机配置的重传次数
  • 超时时间
  • 已重传次数
  • 待确认的告警数
  • 已发送告警数目
  • dropped（由于待确认队列满而删除的报文数目）
  • failed（由于没有收到确认消息而删除的报文数目）
  • confirmed（收到目的主机响应的报文数目）

使用实例

# 显示配置的全局Inform参数和指定目标主机的Inform参数。

<Quidway> display snmp-agent inform
Global config: resend-times 3, timeout 15s, pending 39
Global status: current notification count 0
Target-host ID: VPN instance/IP-Address/Security name
a/1.1.1.1/public:
    Config: resend-times 3, timeout 15s
    Status: retries 0, pending 0, sent 0, dropped 0, failed 0, confirmed 0

参数

描述

Global config

全局参数信息

Global status

全局报文统计信息

current notification count

当前待确认的告警条数

resend-times

配置的重传次数

timeout

配置的超时时间

Target-host ID: VPN instance/IP-Address/Security name

目标主机的标识，由VPN名/主机IP地址/安全名组成

Config

针对主机的配置信息

Status

针对该主机的告警报文统计信息

retries

向该主机重发告警的总次数

pendings

配置的待确认队列中等待响应的最大告警数

sent

成功发送告警条数

dropped

由于告警队列满而删除告警数

failed

重传次数内没有收到响应告警数

confirmed

收到目标主机响应的报文数目
