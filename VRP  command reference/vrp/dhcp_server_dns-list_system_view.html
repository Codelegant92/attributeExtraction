dhcp server dns-list（系统视图）

命令功能

dhcp server dns-list命令用来指定多个接口所连接的DHCP客户端的DNS服务器IP地址。

undo dhcp server dns-list命令用来删除分配的DNS服务器的IP地址。

缺省情况下，未配置DNS服务器的IP地址。

命令格式

dhcp server dns-list ip-address &<1-8> { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } & <1-10> }

undo dhcp server dns-list { ip-address | all } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 to [ vlan-id2 ] } & <1-10> }

参数说明

ip-address：指定DNS服务器的IP地址，一条命令中最多可以指定8个DNS服务器的IP地址，用空格分隔。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

interface interface-type interface-number：指定接口类型和接口编号。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

all：undo命令中的前一个all表示所有指定的DNS服务器地址，后一个all表示所有接口。

使用指南

实现通过域名方式访问主机，dhcp server dns-list命令用来配置指定范围内的多个接口所连接的DHCP客户端的DNS服务器IP地址。

在目前的实现中，每个DHCP地址池最多可以设置8个DNS服务器地址。

使用实例

# 为GE2/0/0.1到GE2/0/0.5的接口DHCP地址池指定DNS服务器的地址为1.1.1.254。

<Quidway> system-view
[Quidway] dhcp server dns-list 1.1.1.254 interface gigabitethernet 2/0/0.1 to gigabitethernet 2/0/0.5
