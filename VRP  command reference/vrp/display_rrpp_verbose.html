display rrpp verbose

命令功能

display rrpp verbose命令用来显示交换机上RRPP配置的详细信息。

如果不指定参数ring-id，将显示指定域所有环的详细信息，否则显示指定环的详细信息。

命令格式

display rrpp verbose domain domain-id [ ring ring-id ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

domain domain-id：RRPP域的ID。整数形式，取值范围是1～8。

ring ring-id：RRPP环的ID。整数形式，取值范围是1～64。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

include：只输出包含匹配字符串的行。

exclude：只输出不包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

从详细信息可以看到各端口的状态。可以通过详细信息分析网络的链路状态，定位网络故障。

表1 display rrpp verbose命令输出信息描述

项目

描述

Domain Index

RRPP域ID。

Control VLAN

RRPP域的控制VLAN。

Protected VLAN

RRPP域保护的VLAN列表，此处显示的是引用的MSTP实例。

MSTP实例所映射的VLAN可以通过命令display stp region-configuration查看。

Hello Timer

Hello定时器超时时间，单位是秒。

Fail Timer

Fail定时器超时时间，单位是秒。

RRPP Ring

RRPP环ID。

Ring Level
RRPP环的级别：

  • 0级代表主环。

  • 1级代表子环。

Node Mode
为设备配置的RRPP节点模式：

  • Master：主节点。

  • Transit：传输节点。

  • Edge：边缘节点。

  • Assistant Edge：辅助边缘节点。

Ring State
RRPP环的状态。主节点有以下状态：

  • Complete：环路连通。

  • Failed：环路故障。

  • Unknown：未知状态。

主节点激活后显示Complete或Fail状态，未激活时显示Unknown状态。传输节点、边缘节点和辅助边缘节点有以下状态：

  • LinkUp：链路状态为Up。

  • LinkDown：链路状态为Down。

  • PreForwarding：预转发状态。

  • Unknown：未知状态。

Is Enabled

标识RRPP环是否使能。

Is Activated

标识RRPP环是否激活。

Primary Port

RRPP环的主端口。

Secondary Port

RRPP环的副端口。

Common Port

RRPP子环边缘节点或辅助边缘节点的公共端口。

Edge Port

RRPP子环边缘节点或辅助边缘节点的边缘端口。

Port Status
RRPP环的端口状态：

  • UNKNOWN：接口板不在位态时，端口处于未知状态。

  • UP：RRPP端口Up且能够转发报文。

  • DOWN：端口未连接线缆时，处于Down状态。

  • BLOCKED：RRPP端口Up但不能转发数据报文。

Track interface

RRPP环上的监视接口。
