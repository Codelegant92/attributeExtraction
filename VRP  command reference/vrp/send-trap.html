send-trap

命令功能

send-trap命令用来配置Trap消息的发送条件。

undo send-trap命令取消已配置的发送条件。

缺省情况下，不发送Trap消息。

命令格式

send-trap { all | [ owd-ds | owd-sd | probefailure | rtd | testcomplete | testfailure ] * }

undo send-trap { all | [ owd-ds | owd-sd | probefailure | rtd | testcomplete | testfailure ] * }

参数说明

all：下面任何一种情况发生时都发送Trap消息。

owd-ds：从目的到源的单向延时超过阈值，发送Trap消息。

owd-sd：从源到目的的单向延时超过阈值，发送Trap消息。

probefailure：当发送测试包失败时发送Trap消息。

rtd：双向延时超过阈值，发送Trap消息。

testcomplete：当测试完成时发送Trap消息。

testfailure：当测试失败时发送Trap消息。

使用指南

只能对Jitter、ICMPJitter、使能接口板发包的Jitter、使能接口板发包的ICMPJitter类型的测试例配置owd-ds、owd-sd。

不能对正在运行的测试例修改此项配置。

使用实例

# 允许测试例user test在测试包单向延时超过阈值、发送失败、测试完成、测试失败情况下都发送Trap消息。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] send-trap all
