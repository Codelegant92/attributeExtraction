group-testperiod

命令功能

group-testperiod命令用来配置NQA测试组的测试周期。

undo group-testperiod命令用来恢复缺省配置。

缺省情况下，NQA测试组的测试周期是60秒。

命令格式

group-testperiod period

undo group-testperiod

参数说明

period：NQA测试组的测试周期。单位是秒，取值范围是1～604800。

使用指南

group-testperiod命令配置的是该测试组的整体测试时间，即，该测试组的所有成员测试例在这个周期内会全部测试完毕。

假设一个测试组包括m个成员测试例，每个测试例需要运行t秒。如果组测试周期设置为s秒，则测试例将分为n批（n = s/t）完成，每批包含的测试例个数为m/n。

因此，group-testperiod命令配置的组测试周期必须大于成员测试例完成一次测试所需的时间，否则测试组不能正常启动。

举例说明：

假设一个测试组包括8个成员测试例，每个测试例需要运行2秒：

  • 如果组测试周期为4秒，则测试例分2（4/2）批完成，每批4（8/2）个；

  • 如果组测试周期为8秒，则测试例分4（8/2）批完成，每批2（8/4）个；

  • 如果组测试周期为16秒，则测试例分8（16/2）批完成，每批1（8/8）个；

  • 如果组测试周期为1秒，测试组不能运行。

类似地，group-testperiod命令配置的组测试周期也必须小于frequency命令配置的自动测试执行间隔。以保证在一个周期内所有成员测试例都能完成一次测试。

说明：

在指定的组测试周期内，如测试组成员过多，可能导致组测试无法正常启动。请根据测试组成员个数，选择合适的组测试周期。

使用实例

# 设置测试组group1的测试周期为60秒。

<Quidway> system-view
[Quidway] nqa test-instance user group1
[Quidway-nqa-user-group1] switch-to group
[Quidway-nqa-user-group1] group-testperiod 60
