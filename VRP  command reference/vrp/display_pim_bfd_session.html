display pim bfd session

命令功能

display pim bfd session命令用来查看PIM BFD session的信息。

命令格式

display pim [ vpn-instance vpn-instance-name | all-instance ] bfd session statistics [ | count ] [ | { begin | exclude | include } regular-expression ]

display pim [ vpn-instance vpn-instance-name | all-instance ] bfd session [ interface interface-type interface-number | neighbor neighbor-address ] * [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称，字符串形式，长度范围是1～31。

all-instance：指定所有实例。

statistics：显示PIM BFD的统计信息。

interface interface-type interface-number：指定需要查看的接口。interface-type interface-number为接口类型和接口号。

neighbor neighbor-address：指定要查看的PIM邻居的IP地址，点分十进制形式。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display pim bfd session命令：

  • 若不指定vpn-instance或all-instance，则显示公网实例中的PIM BFD session。

  • 若不指定interface或neighbor，则显示指定实例中所有接口上的PIM BFD session。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

表1 display pim bfd session命令输出信息描述

项目

描述

GigabitEthernet6/2/2 (10.1.2.2)

PIM接口名（IP地址）。

Neighbor

PIM邻居IP地址。

ActTx(ms)

实际运行的最小发送间隔。

ActRx(ms)

实际运行的最小接收间隔。

ActMulti

实际运行的本地检测倍数。

Local/Remote

BFD会话的本地/远端标识符。

State
PIM BFD session的状态：

  • UP：BFD session已经建立成功，周期性交互检测报文。

  • Init：已经能够与对端系统通信，本端希望使会话进入Up状态。

  • Admin down：管理性Down（在BFD会话视图下执行Shutdown）。

  • BFD global disable：没有全局使能BFD。

  • BFD session number exceed：BFD session标识符越界。

  • Detect down：检测时间超时没有收到BFD报文。

  • Neighbour down：收到邻居发送的State为Down的报文。

  • Receive admin down：收到邻居发送的State为AdminDown的报文。

Total 1 PIM BFD session in this instance

公网或者VPN实例中PIM BFD会话的总数。

Total 0 PIM BFD session up

公网或VPN实例中状态为Up的PIM BFD会话数。

Total 1 PIM BFD session down

公网或VPN实例中状态为Down的PIM BFD会话数，包括除状态为Up的PIM BFD会话之外的所有PIM BFD会话。
