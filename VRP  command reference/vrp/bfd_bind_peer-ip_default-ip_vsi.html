bfd bind peer-ip default-ip vsi

命令功能

bfd bind peer-ip default-ip vsi命令用来创建基于VSI PW的BFD会话。

undo bfd cfg-name命令用来删除指定的BFD会话，同时取消BFD的绑定配置。

命令格式

bfd cfg-name bind peer-ip default-ip vsi vsi-name peer peer-address [ vc-id vc-id ]

undo bfd cfg-name

参数说明

cfg-name：BFD配置名，不区分大小写，字符串类型，长度范围是1～15。

peer-ip default-ip：BFD会话绑定默认组播IP。

vsi vsi-name：VSI实例的名称，不区分大小写，字符串类型，长度范围是1～31。

peer peer-address：VSI的PW对端IP地址，点分十进制形式。

vc-id vc-id：虚电路的唯一标识，一般用于两端VSI ID不同但要求互通的情况。参数vc-id不能与本端其他VSI配置的VSI ID相同，也不能与本端其他negotiate-vc-id指定的VSI ID相同，即vc-id是没有被用过的虚电路号，十进制整数形式，取值范围为1～4294967295。

使用指南

创建基于VSI PW的BFD会话时，必须全局使能BFD，而且VSI PW必须已经存在。其中bfd bind peer-ip default-ip vsi命令中指定的VSI必须是管理VSI，否则无法创建基于VSI PW的BFD会话。

创建基于VSI PW的BFD会话后，设备之间的BFD报文可以通过VSI进行转发。

使用实例

# 创建基于VSI v100的BFD会话。

<Quidway> system-view
[Quidway] mpls l2vpn 
[Quidway-l2vpn] quit
[Quidway] vsi v100 static
[Quidway-vsi-v100] pwsignal ldp
[Quidway-vsi-v100-ldp] vsi-id 100
[Quidway-vsi-v100-ldp] quit
[Quidway-vsi-v100] admin-vsi
[Quidway-vsi-v100] quit
[Quidway] bfd
[Quidway-bfd] quit
[Quidway] bfd bfd1 bind peer-ip default-ip vsi v100 peer 3.3.3.3
