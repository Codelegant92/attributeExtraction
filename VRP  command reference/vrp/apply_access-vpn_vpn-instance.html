apply access-vpn vpn-instance

命令功能

apply access-vpn vpn-instance命令用来指定VPN实例，根据此VPN实例的路由表转发报文。

undo apply access-vpn vpn-instance命令用来在策略节点中删除一个或多个指定的VPN实例。

命令格式

apply access-vpn vpn-instance vpn-instance-name &<1-6>

undo apply access-vpn vpn-instance [ vpn-instance-name &<1-6> ]

参数说明

vpn-instance-name：VPN实例的名称。区分大小写。最多可以输入六个VPN实例名。

视图

Policy-Based-Route视图

缺省级别

2：配置级

使用指南

缺省情况下，PE上的VPN报文只根据该VPN的路由表进行转发。如果要查找其他VPN的路由表进行转发，可以配置策略路由，指定该报文可以查找哪些VPN实例的路由表进行转发。

一条策略的一个节点中最多可以设置六个VPN实例。当满足匹配条件后，将根据第一个可用的VPN实例路由表进行转发。

如果在使用undo apply access-vpn vpn-instance命令时没有指定VPN实例名，将从转发信息中删除所有的VPN实例。

使用实例

# 在策略节点中设置使用名为vpn1和vpn2的VPN实例进行转发。

<Quidway> system-view
[Quidway] policy-based-route policy1 permit node 10
[Quidway-policy-based-route-policy1-10] apply access-vpn vpn-instance vpn1 vpn2

# 在策略节点中删除vpn1，即，不再使用vpn1的路由信息指导转发。

<Quidway> system-view
[Quidway] policy-based-route policy1 permit node 10
[Quidway-policy-based-route-policy1-10] undo apply access-vpn vpn-instance vpn1
