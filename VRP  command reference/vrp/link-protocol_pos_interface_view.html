link-protocol（POS接口视图）

命令功能

link-protocol命令用来设置接口的链路层协议。

缺省情况下，POS接口使用PPP作为链路层协议。

命令格式

link-protocol { fr [ ietf | nonstandard | mfr interface-number ] | hdlc | ppp }

参数说明

fr：使用帧中继作为接口的链路层协议。

ietf：IETF标准封装，按照RFC1490规定的格式进行封装，为缺省封装格式。

nonstandard：非标准兼容的封装格式。

mfr interface-number：使用MFR作为接口的链路层协议。配置此参数时，指定的MFR接口必须存在。

hdlc：使用HDLC作为接口的链路层协议。

ppp：使用PPP作为接口的链路层协议。

缺省级别

使用指南

接口相关的参数都需要在接口视图下配置，只有配置了正确的链路层协议才能配置该协议相关的参数。

使用实例

# 设置POS接口1/0/0的链路层协议为HDLC。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] link-protocol hdlc
