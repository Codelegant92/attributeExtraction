multicast-channel

命令功能

multicast-channel命令用来使能组播CAC功能，并进入multicast-channel视图。

undo multicast-channel命令用来关闭组播CAC功能，同时删除所有与组播CAC功能相关的配置。

命令格式

multicast-channel [ vpn-instance vpn-instance-name ]

undo multicast-channel [ vpn-instance vpn-instance-name ]

参数说明

vpn-instance vpn-instance-name：VPN实例的名称。字符串形式，区分大小写，长度范围是1～31。

视图

系统视图

缺省级别

2：配置级

使用指南

该命令将作为组播CAC功能的开关，在使用组播CAC功能前必须配置。

在打开组播CAC功能前，系统必须满足以下条件：

  • 使能组播路由。
  • 没有配置PIM-DM的接口存在。
  • 路由器上不存在IGMP only-link接口。
说明：

IGMP only-link接口是指接口使能了IGMP或者配置了静态组播组，但没有使能PIM-SM。

使用实例

# 在公网下进入打开组播CAC功能，进入multicast-channel视图。

<Quidway> system-view
[Quidway] multicast routing-enable
[Quidway] multicast-channel
[Quidway-multicast-channel]
