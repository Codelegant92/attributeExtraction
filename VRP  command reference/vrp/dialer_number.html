dialer number

命令功能

dialer number命令用来设定去往单个对端的拨号串。

undo dialer number命令用来删除已设定的拨号串。

缺省不设定去往对端的拨号串。

命令格式

dialer number dial-number

undo dialer number

参数说明

dial-number：去往对端的拨号串。字符串形式，不支持空格，区分大小写，长度为1～30。

视图

接口视图、Dialer接口视图

缺省级别

2：配置级

使用指南

在配置该命令前，必须先执行命令dialer user设置对端用户名，或执行命令dialer enable-circular使能轮询DCC。

当轮询DCC的Dialer接口作为主叫端，并且只呼叫一个目的地址或缺省地址时，使用此命令。此命令在满足下列条件之一时有效：

  • 接口没有配置dialer route命令。

  • 在dialer route命令没有配置发送报文的下一跳地址。

在共享DCC中，Dialer接口封装PPP时，此时可以选择配置dialer number命令，但必须配置dialer user命令。

当Dialer接口封装帧中继，则主叫端和被叫端都必须配置dialer number命令，无需配置dialer user命令，被叫侧该命令用来判断是否对呼叫进行应答。

说明：

若配置中没有配置dialer-group命令，则DCC不会拨号。

使用共享DCC，主叫端不同拨号接口允许存在重复的dialer number配置；在被叫端，dialer number配置不能重复，否则呼叫将失败。使用轮询DCC，主叫端和被叫端的dialer number配置都允许重复。

使用实例

# 设定接口Dialer1去往对端的拨号串为“11111”。

<Quidway> system-view
[Quidway] interface dialer 1
[Quidway-Dialer1] dialer number 11111
