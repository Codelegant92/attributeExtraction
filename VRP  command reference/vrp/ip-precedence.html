ip-precedence

命令功能

ip-precedence命令用来使用ip-precedence设置PVC承载的IP包的优先级。

undo ip-precedence命令用来删除PVC上承载的IP包的优先级的设置。

命令格式

ip-precedence { pvc-name [ vpi/vci ] | vpi/vci } { min [ max ] | default }

undo ip-precedence { pvc-name [ vpi/vci ] | vpi/vci }

参数说明

pvc-name：PVC名，最长为16个字符，在ATM接口上保持唯一，不区分大小写，并且不能是合法的VPI/VCI值对，如“1/20”就不允许作为PVC名。pvc-name对应的PVC必须已经建立。

vpi/vci：vpi为ATM网络虚路径标识（Virtual Path Identifier，VPI），取值范围是0～255；vci为ATM网络虚通道标识（Virtual Channel Identifier，VCI），取值范围是0～2047。通常，vci取值0到31保留用于特定用途，建议用户不要使用。vpi/vci对应的PVC必须已经建立。

min：该PVC承载的IP包的最小优先级。取值范围是0～7。

max：该PVC承载的IP包的最大优先级。取值范围是0～7。

default：该PVC承载缺省级别的包。

使用指南

只能对该PVC-Group内的PVC进行设置，指定的最小优先级min必须小于或等于指定的最大优先级max。

使用实例

# 设置一条名为“huawei”、VPI/VCI为1/32的PVC承载优先级为0～3的IP包。

<Quidway> system-view
[Quidway] interface atm 1/0/0
[Quidway-ATM1/0/0] pvc huawei 1/32
[Quidway-atm-pvc-Atm1/0/0-1/32-huawei] quit
[Quidway-ATM1/0/0] pvc-group huawei 1/32
[Quidway-atm-pvc-group-Atm1/0/0-1/32-huawei] ip-precedence huawei 1/32 0 3
