lsp-trigger

命令功能

lsp-trigger命令用来设置触发建立LSP的策略，通过32位地址的IP路由或IP地址前缀列表定义的IGP路由可以触发LDP建立LSP。

undo lsp-trigger命令用来恢复缺省设置。

缺省情况下，根据32位地址的IP路由触发LDP建立LSP。

命令格式

lsp-trigger { all | host | ip-prefix ip-prefix-name | none }

undo lsp-trigger

参数说明

all：所有静态路由和IGP路由项触发建立LSP。

host：32位地址的IP路由触发建立LSP。

ip-prefix ip-prefix-name：根据IP地址前缀列表触发建立LSP。字符串形式，不支持空格，长度范围是1～169。

none：不触发建立LSP。

缺省级别

2：配置级

使用指南

本命令只对公网的Ingress LSP和Egress LSP以及私网的Ingress LSP有效，如果要配置Transit LSP的过滤策略可以执行propagate mapping命令。

lsp-trigger和lsp-trigger bgp-label-route命令都用于配置LDP LSP的触发建立策略。前者适用于静态路由和IGP路由；后者仅适用于带标签的公网BGP路由。

说明：

LDP GR期间修改LSP触发策略的操作无效。

使用实例

# 设置根据所有静态路由和IGP路由项来触发建立LSP。

<Quidway> system-view
[Quidway] mpls
[Quidway-mpls] lsp-trigger all
