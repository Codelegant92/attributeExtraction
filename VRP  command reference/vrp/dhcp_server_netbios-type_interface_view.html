dhcp server netbios-type（接口视图）

命令功能

netbios-type命令用来配置当前接口的DHCP客户端NetBIOS节点类型。

undo netbios-type命令用来恢复缺省设置。

缺省情况下，不指定客户端的节点类型。

命令格式

dhcp server netbios-type { b-node | h-node | m-node | p-node }

undo dhcp server netbios-type { b-node | h-node | m-node | p-node }

参数说明

b-node：b类节点，“b”代表广播（broadcast），即，此类节点采用广播的方式获取映射关系。

p-node：p类节点，“p”代表端到端（peer-to-peer），即，此类节点采用与NetBIOS服务器通信的方式获取映射关系。

m-node：m类节点，“m”代表混合（mixed），是具有部分广播特性的p类节点。

h-node：h类节点，“h”代表混合（hybrid），是具备“端对端”通信机制的b类节点。

使用指南

DHCP客户端在广域网上使用NetBIOS协议时，需要在主机名和IP地址之间建立映射关系，为当前接口的DHCP客户端配置NetBIOS节点类型。

使用实例

# 配置GE 1/0/0的DHCP地址池为其客户端分配的NetBIOS节点类型为p-node。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] dhcp server netbios-type p-node
