peer static-upe

命令功能

peer static-upe命令用于在H-VPLS下配置VSI的静态UPE对等体。

undo peer static-upe命令用于在H-VPLS下删除VSI的静态UPE对等体。

缺省情况下，在H-VPLS下VSI实体没有静态UPE对等体。

命令格式

peer peer-address [ negotiation-vc-id vc-id ] [ tnl-policy policy-name ] static-upe trans trans-label recv recv-label

undo peer peer-address [ negotiation-vc-id vc-id ]

参数说明

peer-address：静态UPE对等体的IPv4地址，通常指定为对端的Loopback地址。

negotiation-vc-id vc-id：虚电路的唯一标识，一般用于两端VSI ID不同但要求互通的情况。参数vc-id不能与本端其他VSI配置的VSI ID相同，也不能与本端其他negotiate-vc-id指定的VSI ID相同，即vc-id是没有被用过的虚电路号，十进制整数形式，取值范围是1～4294967295。

tnl-policy policy-name：用于该静态UPE对等体的隧道策略名称。字符串形式，不支持空格，区分大小写，长度范围是1～19。

static-upe：用于标识该静态UPE对等体是否是用户端的PE，该参数适用于HVPLS（Hierarchical Virtual Private LAN Service）。

trans trans-label：手工配置的本端设备发往对等体的外层标签值。是静态二层VC的出标签，十进制整数形式，取值范围是16～1023。

recv recv-label：手工配置的对等体发往本端设备的外层标签值。是静态二层VC的入标签，十进制整数形式，取值范围是16～1023。

使用指南

配置VSI的对等体之前，需要先配置VSI-ID。

在peer命令中，如果选择关键字upe，则表明对等体属于用户端，而不在水平分割的范围内。

如果配置VSI的对等体时使用了参数negotiation-vc-id vc-id，并且指定的vc-id与使用vsi-id命令配置的vsi-id不同，那么使用命令undo peer删除该VSI的对等体时，仍需携带参数negotiation-vc-id vc-id，否则将无法正常删除该VSI的对等体。

使用实例

# 配置当前VSI实例的静态UPE对等体。发送与接收的外层标签值都为100。

<Quidway> system-view
[Quidway] vsi company1
[Quidway-vsi-company1] pwsignal ldp
[Quidway-vsi-company1-ldp] peer 3.3.3.3 static-upe trans 100 recv 100
