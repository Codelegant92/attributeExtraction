diffserv-mode

命令功能

diffserv-mode命令用来配置MPLS的差分服务模式，实现端到端的QoS保证。

缺省情况下，MPLS的差分服务模式为Uniform。

命令格式

diffserv-mode { pipe service-class color | short-pipe service-class color [ domain ds-name] | uniform }

undo diffserv-mode

参数说明

pipe：指定MPLS的差分服务模式为Pipe。

short-pipe：指定MPLS的差分服务模式为Short-pipe。

uniform：指定MPLS的差分服务模式为Uniform。

service-class：指定报文从私网进入公网时的服务类型。取值为cs7、cs6、ef、af4、af3、af2、af1、be。

color：指定报文从私网进入公网时的颜色，用来标记报文的丢弃优先级。取值为green、yellow、red。

domain ds-name：Short-pipe模式时，指定出节点上采用指定PHB的DS域名称。

使用指南

在接口视图和Tunnel接口视图下，此命令的格式为：

diffserv-mode { pipe service-class color | uniform }

在接口视图下，设置绑定VLL接口的diffserv模式。

在Tunnel接口视图下，设置Tunnel接口的diffserv模式。

当配置报文从私网进入公网的服务类型时：
  • 需要传送信令协议，服务类型设置为：cs7和cs6。
  • 需要保证低时延的业务，服务类型设置为：ef。
  • 需要保证带宽的关键数据业务，服务类型设置为：af4、af3、af2或af1。
  • 不需要严格QoS保证的尽力发送业务，服务类型设置为：be。

上述服务类型的报文按照发送优先级从高到低的顺序排列。

报文从私网进入公网的颜色，按照丢弃优先级从低到高的顺序排列为green、yellow和red。

使用实例

# 配置MPLS diffserv模式为Pipe，服务类型为af1，颜色为green。

<Quidway> system-view
[Quidway] ip vpn-instance vpna
[Quidway-vpn-instance-vpna] diffserv-mode pipe af1 green
