transform

命令功能

transform命令用来设置提议采用的安全协议。

undo transform命令用来恢复缺省的安全协议。

缺省情况使用esp，即RFC2406规定的ESP协议。

命令格式

transform { ah | ah-esp | esp | ah-ipcomp | esp-ipcomp | ah-esp-ipcomp }

undo transform

参数说明

ah：设置采用RFC2402规定的AH协议。

ah-esp：设置先用ESP协议对报文进行保护，再用AH协议进行保护。

esp：设置采用RFC2406规定的ESP协议。

ipcomp：采用RFC2393规定的压缩协议，不支持单独使用的ipcomp。根据RFC建议，在组合应用时，对报文先进行压缩，再进行esp协议和/或ah协议处理（该参数只使用于NE16E、NE08E、NE05产品并由加密卡硬件提供）。

使用指南

如果采用ESP协议，则缺省的加密算法为DES，认证算法为MD5。

如果采用AH协议，则缺省的认证算法为MD5。

如果指定参数ah-esp，则AH缺省的认证算法为MD5，ESP协议缺省使用的加密算法为DES，不使用认证算法。

如果采用同时使用ipcomp协议，则压缩算法为LZS。

AH协议提供数据源认证、数据完整性校验和防报文重放功能。

ESP协议提供数据源认证、数据完整性校验、防报文重放和数据加密功能。

IPCOMP协议提供数据压缩算法。

在安全隧道的两端，IPsec提议所使用的安全协议需要匹配。

使用实例

# 设置一个采用AH协议的提议。

<Quidway> system-view
[Quidway] ipsec proposal prop1
[Quidway-ipsec-proposal-prop1] transform ah
