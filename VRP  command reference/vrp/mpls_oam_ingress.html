mpls oam ingress

命令功能

mpls oam ingress命令用来在LSP的入节点配置MPLS OAM参数。

undo mpls oam ingress命令用来删除指定的MPLS OAM参数。

缺省情况下，检测报文的类型为CV，发送频率固定为1秒；反向通道采用共享方式。

命令格式

mpls oam ingress tunnel tunnel-number [ type { cv | ffd frequency ffd-fre } ] [ backward-lsp { lsp-name lsp-name | lsr-id rev-ingress-lsr-id tunnel-id rev-tunnel-id | share } ] [ packet-priority priority ]

undo mpls oam ingress { all | tunnel interface-number }

参数说明

tunnel tunnel-number:：隧道的接口编号，代表启用MPLS OAM检测功能的隧道。

type：设置OAM检测报文的类型，可选输入为CV、FFD。

ffd-fre：设置FFD报文发送频率，仅针对FFD报文。可选输入为10、20、50、100、200和500，单位为毫秒。

backward-lsp：设置反向通道LSP的共享类型，分共享和独占两种。独占方式提供两种LSP的标识“lsp-name”和“lsr-id tunnel-id”。

lsp-name lsp-name：静态LSP名称。字符串形式，不支持空格，长度范围是1～19。

lsr-id rev-ingress-lsr-id：反向通道入节点的LSR ID。

tunnel-id rev-tunnel-id：反向通道的隧道ID。整数形式，取值范围是1～65534。

share：设置反向LSP为共享。

packet-priority：设置CV和FFD报文发送的优先级。

priority：CV和FFD报文发送的优先级。整数形式，取值范围是0～7，值越大，表示优先级越高。缺省值是0。

使用指南

配置MPLS OAM参数前，必须先执行mpls oam命令全局使能MPLS OAM功能。

如果反向通道是静态LSP或静态CR-LSP，则反向通道必须采用独占方式。

如果执行static-lsp egress或static-cr-lsp egress命令创建反向LSP时，指定了参数lsrid ingress-lsr-id和tunnel-id tunnel-id，则在此步骤中配置反向通道可以选用参数lsr-id rev-ingress-lsr-id和tunnel-id rev-tunnel-id，否则必须用参数lsp-name lsp-name。

配置OAM参数后，需要执行mpls oam ingress enable命令手工启动OAM。

使用实例

# 在Tunnel1/0/0接口上配置MPLS OAM参数：检测类型为FFD，速率100毫秒，反向LSP为共享方式。

<Quidway> system-view
[Quidway] mpls oam ingress tunnel 1/0/0 type ffd frequency 100
