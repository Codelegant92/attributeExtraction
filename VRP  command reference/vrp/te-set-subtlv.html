te-set-subtlv

命令功能

te-set-subtlv命令用来设置携带DS-TE参数的各种子TLV的TLV类型。

undo te-set-subtlv命令用来恢复缺省设置。

缺省情况下，带宽约束子TLV为252；本地过预订倍数的子TLV为253；子池未预订带宽的子TLV为251。

命令格式

te-set-subtlv { bw-constraint value | lo-multiplier value | unreserve-bw-sub-pool value }*

undo te-set-subtlv { bw-constraint | lo-multiplier | unreserve-bw-sub-pool }*

参数说明

bw-constraint value：指定带宽约束的子TLV值。value为整数形式，取值范围是19～254。

lo-multiplier value：指定本地过预订倍数LOM（Local Overbooking Multipliers）的子TLV值。value为整数形式，取值范围是19～254。

unreserve-bw-sub-pool value：指定子池未预订带宽的子TLV值。value为整数形式，取值范围是19～254。

使用指南

由于DS-TE参数的各种子TLV的类型尚未形成标准，当使用不同厂商提供的设备互联时，用户需要手工配置这些子TLV值。配置后，TEDB（Traffic Engineering DataBase）将重新生成，进而导致TE隧道重建。

使用实例

# 配置IS-IS进程1的子TLV类型值。

<Quidway> system-view
[Quidway] isis 1
[Quidway-isis-1] te-set-subtlv bw-constraint 200 lo-multiplier 201 unreserve-bw-sub-pool 202
