igmp prompt-leave

命令功能

igmp prompt-leave命令用来在接口上配置立刻离开组。当接口接收到针对某组播组的离开消息时，不发送最后组成员查询消息，立即删除该组记录。

undo igmp prompt-leave命令用来恢复缺省配置。

缺省情况下，IGMP在接收到主机发送的离开消息后发送最后组成员查询消息。

命令格式

igmp prompt-leave [ group-policy basic-acl-number ]

undo igmp prompt-leave

参数说明

basic-acl-number：表示基本访问控制列表，该列表定义了一个组播组范围。整数形式，取值范围是2000～2999。

使用指南

本命令只适用于IGMPv2。

如果未使用group-policy basic-acl-number 参数，则所有的组都将执行立即离开。

使用group-policy basic-acl-number参数之前，需要配置待引用的ACL，接口将按照该ACL过滤收到的Leave报文。

  • 如果主机要离开的组在ACL允许的范围之中，则路由器不发送最后组成员查询消息，立即删除该组记录。

  • 如果主机要离开的组在ACL允许的范围之外，则路由器发送最后组成员查询消息。

igmp prompt-leave命令与acl命令配合使用。在ACL视图下，通过rule命令的source参数定义主机将要离开的组播组地址。

使用实例

# 创建编号为2005的ACL，配置路由器与主机相连的接口GE1/0/0在收到主机发送的针对组播组225.1.0.0/16的IGMP离开消息后，立即删除该组记录。

<Quidway> system-view
[Quidway] acl number 2005
[Quidway-acl-basic-2005] rule permit source 225.1.0.0 0.0.255.255
[Quidway-acl-basic-2005] quit
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp prompt-leave group-policy 2005
