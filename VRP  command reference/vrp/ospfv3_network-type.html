ospfv3 network-type

命令功能

ospfv3 network-type命令用来设置OSPFv3接口的网络类型。

undo ospfv3 network-type命令用来恢复OSPFv3接口缺省的网络类型。

缺省情况下，接口的网络类型根据物理接口而定。以太网接口的网络类型为broadcast，串口和POS口（封装PPP协议或HDLC协议时）网络类型为p2p，ATM和Frame-relay接口的网络类型为nbma。

命令格式

ospfv3 network-type { broadcast | nbma | p2mp [ non-broadcast ] | p2p } [ instance instance-id ]

undo ospfv3 network-type [ instance instance-id ]

参数说明

broadcast：将接口的网络类型改为广播。

nbma：将接口的网络类型改为NBMA。

p2mp：将接口的网络类型改为点到多点。

non-broadcast：将接口的网络类型改为非广播的点到多点。

p2p：将接口的网络类型改为点到点。

instance instance-id：指定接口所属的实例ID。整数形式，取值范围是0～255。

使用实例

# 将接口POS 1/0/0设置为NBMA类型。
