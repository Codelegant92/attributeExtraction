qinq protocol

命令功能

qinq protocol命令用来配置接口的外层Tag的以太封装类型值。取值范围是0x0600～0xFFFF，缺省值是0x8100。

undo qinq protocol用来恢复接口的外层Tag的缺省以太封装类型值（0x8100）。

缺省情况下，接口的外层Tag的以太封装类型值为0x8100。

命令格式

qinq protocol ethertype-value

undo qinq protocol

参数说明

ethertype-value：接口的外层Tag的以太封装类型值。十六进制整数形式，取值范围是0x600～0xFFFF。

视图

Ethernet接口视图、GE接口视图、Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

在QinQ封装中，各个设备厂商的内层TPID的EtherType的值为0x8100，但是对于外层TPID的EtherType，各个厂商所使用的值不相同。

在IEEE 802.1ad中规定外层TPID的 EType字段的定义为0x88a8。

为了使不同厂商的设备实现互通，需要实现外层TPID的 EtherType的兼容，通过qinq protocol ethertype-value命令可以配置外层TPID的 EtherType。

qinq protocol命令在二层接口和三层接口上都能使用。

使用实例

# 设置接口的外层Tag的以太封装类型值为0x9100。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] qinq protocol 9100
