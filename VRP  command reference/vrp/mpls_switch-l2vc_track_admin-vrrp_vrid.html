mpls switch-l2vc track admin-vrrp vrid

命令功能

mpls switch-l2vc track admin-vrrp vrid命令把交换VLL的PW与管理VRRP进行绑定。

undo mpls switch-l2vc track admin-vrrp vrid命令用来删除交换VLL的PW与管理VRRP的绑定关系。

缺省情况下，没有配置交换VLL的PW与任何管理VRRP的绑定。

命令格式

mpls switch-l2vc ip-address vc-id encapsulation { ethernet | ip-interworking | ip-layer2 | vlan } track admin-vrrp interface interface-type interface-number vrid virtual-router-id

undo mpls switch-l2vc ip-address vc-id encapsulation { ethernet | ip-interworking | ip-layer2 | vlan } track admin-vrrp

参数说明

ip-address：PW对端设备的LSR-ID。

vc-id：二层虚电路的ID。十进制整数形式，取值范围是1～4294967295。

encapsulation：PW的封装类型。

ethernet：PW的封装为以太类型。

vlan：PW的封装为VLAN类型。

ip-interworking：如果进行异种介质互通的设备都是华为设备，选择参数 ip-interworking来显示交换PW的信息。

ip-layer2：如果与其他设备制造商的设备进行异种介质互通，选择参数ip-layer2来显示交换PW的信息。

virtual-router-id：管理VRRP备份组号，十进制整数形式，取值范围是1～255。

interface-type interface-number：绑定了管理VRRP的接口类型和编号。

使用指南

使用mpls switch-l2vc track admin-vrrp vrid命令前，需要先使用mpls switch-l2vc命令，创建交换VLL连接。

一个交换VLL只能绑定到一个管理VRRP中。

交换VLL的PW与管理VRRP接口不能配置在不同的接口板，而必须配置在同一物理接口的不同子接口，否则管理VRRP的状态不能反映PW的真实状态。

使用实例

# 配置VRRP备份组100，虚拟IP为 192.168.0.100。并配置该VRRP备份组为管理VRRP。指定的VC ID为100，在NPE上配置VRRP和业务VLL中的PW绑定。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ip address 192.168.0.1 255.255.255.0
[Quidway-GigabitEthernet1/0/0] vrrp vrid 100 virtual-ip 192.168.0.100
[Quidway-GigabitEthernet1/0/0] admin-vrrp vrid 100
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] mpls l2vpn
[Quidway-l2vpn] quit
[Quidway] mpls switch-l2vc 1.1.1.1 200 between 2.2.2.2 100 encapsulation ppp 
[Quidway] mpls switch-l2vc 1.1.1.1 200 encapsulation ethernet track admin-vrrp interface gigabitethernet 1/0/0 vrid 100
