display igmp routing-table

命令功能

display igmp routing-table命令用来查看IGMP路由表信息。

命令格式

display igmp [ vpn-instance vpn-instance-name | all-instance ] routing-table [ group-address [ mask { group-mask | group-mask-length } ] | source-address [ mask { source-mask | source-mask-length } ] ] * [ static ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称，字符串形式，长度范围是1～31。

all-instance：指定所有实例。

group-address：表示组播组地址，用来指定一个组播组，显示该组对应的路由表信息。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

mask：指定组播组或源的地址掩码。

group-mask：表示组播组地址掩码。点分十进制形式。

group-mask-length：表示组播组地址掩码长度。整数形式，取值范围是4～32。

source-address：表示组播源地址，用来指定一个组播源，显示该源对应的路由表信息。点分十进制形式。

source-mask：表示组播源地址掩码。点分十进制形式。

source-mask-length：表示组播源地址掩码长度。整数形式，取值范围是0～32。

static：表示静态加入的组播组信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display igmp routing-table命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 若指定了group-address参数则仅显示和该组播组相关的IGMP路由表项。

  • 若指定了source-address参数则仅显示和该组播源相关的IGMP路由表项。

  • 若指定了static，则仅显示通过静态加入组播组生成的IGMP路由表项。

  • IGMP路由表项包括(S,G)或(*,G)两种，包含了下游接口列表（Downstream List），即下游接口连接组播数据的接收者。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
