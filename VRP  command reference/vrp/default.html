default

命令功能

default命令用来配置引入外部路由时的缺省参数，包括OSPF引入外部路由的开销、类型（Type1或Type2）、标记、路由数量。

undo default命令用来恢复各项的缺省值。

缺省情况下，OSPF引入外部路由的缺省度量值为1，一次可引入外部路由数量的上限为2147483647，引入的外部路由类型为Type2，设置缺省标记值为1。

命令格式

default { cost { cost | inherit-metric } | limit limit | tag tag | type type } *

undo default { cost | limit | tag | type } *

参数说明

cost cost：指定OSPF引入的外部路由的缺省度量值。整数形式，取值范围是0～16777214。

inherit-metric：引入路由的开销值为路由自带的cost值。如果没有指定开销值，则使用default命令设置的缺省开销值。

limit limit：指定单位时间内引入外部路由上限的缺省值。整数形式，取值范围是1～2147483647。

tag tag：指定外部路由的标记。整数形式，取值范围是0～4294967295。

type type：指定外部路由类型，取值为1或2。

使用指南

路由标记可以用来标识协议相关的信息，如OSPF接收BGP时用来区分自治系统的编号。

可以通过以下三条命令设置引入路由的开销值，其优先级依次递减：

  • 通过apply cost命令设置路由的开销值。
  • 通过import-route（OSPF）命令设置引入路由的开销值。
  • 通过default命令设置引入路由的缺省开销值。

使用实例

# 设置路由开销、类型、标记和数量上限的缺省值。

<Quidway> system-view
[Quidway] ospf 100
[Quidway-ospf-100] default cost 10 limit 20000 tag 100 type 2
