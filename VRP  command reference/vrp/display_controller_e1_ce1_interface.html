display controller e1（CE1接口）

命令功能

display controller e1命令用来查看CE1接口的状态。

命令格式

display controller e1 [ controller-number ]

参数说明

controller-number：接口号。如果不指定接口编号，将显示所有CE1接口的状态。

视图

所有视图

缺省级别

1：监控级

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和报文统计信息等。

使用实例

# 查看CE1接口1/0/0的状态信息。

<Quidway> display controller e1 1/0/0
E1 1/0/0 current state : DOWN
Work Mode : Channelized E1 - 4 port 75 OHM unbalanced
Framing : NO-CRC4   Line Code : HDB3
Clock : Slave   Loopback : None
CPLD version : 0
Alarm : Loss of Signal
Error : 0 FERR, 0 CERR, 0 FEBE, 0 LCV
表1 display controller e1命令输出信息描述

项目

描述

E1 1/0/0 current state :
CE1接口当前的物理状态：

  • UP：物理状态正常启动。

  • DOWN：物理连接不正常。

  • Administratively down：如果管理员在该接口下执行了shutdown命令，将显示状态为Administratively down。

Work Mode :

线缆工作模式和线缆类型。

Frameing

CE1接口的帧格式：请参见frame-format（CE1接口视图）。

Line Code

CE1接口的线路编解码格式：请参见code（CE1接口视图）。

clock :

时钟模式。可以通过clock（CE1接口视图）设置为master或者slave。

loopback : None

没有使能CE1接口环回功能。只有在进行某些特殊功能测试时，才将CE1接口设为环回。

CPLD version:

复杂可编程逻辑器件（CPLD）的版本信息。

Alarm:Error:
告警类型和错误类型。可能出现的错误类型有以下几类。

  • LOS：Loss of Signal，帧同步丢失。

  • LOF：Loss of Frame，帧丢失。

  • CERR：CRC Error，循环冗余校验错。

  • LCV：Line Code Violation，线路码错误。

  • FEBE：Far End Block Error，远端块错。这种错误只有在E1通道采用CRC4的帧格式时才可能出现。
