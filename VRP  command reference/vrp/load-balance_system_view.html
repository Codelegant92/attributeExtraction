load-balance（系统视图）

命令功能

load-balance命令用来配置IP报文转发的负载分担方式。

undo load-balance命令用来IP报文转发的负载分担方式恢复为缺省设置。

缺省情况下，设备对所有IP报文进行逐流负载分担。

命令格式

load-balance { flow | packet } [ all | slot slot-id ]

在集中式环境上的命令格式为：

load-balance { flow | packet }

参数说明

flow：配置IP报文转发按流进行负载分担。

packet：配置IP报文转发按包进行负载分担。

all：将配置应用于所有的接口板。

slot slot-id：指定接口板的槽号。

使用指南

如果按流进行负载分担，转发报文时会根据报文的五元组（五元组即：协议类型、源IP地址及掩码、目的IP地址及掩码、源端口范围、目的端口范围），采用HASH算法计算出一个值，然后根据这个值选择一条链路进行转发。

缺省情况下，IP报文转发按流进行负载分担。

说明：

load-balance命令对MPLS转发同样有效，此命令在MPLS中的使用请参见《VRP 命令参考》“MPLS命令”。

使用实例

# 配置路由设备IP报文转发采用按包负载分担方式。

<Quidway> system-view
[Quidway] load-balance packet slot 1
