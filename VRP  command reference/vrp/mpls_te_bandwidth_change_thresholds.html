mpls te bandwidth change thresholds

命令功能

mpls te bandwidth change thresholds命令用来设置MPLS TE隧道占用的物理链路带宽变化比率的泛洪阈值。

undo mpls te bandwidth change thresholds命令用来恢复缺省设置。

缺省情况下，当一条链路上为MPLS TE隧道保留的带宽与TEDB（Traffic Engineering DataBase）中的链路剩余带宽的比值等于或大于10%，或MPLS TE隧道释放的带宽与TEDB中剩余带宽的比值等于或大于10%时，IGP将对该链路信息进行泛洪，CSPF更新TEDB。

命令格式

mpls te bandwidth change thresholds { down | up } percent

undo mpls te bandwidth change thresholds { down | up }

参数说明

down：MPLS TE隧道占用带宽的泛洪阈值。当MPLS TE隧道占用的带宽与TEDB中的链路剩余带宽的比值大于或等于此阈值时，IGP将进行泛洪，CSPF更新TEDB。

up：MPLS TE隧道释放带宽的泛洪阈值。当MPLS TE隧道释放的带宽与TEDB中的链路剩余带宽的比值大于或等于此阈值时，IGP将进行泛洪，CSPF更新TEDB。

percent：带宽阈值百分比。整数形式，取值范围是0～100，缺省值为10。

使用指南

链路带宽变化很小时对带宽进行泛洪会浪费网络资源。例如某条链路带宽为100M，在此链路上建立100条1M的TE隧道时，则需要进行100次泛洪。如果设置泛洪阈值为10％，则建立第1～9条时不进行泛洪；当建立第10条时才对第1～10条所占用的10M带宽进行泛洪。当建立第11～18条隧道时不进行泛洪，当建立第19条时才泛洪。依此类推。

使用实例

# 对于接口GE1/0/0，当接口上建立的MPLS TE隧道保留的带宽超过链路总带宽的25%，或接口上MPLS TE隧道释放的带宽超过链路总带宽的10%时，对该链路的信息进行泛洪。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] mpls te bandwidth change thresholds up 25
[Quidway-GigabitEthernet1/0/0] mpls te bandwidth change thresholds down 10
