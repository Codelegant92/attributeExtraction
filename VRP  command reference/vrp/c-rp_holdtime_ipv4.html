c-rp holdtime（IPv4）

命令功能

c-rp holdtime命令用来配置BSR等待接收C-RP发送Advertisement消息的超时时间。

undo c-rp holdtime命令用来恢复该配置参数的缺省值。

缺省情况下，BSR等待接收C-RP发送Advertisement消息的超时时间是150秒。

命令格式

c-rp holdtime interval

undo c-rp holdtime

参数说明

interval：表示BSR等待接收该C-RP发送Advertisement消息的超时时间。整数形式，取值范围是1～65535，单位是秒。缺省值是150秒。

缺省级别

2：配置级

使用指南

在C-RP上配置该超时参数后，C-RP会在Advertisement消息中封装该参数并传输给BSR。BSR从消息中获得该超时参数并启动定时器，如果超过超时时间BSR仍旧没有收到C-RP发送的宣告消息，则认为目前网络中的C-RP失效或不可达。

该命令和c-rp interface-type interface-number holdtime hold-interval命令功能完全相同，差别只是作用范围不同。c-rp holdtime interval命令具有全局性，对整个路由器有效；c-rp interface-type interface-number holdtime hold-interval命令只对指定接口有效。

系统优先采用c-rp interface-type interface-number holdtime hold-interval命令的配置。当未执行c-rp interface-type interface-number holdtime hold-interval命令时，才采用c-rp holdtime interval命令的配置。

使用实例

# 在公网实例下，在C-RP上配置发送宣告消息的超时时间为100秒。

<Quidway> system-view
[Quidway] pim
[Quidway-pim] c-rp holdtime 100
