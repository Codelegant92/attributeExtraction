fr switch

命令功能

fr switch命令用来对交换PVC配置备份功能。

undo fr switch命令用来删除指定的交换PVC。

缺省情况下，没有用于帧中继交换的PVC。

命令格式

fr switch name [ interface interface-type in-interface-number dlci in-dlci interface interface-type out-interface-number dlci out-dlci [ [ standby pvc-name ] group group-number ] ]

undo fr switch name

参数说明

name：用于帧中继交换的PVC名称。字符串形式，长度是1～31。

interface interface-type in-interface-number dlci in-dlci：PVC入侧的接口的类型、接口编号和DLCI值。dlci-number取值范围是16～1007。当使用帧中继PVC或Tunnel接口备份帧中继链路时，入接口必须和主用链路的入接口为同一接口，否则无法配置成功。

interface interface-type out-interface-number dlci out-dlci：PVC出侧的主用或备用PVC的接口类型、接口编号和DLCI值。dlci-number取值范围是16～1007。接口类型可以是帧中继接口或Tunnel接口。

standby pvc-name：将配置的PVC作为名是“pvc-name”的交换PVC的备用链路，pvc-name为字符串形式，长度是1～31。

group-number：此交换PVC所在备份组的组号。取值范围是1～256。备用PVC必须和主用PVC在同一个备份组中。

使用指南

配置时，必须先配置接口类型为DCE或NNI，且接口上必须配置好DLCI。

配置时，如果不使用standby参数，但使用group指定了所属的备份组，则表示此PVC是主用链路；如果使用standby参数，则表示此PVC作为备用链路。主备链路是一一对应的，如果对同一条备用链路多次配置主用链路，新的配置将覆盖以前的配置。

对交换PVC配置备份功能时，如果指定的备份组不存在，则系统将自动创建一个使用指定组号的备份组，并将此交换PVC加入该备份组。

删除交换PVC时，如果此交换PVC是所属备份组的最后一条链路，系统将自动删除此备份组。

使用实例

# 创建一条名为pvc2的交换PVC，指定它作为名为pvc1的交换PVC的备份链路，它们都属于PVC备份组1。

<Quidway> system-view
[Quidway] fr switch pvc2 interface serlai 1/0/0 dlci 100 interface serial 2/0/0 dlci 200 standby pvc1 group 1
