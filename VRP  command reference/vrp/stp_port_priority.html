stp port priority

命令功能

stp port priority命令用来配置当前端口在指定生成树实例上的优先级。

undo stp port priority命令用来恢复当前端口在指定生成树实例上的优先级的缺省值。

命令格式

stp [ instance instance-id ] port priority priority

undo stp [ instance instance-id ] port priority

参数说明

instance instance-id：指定生成树实例。instance-id表示生成树实例的编号，取值范围是0～48，取值为0表示的是CIST。

priority priority：端口的优先级。取值范围是0～240，步长为16，如0、16、32等。缺省值是128。

视图

Ethernet接口视图、GE接口视图、Eth-Trunk接口视图、VE接口视图

缺省级别

2：配置级

使用指南

在使用stp port priority命令或undo stp port priority命令时，如果参数instance-id取值为0或者不输入instance instance-id参数，表示配置的是当前端口在CIST上的优先级。

端口优先级可以影响端口在指定生成树实例上的角色。用户可以在不同MSTI上对同一端口配置不同的优先级，从而使不同VLAN的流量沿不同的物理链路转发，完成按VLAN负载分担的功能。端口优先级的改变时，MSTP会重新计算端口的角色并进行状态迁移。

使用实例

# 配置以太网接口GE 1/0/0在生成树实例2上的优先级为16。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] portswitch
[Quidway-GigabitEthernet1/0/0] stp instance 2 port priority 16
