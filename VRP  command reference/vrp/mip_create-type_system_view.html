mip create-type（系统视图）

命令功能

mip create-type命令用来配置全局或指定接口的MIP生成规则。

undo mip create-type命令用来将指定接口的MIP生成规则恢复为缺省值。

缺省情况下，全局的MIP生成规则为none，接口按全局MIP生成规则生成MIP。

命令格式

mip create-type { default | explicit | none } [ interface interface-type interface-number ]

undo mip create-type interface interface-type interface-number

参数说明

default：指定MIP的生成规则为default类型，即如果指定接口不存在更高级别的MEP，并且不存在更低级别的MIP，则可以在该接口上创建MIP。在本类型下，接口上没有配置MEP也可创建MIP。

explicit：指定MIP的生成规则为explicit类型，即如果在指定接口下存在更低级的MEP但不存在更高级别的MEP，而且不存在更低级别的MIP，则可以创建MIP。在本类型下，接口上只有已配置了更低级别的MEP才可能创建MIP。

none：指定MIP的生成规则为none类型，即不自动创建MIP。

interface interface-type interface-number：表示配置指定接口的MIP生成规则。interface-type表示接口类型，取值包括：ethernet、eth-trunk和gigabitethernet。interface-number表示接口编号。不指定本参数时，表示配置全局的MIP生成规则。

使用指南

说明：
本命令只适用于draft7草案版本中。

配置了MIP的生成规则为default类型或explicit类型后，设备会根据规则自动创建MIP。

在使用cfm enable（系统视图）命令全局使能CFM功能后才能成功使用本命令。

一般情况下，在对MA内的非MEP节点进行802.1ag MAC Ping/Trace操作时，才需要配置MIP节点。

使用实例

# 配置以太网接口GigabitEthernet1/0/1的MIP生成规则为default。

<Quidway> system-view
[Quidway] cfm enable
[Quidway] mip create-type default interface gigabitethernet1/0/1
