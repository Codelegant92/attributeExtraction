display fr interface

命令功能

display fr interface命令用来查看使用帧中继协议的接口状态，包括接口是作为DTE还是DCE，以及物理状态和链路层协议状态，对于子接口，将显式子接口类型和链路层协议状态。

命令格式

display fr interface [ interface-type interface-number ]

参数说明

interface-type：接口类型。不指定口类型则显示所有帧中继接口的信息。

interface-number：接口编号。可以指定主接口，接口编号规则是槽号/卡号/接口号。也可以指定子接口，接口编号规则是槽号/卡号/接口号.子接口号。不指定接口编号则显示指定接口类型的全部接口信息。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行接口的故障诊断等。

使用实例

# 查看所有帧中继接口的状态。

<Quidway> display fr interface
MFR12/0/0, DTE, physical down, protocol down
MFR12/0/0.1, multi-point, protocol down
Serial12/0/0:0, DTE, physical down, protocol down
Serial12/0/3:0, DTE, physical up, protocol up
表1 display fr interface输出信息描述

项目

描述

MFR12/0/0

接口类型和接口编号。

DTE
接口的工作方式。

  • DTE：用户侧设备接口工作方式。

  • DCE：网络侧设备接口工作方式。

  • NNI：交换机之间的接口工作方式。

physical down

物理链路状态为Down。可能是线缆没有插好。

protocol down

网络协议状态为Down。可能是因为没有配置DLCI。

multi-point

子接口类型是点到多点。
