add hop

命令功能

add hop命令用来在显式路径中插入指定节点。

缺省情况下，显式路径采用include strict模式。

命令格式

add hop ip-address1 [ exclude | include [ loose | strict ] ] { after | before } ip-address2

参数说明

ip-address1：新增节点上接口的IP地址，点分十进制形式。

exclude：显式路径不能经过ip-address1指定的链路。

include：显式路径必须经过ip-address1指定的链路。

loose：松散显式路径，ip-address1指定的新节点与ip-address2指定的节点可以不是直连的。

strict：严格显式路径，ip-address1指定的新节点与ip-address2指定的节点必须是直连的。

after：表示在ip-address2后插入ip-address1指定的节点。

before：表示在ip-address2前插入ip-address1指定的节点。

ip-address2：显式路径中节点接口的IP地址或节点Router ID，点分十进制形式。

使用指南

使用add hop命令在显式路径中插入节点的时候，必须保证ip-address2存在于该显式路径中，且保证本设备上没有任何Tunnel接口在使用该显式路径。

使用实例

# 配置显式路径p1在经过3.3.10.5后，不能经过3.3.29.3。

<Quidway> system-view
[Quidway] explicit-path p1
[Quidway-explicit-path-p1] add hop 3.3.29.3 exclude after 3.3.10.5
