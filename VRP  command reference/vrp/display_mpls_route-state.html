display mpls route-state

命令功能

display mpls route-state命令用来查看LSP对应的路由信息。

命令格式

display mpls route-state [ vpn-instance vpn-instance-name ] [ { exclude | include } { idle | ready | settingup } * | destination-address mask-length ] [ verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vpn-instancevpn-instance-name：VPN实例名称。字符串形式，不支持空格，区分大小写，长度范围是1～31。

exclude：输出不包含指定的路由状态信息。

include：输出包含指定的路由状态信息。

idle：空闲的路由状态，即路由还没有触发建立LSP。

ready：就绪的路由状态，即路由已经触发建立了LSP。

settingup：等待LSP建立的路由状态，即信令协议正在创建LSP。

destination-address：目的地址。

mask-length：目的地址的掩码长度。取值范围是0～32。

verbose：显示LSP对应路由的详细信息。

| ：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

如果不指定VPN实例，将显示公网实例的路由信息；如果不指定目的地址和掩码，将显示所有路由信息。

使用实例

# 查看所有路由和LSP的对应情况。

项目

描述

Dest/Mask

目的IP地址及掩码长度

Next-Hop

下一跳IP地址

Out-Interface

出接口

State
MPLS控制平面路由状态：

  • IDLE:空闲的路由状态，即路由还没有触发建立LSP。

  • SETTINGUP:等待LSP建立的路由状态，即信令协议正在创建LSP。

  • READY：就绪的路由状态，即路由已经触发建立了LSP。静态LSP只有“READY”状态。

LSP

能到达该目的地址的LSP数目。

VRF

VPN索引号，“0”表示公网。

Type
路由类型：

  • B：BGP路由。

  • I：IGP路由。

  • L：带标签的公网BGP路由。

  • O：Original BGP路由。

  • U：未知类型的路由（如组播路由）。

LspIndex

该路由触发建立的LSP的索引号。

InLabel

该路由触发建立的LSP的入标签。

OutLabel

该路由触发建立的LSP的出标签。
