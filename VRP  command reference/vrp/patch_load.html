patch load

命令功能

patch load命令用来在单板上加载补丁包中与单板匹配的补丁。

缺省情况下，在主用主控板上加载补丁。

命令格式

patch load file-name [ all [ active | run ] | slave | slot slot-id ]

参数说明

file-name：补丁包的存放路径及文件名。字符串形式，格式为[ drive-name ][ file-name ]，长度范围是1～48。如果未指定drive-name，则此值为默认的设备名。

all：为所有单板加载补丁。

active：加载后直接激活补丁。

run：加载后直接运行补丁。

slave：为备用主控板加载补丁。

slot slot-id：为指定的单板加载补丁。

使用指南

在加载补丁之前系统要对补丁包进行解析，检查补丁包中补丁文件的合法性，并获取补丁文件的属性(包括补丁类型、单板类型、版本信息)。

为单板加载补丁时，系统会根据补丁文件的属性在补丁包中查找匹配的补丁文件，查找成功，则进行加载操作。如果补丁包中没有适合某类型单板的补丁，则不加载。

执行patch load all命令，将补丁包里各种类型的补丁加载到相应的单板上(包括主用主控板、备用主控板和所有接口板)。如果指定参数，在补丁加载后将直接达到激活态或运行态，该操作将影响所有单板。

补丁文件必须在主控板根目录下。

当备用主控板正在注册中且尚未注册成功时，如果进行补丁加载操作，则系统会提示：是否确认继续执行补丁操作。

说明：

目前此命令仅在NE40E、NE80E、NE5000E上可用。
