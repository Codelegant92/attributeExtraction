ripng metricout

命令功能

ripng metricout命令用来设置接口发送RIPng路由增加的度量值。

undo ripng metricout命令用来恢复该度量值为缺省值。

缺省情况下，接口发送RIPng路由增加的度量值为1。

命令格式

ripng metricout { value | { acl6-number | ipv6-prefix ipv6-prefix-name } value1 }

undo ripng metricout

参数说明

value：对发送的路由增加的度量值。整数类型，取值范围是1～15。缺省值为1。

acl6-number：指定基本ACL6的编号。整数类型，取值范围是1000～3999。

ipv6-prefix ipv6-prefix-name：指定IPv6地址前缀列表的名称。该名称必须唯一。字符串形式，不支持空格，区分大小写，长度范围是1～169。

value1：对通过ACL6或ipv6-prefix过滤的路由增加度量值。整数类型，取值范围是2～15。

使用指南

当发布一条路由时，发送度量值会在发布该路由之前附加在这条路由上。因此，增加一个接口的发送度量值，该接口发送的RIPng路由权值也会相应增加。但路由表中的度量值不会发生改变。

当用ACL6或ipv6-prefix方式来设置接口发送RIPng路由增加的度量值时，指定value1为通过过滤策略的RIPng路由增加的度量值，没有通过过滤的RIPng路由增加的度量值为1。

使用实例

# 设置RIPng发送度量值为12。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ripng metricout 12

# 设置用编号为2050的ACL6过滤的RIPng路由的发送度量值为12。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ripng metricout 2050 12

# 设置用IPv6地址前缀列表p1过滤的RIPng路由的发送度量值为12。

<Quidway> system-view
[Quidway]ip ipv6-prefix p1 permit 1::1 1::1 128
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ripng metricout ipv6-prefix p1 12
