mac-address static vsi

命令功能

mac-address static vsi命令用来配置静态MAC地址表项。该表项中的出接口已经与指定的VSI绑定。

undo mac-address命令用来删除MAC地址表项。

缺省情况下，系统没有配置任何静态MAC地址表项。

命令格式

mac-address static mac-address interface-type interface-number vsi vsi-name [ pe-vid pe-vid [ ce-vid ce-vid ] ]

undo mac-address mac-address vsi vsi-name

undo mac-address { all | dynamic | static }

参数说明

static：静态表项，不被老化。当收到指定地址的帧后，直接通过出接口转发。配置并保存后，系统复位或接口板热插拔表项不丢失。

mac-address：单播MAC地址，格式为H-H-H。其中H为4位的十六进制数，可以输入1～4位，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。MAC地址不可设置为FFFF-FFFF-FFFF。

interface-type interface-number：接口类型及编号，可以是Ethernet接口、Ethernet子接口、GE接口、GE子接口、Eth-Trunk接口或Eth-Trunk子接口。本命令中的接口是三层接口，并且已经绑定到VSI。

vsi vsi-name：指定的VSI实例的名称。字符串形式，不支持空格，区分大小写，取值范围是1～31。

pe-vid pe-vid：用户外层VLAN的Tag值。十进制整数形式，取值范围是1～4094。

ce-vid ce-vid：用户内层VLAN的Tag值。十进制整数形式，取值范围是1～4094。

dynamic：动态表项，可以被老化，系统复位或接口板热插拔后丢失。

all：所有表项。

缺省级别

2：配置级

使用指南

使用命令mac-address static mac-address interface-type interface-number vsi vsi-name时，命令中接口必须是三层接口，并且该三层接口已经和VSI绑定。

undo mac-address { all | dynamic | static }命令仅可以在系统视图下执行。

使用命令mac-address static mac-address interface-type interface-number vsi vsi-name pe-vid pe-vid时，命令中接口必须是子接口，并且该子接口配置了dot1q终结和绑定了VSI。

使用命令mac-address static mac-address interface-type interface-number vsi vsi-name pe-vid pe-vid ce-vid ce-vid时，命令中接口必须是子接口，并且该子接口配置了QinQ终结和绑定了VSI。

使用实例

# 为名称为abc的VSI添加一个静态MAC地址表项，当收到目的MAC为0011-2233-4455的帧时，在名称为abc的VSI中转发。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit 
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vsi abc

# 接口GE1/0/0.1配置了dot1q终结子接口接入VPLS。在该子接口上配置静态MAC地址表项，当收到目的MAC为0011-2233-4455且属于VLAN10的帧时，去掉该帧的Tag后，在名称为abc的VSI中转发该帧。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit
[Quidway] interface gigabitethernet 1/0/0 
[Quidway-GigabitEthernet1/0/0] mode user-termination
[Quidway-GigabitEthernet1/0/0] quit 
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 dot1q-termination
[Quidway-GigabitEthernet1/0/0.1] dot1q termination vid 10
[Quidway-GigabitEthernet1/0/0.1] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0.1] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0.1 vsi abc pe-vid 10

# 接口GE1/0/0.1配置了QinQ终结子接口接入VPLS。在该子接口上配置静态MAC地址表项，当收到目的MAC为0011-2233-4455且带有两层Tag（外层Tag为10，内层Tag为100）的帧时，去掉该帧的两层Tag后，在名称为abc的VSI中转发该帧。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] mode user-termination
[Quidway-GigabitEthernet1/0/0] quit 
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 qinq-termination
[Quidway-GigabitEthernet1/0/0.1] qinq termination pe-vid 10 ce-vid 100 to 200
[Quidway-GigabitEthernet1/0/0.1] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0.1] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0.1 vsi abc pe-vid 10 ce-vid 100
