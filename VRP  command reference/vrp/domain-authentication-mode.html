domain-authentication-mode

命令功能

domain-authentication-mode命令用来设置IS-IS路由域按照预设的方式和密码验证收到的Level-2路由信息报文（LSP和SNP）。

undo domain-authentication-mode命令使IS-IS不验证收到的上述报文。

缺省情况下，系统不对产生的Level-2路由信息报文封装认证信息，也不会验证收到的Level-2路由信息报文。

命令格式

domain-authentication-mode { simple password | md5 password-key } [ ip | osi ]

domain-authentication-mode { simple password | md5 password-key } [ ip | osi ] [ snp-packet { authentication-avoid | send-only } | all-send-only ]

undo domain-authentication-mode

参数说明

simple：密码以纯文本方式发送。

password：明文模式下的认证密码，字符串类型，可以为字母或数字，长度范围是1～16。

md5：密码通过MD5加密后发送。

password-key：MD5模式下的认证密码，字符串类型，可以为字母或数字，不支持空格。其中，认证密码可以是长度为1～16的明文；也可以是长度为24的密文。

ip：IP认证密码。

osi：OSI认证密码。

snp-packet：配置SNP报文相关的验证。

authentication-avoid：不对产生的SNP封装认证信息，也不检查收到的SNP。只对产生的LSP封装认证信息，并检查收到的LSP。

send-only：对产生的LSP和SNP封装认证信息，只检查收到的LSP，不检查收到的SNP。

all-send-only：对产生的LSP和SNP封装认证信息，不检查收到的LSP和SNP。

使用指南

此命令只在Level-2或Level-1-2路由器上配置有效。

通过这条命令，可以丢弃和此命令设定的路由域验证密码不符的Level-2路由报文；同时也会以指定的方式在本节点发送的所有Level-2路由信息报文中插入所设定的路由域验证密码。

如果设置了密码，但没有指定参数ip和osi，系统默认为osi。

该验证仅对设置端起作用，没有设置认证的验证端仍能接受带验证密码的LSP。

认证密码用ip或osi不受实际的网络环境影响。

说明：

如果通过isis circuit-type命令将广播接口模拟为P2P接口或者通过undo isis circuit-type命令将该接口恢复为广播接口，则IS-IS域认证恢复为缺省配置。

使用实例

# 当用户需要对Level-2路由信息报文进行验证时，可以通过simple方式，密码设为huawei。

<Quidway> system-view
[Quidway] isis 1
[Quidway-isis-1] domain-authentication-mode simple huawei
