cluster-member get

命令功能

cluster-member get命令用来指定成员交换机获取指定名称的配置文件、版本文件、补丁文件、PAF文件或者License文件。

命令格式

cluster-member [ group-by { device-type device-type | ip {ip-addres [ to ip-addres ] } &<1-10> | member-number { member-number [ to member-number ] } &<1-10> } ] get { configuration-file | system-software | patch | paf | license } file-name [ ip ftp-ip-address username user-name password password ]

参数说明

group-by：指定成员交换机的分类方式。目前支持三种分类方式：设备类型、IP地址和成员编号。不选择该参数时，默认对所有的成员交换机进行操作。

device-type device-type：指定成员交换机的设备类型。字符串形式，不区分大小写，不支持空格，长度范围是1～100。一次只能指定一种设备类型。目前支持的设备类型由NTDP协议收集得到，例如NE40E，CX200B等。

ip-address：指定成员交换机的IPv4地址。可以同时指定一个或者多个成员交换机的地址和地址范围。

member-number：指定成员交换机的成员编号。十进制整数形式，取值范围是1～255。可以同时指定一个或者多个成员交换机的成员编号。

file-name：配置文件、版本文件、PAF文件、License文件或者补丁文件的名称。字符串形式，不区分大小写，长度范围是1～50。

ip ftp-ip-address：FTP服务器的IPv4地址，点分十进制形式。

username user-name：FTP服务器的用户名称。字符串形式，区分大小写，不支持空格，长度范围是1～20。

password password：FTP服务器的用户密码。字符串形式，区分大小写，不能输入空格和问号等命令行专用字符，长度范围是1～8。

使用指南

本命令只能在命令交换机上执行。

指定成员交换机的成员编号时不能超过集群支持的最大编号。

使用本命令时，如果不指定FTP服务器的地址、用户名称和用户密码，则成员交换机将使用命令交换机上配置的默认FTP服务器信息。

说明：

命令交换机上默认的FTP服务器配置用来支持HGMP的即插即用功能。具体配置方法请参考cluster-plug-play ip username password命令。

使用实例

# 指定集群中所有的NE40E设备从FTP服务器100.1.1.1上获取版本文件。版本文件的名称是hgmp.cc，FTP服务器的用户名称是user1，密码是1234。

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] cluster-member group-by device-type ne40e get system-software hgmp.cc ip 100.1.1.1 username user1 password 1234

# 指定集群中IP地址是10.1.1.1～10.1.1.100的成员交换机从FTP服务器100.1.1.1上获取系统文件。系统文件的名称是hgmp.cfg，FTP服务器的用户名称是user1，密码是1234。

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] cluster-member group-by ip 10.1.1.1 to 10.1.1.100 get configuration-file hgmp.cfg ip 100.1.1.1 username user1 password 1234

# 指定集群中所设备编号从1～100的成员交换机获取补丁文件。补丁文件的名称是hgmp.pat，FTP服务器使用命令交换机上默认的FTP配置信息。

<Quidway> system-view
[Quidway] cluster
[Quidway-cluster] cluster-member group-by member-number 1 to 100 get patch hgmp.pat
