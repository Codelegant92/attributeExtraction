esp encryption-algorithm

命令功能

esp encryption-algorithm命令用来设置ESP协议采用的加密算法。

undo esp encryption-algorithm命令用来设置ESP协议不对报文进行加密。

缺省情况使用des，即DES加密算法。

命令格式

esp encryption-algorithm { 3des | des | aes }

undo esp encryption-algorithm

参数说明

des：即数据加密标准DES（Data Encryption Standard），是国际通用的加密算法，密钥长度56位。

3des：即3DES（Triple DES），也是一种国际通用的加密算法，密钥长度168位。

aes：使用AES（Advanced Encryption Standard，高级加密标准）做为加密算法（该参数只使用于NE16E、NE08E、NE05产品并由加密卡硬件提供）。

使用指南

对于保密及安全性要求非常高的地方采用3DES算法可以满足需要，但3DES加密速度比较慢；对于普通的安全要求，DES算法则可以满足需要。

ESP协议允许对报文同时进行加密和认证，或只加密，或只认证。

注意：undo esp encryption-algorithm命令不是恢复加密算法为缺省算法，而是设置加密算法为空，即不加密。当加密算法不为空时，undo esp encryption-algorithm命令才起作用。

ESP协议采用的加密算法和认证算法不能同时设置为空。

在安全隧道的两端设置的安全策略所引用的提议必须设置成采用同样的加密算法。

使用实例

# 设定提议prop1采用ESP协议并使用3DES加密算法。

<Quidway> system-view
[Quidway] ipsec proposal prop1
[Quidway-ipsec-proposal-prop1] transform esp
[Quidway-ipsec-proposal-prop1] esp encryption-algorithm 3des
