max bandwidth-affected-linknumber

命令功能

max bandwidth-affected-linknumber命令用来配置Eth-Trunk口链路UP的上限阈值。

命令格式

max bandwidth-affected-linknumber link-number

参数说明

link-number：链路UP的上限阈值，取值范围是1～16，缺省值是1。

视图

Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

当Eth-Trunk口处于UP状态的成员端口数达到上限阈值后，之后再发生的成员端口链路状态改变不影响Trunk的带宽，即，成员端口链路UP不会使Trunk的带宽增加，成员端口链路Down也不会使Trunk的带宽减少。

Eth-Trunk成员链路UP的上限阈值不能小于成员链路UP的下限阈值。

说明：

只有Eth-Trunk处于二层模式时，才能使用max bandwidth-affected-linknumber命令。

使用实例

# 配置当5条成员链路处于UP状态后，Eth-Trunk 1的状态不再受其他成员链路UP/Down的影响。

<Quidway> system-view
[Quidway] interface eth-trunk 1
[Quidway-Eth-Trunk1] portswitch
[Quidway-Eth-Trunk1] max bandwidth-affected-linknumber 5
