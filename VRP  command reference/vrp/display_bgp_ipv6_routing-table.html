display bgp ipv6 routing-table

命令功能

display bgp ipv6 routing-table命令用来查看BGP IPv6的路由信息。

命令格式

display bgp ipv6 routing-table [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table ipv6-address prefix-length [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table as-path-filter as-path-filter-number [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table community [ community-number | aa:nn ] &<1-13> [ internet | no-advertise | no-export | no-export-subconfed ] * [ whole-match ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table community-filter { { community-filter-name | basic-community-filter-number } [ whole-match ] | advanced-community-filter-number } [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table different-origin-as [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table peer { ipv6-address | ipv4-address } { advertised-routes | received-routes [ active ] } [ | count ] [ | { begin | exclude | include } regular-expression ]

display bgp ipv6 routing-table regular-expression as-regular-expression

参数说明

ipv6-address：指定要显示的对等体的IPv6地址。

prefix-length：指定前缀长度。

as-path-filter：用来显示匹配指定过滤器的路由。

as-path-filter-number：指定匹配的AS路径过滤器号。

regular-expression as-regular-expression：为匹配的AS正则表达式。字符串类型，取值范围是1～80。

different-origin-as：用来显示具有相同目的地址但不同源AS号的路由。

community-filter：用来显示匹配指定的BGP团体属性过滤器的路由信息。

community-filter-name：团体属性过滤器名称。

basic-community-filter-number：指定基本团体属性过滤器号。

advanced-community-filter-number：指定高级团体属性过滤器号。

community：用来查看路由表中指定BGP团体的路由信息。

community-number：指定的团体号。

aa:nn：指定的团体号。最多可以设置13个团体号。

internet：显示带有Internet团体属性的BGP路由。

no-advertise：显示带有No-Advertise团体属性的BGP路由。

no-export：显示带有No-Export团体属性的BGP路由。

no-export-subconfed：显示带有No-Export-Subconfed团体属性的BGP路由。

whole-match：精确匹配。

peer：查看指定对等体的路由信息。

ipv4-address：指定要显示的对等体的IPv4地址。

advertised-routes：显示发布给指定对等体的路由信息。

received-routes：显示从指定对等体收到的路由信息。

active：显示从指定对等体收到的活跃路由信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

通过指定不同的参数可以只查看特定的路由信息。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
