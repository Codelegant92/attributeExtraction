display mpls l2vpn connection

命令功能

display mpls l2vpn connection命令用来查看Kompella方式的L2VPN连接信息。

命令格式

display mpls l2vpn connection vpn-name [ remote-ce ce-id | down | up | verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

display mpls l2vpn connection [ summary | interface interface-type interface-number ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vpn-name：用于显示的指定的VPN名称。字符串格式，不支持空格，区分大小写，长度范围是1～31。如果不指定，将显示所有L2VPN信息。

remote-ce ce-id：指定将要显示其远程连接信息的CE。

down：状态为Down的连接的详细信息。如果不指定Down，将显示两种状态的连接的详细信息。

up：状态为Up的连接的详细信息。如果不指定Up，将显示两种状态的连接的详细信息。

verbose：显示连接的详细信息。该参数只有在查看VPN内所有连接时有效。

summary：显示连接的摘要统计信息。

interface interface-type interface-number：显示特定接口类型和编号的连接信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

配置完Kompella方式的L2VPN时，可以使用指定参数vpn-name的display mpls l2vpn connection命令查看该连接是否处于Up状态。

display mpls l2vpn connection命令还可以用来查看本设备上所有的Kompella方式L2VPN连接信息。

使用实例

# 查看所有Kompella方式的L2VPN连接。

<Quidway> display mpls l2vpn connection
1 total connections,
connections: 1 up, 0 down, 0 local, 1 remote, 0 unknown
VPN name: vpn1,
1 total connections,
connections: 1 up, 0 down, 0 local, 1 remote, 0 unknown
  CE name: ce1, id: 1,
  Rid type status peer-id         route-distinguisher intferface           primary or not
  2   rmt  up     3.3.3.9         100:1               GigabitEthernet1/0/1  primary
表1 display mpls l2vpn connection命令输出信息描述

项目

描述

1 total connections,connections: 1 up, 0 down, 0 local, 1 remote, 0 unknown

前两行的显示信息表示本设备上建立的所有Kompella方式的连接信息，包括连接的总数、处于up状态的连接数、down状态的连接数、本地连接数、远程连接数、未知的连接数。

vpn-name

已创建的VPN实例的名称。

1 total connections,connections: 1 up, 0 down, 0 local, 1 remote, 0 unknown

该VPN的连接信息，包括连接的总数、处于up状态的连接数、down状态的连接数、本地连接数、远程连接数、未知的连接数。

CE name:

CE连接的名称。

id:

该CE连接的CE ID，用于在一个VPN中唯一确定一个CE。

Rid

与本CE建立连接的远端CE ID。

type
该CE连接的类型：

  • rmt：表示该连接的类型为远程连接；

  • loc：表示该连接的类型为本地连接；

  • ---：表示目的CE不存在或者创建远程连接时，信令连接建立不成功。

status
该CE连接的状态：

  • up：表示该连接建立成功；

  • down：表示该连接建立不成功，可能是配置不正确或关联的接口为down导致的。

peer-id

与本端建立会话连接的对端IP地址。

route-distinguisher

该CE所属VPN的路由标志符。

interface

与该CE相关联的接口（AC接口）。

primary or not

该VC连接是主用VC还是备用VC。

项目

描述

conn-type
L2VPN连接类型：

  • local：本地连接；

  • remote：远程连接；

  • unknown：未知的L2VPN连接。出现的原因可能是目的CE不存在；或者创建远程连接时，信令连接建立不成功。

local vc state

本端的VC状态。up表示该连接建立成功；down表示连接建立失败，可能是配置不正确或相关联的接口为down。

remote vc state

目的端的VC状态。up表示该连接建立成功；down表示连接建立失败，可能是配置不正确或相关联的接口为down。

local ce-id

本端的CE-ID。

local ce name

本端CE的名称。

remote ce-id

与本CE建立连接的目的CE-ID。

intf(state,encap)

连接关联的本端接口的状态和封装类型。

peer id

与该接口关联的CE建立会话连接的远端IP地址。

route-distinguisher

与该接口关联的CE所属的VPN的路由标志符。

local vc label

本端系统产生的标签。

remote vc label

远端标签，是远端LDP分发给本端的标签。

tunnel policy

应用到该CE连接的隧道策略。

primary or secondary

该VC是主用VC还是备用VC。

forward entry exist or not

是否存在转发表项。

forward entry active or not

转发表项是否处于活跃状态（活跃状态表示可以转发）。

manual fault set or not

是否手工设置了PW故障。

AC OAM state

本端AC的OAM状态。

BFD for PW session index

检测PW状态的BFD会话的索引值。

BFD for PW state

检测PW状态的BFD会话的状态。

BFD for LSP state

检测LSP状态的BFD会话的状态。

Local C bit is not set

表示该L2VPN没有使能控制字。如果使能了控制字，则不显示该行信息。

tunnel type

隧道的类型。该隧道用于在公网上传输该CE的用户数据。

id

隧道的Tunnel ID。该隧道用于在公网上传输该CE的用户数据。
