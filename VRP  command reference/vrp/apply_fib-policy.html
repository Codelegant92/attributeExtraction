apply fib-policy

命令功能

apply fib-policy命令用来配置接口板的IPv4 FIB策略。

undo apply fib-policy命令用来取消指定接口板的指定IPv4 FIB策略。

缺省情况下，没有配置任何Pv4 FIB策略。

命令格式

apply fib-policy { no-route | public | vpn-instance vpn-instance-name &<1-6> } slot slot-number

undo apply fib-policy { all | no-route | public | vpn-instance vpn-instance-name &<1-6> } slot slot-number

参数说明

no-route：不接收主控板下发的任何IPv4转发表项。

public：接收主控板下发的IPv4公网转发表项。

vpn-instance vpn-instance-name：IPv4 VPN实例的名称，字符串形式，区分大小写，长度范围是1～31。每次执行该命令时最多可以输入六个VPN实例名。每个接口板上最多可以对16个VPN实例配置IPv4 FIB策略。配置时，该实例名必须已经存在。

slot slot-number：需要配置IPv4 FIB策略的接口板编号。

all：取消接口板上配置的所有IPv4 FIB策略。

使用指南

同一接口板上不能同时存在以下两种配置：

  • 不接收主控板下发的任何IPv4转发表项（no-route）和接收IPv4公网转发表项（public）

  • 不接收主控板下发的任何IPv4转发表项（no-route）和接收IPv4 VPN实例转发表项（vpn-instance）

但是可以同时存在接收IPv4公网转发表项（public）和接收IPv4 VPN实例转发表项的配置（vpn-instance）。

执行命令apply fib-policy后，必须对接口板的FIB表项进行刷新才能使配置生效。

使用实例

# 2号接口板只接收主控板下发的IPv4公网转发表项。

<Quidway> system-view
[Quidway] apply fib-policy public slot 2
