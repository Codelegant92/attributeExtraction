rtpq

命令功能

rtpq命令用来配置应用RTP优先队列（Realtime Transport Protocol Priority Queue）。

undo rtpq命令用来取消应用。

命令格式

rtpq start-port min-rtp-port-number end-port max-rtp-port-number bandwidth bandwidth [ cbs cbs ]

undo rtpq

参数说明

min-rtp-port-number：目的UDP端口的下限。取值范围是2000～65535。

max-rtp-port-number：目的UDP端口的上限。取值范围是2000～65535。

bandwidth：RTP队列的带宽，单位是kbit/s，取值范围是8～1000000。

cbs：承诺突发尺寸，取值范围是1500～2000000bytes，缺省值为1500 bytes。

使用指南

该命令创建一个严格优先队列，只有在RTP指定的start-port和end-port端口范围内、且目的端口号为偶数的UDP报文才能进入RTP优先队列。

将配置了RTPQ的帧中继class应用到PVC上，将在该PVC上创建一个严格优先队列，目的UDP端口在RTPQ指定的端口范围内的报文将进入RTPQ优先队列。虚电路发生拥塞时，该队列的报文绝对优先发送，但不超过配置的带宽；虚电路没有发生拥塞时，指定端口范围内的RTP报文可以占用虚电路上可用带宽。一般可将VoIP使用的UDP端口范围配置为[16384，32767]。

使用实例

# 在名为test1的帧中继class上配置RTP优先队列，带宽为20kbit/s。

<Quidway> system-view
[Quidway] fr class test1
[Quidway-fr-class-test1] rtpq start-pot 16383 end-port 16384 bandwidth 20
