igmp static-group

命令功能

igmp static-group命令用来配置该接口以静态方式加入组播组。

undo igmp static-group命令用来删除以静态方式加入的组播组。

缺省情况下，接口未以静态方式加入任何组播组。

命令格式

igmp static-group group-address [ source source-address ]

undo igmp static-group { all | group-address [ source source-address ] }

参数说明

all：表示该接口以静态方式加入的所有组播组。

group-address：表示组播组地址。点分十进制形式，取值范围是224.0.1.0～239.255.255.255。

source source-address：用来指定一个组播源。source-address表示组播源地址。点分十进制形式。

使用指南

如果组播组地址在SSM范围内，可通过source source-address参数指定组播源地址，在接口加入组的同时指定源地址。

执行此命令后，路由器认为该接口上始终连接着组成员主机，并持续向该接口所在网段转发符合条件的组播报文。

配置组播CAC功能后，要想配置静态组播组，必须先使能PIM-SM。

使用实例

# 配置路由器GE1/0/0接口以静态方式加入组播组。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp static-group 224.1.1.1

# 配置路由器在GE1/0/0接口上转发从组播源192.168.11.1到组播组232.1.1.1的报文。

<Quidway> system-view
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] igmp static-group 232.1.1.1 source 192.168.11.1
