display mpls te tunnel

命令功能

display mpls te tunnel命令用来查看隧道信息。

命令格式

display mpls te tunnel [ destination ip-address ] [ lsp-id ingress-lsr-id session-id local-lsp-id] [lsr-role { all | egress | ingress | remote | transit } ] [ name tunnel-name ] [ { incoming-interface | interface | outgoing-interface } interface-type interface-number ] [ te-class0 | te-class1 | te-class2 | te-class3 | te-class4 | te-class 5 | te-class6 | te-class7 ] [ verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

display mpls te tunnel { stale-incoming-interface | stale-interface | stale-outgoing-interface } interface-index [ verbose ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

destination ip-address：只显示目的地址为指定IP地址的隧道。

lsp-id：显示指定LSP标识的隧道。

ingress-lsr-id：入节点的LSR-ID，点分十进制格式。

session-id：会话标识。整数形式，取值范围是0～65535。

local-lsp-id：本地LSP标识号。整数形式，取值范围是0～65535。

lsr-role：以本设备在隧道中担任的角色来显示隧道信息。

all：显示所有隧道。

egress：显示以本设备为出节点的隧道，即，在本设备终止的隧道。

ingress：显示以本设备为入节点的隧道，即，本设备发起的隧道。

remote：显示非本设备发起的隧道，即，在本设备终止或穿过本地的隧道。

transit：显示以本设备为中间节点的隧道，即，穿过本设备的隧道。

name tunnel-name：显示具有指定名称的隧道，tunnel-name为字符串形式，长度为1～255。tunnel-name的取值必须是已经存在的隧道名称，并且，该名称的形式必须与配置文件中的名称形式完全一致，包括大小写和空格。例如，如果配置文件中的Tunnel接口信息为“interface Tunnel1/0/0”，此处指定隧道名称时，只能用“Tunnel1/0/0”，不能更改大小写或增加空格，“tunnel1/0/0”或“Tunnel 1/0/0”都是错误的写法。隧道名称应当包含在信令消息中，这样所有的跳都可以获得。

incoming-interface interface-type interface-number：显示使用指定入接口的所有隧道。

outgoing-interface interface-type interface-number：显示使用指定出接口的所有隧道。

interface interface-type interface-number：显示使用指定类型和编号的接口作为出接口或入接口的所有隧道。

te-class0 ～ te-class7：显示指定TE-Class的隧道。

verbose：显示详细信息。

stale-incoming-interface：显示处于stale状态的指定入接口的隧道信息。

stale-interface：显示处于stale状态的隧道的信息。

stale-outcoming-interface：显示处于stale状态的指定出接口的隧道信息。

interface-index：指定stale接口的索引号。十六进制整数形式，取值范围是1～FFFFFFFF。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

项目

描述

No

表项的编号

Tunnel-Name

隧道的名称

TunnelIndex

隧道的索引号

LSP Index

LSP的索引号

Session ID

会话标识符，也称为Tunnel ID

LSP ID

该LSP的标识符

Lsr Role
该LSR的角色：
  • Ingress：本节点为该隧道的入节点
  • Egress：本节点为该隧道的出节点
  • Transit：本节点为该隧道的中间节点

Ingress LSR ID

该LSP的入节点标识符

Egress LSR ID

该LSP的出节点标识符

In-Interface

该LSP在本节点的入接口

Out-Interface

该LSP在本节点的出接口

Sign-Protocol
隧道协议：
  • Static CR：静态CR-LSP
  • RSVP TE：以RSVP TE为信令

Resv Style

隧道的资源预留风格。如果是静态CR-LSP，则没有预留风格；如果是RSVP信令，则显示为SE和FF

IncludeAnyAff

生效的亲和属性掩码，缺省值是0x0

ExcludeAnyAff

未生效的亲和属性，缺省值是0x0

IncludeAllAff

生效的亲和属性，缺省值是0x0

ER-Hop Table Index

显式路由表的索引

AR-Hop Table Index

实际路由表的索引

C-Hop Table Index

CSPF计算出的路由表索引

PrevTunnelIndexInSession

同一个会话中前一个隧道表项的索引

NextTunnelIndexInSession

同一个会话中下一个隧道表项的索引

PSB Handle

链路状态块PSB（Path State Block）的句柄

Created Time

该隧道创建时间

Bandwidth Reserved Flag
带宽预留标识：
  • Unreserved：没有给任何CT预留带宽
  • Reserved：给一个或多个CT预留了带宽

CT0 Bandwidth(Kbit/sec)～CT7 Bandwidth(Kbit/sec)

给CT0～CT7预留的带宽值

Setup-Priority

隧道的建立优先级

Hold-Priority

隧道的保持优先级

Primary LSP Info

主LSP的相关信息

TE Attribute Flag

TE隧道属性的需求标识

Protected Flag

保护主隧道的需求标识

Bypass In Use

与该主隧道绑定的旁路隧道。“Not Exists”表示该主隧道还没有绑定任何旁路隧道

Bypass Tunnel Id

旁路隧道的隧道标识符

BypassTunnel

旁路隧道的名称

Bypass Lsp ID

旁路隧道的LSP标识符

FrrNextHop

重路由的下一跳

ReferAutoBypassHandle

对应的自动旁路隧道句柄

FrrPrevTunnelTableIndex

FRR前一个隧道表项的索引值

FrrNextTunnelTableIndex

FRR下一个隧道表项的索引值

Bypass Attribute

旁路隧道的属性。“(Not configured)”表示用户没有配置旁路隧道属性，旁路隧道继承主隧道的属性

Setup Priority

旁路隧道的建立优先级

Hold Priority

旁路隧道的保持优先级

HopLimit

旁路隧道的最大路径跳数限制值

IncludeAnyGroup

生效的旁路隧道亲和属性掩码

ExcludeAnyGroup

未生效的旁路隧道亲和属性

IncludeAllGroup

生效的旁路隧道亲和属性

Bypass Unbound Bandwidth Info

旁路隧道未绑定的带宽信息

CT0 Unbound Bandwidth～CT7 Unbound Bandwidth

未预留但可预留给CT0～CT7的带宽

NextSessionTunnelIndex

旁路隧道表项的前一个索引

PrevSessionTunnelIndex

旁路隧道表项的后一个索引

NextLspId

Bypass LSP ID的下一个表项值

PrevLspId

Bypass LSP ID的前一个表项值
