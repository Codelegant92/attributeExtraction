ip address

命令功能

ip address命令用来配置接口的IP地址。

undo ip address命令用来删除接口的IP地址。

缺省情况下，在接口上没有配置IP地址。

命令格式

ip address ip-address { mask | mask-length } [ sub ]

undo ip address ip-address { mask | mask-length } [ sub ]

参数说明

ip-address：指定接口IP地址。点分十进制格式。

mask：指定子网掩码。点分十进制格式。

mask-length：指定掩码长度。整数形式，取值范围是0～32。

sub：配置从IP地址。为了实现一个接口下的多个子网之间能够通信，需要在接口上配置从IP地址。

使用指南

ip address命令用来给路由设备上的接口配置IP地址和掩码，实现和网络的互连。有时为了使路由设备的一个接口能够与多个子网相连，可以在一个接口上配置多个IP地址，其中一个为主IP地址，其余为从IP地址。当配置主IP地址时，如果接口上已经有主IP地址，则原主IP地址被删除，新配置的IP地址成为主IP地址。

undo ip address命令不带任何参数表示删除该接口的所有IP地址。undo ip address ip-address mask表示删除主IP地址，undo ip address ip-address mask sub表示删除从IP地址。在删除主IP地址前必须先删除完所有的从IP地址。

VRP支持同一设备网段地址的空间重叠，以节约地址空间。

  • 在同一设备不同接口上可以配置网段重叠但不能完全相同的IP地址。例如，设备上某接口配置了地址20.1.1.1/16后，另一接口若配置20.1.1.2/24，此时会有提示信息，但配置仍然成功；若另一接口配置20.1.1.2/16，系统会提示地址冲突，配置失败。
  • 同一接口的主从IP地址之间可以配置网段重叠的IP地址。例如，在接口上配置了主IP地址20.1.1.1/24后，若配置从IP地址20.1.1.2/16 sub，此时会有提示信息，但配置仍然成功。
  • 同一设备不同接口的主从IP地址之间可以配置网段重叠但不能完全相同的IP地址。例如，设备上某接口配置了地址20.1.1.1/16后，另一接口若配置20.1.1.2/24 sub，此时会有提示信息，但配置仍然成功。

关于IP地址的详细介绍，请参见《VRP 配置指南 IP业务》“01 IP地址配置”。

使用实例

# 为接口POS1/0/0配置主IP地址为129.102.0.1，从IP地址为202.38.160.1，子网掩码都为255.255.255.0。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] ip address 129.102.0.1 255.255.255.0
[Quidway-Pos1/0/0] ip address 202.38.160.1 255.255.255.0 sub
