register-policy（IPv6）

命令功能

register-policy命令用来配置RP过滤由组播源端DR发送的Register注册报文的规则。

undo register-policy命令用来恢复缺省配置。

缺省情况下，未配置注册报文过滤规则。

命令格式

register-policy advanced-acl6-number

undo register-policy

参数说明

advanced-acl6-number：表示高级访问控制列表序号，用来过滤组播源或组的地址。整数形式，取值范围是3000～3999。

使用指南

注意：

RP只接收匹配ACL中允许状态的注册报文。指定一个未定义ACL编号会使RP拒绝所有的注册报文。

为了防止非法注册报文攻击，可以根据报文过滤规则来接受或拒绝和规则匹配的注册报文。

如果ACL拒绝某个（S, G）转发项，或者对其没有定义操作，RP向源端DR发送注册停止消息，停止这个组播数据流的注册。

请参见IPv4 PIM配置命令。

crp-policy命令与acl命令配合使用。在ACL6视图下，通过rule命令的source参数定义组播源地址，destination参数定义组播组地址。

使用实例

# 配置RP接受网段3:1::/64上的组播源发送给组播地址为FF02:13::/64的注册报文。

<Quidway> system-view
[Quidway] acl ipv6 3000
[Quidway-acl6-adv-3000] rule permit ipv6 source 3:1:: 64 destination ff02:13:: 64
[Quidway-acl6-adv-3000] quit
[Quidway] pim-ipv6
[Quidway-pim6] register-policy 3000
