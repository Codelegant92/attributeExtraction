peer bfd

命令功能

peer bfd命令用来为对等体/组配置指定的BFD检测参数。

undo peer bfd命令用来恢复BFD检测参数的缺省值。

命令格式

peer { group-name | ipv4-address } bfd { min-tx-interval min-tx-interval | min-rx-interval min-rx-interval | detect-multiplier multiplier } *

undo peer { group-name | ipv4-address } bfd { min-tx-interval | min-rx-interval | detect-multiplier } *

参数说明

group-name：对等体组的名称。

ipv4-address：对等体的IPv4地址。

min-tx-interval min-tx-interval：BFD发送检测报文的间隔。整数形式，取值范围是10～1000，单位是毫秒。缺省值是100毫秒。

min-rx-interval min-rx-interval：BFD接收检测报文的间隔。整数形式，取值范围是10～1000，单位是毫秒。缺省值是300毫秒。

detect-multiplier multiplier：指定本地检测时间倍数。整数形式，取值范围是3～50。缺省值是3。

缺省级别

2：配置级

使用指南

BGP对等体的状态是Established时，BFD会话才能被创建。

Peer级和Group级的BFD配置遵从个体优先的原则，即Peer上的配置优先级要高于Group上的配置。如果Peer上没有配置BFD，且Peer所在的Group使能了BFD功能，则Peer会从从其加入的Group中继承BFD配置。

说明：

检测参数的缺省值是动态的从BFD模块获得的，不同产品的缺省值可能不同。

使用实例

# 为对等体2.2.2.9配置BFD功能及检测参数。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] peer 2.2.2.9 bfd min-tx-interval 100 min-rx-interval 100 detect-multiplier 5
