dialer call-in

命令功能

dialer call-in命令用来配置按照ISDN主叫识别号码进行ISDN回呼。

undo dialer call-in命令用来取消该设置。

缺省情况下，未配置按照ISDN主叫识别号码进行ISDN回呼。

命令格式

dialer call-in remote-number [ callback ]

undo dialer call-in remote-number [ callback ]

参数说明

remote-number：用来匹配对端的呼入号码，字符“*”通配任意字符，长度范围是1～30。

callback：在回呼Server端，呼入号码将与包含该关键字的dialer call-in命令相匹配，从而发起回呼。

缺省级别

2：配置级

使用指南

当应用ISDN主叫识别回呼时，在Server端必须配置本命令。在共享DCC中，由于目前同时支持Dialer接口封装PPP和帧中继协议，ISDN接口可根据对应Dialer接口来动态封装链路层协议。

被呼端首先依据主叫号码与dialer number命令来匹配查找对应Dialer接口。命令dialer call-in用来对ISDN拨入号码进行预处理，以确定该号码用户是否应被接入，如果PBX交换机没有提供主叫号码则直接拒绝该呼叫。

使用实例

# 设置轮询DCC方式下回呼号码为8810152的用户。

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] dialer enable-circular
[Quidway-Serial1/0/0] dialer route ip 100.1.1.2 8810152
[Quidway-Serial1/0/0] dialer call-in 8810152 callback
