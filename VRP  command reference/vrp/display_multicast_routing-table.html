display multicast routing-table

命令功能

display multicast routing-table命令用来查看IP组播路由表信息。

命令格式

display multicast { vpn-instance vpn-instance-name | all-instance } routing-table [ group-address [ mask { group-mask | group-mask-length } ] | source-address [ mask { source-mask | source-mask-length } ] | incoming-interface { interface-type interface-number | register } | outgoing-interface { include | exclude | match } { interface-type interface-number | register | none } ] * [ | count ] [ | { begin | include | exclude } regular-expression ]

display multicast routing-table [ group-address [ mask { group-mask | group-mask-length } ] | source-address [ mask { source-mask | source-mask-length } ] | incoming-interface { interface-type interface-number | register } | outgoing-interface { include | exclude | match } { interface-type interface-number | vpn-instance vpn-instance-name | register | none } ] * [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

group-address：表示组播组地址。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

mask：指定组播组或源的地址掩码。

group-mask：表示组播组地址掩码。点分十进制形式。

group-mask-length：表示组播组地址掩码长度。整数形式，取值范围是4～32。

source-address：表示组播源地址。点分十进制形式。

source-mask：表示组播源地址掩码。点分十进制形式。

source-mask-length：表示组播源地址掩码长度。整数形式，取值范围是0～32。

incoming-interface：表示组播路由项的入接口。

interface-type interface-number：表示接口类型和接口编号。

register：表示PIM-SM的注册接口。

outgoing-interface：表示组播路由项的出接口。

include：显示下游接口列表中包含指定出接口的（S，G）表项。

exclude：显示下游接口列表中不包含指定出接口的（S，G）表项。

match：显示下游接口列表中只有一个接口且该接口与指定出接口相同的（S，G）表项。如果没有指定接口，则显示下游接口列表为空的（S，G）表项。

none：表示出接口列表为空。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

该命令显示组播路由表的信息，而display multicast forwarding-table命令显示组播转发表的信息。

若不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

组播转发表项与组播路由表项保持一致。

  • 每一个（S，G）项有一个唯一的Upstream（上游），表示组播数据应该从此接口到达。

  • 每一个（S，G）项有一个Downstream List（下游接口列表），表示组播数据应该从哪些接口向外转发。

使用display multicast routing-table命令：

  • 若不指定vpn-instance或all-instance，则只显示公网实例的组播路由表信息。

  • 如果指定group-address，则显示指定组播组对应的组播路由表信息。

  • 如果指定group-address mask ，则显示指定地址范围内的组播组对应的组播路由表信息。

  • 如果指定source-address，则显示指定组播源对应的组播路由表信息。

  • 如果指定source-address mask，则显示指定地址范围内的组播源对应的组播路由表信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
