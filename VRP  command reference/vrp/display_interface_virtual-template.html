display interface virtual-template

命令功能

display interface virtual-template命令用来查看虚拟接口模板的状态。

命令格式

display interface virtual-template [ vt-number ] [ | { begin | exclude | include } regular-expression ]

参数说明

virtual-template：虚拟接口模板。

vt-number：虚拟接口模板的编号，取值范围是0～1023。不指定编号将显示所有虚拟接口模板状态。

|：使用正则表达式过滤输出的显示信息。

begin：显示以包含指定正则表达式的行开始的配置信息。

exclude：显示不包含指定正则表达式的行的配置信息。

include：显示包含指定正则表达式的行的配置信息。

regular-expression：正则表达式。字符串形式，支持空格，区分大小写。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

表1 display interface Virtual-Template命令输出信息描述

项目

描述

Virtual-Template1 current state :
Virtual-Template接口当前的物理状态。

  • UP：物理状态正常启动。

  • DOWN：物理连接不正常。

  • Administratively down：如果网络管理员在该接口执行shutdown（接口视图）命令，将显示状态为Administratively down。

Line protocol current state :

显示该接口的链路协议状态。

  • UP：接口的链路协议处于正常的启动状态。

  • DOWN：接口的链路协议不正常，或者没有在此接口配置IP地址。

Link layer protocol is

链路连接协议。

Internet Address is

接口配置的IP地址。如果接口上没有配置IP地址，则显示为“Internet protocol processing : disabled”。

LCP initial

LCP初始化。

Description :

对接口的说明，支持空格，区分大小写，最多可输入242个字符。该说明信息便于用户了解该接口的作用。

The Maximum Transmit Unit is

接口的最大传输单元（MTU）。例如串口缺省值是1500字节。长度大于MTU的报文，将会被分片后再发送。如果设置了不准分片，会被丢弃。

Physical is None

没有物理设备。逻辑接口在物理上是不可见的。

Output queue : (Urgent queue : Size/Length/Discards) Output queue : (Protocol queue : Size/Length/Discards)Output queue : (FIFO queuing : Size/Length/Discards)
接口输出队列的当前状态，通常包括三类输出队列（Output queue）：

  • 紧急队列（Urgent queue），链路层的协议报文，比如PPP的协商报文和Keepalive报文等将进入这个队列。

  • 协议队列（Protocol queue），IP优先级为6的报文将进入协议队列。

  • 普通的报文队列（FIFO queue），根据接口上应用的队列类型，可能是FIFO（First In First Out Queue）、PQ（Priority Queue）、CQ（Custom Queue）或CBQ（Class-based Queue）等。

当发生拥塞时，接口首先发送紧急队列中的报文，然后是协议队列中的报文，最后发送普通队列中的报文。对于各种发送队列，显示为Size/Length/Discards形式的数字，这里各字段含义如下：

  • Size ：队列中的分组数。

  • Length：用分组表示的最大队列的长度。

  • Discards：由于队列已满而丢失的分组数。

通过检查Discards在一段时间内与Size和Length的关系，可以考察该接口的性能是否满足要求。例如：当Discards的值较大时可能表示路由器正忙于进行其他工作，而无法及时处理进入的分组，如果这种情况持续很长的时间，则通常表示需要一个更强大的路由器以满足工作的要求。

Statistics last cleared:

上次清除本接口统计信息的时间。

Last 300 seconds input rate

Last 300 seconds output rate

这一字段显示了在前300秒内通过接口接收和发送的字节速率和报文速率。
