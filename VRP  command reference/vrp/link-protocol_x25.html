link-protocol x25

命令功能

link-protocol x25命令用来将指定的接口用X.25协议。

缺省情况下，接口的链路层协议为PPP。当接口使用X.25协议时，缺省工作方式为DTE IETF。

命令格式

link-protocol x25 [ dce | dte ] [ ietf | nonstandard ]

参数说明

dce：设定该接口的工作方式为DCE。

dte：设定该接口的工作方式为DTE。

letf：按照IETF RFC 1356的标准规定在X.25网络上封装IP或其它的网络协议。

nonstandard：非标准的在X.25网络上封装IP或其它网络协议。

使用指南

如果不使用X.25交换功能，当两台路由器使用X.25协议直接背靠背相连时，应该令一台工作在DTE方式，一台工作在DCE方式。并且保证两端设备接口的数据报格式相同。

当两台路由器通过X.25公共分组网相连时，则两台路由器一般都应该工作在DTE方式。

如果使用X.25交换功能，则一般路由器应该工作在DCE方式。

在通常的应用中，如果没有特殊的要求，请选用数据报的IETF格式。

使用实例

# 指定接口Serial1/0/0的链路层协议为X.25，工作在DTE IETF方式。

<Quidway> system-view
[Quidway] interface serial 1/0/0
[Quidway-Serial1/0/0] link-protocol x25 dte ietf
