igmp-snooping group-policy

命令功能

igmp-snooping group-policy命令用来配置当前VLAN/VSI的组播组策略，即当前VLAN/VSI内主机允许加入的组播组。

undo igmp-snooping group-policy命令用来取消该配置。

缺省状况下，VLAN/VSI无组播策略，即VLAN/VSI内的主机可以加入任何组播组。

命令格式

igmp-snooping group-policy basic-acl-number [ version number ]

undo igmp-snooping group-policy

参数说明

basic-acl-number：基本访问控制列表编号，取值范围是2000～2999，用来指定该VLAN/VSI内主机允许加入的组播组。

version number：表示只对指定版本的IGMP报文应用组播组策略。取值为1或者2。

视图

VLAN视图、VSI视图

缺省级别

2：配置级

使用指南

通过配置VLAN/VSI的组播组策略，可以限制VLAN/VSI内主机对IP组播组的访问。在路由器上配置针对某VLAN/VSI的组播组策略后，当路由器从该VLAN/VSI内收到的用户主机请求加入组播组的Report消息时，处理方法如下：

  • 当该请求符合VLAN/VSI的组播组策略时，路由器为该组播组维护出端口信息。

  • 当该请求不符合VLAN/VSI的组播组策略时，路由器不会为该组播组维护出端口信息。

如果不指定应用组播组策略的IGMP报文的版本，则无论收到哪个版本（V1或者V2）的IGMP报文，都应用该组播组策略。

在VLAN视图下配置时，以下情况会导致配置不成功：

  • VLAN已经绑定了VSI；

  • VLAN内包含VE接口。

使用实例

# 限定VLAN 2内的主机只能加入组播组225.1.1.123。

<Quidway> system-view
[Quidway] acl number 2008
[Quidway-acl-basic-2008] rule permit source 225.1.1.123 0
[Quidway-acl-basic-2008] quit
[Quidway] vlan 2
[Quidway-vlan2] igmp-snooping group-policy 2008
