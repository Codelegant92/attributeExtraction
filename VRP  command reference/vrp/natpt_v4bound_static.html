natpt v4bound static

命令功能

natpt v4bound static命令用来配置IPv4到IPv6地址的静态映射，映射使用的是IPv4源地址。而实际上，缺省情况下添加了一条反方向的等效规则，因此当IPv6地址转化为IPv4地址时同样适用。

undo natpt v4bound static命令用来删除IPv4到IPv6地址的静态映射。

命令格式

natpt v4bound static ipv4-address ipv6-address

undo natpt v4bound static ipv4-address ipv6-address

natpt v4bound static v6server protocol { tcp | udp } ipv4-address ipv4-port-number ipv6-address ipv6-port-number

undo natpt v4bound static v6server protocol { tcp | udp } ipv4-address ipv4-port-number ipv6-address ipv6-port-number

参数说明

ipv4-address：指定需要映射的IPv4源地址，点分十进制形式。

ipv4-port-number：指定IPv4的端口号。整数形式，取值范围是1～12287。

ipv6-address：指定IPv6目的地址，格式为X.X::X.X。

ipv6-port-number：指定IPv6的端口号。整数形式，取值范围是1～12287。

使用指南

IPv6地址的前缀可以不同于NAT-PT前缀，但是建议与NAT-PT前缀相同。

如果指定了参数“v6server”，就意味着从IPv4侧到目的地址的静态映射。因为DNS-ALG不支持PAT，它只映射IPv4地址而不含端口号。如果IPv4地址池里没有足够IPv4的公网地址，DNS-ALG就会溢出。此时这个命令就可以用来配置从IPv4侧到目的地址的静态PAT映射，节省了IPv4公网地址。

说明：

  • 如果IPv4源地址映射的IPv6地址的前缀与配置的NAT-PT前缀不同，应谨慎对待，因为转换的IPv6报文有可能返回不成功。

  • 使用undo natpt v4bound static命令可能会影响某些动态映射的转换。

使用实例

# 配置IPv4到IPv6地址的静态映射。

<Quidway> system-view
[Quidway] natpt v4bound static 2.3.4.9 2001::1
