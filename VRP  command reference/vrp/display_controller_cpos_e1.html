display controller cpos e1

命令功能

display controller cpos e1命令用来查看指定CPOS的指定E1的物理层配置信息。

命令格式

display controller cpos cpos-number e1 e1-number

参数说明

e1-number：CPOS的E1通道号。整数形式，取值范围是1～63。

使用指南

与display controller cpos命令相比，本命令可以显示对应的低阶通道的错误和告警信息，以及E1帧的错误和告警信息。

项目

描述

Cpos12/0/0 current state :

CPOS接口当前的物理状态。

Frame-format SDH, multiplex AU-4, clock master, loopback not set

CPOS接口的物理层信息：帧格式为SDH、采用AU-4的复用路径、主时钟模式（使用内部时钟信号）、没有使能环回。

Tx: J0: 0x01, J1: "NetEngine", C2: 0x02

发送的开销字节。

Rx: J0: 0x01, J1: "NetEngine", C2: 0x02

接收的开销字节。

regenerator section:

再生段的告警和错误统计。

Multiplex section:

复用段的告警和错误统计。

Higher order path(VC-3-x):

高阶通道的告警和错误统计，VC-4-1表示采用AU-4的复用路径，只有一个高阶通道VC-4。

Alarm:Error:

告警统计和错误统计。可能出现的错误类型有以下几类。

  • FRED：Receive Loss of Basic Frame Alignment，接收到基本帧丢失位。也可以认为是收到有红色告警错误的帧。

  • COFA：Change of Frame Alignment，帧对齐改变。

  • SEF：Severely Errored Frame，严重错帧，连续4个帧同步错误将产生一个SEF。

  • FERR：Framing Bit Error，指有Ft/Fs/FPS/FAS错误的帧。

  • CERR：CRC Error，循环冗余校验错。

  • FEBE：Far End Block Error，远端块错。这种错误只有在E1通道采用CRC4的帧格式时才可能出现。

  • BERR：PRBS Bit Error（随机码测试位错，只用于测试）。

  • BIP：Bit-Interleaved Parity，比特交叉奇偶校验。

  • REI：Remote Error Indication，远端错误指示。

上面前三种错误（FRED、COFA、SEF）统称为Alarm Error，简写为AERR。

Cpos12/0/0 CE1 1 is up

CPOS接口12/0/0的E1通道1当前的物理状态。

E1 framer(1-1-1-1):

表示E1的复用过程。1-1-1-1依次表示此E1通道所属的VC-4编号、TUG-3编号、TUG-2编号和TU-12编号。
