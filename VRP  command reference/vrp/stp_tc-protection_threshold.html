stp tc-protection threshold

命令功能

stp tc-protection threshold命令用来配置MSTP进程在收到TC类型BPDU报文后，单位时间内，处理TC类型BPDU报文并立即刷新转发表项的阈值。

undo stp tc-protection threshold命令用来恢复MSTP进程在收到TC类型BPDU报文后，单位时间内，处理TC类型BPDU报文并立即刷新转发表项的缺省值。

缺省情况下，MSTP进程在收到TC类型BPDU报文后，单位时间内，处理TC类型BPDU报文并立即刷新转发表项的缺省值是3。

命令格式

stp tc-protection threshold threshold

undo stp tc-protection threshold

参数说明

threshold threshold：MSTP进程在收到TC类型BPDU报文后，单位时间内，处理TC类型BPDU报文并立即刷新转发表项的阈值。十进制整数形式，取值范围是1~255。缺省值为3。

使用指南

配置stp tc-protection命令后，在单位时间内，MSTP进程处理TC类型BPDU报文的次数，由stp tc-protection threshold命令指定。缺省的单位时间是2秒，缺省的处理次数是3次。

说明：

单位时间的取值与MSTP的Hello Time一致，可以通过命令stp timer hello配置。

如果在单位时间内，MSTP进程在收到TC类型BPDU报文数量大于stp tc-protection threshold命令中指定的阈值，那么MSTP进程只会处理阈值指定的次数。对于其他超出阈值的TC类型BPDU报文，定时器到期后，MSTP进程只对其统一处理一次。

使用实例

# 配置MSTP进程0和MSTP进程1处理TC类型BPDU报文并立即刷新转发表项的阈值为5。

<Quidway> system-view
[Quidway] stp tc-protection threshold 5
[Quidway] stp process 1
[Quidway-stp-process-1] stp tc-protection threshold 5
