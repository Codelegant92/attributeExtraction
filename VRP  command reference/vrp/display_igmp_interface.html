display igmp interface

命令功能

display igmp interface命令用来查看接口上IGMP配置和运行信息。

命令格式

display igmp [ vpn-instance vpn-instance-name | all-instance ] interface [ interface-type interface-number | up | down ] [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称，字符串形式，长度范围是1～31。

all-instance：指定所有实例。

interface-type interface-number：表示路由器的接口类型和接口号，用来指定具体的接口。

up：IGMP接口的IP协议状态是Up，IGMP协议的状态是Active。

down：IGMP接口的IP协议状态是Down，IGMP协议的状态是Inactive。

verbose：表示IGMP的详细信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

使用display igmp interface命令时：

  • 如果不指定vpn-instance或all-instance，则只显示公网实例的相关信息。

  • 若指定了interface-type interface-number，则仅显示在该接口下的IGMP配置和运行信息。如果该接口未使能IGMP，则出现提示信息。若不指定interface-type interface-number则显示所有运行IGMP的接口的信息。

  • 如果指定up，则仅显示所有IGMP协议为Active状态的IGMP接口参数。
  • 如果指定down，则仅显示所有IGMP协议为Inactive状态的IGMP接口参数。

  • 若指定了verbose则显示接口运行IGMP的各种状态参数。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

Interface information of VPN-Instance

IGMP接口信息所属的VPN实例。

GigabitEthernet6/0/0(11.1.0.1)

接口编号(IP地址)。

IGMP is enabled

接口已使能IGMP。

Current IGMP version

该接口使能的IGMP版本号。

IGMP state

IGMP接口状态，up或down。

IGMP group policy

IGMP组策略的ACL编号，用于限制该接口上可以加入的组。可通过igmp group-policy命令进行配置。

IGMP limit

IGMP表项限制值。

except

ACL定义的组播组不受IGMP表项限制值的限制。

Value of query interval for IGMP (negotiated)

非查询器通过协商得到的查询消息的时间间隔，单位是秒。该协商值只存在于IGMPv3中。

Value of query interval for IGMP (configured)

接口上配置发送IGMP查询消息的时间间隔，单位是秒。

Value of other querier timeout for IGMP

其它IGMP查询器存活时间，每秒减1，超时值可以通过igmp timer other-querier-present进行配置。作为查询器的接口上取值为0。

Value of maximum query response time for IGMP

IGMP查询报文中包含的最大响应时间。

Value of last member query time

最后成员查询时间。最后成员查询时间 = 最后成员查询时间间隔 × 健壮系数。在IGMPv1中，未定义“最后成员查询时间”。

Value of last member query interval

最后成员查询时间间隔，单位是秒。在IGMPv1中，未定义“最后成员查询时间间隔”。

Value of startup query interval

该接口作为查询器刚启动时的查询间隔，单位是秒。在IGMPv1中，未定义“查询器启动时发送查询消息的查询间隔”。

Value of startup query count

该接口作为查询器启动时发送查询消息的次数。在IGMPv1中，未定义“查询器启动时发送查询消息的次数”。

General query timer expiry (hours:minutes:seconds)

普遍组查询定时器的超时时间。

Querier for IGMP

IGMP查询器。在IGMP版本1中，查询器的选择由组播路由协议决定；在IGMPv2中，共享网段上IP地址最低的路由器充当查询器。

IGMP activity: 0 joins, 0 leaves
接口上的活跃的组成员关系：

  • joins表示加入过的IGMP组个数，有新的组加入时增加1，组离开时该值不变。

  • leaves表示离开的组个数，有组离开时增加1。

Robustness (negotiated)

非查询器通过协商得到的健壮系数。该协商值只存在于IGMPv3中。

Robustness (configured)

接口上配置的健壮系数。

Require-router-alert

是否要求接收到的IGMP报文携带Router Alert选项。

Send-router-alert

发送的IGMP报文是否携带Router Alert选项。

Prompt-leave

是否使能立即离开。

SSM-Mapping

是否使能SSM-Mapping功能。

Startup-query-timer-expiry
启动时该接口作为查询器的查询间隔定时器状态：
  • off：查询消息发送完毕。
  • on：查询消息未发送完毕。

Other-querier-present-timer-expiry
其他查询器的存在时间定时器状态：
  • off：接口认为自己是查询器，不存在其他查询器。
  • on：接口不是查询器，存在其他查询器。

Total 1 IGMP Group reported

该接口上动态加入的IGMP组个数，不包括静态加入的IGMP组。有新组播组动态加入时增加1，有组播组离开时减1。
