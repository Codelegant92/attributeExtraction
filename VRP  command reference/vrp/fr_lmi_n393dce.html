fr lmi n393dce

命令功能

fr lmi n393dce命令用来配置DCE侧N393。

undo fr lmi n393dce命令用来恢复缺省值。

命令格式

fr lmi n393dce n393-value

undo fr lmi n393dce

参数说明

n393-value：事件计数器。整数形式，取值范围是1～10。缺省值是4。

视图

帧中继接口视图、MFR接口视图

缺省级别

2：配置级

使用指南

DCE设备每隔一定的时间（间隔由T392决定）要求DTE设备发送一个状态请求报文（Status-Enquiry报文）。如果DCE在规定时间内没有收到状态请求报文，DCE就记录该错误，且错误次数加1。如果错误次数超过门限，DCE设备就认为物理通路不可用，所有的虚电路都不可用。

N392和N393一起定义了“错误门限”。其中N393表示被观察的总事件数，N392表示在被观察的总事件数中发生的错误门限。也就是说，如果DCE设备在N393个事件中，发生错误次数达到N392，DCE设备就认为错误次数达到门限，且认为物理通路不可用，所有的虚电路都不可用。

DCE方的N392应小于DCE方的N393。

使用实例

# 配置帧中继接口Serial1/0/0:0工作在DCE方式，并配置N392为5，N393为6。

<Quidway> system-view
[Quidway] interface Serial1/0/0:0
[Quidway-Serial1/0/0:0] link-protocol fr
[Quidway-Serial1/0/0:0] fr interface-type dce
[Quidway-Serial1/0/0:0] fr lmi n392dce 5
[Quidway-Serial1/0/0:0] fr lmi n393dce 6
