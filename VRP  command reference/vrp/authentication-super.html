authentication-super

命令功能

authentication-super命令用来配置当前认证方案使用的用户级别提升认证模式。

缺省情况下，认证模式为super。

命令格式

authentication-super { hwtacacs | super }* [ none ]

authentication-super none

参数说明

hwtacacs：采用HWTACACS模式对用户级别提升进行认证。

super：采用路由器本地验证的方式对用户级别提升进行处理。

none：不进行认证，直接让用户更改用户级别。

视图

认证方案视图

缺省级别

3：管理级

使用指南

如果在一个认证方案中使用多个用户级别提升认证方式，将按照配置的顺序进行认证。

  • 当配置的认证方式是先本地认证后远端认证时

    如果登录的帐号在本地没有创建，但是在远端服务器上是存在的，认证模式将由本地认证转入远端认证。

  • 当配置的认证方式是先远端认证后本地认证时

    如果登录的帐号在远端服务器上没有创建，但是在本地是存在的，经过远端认证时，将被认为认证失败，不再转入本地认证。

    只有在远端认证服务器Down时，才会转入本地认证。

采用HWTACACS模式对用户级别提升进行认证时，用户的口令需要在TACACS Server上预先设置完成。

如果选用了不认证（none），它必须做为最后一种认证模式。

使用实例

# 配置名为scheme1的认证方案，采用的用户级别提升认证方法为super认证。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authentication-scheme scheme1
[Quidway-aaa-authen-scheme1] authentication-super super

# 配置名为scheme2的认证方案，用户级别提升认证首次采用hwtacacs认证，二次认证为super认证，若这两种认证都没有响应则让用户通过。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] authentication-scheme scheme2
[Quidway-aaa-authen-scheme2] authentication-super hwtacacs super none
