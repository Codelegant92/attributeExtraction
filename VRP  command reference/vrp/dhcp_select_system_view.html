dhcp select（系统视图）

命令功能

dhcp select命令用来设置指定范围内的多个接口对目的地址是本机的DHCP报文的处理模式。

undo dhcp select命令用来恢复缺省设置。

缺省情况下，目的地址是本机的DHCP报文将被送到内部服务器，从全局地址池中分配地址（global方式）。

命令格式

dhcp select { global | interface | relay } { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

undo dhcp select { all | interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ] | interface interface-type interface-number | vlan { vlan-id1 [ to vlan-id2 ] } &<1-10> }

参数说明

global：将DHCP报文发送到本地的DHCP服务器，由本地DHCP服务器从全局地址池分配地址。

interface：将DHCP报文发送到本地的DHCP服务器，由本地DHCP服务器从接口地址池分配地址。

relay：将DHCP报文通过中继发送到外部的DHCP服务器，由外部DHCP服务器分配地址。

all：表示所有接口。

interface interface-type interface-number.sub-interface-number1 [ to interface-type interface-number.sub-interface-number2 ]：用关键字to连接两个接口，表示子接口号是在两个子接口之间的所有子接口（包含这两个子接口）。

vlan vlan-id1 [ to vlan-id2 ]：用关键字to连接两个VLAN，表示在两个VLAN ID之间的所有VLAN（包含这两个VLAN）。该参数用于有交换式以太网接口的设备。VLAN ID的取值范围是1～4094，vlan-id2的值必须大于vlan-id1。

使用指南

dhcp select命令用来设置指定范围内的多个接口对目的地址是本机的DHCP报文的处理模式。

说明：

“DHCP配置命令”中，与VLAN接口有关的命令行只可以在S8016、NE40、NE40E、NE80E、NE5000E上支持，其他硬件平台不支持。

使用实例

# 配置GE1/0/0.1到GE1/0/0.5之间的接口，对于目的地址是本机的DHCP报文，从内部服务器的接口地址池中分配地址。

<Quidway> system-view
[Quidway] dhcp select interface interface gigabitethernet 1/0/0.1 to gigabitethernet 1/0/0.5
