authentication-mode（OSPF区域视图）

命令功能

authentication-mode命令用来指定OSPF区域所使用的验证模式及验证口令。配置该命令相当于在指定区域所有路由器接口下使用相同的验证。

undo authentication-mode命令用来取消该区域已配置的验证模式。

缺省情况下，没有配置区域验证模式。

命令格式

authentication-mode simple [ [ plain ] plain-text | cipher cipher-text ]

authentication-mode { md5 | hmac-md5 } [ key-id { plain plain-text | [cipher ] cipher-text } ]

参数说明

simple：使用简单验证模式。

plain：明文口令类型。只能键入明文口令，在查看配置文件时以明文方式显示口令。对于简单验证模式，当此参数缺省时，默认为plain类型。

plain-text：指定明文验证字。simple模式下是1～8字节字符串；md5、hmac-md5模式是1～16字节字符串。

cipher：密文口令类型。对于MD5/HMAC-MD5验证模式，当此参数缺省时，默认为cipher类型。

cipher-text：指定密文验证字。simple模式下，1～8字节字符串对应明文，24字节对应密文；md5、hmac-md5模式下1～16字节字符串对应明文，24字节对应密文。

md5：使用MD5密文验证模式。

hmac-md5：使用hmac-md5密文验证模式。

key-id：认证ID值。整数形式，取值范围是1～255。

使用指南

一个区域中所有路由器的验证模式必须相同（不验证、使用简单验证、使用MD5密文验证），验证口令也必须相同。

该命令的优先级比接口验证方式（ospf authentication-mode）的优先级低。

验证字中不允许包含空格。

使用实例

# 指定OSPF区域0支持MD5密文验证模式。

<Quidway> system-view 
[Quidway] ospf 100
[Quidway-ospf-100] area 0
[Quidway-ospf-100-area-0.0.0.0] authentication-mode md5 1 cipher abc
