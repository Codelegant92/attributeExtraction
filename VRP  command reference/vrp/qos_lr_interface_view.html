qos lr（接口视图）

命令功能

qos lr命令用来限制接口的向外发送数据的速率。

undo qos lr命令用来取消限制。

命令格式

qos lr cir cir [ cbs cbs [ ebs ebs ] ]

undo qos lr

参数说明

cir：承诺信息速率。

cir: 承诺信息速率的值，取值范围是8000bps~155000000bps。

cbs：承诺突发尺寸，实际平均速率在承诺速率以内时的突发流量。

cbs：承诺突发尺寸的值，取值范围是15000～155000000bit。cbs的取值应大于等于cir取值的1/2，当cir>30000bit/s时，缺省取值是cir的1/2，当cir<30000bit/s时，缺省取值是15000。

ebs：过度突发尺寸。

ebs：过度突发尺寸，取值范围是0～155000000bit。

缺省为0，即只采用一个令牌桶监管。

使用指南

qos lr命令还可以应用于Tunnel接口，实现对Tunnel接口的流量限速，结合各种队列调度算法，可以实现Tunnel接口的拥塞管理。在为Tunnel接口配置队列之前，必须先在Tunnel接口上应用LR。反之，在删除Tunnel接口上的qos lr命令时，必须先删除Tunnel接口上的队列配置。

使用实例

# 下面的命令将对物理接口GE6/0/0发出的报文进行限速。

<Quidway> system-view
[Quidway] interface GigabitEthernet6/0/0
[Quidway-GigabitEthernet6/0/0] qos lr cir 38400 cbs 76800 ebs 0
