display mpls te hot-standby state

命令功能

display mpls te hot-standby state用来显示全部或者某个热备份隧道的热备份状态。

命令格式

display mpls te hot-standby state { all [ verbose ] | interface tunnel interface-name } [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

all：查看所有热备份隧道的热备份状态。

verbose：查看详细信息。

interface tunnel interface-name：查看指定隧道的热备份状态。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

需要把隧道类型配置为TE隧道且使能热备份。如果Tunnel接口被删除或者切换成其他模式，则此命令无效。

使用实例

# 查看Tunnel1/0/0的热备份状态。

<Quidway> display mpls te hot-standby state interface Tunnel 1/0/0

表1 display mpls te hot-standby state命令输出信息描述

项目

描述

tunnel name

隧道名称

session id

会话标识符，即配置的Tunnel ID

main LSP token

主CR-LSP的索引值

hot-standby LSP token

备份CR-LSP的索引值

HSB switch result

隧道切换结果：

  • main LSP：数据切换到主CR-LSP
  • hot-standby LSP：数据切换到备份CR-LSP
  • Best-Effort LSP：数据切换到逃生路径上

WTR

表示回切时间间隔。主隧道故障恢复时，系统等待一段时间后再将数据从备份CR-LSP或逃生路径回切到主CR-LSP，这段时间称为回切时间间隔
