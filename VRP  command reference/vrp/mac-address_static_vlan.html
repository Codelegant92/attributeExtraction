mac-address static vlan

命令功能

mac-address static vlan命令用来配置静态MAC地址表项。该表项中的出端口属于某个指定的VLAN。

undo mac-address命令用来删除的MAC地址表项。

缺省情况下，系统没有配置任何静态MAC地址表项。

命令格式

mac-address static mac-address interface-type interface-number vlan vlan-id [ ce-vlan ce-vlan ]

mac-address static mac-address virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } vlan vlan-id

undo mac-address { all | dynamic | static } [ interface-type interface-number ] [ vlan vlan-id ]

undo mac-address { all | dynamic | static } virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } vlan vlan-id

参数说明

static：静态表项，不被老化。当收到指定地址的帧后，直接通过出接口转发。配置并保存后，系统复位或接口板热插拔表项不丢失。

mac-address：单播MAC地址，格式为H-H-H。其中H为4位的十六进制数，可以输入1～4位，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。MAC地址不可设置为FFFF-FFFF-FFFF。

interface-type interface-number：接口类型及编号，可以是Ethernet接口、GE接口或Eth-Trunk接口。本命令中的接口必须是属于指定VLAN的二层端口。

vlan vlan-id：VLAN ID，十进制整数形式，取值范围是1～4094。

ce-vlan ce-vlan：配置VLAN Mapping时，接口收到的用户VLAN ID。十进制整数形式，取值范围是1～4094。

dynamic：动态表项，可以被老化，系统复位或接口板热插拔后丢失。

all：所有表项。

使用指南

使用命令mac-address static mac-address interface-type interface-number vlan vlan-id时，命令中的接口必须是二层端口，并且该端口已经通过port defaut、trunk或者VLAN Stacking方式加入到指定的VLAN中。

使用命令mac-address static mac-address interface-type interface-number vlan vlan-id ce-vlan ce-vlan时，命令中接口必须是二层端口，并且该端口已经通过VLAN Mapping方式加入到指定的VLAN中。

使用命令mac-address static mac-address virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } vlan vlan-id时，命令中的VE接口必须是二层端口，并且该VE端口已经通过port defaut、trunk或者VLAN Stacking方式加入到指定的VLAN中。

undo mac-address { all | dynamic | static } [ interface-type interface-number ] [ vlan vlan-id]命令和undo mac-address { all | dynamic | static } virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/vci ] | vpi/vci } vlan vlan-id命令仅可以在系统视图下执行。

使用实例

# 在配置静态MAC地址表项，该表项中的出端口属于某个VLAN：

  • 当收到目的MAC为0011-2233-4455且属于VLAN10的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以port default方式加入VLAN10。

  • 当收到目的MAC为0011-2233-4466且属于VLAN20的帧时，使用出端口GE1/0/1转发，其中GE1/0/1以trunk方式加入VLAN20。

  • 当收到目的MAC为0011-2233-4477且属于VLAN30的帧时，使用出端口GE1/0/2转发，其中GE1/0/2以stacking方式加入VLAN30。

<Quidway> system-view
[Quidway] vlan 10
[Quidway-vlan10] quit
[Quidway] vlan 20
[Quidway-vlan20] quit
[Quidway] vlan 30
[Quidway-vlan30] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] portswitch
[Quidway-GigabitEthernet1/0/0] port default vlan 10
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/1
[Quidway-GigabitEthernet1/0/1] portswitch
[Quidway-GigabitEthernet1/0/1] port trunk allow-pass vlan 20
[Quidway-GigabitEthernet1/0/1] quit
[Quidway] interface gigabitethernet 1/0/2
[Quidway-GigabitEthernet1/0/2] portswitch
[Quidway-GigabitEthernet1/0/2] port vlan-stacking outside-vlan 100 stack-vlan 30
[Quidway-GigabitEthernet1/0/2] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vlan 10
[Quidway] mac-address static 0011-2233-4466 gigabitethernet 1/0/1 vlan 20
[Quidway] mac-address static 0011-2233-4477 gigabitethernet 1/0/2 vlan 30

# 配置静态MAC地址表项，该表项中的出端口上启用了VLAN mapping功能：

  • 当收到目的MAC为0011-2233-4455且属于VLAN100的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以mapping方式加入VLAN100。当从出端口GE1/0/0转发该帧时，VLAN100将被替换成用户VLAN10。

  • 当收到目的MAC为0011-2233-4466且属于VLAN100的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以mapping方式加入VLAN100。当从出端口GE1/0/0转发该帧时，VLAN100将被替换成用户VLAN11。

<Quidway> system-view
[Quidway] vlan 100
[Quidway-vlan100] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] port vlan-mapping outside-vlan 10 to 20 map-vlan 100
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vlan 100 ce-vlan 10
[Quidway] mac-address static 0011-2233-4466 gigabitethernet 1/0/0 vlan 100 ce-vlan 11

# 根据指定端口删除静态MAC地址表项。

[Quidway] undo mac-address static gigabitethernet 1/0/0

# 根据指定端口和VLAN删除静态MAC地址表项。

[Quidway] undo mac-address static gigabitethernet 1/0/0 vlan 1

# 根据指定端口和PVC删除静态MAC地址表项。

[Quidway] undo mac-address static virtual-ethernet 1/0/0 atm 1/0/0 pvc pvc1
