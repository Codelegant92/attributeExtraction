display ospfv3 error

命令功能

display ospfv3 error命令用来查看OSPFv3的错误信息。如果不指定OSPFv3进程号，该命令将显示所有OSPFv3进程的错误信息。

命令格式

display ospfv3 [ process-id ] error [ lsa ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ospfv3 error [ packet ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

process-id：OSPFv3进程号。整数形式，取值范围是1～65535。

packet：显示最近一次接收的错误报文信息。只有当没有指定进程号时，该参数才可以配置。

lsa：显示LSA错误信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

在检查OSPFv3的故障原因时，可执行此命令获取错误相关信息。用户可以根据这些信息进行OSPFv3的故障诊断等。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

General Packet Errors

标题栏：普通报文错误

Bad packet

解析的报文有错误,包括长度域的校验等

Bad version

OSPF版本号错误，即不是2

Bad area id

收到的报文中的Area ID不匹配（当收到报文的Area ID和本地不一致时，除了报文来自于Area0是正确的，其他不一致的情况都算错误）

Bad virtual link

Vlink收到非法报文

Packet too small

收到的报文长度与IP报文头长度加上报文长度域之和不相等.

Transmit error

向Socket发送报文失败

Interface down

OSPFv3接口Down的次数

Unknown neighbor

对于NBMA、sham-link和v-link网络而言，收到非OSPFv3邻居发来的OSPFv3报文

Hello Packet Errors

标题栏：HELLO报文错误

Hello timer mismatch

Hello时间间隔不一致

Dead timer mismatch

Dead时间间隔不一致

Extern option mismatch

Hello报文的扩展属性不一致

Router id confusion

Route ID重复

Virtual neighbor unknown

报文的Route ID与vlink配置的邻居Route ID不一致

DD Packet Errors

标题栏：DD报文错误

Neighbor state low
下面几种情况会报错：

  • 收到DD报文但邻居状态低于2-way

  • 收到LSR报文但邻居状态低于Exchange

  • 收到LSU报文但邻居状态低于Exchange

  • 收到ACK报文但邻居状态低于Exchange

Extern option mismatch

DD报文选项不匹配

MTU option mismatch

使能了OSPFv3接口的MTU检查，并且收到的DD报文的MTU值大于接收接口的MTU值

Unknown LSA Type

未知的LSA类型

LS Ack packet Errors

标题栏：LS ACK报文错误

Bad ack

收到错误的ACK报文的次数

Duplicate ack

收到重复ACK报文的次数

LS REQ packet Errors

标题栏：LS REQ报文错误

Bad request

协议中的BadRequest事件

LS UPD packet Errors

标题栏：LS UPD报文错误

Received less recent LSA

收到了比本地copy更旧的LSA

Paf License Errors

PAF认证错误

MaxExternal Routes Reached

收到的外部路由达到最大外部路由数量的上限

Max Nbrs Reached

邻居数量达到最大邻居数量的上限
