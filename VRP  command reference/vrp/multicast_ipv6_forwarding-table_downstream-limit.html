multicast ipv6 forwarding-table downstream-limit

命令功能

multicast ipv6 forwarding-table downstream-limit命令用来设置转发表中每条路由出接口的最大数目。

undo multicast ipv6 forwarding-table downstream-limit命令用来恢复出接口的限制数为缺省值。

缺省情况下，转发表中每条路由出接口的最大数目是128。

命令格式

multicast ipv6 forwarding-table downstream-limit limit-value

undo multicast ipv6 forwarding-table downstream-limit

参数说明

limit-value：指定组播转发表中每条路由出接口的最大数目。取值范围是0～128。

使用指南

如果设置的出接口数小于目前存在的出接口数，超出的出接口数必须由组播路由协议删除。同时，路由中不能再增加新的出接口。

请参见IPv4 PIM配置命令。

使用实例

# 设置组播转发表中一条路由的最大出接口数为120。

<Quidway> system-view
[Quidway] multicast ipv6 forwarding-table downstream-limit 120
