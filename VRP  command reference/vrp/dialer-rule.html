dialer-rule

命令功能

dialer-rule命令用来设定控制DCC呼叫发生的数据报文的条件。

undo dialer-rule命令用来取消该设置。

缺省不设定控制DCC呼叫发生的数据报文的条件。

命令格式

dialer-rule dialer-number { protocol-name { deny | permit } | acl acl-number }

undo dialer-rule dialer-number

参数说明

dialer-number：拨号访问组（dialer access group）的序号。整数形式，取值范围是1～255。与DCC接口配置中的dialer-group group-number相对应。

protocol-name：网络协议名，为IP等。

deny：表示禁止相应协议的数据报文。

permit：表示允许相应协议的数据报文。

acl acl-number：拨号访问组对应的acl序号，取值范围是2000～3999。

使用指南

通过此命令设定一个拨号访问组对应的发送控制，通过dialer-group命令将Dialer接口置于一个拨号访问组中，从而完成对一个DCC接口的报文发送控制。

若一个dialer-group找不到对应的dialer-rule，DCC将报文作为Uninteresting报文。

使用实例

# 设置dialer-rule。

<Quidway> system-view
[Quidway] acl number 3001
[Quidway-acl-adv-3001] rule permit ip source 0.0.0.0 255.255.255.255 destination 0.0.0.0 255.255.255.255
[Quidway-acl-adv-3001]  quit
[Quidway] dialer-rule 1 acl 3001
[Quidway] interface serial1/0/0
[Quidway-Serial1/0/0] dialer-group 1
