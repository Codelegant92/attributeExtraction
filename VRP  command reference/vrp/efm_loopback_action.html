efm loopback action

命令功能

efm loopback action命令用来配置当前接口对环回返回的报文的处理方式。

缺省情况下，接口丢弃返回的报文。

命令格式

efm loopback action { forward | discard }

参数说明

forward：指定当前接口转发返回的报文。

discard：指定当前接口丢弃返回的报文。

视图

Ethernet接口视图、GE接口视图

缺省级别

2：配置级

使用指南

只有先使用efm loopback命令使接口进入远端环回状态后，再使用本命令才能配置成功。

通常情况下，配置接口丢弃返回的报文即可。只有当需要检测MAC tunnel的丢包率和时延时，才需要将入隧道侧业务实例绑定的接口配置为转发返回的报文。详细的配置方法，请参考《VRP 配置指南-局域网与城域网接入》。

使用实例

# 配置接口GigabitEthernet1/0/0进行远端环回，且该接口转发返回的报文。

<Quidway> system-view
[Quidway] efm enable
[Quidway] interface gigabitethernet1/0/0
[Quidway-GigabitEthernet1/0/0] efm enable
[Quidway-GigabitEthernet1/0/0] efm loopback start
[Quidway-GigabitEthernet1/0/0] efm loopback action forward
