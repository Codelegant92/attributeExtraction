local-user service-type

命令功能

local-user service-type命令用来设置本地用户的接入类型。

undo local-user service-type命令用来恢复缺省配置。

缺省情况下，本地用户可以使用所有的接入类型。

命令格式

local-user user-name service-type { bind | ftp | ppp | ssh | telnet | terminal | web | x25-pad } *

undo local-user user-name service-type

参数说明

user-name：用户名，字符串形式，长度范围是1～64。

bind：设置用户为绑定认证用户。

ftp：FTP用户。

ppp：通过PPP方式接入的用户。

ssh：SSH用户。

telnet：Telnet用户，通常指网络管理员。

terminal：终端用户。

web：设置用户为WEB认证用户。

x25-pad：X25-PAD用户。

缺省级别

3：管理级

使用指南

系统提供对用户的接入类型管理，即对所有用户配置一定的接入类型，只有用户的接入方式匹配系统配置给该用户的类型，用户才能登录。

使用实例

# 设置本地用户hello@163.net的接入类型为SSH。

<Quidway> system-view
[Quidway] aaa
[Quidway-aaa] local-user hello@163.net service-type ssh
