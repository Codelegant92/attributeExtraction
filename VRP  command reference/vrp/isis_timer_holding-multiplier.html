isis timer holding-multiplier

命令功能

isis timer holding-multiplier命令用来设置宣告邻居失效前IS-IS没有收到的邻居Hello报文数目。

undo isis timer holding-multiplier命令用来恢复缺省设置。

命令格式

isis timer holding-multiplier number [ level-1 | level-2 ]

undo isis timer holding-multiplier [ level-1 | level-2 ]

参数说明

number：宣告邻居失效前IS-IS没有收到的邻居Hello报文数目。整数形式，取值范围是3～1000。缺省值为3。

level-1：宣告邻居失效前IS-IS没有收到的Level-1邻居的Hello报文数目。

level-2：宣告邻居失效前IS-IS没有收到的Level-2邻居的Hello报文数目。

说明：

如果没有指定级别，则默认级别为Level-1和Level-2。

参数level-1和level-2仅在广播接口上是可配置的，而且必须先在接口上使能IS-IS。

使用指南

对广播网络，可以分别设置Level-1和Level-2宣告邻居失效前IS-IS没有收到的Hello报文数目。对点到点链路，由于只有一种Hello报文，所以不需要使用参数level-1和level-2。

实际上，Hello报文数目的设置就是设置Hold down时间（保持时间）。如果路由器在Hold down时间内没有收到对端路由器发送的Hello报文，就认为对端路由器已经失效。Hold down时间基于接口配置，同一路由器的不同接口可以设置不同的值。

说明：

如果通过isis circuit-type命令将广播接口模拟为P2P接口或者通过undo isis circuit-type命令将该接口恢复为广播接口，则宣告邻居失效前IS-IS没有收到的邻居Hello报文的数目恢复为缺省值。

使用实例

# 指定接口GigabitEthernet1/0/0上宣告邻居失效前没有收到的Level-2 Hello报文数目为6。

<Quidway> system-view
[Quidway] interface GigabitEthernet 1/0/0
[Quidway-GigabitEthernet1/0/0] isis timer holding-multiplier 6 level-2
