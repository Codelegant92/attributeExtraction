multicast ipv6 forwarding-table route-limit

命令功能

multicast ipv6 forwarding-table route-limit命令用来配置IPv6组播转发表中允许的最大路由数目。

undo multicast ipv6 forwarding-table route-limit命令用来恢复组播转发表中的最大路由数为缺省值。

缺省情况下，IPv6组播转发表中允许的最大路由数目是8192。

命令格式

multicast ipv6 forwarding-table route-limit limit-value

undo multicast ipv6 forwarding-table route-limit

参数说明

limit-value：指定转发表中允许的最大路由数目。取值范围是0～8192。

使用指南

如果新配置的路由数小于目前存在的数目，超出限制的表项不会被立即删除。该数量限制在已有表项老化后重新向组播转发表中添加表项时才会生效，或者在执行reset multicast ipv6 forwarding-table all命令后生效。

使用实例

# 设置转发表中的最大路由数为1024。

<Quidway> system-view
[Quidway] multicast ipv6 forwarding-table route-limit 1024
