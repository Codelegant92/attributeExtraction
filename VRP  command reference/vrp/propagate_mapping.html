propagate mapping

命令功能

propagate mapping for ip-prefix命令用来使用IP前缀列表对LDP收到的路由进行过滤，只有通过指定前缀列表的路由才会被本LDP用来建立transit LSP。

undo propagate mapping命令用来恢复为缺省配置。

缺省情况下，LDP在建立transit LSP时，不对收到的路由进行过滤。

命令格式

propagate mapping for ip-prefix ip-prefix-name

undo propagate mapping

参数说明

ip-prefix ip-prefix-name：用于路由过滤的IP前缀列表名称。字符串形式，不支持空格，长度范围是1～169。

视图

缺省级别

2：配置级

使用指南

LDP GR期间修改transit LSP建立策略的操作无效。

使用实例

# 配置只有通过前缀列表policy1过滤的路由才可以用于建立Transit LSP。

<Quidway> system-view
[Quidway] mpls ldp
[Quidway-mpls-ldp] propagate mapping for ip-prefix policy1
