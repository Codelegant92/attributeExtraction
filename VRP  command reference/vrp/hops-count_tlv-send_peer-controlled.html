hops-count tlv-send peer-controlled

命令功能

hops-count tlv-send peer-controlled命令用来配置在对端设备不支持环路检测TLV的情况下，不向对端发送环路检测TLV。

undo hops-count tlv-send peer-controlled用来恢复缺省配置。

缺省情况下，设备向对端发送hop count TLV。

命令格式

hops-count tlv-send peer-controlled

undo hops-count tlv-send peer-controlled

参数说明

无

视图

MPLS-LDP-VPN实例视图、MPLS-LDP视图

缺省级别

2：配置级

使用指南

hops-count tlv-send peer-controlled命令只对修改后建立的LSP有效，即该命令的配置不影响已经建立的LSP。对于已经建立的LSP，必须先对所有接口执行undo mpls ldp命令，然后在用户视图下执行reset mpls ldp命令，该命令才能生效。

使用实例

# 在对端设备不支持路径向量TLV的情况下，不向对端发送hop count TLV。

<Quidway> system-view
[Quidway] mpls ldp
[Quidway-mpls-ldp] hops-count tlv-send peer-controlled
