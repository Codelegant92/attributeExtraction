display rrpp statistics

命令功能

display rrpp statistics命令用于显示RRPP报文统计信息。

命令格式

display rrpp statistics domain domain-id [ ring ring-id ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

domain domain-id：RRPP域的ID。整数形式，取值范围是1～8。

ring ring-id：RRPP环的ID。整数形式，取值范围是1～64。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

include：只输出包含匹配字符串的行。

exclude：只输出不包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用此命令可以查看到各端口的各种报文统计数量，可以用来分析定位网络故障。

表1 display rrpp statistics命令输出信息描述

项目

描述

RRPP Ring

RRPP环ID。

Ring Level
RRPP环的级别：

  • 0级代表主环。

  • 1级代表子环。

Node Mode
为设备配置的RRPP节点模式：

  • Master：主节点。

  • Transit：传输节点。

  • Edge：边缘节点。

  • Assistant Edge：辅助边缘节点。

Is Activated

标识RRPP环是否激活。

Primary Port

RRPP环的主端口。

Secondary Port

RRPP环的副端口。

Common Port

RRPP子环边缘节点或辅助边缘节点的公共端口。

Edge Port

RRPP子环边缘节点或辅助边缘节点的边缘端口。

Packet Direct
表示RRPP报文的方向：

  • Send：发送的报文。

  • Rcv：接收的报文。

HEALTH

健康检测报文。由主节点发起，用于对网络进行环路完整性检测。

LINK DOWN

链路Down报文。由传输节点、边缘节点或者辅助边缘节点发起，用于通知主节点环路上有链路状态转为Down，物理环路消失。在主节点上只有接收计数，在其它节点上只有发送计数。

COMMON FDB

刷新FDB(Forwarding Database)报文。由主节点发起，用于通知传输节点、边缘节点和者辅助边缘节点更新各自MAC地址转发表和ARP表。在主节点上只有发送计数，在其它节点上只有接收计数。

COMPLETE FDB

环网恢复刷新FDB报文。由主节点发起，用于通知传输节点、边缘节点和辅助边缘节点更新各自MAC地址转发表和ARP表，同时通知传输节点放开临时阻塞端口。在主节点上只有发送计数，在其它节点上只有接收计数。

EDGE HELLO

主环完整性检查报文。由子环的边缘节点发起，同子环的辅助边缘节点接收。子环通过此报文检查其所在域主环的环路完整性。只有边缘节点上有发送计数，辅助边缘节点上有接收计数。

MAJOR FAULT

主环故障通知报文，由子环的辅助边缘节点发起。当子环的辅助边缘节点在规定时间内收不到边缘节点发送的EDGE-HELLO报文时，向边缘节点发送此报文，报告子环报文在主环上无法透传。只有边缘节点上有接收计数，辅助边缘节点上有发送计数。

Packet Total

表示发送或者接收的报文总数。
