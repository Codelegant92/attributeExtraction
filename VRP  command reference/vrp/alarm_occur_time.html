alarm occur time

命令功能

alarm occur time命令用来配置告警产生的防抖时间。

缺省情况下，告警产生的防抖时间为2500毫秒。

命令格式

alarm occur time time

参数说明

time time：告警产生的防抖时间，取值范围是0～30000，单位是毫秒，步长为500毫秒。如500、1000、1500等，缺省值为2500毫秒。

视图

MA视图、Mac-tunnel隧道视图

缺省级别

2：配置级

使用指南

现如今，802.1ag协议分为draft7草案版本和standard2007标准版本，本命令只支持standard2007标准版本使用。

对于每个MA下的所有RMEP有两个定时器，分别是：

  • 一个是告警产生定时器，周期是用户配置的告警产生防抖时间；

  • 另一个是告警恢复定时器，周期是用户配置的告警恢复防抖时间。

当RMEP检测到告警时，告警定时器开始启动，等到定时器的时间到，上报告警；当每RMEP检测到告警恢复时，告警恢复定时器开始启动，等到定时器时间到，上报告警恢复。

如果RMEP频繁的检测到告警和告警恢复信号，那么就会产生告警振荡。

通过配置告警产生的防抖时间可以抑制告警振荡。

使用实例

# 配置告警产生防抖时间为2000毫秒。

<Quidway> system-view
[Quidway] cfm md md1
[Quidway-md-md1] ma ma1
[Quidway-md-md1-ma-ma1] alarm occur time 2000
