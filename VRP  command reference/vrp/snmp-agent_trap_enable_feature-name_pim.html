snmp-agent trap enable feature-name pim

命令功能

snmp-agent trap enable feature-name pim命令用来将打开PIM相关事件的告警开关。

undo snmp-agent trap enable feature-name pim trap-name命令用来关闭PIM相关事件的告警开关。

命令格式

snmp-agent trap enable feature-name pim { non-excessive all | trap-name { neighbor-loss | invalid-join-prune | rp-mapping-change | interface-election | invalid-register } }

undo snmp-agent trap enable feature-name pim { [ non-excessive | excessive ] all | trap-name { neighbor-loss | invalid-join-prune | rp-mapping-change | interface-election | invalid-register } }

参数说明

non-excessive all：打开PIM模块所有非海量事件的告警开关，包括neighbor-loss、invalid-join-prune、rp-mapping-change、interface-election和invalid-register。

trap-name：打开指定类型事件的告警开关。

neighbor-loss：打开PIM的邻居丢失告警开关。

invalid-join-prune：打开PIM的收到无效的Join/Prune报文告警开关。

rp-mapping-change：打开PIM的RP-Mapping 改变告警开关。

interface-election：打开PIM的接口当选DR告警开关。

invalid-register：打开PIM的收到无效注册报文告警开关。

non-excessive：关闭PIM模块所有非海量事件的告警开关。

excessive：关闭PIM模块所有海量事件的告警开关。

all：关闭PIM模块所有事件的告警开关。

使用指南

目前定义的PIM非海量事件有：邻居丢失、收到无效的Join/Prune报文、RP-Mapping改变、接口当选DR、收到无效注册报文。

当用户希望打开PIM模块所有事件的告警开关时，可以选定non-excessive all批量处理；当用户希望打开某个或几个事件的告警开关时，可以指定trap-name。

使用实例

# 在系统视图下打开PIM邻居丢失告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name pim trap-name neighbor-loss

# 在系统视图下打开PIM所有非海量事件的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name pim non-excessive all

# 在系统视图下关闭PIM邻居丢失告警开关。

<Quidway> system-view
[Quidway] undo snmp-agent trap enable feature-name pim trap-name neighbor-loss
