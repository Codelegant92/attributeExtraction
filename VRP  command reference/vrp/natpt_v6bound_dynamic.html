natpt v6bound dynamic

命令功能

natpt v6bound dynamic命令用来为IPv6侧到IPv4侧的报文配置动态映射。

undo natpt v6bound dynamic命令用来取消已配置的动态映射。

命令格式

natpt v6bound dynamic { acl6 acl6-number | prefix natpt-prefix } { address-group group-number [ no-pat ] | interface interface-type interface-number }

undo natpt v6bound dynamic { acl6 acl6-number | prefix natpt-prefix } { address-group group-number | interface interface-type interface-number }

参数说明

acl6-number：指定IPv6访问控制列表编号。整数形式，取值范围是2000～2999，即为基本ACL。

natpt-prefix：指定NAT-PT前缀，前缀长度只能为96位。

group-number：指定IPv4地址组。整数形式，取值范围是1～31。

interface-type：指定IPv4接口类型。

interface-number：指定IPv4接口号。

no-pat：不需要端口转换。缺省情况下，使能端口转换。

使用指南

可以通过指定ACL6规则或NAT-PT前缀，对需要转换的报文进行标识。

  • 若指定ACL6规则，则只对ACL6规则标识的报文进行NAT-PT转换。

  • 若指定NAT-PT前缀，则只对目的地址中包含前缀的报文进行NAT-PT转换。

可以通过指定IPv4地址组或接口，对报文的源地址进行转换。

  • 若指定IPv4地址组，则从地址组中选取IPv4地址对报文的源地址进行转换。

  • 若指定接口，则用接口的IPv4地址对报文的源地址进行转换。

使用实例

# 将IPv6报文的源地址转换为GE 1/0/0的接口地址，基本ACL6编号为2001。

<Quidway> system-view
[Quidway] natpt v6bound dynamic acl6 2001 interface gigabitethernet 1/0/0

# 如果IPv6报文的目的地址含有NAT-PT前缀2001::/96，其源地址可转换为地址组1中的任意空闲IPv4地址，端口无需转换。

[Quidway]  natpt v6bound dynamic prefix 2001:: address-group 1 no-pat
