reset multicast routing-table

命令功能

reset multicast routing-table命令用来清除组播路由表的路由项。转发表中相应的转发项将同时被删除。

命令格式

reset multicast [ vpn-instance vpn-instance-name | all-instance ] routing-table all

reset multicast [ vpn-instance vpn-instance-name | all-instance ] routing-table { group-address [ mask { group-mask | group-mask-length } ] | source-address [ mask { source-mask | source-mask-length } ] | incoming-interface { interface-type interface-number | register } } *

参数说明

vpn-instance vpn-instance-name：指定VPN实例，vpn-instance-name表示VPN实例的名称。字符串形式，长度范围是1～31。

all-instance：指定所有实例。

all：表示组播路由表中的所有路由项。

group-address：表示组播组地址。点分十进制形式，取值范围是224.0.0.0～239.255.255.255。

mask：指定组播组或源的地址掩码。

group-mask：表示组播组地址掩码。点分十进制形式。

group-mask-length：表示组播组地址掩码长度。整数形式，取值范围是4～32。

source-address：表示组播源地址。点分十进制形式。

source-mask：表示组播源地址掩码。点分十进制形式。

source-mask-length：表示组播源地址掩码长度。整数形式，取值范围是0～32。

incoming-interface：表示组播路由项的入接口。

interface-type interface-number：表示路由器的接口类型和接口号，用来指定具体的接口。

register：表示PIM-SM的注册接口。

使用指南

使用reset multicast routing-table命令：

  • 如果不指定vpn-instance或all-instance，则只删除公网实例的路由表项。

  • 如果指定group-address，则仅删除该组对应的路由表项。

  • 如果指定group-address mask，则删除该组地址范围对应的路由表项。

  • 如果指定source-address，则仅删除该组播源对应的路由表项。

  • 如果指定source-address mask，则删除该源地址范围对应的路由表项。

  • 参数group-address和source-address的次序可以颠倒。必须保证group-address和source-address合法，否则系统会提示输入错误。

  • 如果指定incoming-interface interface-type interface-number，则仅删除上游接口为指定接口的路由表项。

  • 如果指定incoming-interface register，则仅删除上游接口为注册接口的路由表项。

使用实例

# 从公网组播路由表中清除组地址为225.5.4.3的路由项。

<Quidway> reset multicast routing-table 225.5.4.3
