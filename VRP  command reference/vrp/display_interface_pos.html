display interface pos

命令功能

display interface pos命令用来查看POS接口的配置和状态信息。

命令格式

display interface pos [ interface-number ][ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

interface-number：接口编号。如果不指定接口编号，将显示所有POS接口的配置及状态信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。字符串格式，支持空格，区分大小写。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

表1 display interface pos命令输出信息描述

项目

描述

Pos1/0/0 current state
出接口的物理链路状态。

  • UP：接口处于正常启动的状态。

  • DOWN：接口在物理上出现故障。

  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Line protocol current state
链路层协议状态。

  • UP：接口的链路协议处于正常的启动状态。

  • DOWN：接口的链路协议不正常，或者没有在此接口配置IP地址。其中damping表示接口因为频繁震荡而被抑制。

The Maximum Transmit Unit

接口的MTU值。

Internet Address

POS 接口的网络地址，即IP地址。

Hold timer

接口的轮询时间。

Link layer protocol

链路层运行的协议：PPP或HDLC。

LCP

LCP协商状态。

Physical layer is POS over STM-1

传输封装类型：SONET或SDH。

Scramble

是否使能加扰：disabled或enabled。

CRC

CRC校验字长度。

clock

时钟工作模式：slave或master。

loopback

环回功能模式：内部自环、外部环回或不使能。

Hardware

POS接口的类型。有155M、2.5G和10G等类型。

section layer:
段。

  • B1表示进行比特间插奇偶校验后错误的报文，其检测时对整个STM-N帧信号进行传输误码检测

line layer:
复用。

  • B2表示比特间插奇偶校验N×24位的（BIP-N×24）字节，其检测是对STM-N帧中的每一个STM-1帧的传输误码情况进行监测。

  • REI（Remote Error Indication）表示线路远端错误指示。

path layer:

通道。B3：通道BIP-8，B3字节负责监测VC4在STM-N帧中传输的误码性能，也就是监测140Mbit/s的信号在STM—N帧中传输的误码性能。监测机理与B1、B2相类似，不过B3是对VC4帧进行BIP-8校验
