filter-policy export（MBGP）

命令功能

filter-policy export命令用来配置对发布的路由进行过滤。只有通过过滤器的路由才被MBGP发布。

undo filter-policy export命令用来取消对发布的路由进行过滤。

缺省情况下，不对发布的路由信息进行过滤。

命令格式

filter-policy { basic-acl-number | ip-prefix ip-prefix-name } export [ protocol [ process-id ] ]

undo filter-policy export [ protocol [ process-id ] ]

参数说明

basic-acl-number：基本ACL序号。

ip-prefix ip-prefix-name：指定过滤路由的地址前缀列表名。

protocol：指定发布路由信息的协议。protocol可以是direct、IS-IS、OSPF、RIP、static等单播路由协议。

process-id：表示单播路由协议的进程号。当单播路由协议为IS-IS、OSPF或RIP时，需要指定process-id。

缺省级别

2：配置级

使用指南

配置filter-policy export命令时：

  • 如果指定protocol参数，将只对引入的这种协议产生的路由进行过滤，对引入的其它协议产生的路由不受影响。

  • 如果没有指定的protocol参数，对引入的任何一个协议产生的路由都要进行过滤。

使用实例

# 使用编号为2001的ACL对所有组播BGP发布的路由信息进行过滤。

<Quidway> system-view
[Quidway] bgp 100
[Quidway–bgp] ipv4-family multicast
[Quidway-bgp-af-multicast] filter-policy 2001 export
