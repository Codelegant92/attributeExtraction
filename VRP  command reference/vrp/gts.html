gts

命令功能

gts命令用来为行为配置流量整形。

undo gts命令用来取消流量整形配置。

命令格式

gts cir cir [ cbs cbs [ ebs ebs [ queue-length queue-length ] ] ]

undo gts

参数说明

cir：承诺信息速率。

cir：流量的平均速率，取值范围是8000～155000000bps。

cbs：承诺突发尺寸。

cbs：承诺突发尺寸，每个时间间隔可发送的位数，取值范围是15000～155000000bit。cbs的取值应大于等于cir取值的1/2，当cir>30000bit/s时，缺省取值是cir的1/2，当cir<30000bit/s时，缺省取值是15000。

ebs：过度突发尺寸，取值范围是0～155000000bit，缺省为0。

ebs：过度突发尺寸的值，取值范围是0～155000000bit，缺省取值是0，即只采用一个令牌桶监管。

queue-length：队列的最大长度，取值范围是1～1024，缺省为50。

使用指南

接口上应用的策略中使用gts时，只能应用到接口的出方向。

接口上应用配置了GTS的策略将导致原有的qos gts命令失效。

如果多次使用该命令在同一个策略的类上配置，最后一次的配置将覆盖前面的配置。

使用实例

# 为行为配置使用GTS，正常流量为38400bit，在第一时间可以有2倍于正常流量的突发流量通过，以后速率小于等于38400bit的流量正常时正常发送，速率大于38400bit时，将进入队列缓存，缓存队列长度为100。
