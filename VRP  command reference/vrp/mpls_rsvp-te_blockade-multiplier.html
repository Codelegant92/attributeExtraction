mpls rsvp-te blockade-multiplier

命令功能

mpls rsvp-te blockade-multiplier命令用来设置阻塞状态超时倍数。

undo mpls rsvp-te blockade-multiplier用来恢复缺省设置。

缺省情况下，阻塞状态超时倍数为4。

命令格式

mpls rsvp-te blockade-multiplier number

undo mpls rsvp-te blockade-multiplier

参数说明

number：阻塞状态倍数，取值范围是3～255。

视图

MPLS视图

缺省级别

2：配置级

使用指南

ResvErr消息在一个节点创建阻塞状态以解决“killer reservation”问题，以转发或创建更小的请求。

阻塞状态超时时间Blockade_Expired_Time＝Kb×refresh-time。其中：

  • “Kb”代表阻塞状态超时倍数

  • “refresh-time”代表节点的刷新周期

假设有一个资源预留请求，在某些节点上没有通过准入控制。有时可能不希望立即删除这个请求，但这个请求也不应该阻止其他资源预留请求。这种情况下，节点将进入阻塞状态（Blockade State），在下游节点生成BSB（Blockade State Block）。如果在阻塞状态超时时间内该节点没有再收到资源预留请求，BSB中相应的状态被删除。

使用实例

# 设定阻塞状态时间倍数为5。

<Quidway> system-view
[Quidway] mpls
[Quidway-mpls] mpls rsvp-te blockade-multiplier 5
