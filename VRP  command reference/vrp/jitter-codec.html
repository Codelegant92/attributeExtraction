jitter-codec

命令功能

jitter-codec命令用来配置NQA Jitter测试例的模拟语音测试的编码类型。

undo jitter-codec用来删除NQA Jitter测试例的模拟语音测试的编码类型。

缺省情况下，NQA Jitter测试例为非模拟语音测试。

命令格式

jitter-codec { g711a | g711u | g729a }

参数说明

g711a：指定模拟语音测试编码类型为g711alaw。

g711u：指定模拟语音测试编码类型为g711ulaw。

g729a：指定模拟语音测试编码类型为g729a。

视图

NQA视图

缺省级别

2：配置级

使用指南

jitter-codec命令只能在Jitter测试例中配置。对于接口板发包的Jitter试例中，可以配置该命令，但在测试例执行时，不会对该值进行统计。

g711a、g711u和g729a指的是VOIP应用中，针对语音模拟信号和数字信号之间转换，所采用的编解码方式。用户可以根据被测试的VOIP应用中，实际编解码方式的不同，而选用不同的编码方式。

使用实例

# 配置模拟语音测试编码类型为g711alaw。

<Quidway>  system-view
[Quidway]  nqa test-instance admin jitter
[Quidway-nqa-admin-Jitter]  jitter-codec g711a
