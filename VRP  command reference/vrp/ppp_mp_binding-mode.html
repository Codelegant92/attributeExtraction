ppp mp binding-mode

命令功能

ppp mp binding-mode命令用来设置MP捆绑的条件。

undo ppp mp binding-mode命令用来恢复MP捆绑条件的缺省值。

缺省情况下，同时根据PPP的验证用户名和终端标识符进行MP捆绑。

命令格式

ppp mp binding-mode { authentication | both | descriptor }

undo ppp mp binding-mode

参数说明

authentication：根据PPP的验证用户名进行MP捆绑。

both：同时根据PPP的验证用户名和终端标识符进行MP捆绑。

descriptor：根据PPP的终端标识符进行MP捆绑。

视图

虚拟接口模板视图、Dialer接口视图

缺省级别

2：配置级

使用指南

用户名是指PPP链路进行PAP或CHAP验证时所接收到的对端用户名；终端标识符是用来唯一标识一台路由器的标志，是指进行LCP协商时所接收到的对端终端标识符。系统可以根据接口接收到的用户名或终端标识符来进行MP捆绑，将拥有相同用户名或终端标识符的接口捆绑在一起。

使用实例

# 仅根据PPP验证的用户名进行MP捆绑。

<Quidway> system-view
[Quidway] interface virtual-template 10
[Quidway-Virtual-Template10] ppp mp binding-mode authentication
