mac-address static

命令功能

mac-address static命令用来配置静态MAC地址表项。

undo mac-address命令用来删除的MAC地址表项。

缺省情况下，系统没有配置任何静态MAC地址表项。

命令格式

mac-address static mac-address interface-type interface-number { vlan vlan-id [ ce-vlan ce-vlan ] | vsi vsi-name [ pe-vid pe-vid [ ce-vid ce-vid ] ] }

mac-address static mac-address virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } { vlan vlan-id | vsi vsi-name }

undo mac-address { all | dynamic | static }

undo mac-address { all | dynamic | static } virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } { vlan vlan-id | vsi vsi-name }

参数说明

static：静态表项，不被老化。配置并保存后，系统复位或接口板热插拔表项不丢失。

mac-address：单播MAC地址，格式为H-H-H。其中H为4位的十六进制数，可以输入1～4位，如00e0、fc01。当输入不足4位时，表示前面的几位为0，如：输入e0，等同于00e0。MAC地址不可设置为FFFF-FFFF-FFFF。

interface-type interface-number [ .subinterface-number ]：接口类型和接口编号，可以是VE接口、Ethernet接口、Ethernet子接口、GE接口、GE子接口、Eth-Trunk接口和Eth-Trunk子接口。

vlan vlan-id：配置VLAN Mapping时，Mapping后的VLAN ID，十进制整数形式，取值范围是1～4094。

vsi vsi-name：指定的VSI实例的名称。字符串形式，不支持空格，区分大小写，取值范围是1～31。

ce-vlan ce-vlan：配置VLAN Mapping时，接口收到的用户VLAN ID，十进制整数形式，取值范围是1～4094。

pe-vid pe-vid：用户外层VLAN的Tag值，十进制整数形式，取值范围是1～4094。

ce-vid ce-vid：用户内层VLAN的Tag值，十进制整数形式，取值范围是1～4094。

virtual-ethernet interface-number：VE接口编号。配置VE接口时，必须配置与该VE接口关联的ATM PVC。VE接口在本命令中必须是属于指定VLAN的二层端口。

atm interface-number：ATM接口编号。该ATM接口的PVC与VE接口必须配置了映射关系。

pvc-name：PVC名，在ATM接口上保持唯一。最长为16个字符，不区分大小写，并且不能是合法的VPI/VCI值对，例如不允许“1/20”作为PVC名。

vpi/vci：vpi为ATM网络虚路径标识（Virtual Path Identifier），取值范围是0～255。vci为ATM网络虚通道标识（Virtual Channel Identifier），取值范围是0～2047。通常，vci取值0到31保留用于特定用途，建议用户不要使用。vpi与vci不能同时为0。

使用指南

同一物理接口可以同时属于多个VLAN，而多个VLAN接口可以与同一个VSI绑定。即，一个物理接口上关联的某个VSI可以绑定多个VLAN，而一个VLAN接口绑定的某个VSI可以关联多个物理接口。因此，在为绑定到VLAN接口的VSI，配置MAC地址静态表项时，需要同时指定物理接口和VLAN接口。

  • 使用命令mac-address static mac-address interface-type interface-number vlan vlan-id时，命令中接口必须是二层接口，并且该接口已经通过port defaut、trunk或者VLAN Stacking方式加入到指定的VLAN中。
  • 使用命令mac-address static mac-address interface-type interface-number vlan vlan-id ce-vlan ce-vlan时，命令中接口必须是二层接口，并且该接口已经通过VLAN Mapping方式加入到指定的VLAN中。
  • 使用命令mac-address static mac-address interface-type interface-number vsi vsi-name时，命令中接口必须是三层接口，并且该三层接口已经和VSI绑定。
  • 使用命令mac-address static mac-address interface-type interface-number vsi vsi-name pe-vid pe-vid时，命令中接口必须是子接口，并且该子接口绑定了VSI或者该子接口配置了dot1q终结和绑定了VSI。
  • 使用命令mac-address static mac-address interface-type interface-number vsi vsi-name pe-vid pe-vid ce-vid ce-vid时，命令中接口必须是子接口，并且该子接口配置了QinQ终结和绑定了VSI。

undo mac-address { all | dynamic | static }和undo mac-address { all | dynamic | static } virtual-ethernet interface-number atm interface-number pvc { pvc-name [ vpi/ vci ] | vpi/vci } { vlan vlan-id | vsi vsi-name }命令只能在系统视图下执行。

使用实例

# 在配置静态MAC地址表项，该表项中的出端口属于某个VLAN：

  • 当收到目的MAC为0011-2233-4455且属于VLAN10的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以port default方式加入VLAN10。

  • 当收到目的MAC为0011-2233-4466且属于VLAN20的帧时，使用出端口GE1/0/1转发，其中GE1/0/1以trunk方式加入VLAN20。

  • 当收到目的MAC为0011-2233-4477且属于VLAN30的帧时，使用出端口GE1/0/2转发，其中GE1/0/2以stacking方式加入VLAN30。

<Quidway> system-view
[Quidway] vlan 10
[Quidway-vlan10] quit
[Quidway] vlan 20
[Quidway-vlan20] quit
[Quidway] vlan 30
[Quidway-vlan30] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] portswitch
[Quidway-GigabitEthernet1/0/0] port default vlan 10
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/1
[Quidway-GigabitEthernet1/0/1] portswitch
[Quidway-GigabitEthernet1/0/1] port trunk allow-pass vlan 20
[Quidway-GigabitEthernet1/0/1] quit
[Quidway] interface gigabitethernet 1/0/2
[Quidway-GigabitEthernet1/0/2] portswitch
[Quidway-GigabitEthernet1/0/2] port vlan-stacking outside-vlan 100 stack-vlan 30
[Quidway-GigabitEthernet1/0/2] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vlan 10
[Quidway] mac-address static 0011-2233-4466 gigabitethernet 1/0/1 vlan 20
[Quidway] mac-address static 0011-2233-4477 gigabitethernet 1/0/2 vlan 30

# 配置静态MAC地址表项，该表项中的出端口上启用了VLAN mapping功能：

  • 当收到目的MAC为0011-2233-4455且属于VLAN100的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以mapping方式加入VLAN100。当从出端口GE1/0/0转发该帧时，VLAN100将被替换成用户VLAN10。

  • 当收到目的MAC为0011-2233-4466且属于VLAN100的帧时，使用出端口GE1/0/0转发，其中GE1/0/0以mapping方式加入VLAN100。当从出端口GE1/0/0转发该帧时，VLAN100将被替换成用户VLAN11。

<Quidway> system-view
[Quidway] vlan 100
[Quidway-vlan100] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] port vlan-mapping outside-vlan 10 to 20 map-vlan 100
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vlan 100 ce-vlan 10
[Quidway] mac-address static 0011-2233-4466 gigabitethernet 1/0/0 vlan 100 ce-vlan 11

# 为名称为abc的VSI添加一个静态MAC地址表项，当收到目的MAC为0011-2233-4455的帧时，在名称为abc的VSI中转发。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0 vsi abc

# 接口GE1/0/0.1配置了dot1q终结子接口接入VPLS。在该子接口上配置静态MAC地址表项，当收到目的MAC为0011-2233-4455且属于VLAN10的帧时，去掉该帧的Tag后，在名称为abc的VSI中转发该帧。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] mode user-termination
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 dot1q-termination
[Quidway-GigabitEthernet1/0/0.1] dot1q termination vid 10
[Quidway-GigabitEthernet1/0/0.1] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0.1] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0.1 vsi abc pe-vid 10

# 接口GE1/0/0.1配置了QinQ终结子接口接入VPLS。在该子接口上配置静态MAC地址表项，当收到目的MAC为0011-2233-4455且带有两层Tag（外层Tag为10，内层Tag为100）的帧时，去掉该帧的两层Tag后，在名称为abc的VSI中转发该帧。

<Quidway> system-view
[Quidway] vsi abc static
[Quidway-vsi-abc] pwsignal ldp
[Quidway-vsi-abc-ldq] vsi-id 1
[Quidway-vsi-abc-ldq] quit
[Quidway-vsi-abc] quit
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] mode user-termination
[Quidway-GigabitEthernet1/0/0] quit
[Quidway] interface gigabitethernet 1/0/0.1
[Quidway-GigabitEthernet1/0/0.1] control-vid 1 qinq-termination
[Quidway-GigabitEthernet1/0/0.1] qinq termination pe-vid 10 ce-vid 100 to 200
[Quidway-GigabitEthernet1/0/0.1] l2 binding vsi abc
[Quidway-GigabitEthernet1/0/0.1] quit
[Quidway] mac-address static 0011-2233-4455 gigabitethernet 1/0/0.1 vsi abc pe-vid 10 ce-vid 100
