domain (RRPP环组视图)

命令功能

domain命令用来配置RRPP环组内的子环。

undo domain命令用来删除RRPP环组中的子环。

命令格式

domain domain-id ring { ring-id1 [ to ring-id2 ] } &<1-10>

undo domain domain-id [ ring { ring-id1 [ to ring-id2 ] } &<1-10> ]

参数说明

domain-id：RRPP域的ID，整数形式，取值范围为1～8。

ring-id：RRPP子环ID，整数形式，取值范围为1～64。

视图

RRPP环组视图

缺省级别

2：配置级

使用指南

一条domain命令中，最多可以使用10次to形式输入10个子环范围。

每个环组最多只能加入15个子环。

注意事项：

  • 只有子环边缘节点或辅助边缘节点才能加入环组。加入同一环组的子环边缘节点都配置在同一台设备上。同样，辅助边缘节点也都配置在同一台设备上。

  • 一个子环只能属于一个环组。

  • 一个环组内的所有子环所在节点应该具有相同的类型：都是边缘节点或者都是辅助边缘节点。

  • 如果需要将激活的环加入环组中，必须先在辅助边缘节点将环加入环组，再在边缘节点将环加入环组。

  • 如果需要将激活的环从环组中删除，必须先在边缘节点将环从环组中删除，再在辅助边缘节点将环从环组中删除。

  • 在同一环组中，所有子环所对应主环的链路必须相同，如果主环链路不相同，环组不能正常运行。

  • 边缘节点环组和对应的辅助边缘节点环组的配置和激活状态必须相同。

使用实例

# 为环组1配置子环。

<Quidway> system-view
[Quidway] rrpp ring-group 1
[Quidway-rrpp-ring-group1] domain 1 ring 1 to 3 5
