hello-option lan-delay（IPv6）

命令功能

hello-option lan-delay命令用来在共享网段上配置传输Prune消息的延迟时间。

undo hello-option lan-delay命令用来恢复配置参数的缺省值。

缺省情况下，在共享网段上传输Prune消息的延迟时间是500毫秒。

命令格式

hello-option lan-delay interval

undo hello-option lan-delay

参数说明

interval：表示当前路由器从收到下游发送的Prune剪枝消息到实施剪枝动作的延迟时间。整数形式，取值范围是1～32767，单位是毫秒。

视图

IPv6 PIM视图

缺省级别

2：配置级

使用指南

当相同链路多台路由器上的剪枝时延不一致时，协商选取其中的最大值。

请参见IPv4 PIM配置命令。

说明：
如果设置的时延太短，在下游路由器还未否决邻居剪枝前，上游路由器就停止转发组播报文。请小心配置。

此命令与接口视图下的pim ipv6 hello-option lan-delay含义是相同的，只是视图不同。缺省情况下，如果接口上没有配置，取IPv6 PIM视图下配置的值；否则取接口视图下配置的值。

使用实例

# 在公网实例中，配置传递Prune剪枝消息的延迟时间为200毫秒。

<Quidway> system-view
[Quidway] pim-ipv6
[Quidway-pim6] hello-option lan-delay 200
