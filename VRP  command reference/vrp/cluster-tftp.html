cluster-tftp

命令功能

cluster-tftp命令用来从HGMP集群公用的TFTP服务器下载文件，或者向HGMP集群公用的TFTP服务器上传文件。

命令格式

cluster-tftp { get get-source-file [ get-destination-file ] | put put-source-file [ put-destination-file ] }

参数说明

get get-source-file：从HGMP集群公用的TFTP服务器下载文件。get-source-file表示被下载文件的文件名，字符串形式，不支持空格，不区分大小写，长度范围是1～46。

get-destination-file：指定文件被下载到本设备后所保存的文件名。字符串形式，不支持空格，不区分大小写，长度范围是1～56。如果不指定本参数，则文件被下载到本设备后所保存的文件名将与被下载文件的文件名一致。

put put-source-file：向HGMP集群公用的TFTP服务器上传文件。put-source-file表示被上传文件的文件名，字符串形式，不支持空格，不区分大小写，长度范围是1～46。

put-destination-file：文件被上传到TFTP服务器后所存的文件名。字符串形式，不支持空格，不区分大小写，长度范围是1～56。如果不指定本参数，则文件被上传到TFTP服务器后所保存的文件名将与被上传文件的文件名一致。

使用指南

本命令只能在成员交换机上执行。

在执行本命令前，需要先在命令交换机上通过tftp-server命令配置本HGMP集群的公用TFTP服务器的IP地址。

对于NE40E、NE80E、NE5000E和CX600产品，目前暂不支持HGMP集群的TFTP功能。

使用实例

# 从集群公用的TFTP服务器下载文件名为test.txt的文件。下载到本设备后，存储的文件名为hgmp.txt。

<HUAWEI_1.Quidway> cluster-tftp get test.txt hgmp.txt
