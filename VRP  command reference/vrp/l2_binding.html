l2 binding

命令功能

l2 binding命令用于将二层接口绑定到VSI实例。

undo l2 binding命令用于撤消接口到VSI实例的绑定。

缺省情况下，接口没有与任何VSI实例进行绑定。

命令格式

l2 binding vsi vsi-name

undo l2 binding vsi vsi-name

参数说明

vsi vsi-name：指定用于与接口绑定的VSI实例的名称。字符串形式，不支持空格，区分大小写，取值范围是1～31。

视图

VLAN接口视图、VE接口视图、Ethernet接口视图、Ethernet子接口视图、GE接口视图、GE子接口视图、Eth-Trunk接口视图、Eth-Trunk子接口视图

缺省级别

2：配置级

使用指南

配置VPLS时，在PE上接入CE时，需要使用l2 binding命令将连接CE的接口与相应的VSI实例绑定。

说明：
  • 当子接口配置了与VSI的绑定关系时，必须先删除子接口和VSI的绑定关系，才能删除子接口。
  • 当接口绑定VSI后，在接口视图下可以配置MTU，但是不生效。PW信令以VSI视图下配置的MTU作为参数进行PW MTU的协商。

在VPLS应用中，不同的CE通过VSI透明地连接在同一局域网网段上，因此，这些CE不能配置相同的IP地址。此外，PE的CE侧接口下不能同时配置l2 binding命令和ip address命令，即：如果接口已经配置了IP地址，那么该接口就不能再与VSI实例绑定；如果接口已经和VSI实例绑定，那么该接口就不能再配置IP地址。同时，PE与CE相连的接口也不要配置与所连接CE相同的网段地址，否则本端CE可能学习到错误的ARP表项，从而导致同一个VSI的CE之间流量丢失。

使用实例

# 将以太网子接口绑定到VSI实例中。

<Quidway> system-view
[Quidway] interface ethernet 2/0/1.1
[Quidway-Ethernet2/0/1.1] l2 binding vsi company1

# 将VLAN接口绑定到VSI实例中。

<Quidway> system-view
[Quidway] interface vlanif 2
[Quidway-Vlanif2] l2 binding vsi company2
