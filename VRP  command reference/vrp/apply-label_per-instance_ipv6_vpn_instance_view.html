apply-label per-instance(IPv6 VPN实例视图)

命令功能

apply-label per-instance命令用来配置当前IPv6 VPN实例下所有发往对端PE的路由都使用本VPN实例的标签值。

undo apply-label per-instance命令用来去使能此特性。

缺省情况下，IPv6 VPN实例下的路由按照每条路由分配一个标签的方式申请标签。

命令格式

apply-label per-instance

undo apply-label per-instance

参数说明

无

视图

IPv6 VPN实例视图

缺省级别

2：配置级

使用指南

注意：
  • 由于改变标签分配方式将导致VPN实例路由重发，请慎重使用apply-label per-instance命令及其undo形式。
  • 对于运营商的运营商（Carriers' Carrier）组网，在目前的版本中，只有二级运营商的PE上支持基于VPN的MPLS标签分配；一级运营商的PE上只支持基于路由的MPLS标签分配。

通常情况下，标签的分配方式是每条路由一个标签（one label per route）。当路由数量比较多时，设备入标签映射表ILM（Incoming Label Map）需要维护的in-segment表项也会增多，从而提高了对设备容量的要求。为了减少ILM表项数目，可以配置apply-label per-instance命令，为每个VPN实例分配一个标签，VPN实例中的所有路由都使用同一个标签。

执行apply-label per-instance命令后，如果之前没有发送过本IPv6 VPN实例的路由，则所有路由使用本IPv6 VPN实例的标签；如果之前发送过本IPv6 VPN实例的路由，但路由标签不是基于IPv6 VPN的，则路由释放原先申请的标签，使用现有IPv6 VPN实例的标签重新发布，即，执行这条命令后，重新发送本IPv6 VPN实例内的所有路由。

执行undo apply-label per-instance命令后，如果之前的路由发布是按照每IPv6 VPN每标签方式，则按照每条路由分配一个标签的方式重新申请标签，并重新发送路由。

使用实例

# 对IPv6 VPN实例try只分配一个标签来发布路由。

<Quidway> system-view
[Quidway] ipv6 vpn6-instance try
[Quidway-vpn-instance-try] apply-label per-instance
