dhcp snooping nomatch-packet（系统视图、接口视图）

命令功能

dhcp snooping nomatch-packet命令用来配置对于在DHCP Snooping绑定表中查不到匹配项的IP、ARP报文的处理方式。

如果在DHCP Snooping绑定表中查不到与该报文对应的绑定表项，那么就可以根据用户的配置决定是丢弃报文还是转发报文。

缺省情况下，对于在DHCP Snooping绑定表中查不到匹配项的IP、ARP报文进行转发处理。

命令格式

dhcp snooping nomatch-packet { arp | ip } action { forward | discard }

参数说明

arp：处理arp报文。

ip：处理IP报文。

forward：转发报文。

discard：丢弃报文。

缺省级别

2：配置级

使用指南

对于配置接口上的ARP/IP报文检查，如果在没有对应的绑定表项时要丢弃报文，必须配置全局下的转发行为为discard，接口下的转发行为也要配置为discard；如果全局或者接口下的转发行为有一个配置为forward，则表示没有对应的绑定表项时转发该报文。

如果用户侧网络配置了很多静态IP地址，可以配置这些分配的IP地址的静态绑定表项，防止用户盗用静态IP地址。但是如果静态IP用户比较多可以使用该命令对静态IP用户没有匹配的报文不检查而进行转发，但是这样无法隔离盗用静态IP地址的非法用户。

使用实例

# 设置丢弃没有匹配的绑定表项的ARP报文。

<Quidway> system-view
[Quidway] dhcp snooping nomatch-packet arp action discard
