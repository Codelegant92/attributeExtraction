alarm

命令功能

alarm entry-number用来配置NQA告警。

缺省情况下，没有配置任何的NQA告警。

命令格式

alarm entry-number { lostpacket-ratio | jitter-average | jitter-ds-average | jitter-sd-average | packetloss-ds | packetloss-sd | rtt-average } { absolute | delta } { falling-threshold threshold-value11 event-entry1 | rising-threshold threshold-value2 event-entry2 } * [description ]

参数说明

entry-number：添加或删除的告警行号。整数形式，取值范围是1～65535。

lostpacket-ratio：丢包率。在所有的测试例中都可以统计到此值。

jitter-average：Jitter的平均值。只针对Jitter测试例、ICMPJitter测试例、PathJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例有此统计值。

jitter-ds-average：目的到源的Jitter平均值。只针对Jitter测试例、ICMPJitter测试例、PathJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例有此统计值。

jitter-sd-average：源到目的的Jitter平均值。只针对Jitter测试例、ICMPJitter测试例、PathJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例有此统计值。

packetloss-ds：目的到源的丢包的数值。只针对Jitter测试例、ICMPJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例有此统计值。

packetloss-sd：源到目的的丢包的数值。只针对Jitter测试例、ICMPJitter测试例、使能接口板发包的Jitter测试例、使能接口板发包的ICMPJitter测试例有此统计值。

rtt-average：往返时间的平均值。在所有的测试例中都可以统计到此值。

absolute：绝对采样。绝对采样是指本次得到的采样值。

delta：相对采样。相对于上一次采样变换的值。

falling-threshold threshold-value1：下限阈值。整数形式，取值范围是0～2147483647。

event-entry1：上限阈值相关的事件表号。整数形式，取值范围是0～65535。

rising-threshold threshold-value2：上限阈值。整数形式，取值范围是0～2147483647。

event-entry2：下限阈值相关的事件表号。整数形式，取值范围是0～65535。

description：关于该告警的相关描述。字符串形式，长度范围是1～127。

使用指南

目前只支持绝对采样。

只有在通过snmp-agent target-host trap address udp-domain ip-address配置发送告警的主机后，NQA产生的告警才能发送到网管端。

使用实例

# 配置丢包率超过配置的阈值后发送告警。

<Quidway> system-view
[Quidway] nqa test-instance admin jitter
[Quidway-nqa-admin-jitter] alarm 10 lost-packet-ratio 2 falling-threshold 10 10 rising-threshold  100 100
