unknown-frame

命令功能

unknown-frame命令用于指定对接收到的不可知帧的处理方式。

缺省情况下，对于不可知帧，在单播和多播的情况下，处理方式都是广播。

命令格式

unknown-frame { unicast | multicast } { broadcast | drop | local-handle }

参数说明

unicast：用于指定类型为单播不可知帧。

multicast：用于指定类型为多播不可知帧。

drop：用于指定对接收到的不可知帧的处理方式是丢弃。

local-handle：用于指定对接收到的不可知帧的处理方式是本地处理。

broadcast：用于指定对接收到的不可知帧的处理方式是广播。

使用指南

对于Kompella方式的VPLS，使用unknown-frame命令前，需要先执行pwsignal命令和route-distinguisher（VSI-BGP视图）命令；对于Martini方式的VPLS，使用unknown-frame命令前，需要先执行pwsignal命令和vsi-id命令。

不可知帧是指VSI实例接收到数据帧后，无法在其MAC表中找到与该数据帧的目的地址相对应的表项。

执行unknown-frame命令后，对于不可知帧，可分为单播和多播两种类型。处理方式包括三种，分别为丢弃、本地处理和广播方式。

使用实例

# 配置当前VSI实例的单播不可知帧的处理方式为丢弃。

<Quidway> system-view
[Quidway] vsi company1 auto
[Quidway-vsi-company1] pwsignal bgp
[Quidway-vsi-company1-bgp] route-distinguisher 100:1
[Quidway-vsi-company1-bgp] quit
[Quidway-vsi-company1] unknown-frame unicast drop
