timer hold（POS口与串行接口视图）

命令功能

timer hold命令用来设置同步串口向对端发送keepalive协商报文的间隔。

undo timer hold命令用来恢复间隔的缺省值。

缺省情况下，间隔是10秒。

命令格式

timer hold hold-time

undo timer hold

参数说明

hold-time：时延。整数形式，取值范围是0～32767，单位是秒。

视图

POS口与串行接口视图

缺省级别

2：配置级

使用指南

当链路状态变为Up后，本端会向对端发送Keepalive协商报文来监控链路状态是否为Up。

链路两端设备的轮询时间应设为相同的值。

如果将两端的轮询时间的间隔都设为0，则禁止链路检测功能。此时，本端检测不到对端链路的状态变化。本端只要物理状态是Up，链路协议状态就会一直处于Up状态。

使用实例

# 设置串口Serial1/0/0:0向对端发送keepalive报文的间隔为5秒。

<Quidway> system-view
[Quidway] interface serial 1/0/0:0
[Quidway-Serial1/0/0:0] timer hold 5
