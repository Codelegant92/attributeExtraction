vpls bgp encapsulation

命令功能

vpls bgp encapsulation命令用于指定本端PE设备接收到封装类型是19的VPLS报文后，对该报文进行重新封装的方式。

VPLS报文的封装方式分为Ethernet和VLAN两种类型。

缺省情况下，系统对接收到的封装类型是19的VPLS报文使用VLAN方式进行重新封装。

命令格式

vpls bgp encapsulation { ethernet | vlan }

参数说明

ethernet：指定对接收到的封装类型是19的VPLS报文使用以太方式进行重新封装。

vlan：指定对接收到的封装类型是19的VPLS报文使用VLAN方式进行重新封装。

使用指南

vpls bgp encapsulation命令只在设备互通的时候使用。

对于Kompella方式的VPLS，最新的RFC上规定VPLS的封装类型是19。使用该命令后，当接收到封装类型是19的VPLS报文后，系统会根据用户的配置对VPLS报文进行重新封装的方式，以实现使用VRP的华为数据通信设备和友商设备的互通。

vpls bgp encapsulation { ethernet | vlan }命令需要和ignore-mtu-match命令相互配合使用，才能实现和友商设备的互通。

AC上的报文封装方式由用户接入方式决定，用户接入方式可以分为两种:VLAN接入和Ethernet接入。其含义如下：

  • VLAN接入：CE发送到PE或PE发送到CE的以太网帧头带有一个VLAN TAG，该TAG是一个ISP为了区分用户而要求用户打上的“服务定界符”。我们把这个作为服务定界符的TAG称为P-TAG（Provider-TAG）。

  • Ethernet接入：CE发送到PE或PE发送到CE的以太网帧头中没有带服务定界符，如果此时帧头中有VLAN TAG，则说明它只是用户报文的内部VLAN TAG，对于PE设备没有意义。我们把这种用户内部VLAN的TAG称为U-TAG(User-TAG)。

使用实例

# 配置对接收到的封装类型是19的VPLS报文使用以太方式进行重新封装。

<Quidway> system-view
[Quidway] mpls lsr-id 1.1.1.1
[Quidway] mpls
[Quidway-mpls] quit
[Quidway] mpls l2vpn
[Quidway] quit
[Quidway] vpls bgp encapsulation ethernet
