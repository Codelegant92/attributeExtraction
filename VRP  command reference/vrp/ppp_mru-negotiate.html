ppp mru-negotiate

命令功能

ppp mru-negotiate命令用来使能链路控制层进行PPP MRU值的协商。

undo ppp mru-negotiate命令用来恢复为缺省的协商方式。

缺省情况下，链路控制层进行PPP IPv4 MRU值的协商。

命令格式

ppp mru-negotiate { ipv4 | ipv6 }

undo ppp mru-negotiate

参数说明

ipv4：表示PPP MRU进行IPv4 MRU的协商。

ipv6：表示PPP MRU进行IPv6 MRU的协商。

缺省级别

2：配置级

使用指南

PPP链路两端的MRU协商必须相同。

如果两个接口的MTU值不同，PPP MRU协商结果将采用较小的MTU值作为链路的MTU值。

说明：

只有在接口下配置了IPv6协议，才可使用ppp mru-negotiate ipv6命令来完成对IPv6报文的MTU协商。

有关IPv6协议的具体配置，请参见《VRP 配置指南 IP业务》中的《IPv6基础配置》。

使用实例

# 配置接口POS1/0/0链路控制层进行PPP MRU值的协商。

<Quidway> system-view
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] mtu 1300
[Quidway-Pos1/0/0] ppp mru-negotiate ipv4
