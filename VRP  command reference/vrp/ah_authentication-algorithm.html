ah authentication-algorithm

命令功能

ah authentication-algorithm命令用来设置IPSec安全提议中AH（Authentication Header）协议采用的认证算法。

undo ah authentication-algorithm命令用来恢复缺省的认证算法。

缺省情况下，在IPSec安全提议中AH协议采用MD5认证算法。

命令格式

ah authentication-algorithm { md5 | sha1 }

undo ah authentication-algorithm

参数说明

md5：设置采用MD5认证算法。

sha1：设置采用SHA-1认证算法。

视图

IPSec提议视图

缺省级别

2：配置级

使用指南

AH协议没有加密的功能，只对报文进行认证。

MD5算法使用128位的密钥，SHA-1算法使用160位的密钥。MD5算法的计算速度比SHA-1算法快，而SHA-1算法的安全强度比MD5算法高。

在安全隧道的两端设置安全策略时，所引用的提议中AH协议必须设置成采用同样的认证算法。

使用实例

# 在配置安全提议prop1时，设定AH协议采用SHA-1算法。

<Quidway> system-view
[Quidway] ipsec proposal prop1
[Quidway-ipsec-proposal-prop1] transform ah
[Quidway-ipsec-proposal-prop1] ah authentication-algorithm sha1
