datasize

命令功能

datasize命令用来配置NQA测试例的报文大小。

undo datasize命令用来恢复缺省配置。

缺省情况下，测试报文的大小为0字节，此时测试报文的实际大小按100字节构造。

命令格式

datasize size

undo datasize

参数说明

size：NQA测试发送的报文大小。单位是字节，取值范围是0～8100。

视图

NQA视图

缺省级别

2：配置级

使用指南

只能对PWE3Ping、MPing、ICMP、UDP、Jitter、LSP Jitter、Path Jitter、ICMP Jitter、Trace、LSP Ping和Jitter类型的测试例配置测试报文大小。

对于ICMP Jitter类型的测试例，只有ICMP echo模式下，可以配置测试报文的大小。

不能修改正在执行的测试例的报文大小。

如果需要模拟64kbit/s的语音流，可以设置报文大小为172字节（160 bytes payload +12 bytes RTP header + 28 bytes IP header and UDP header），每隔20ms发一个包。这样，一分钟可以发送3000个报文。

使用实例

# 设置测试例user test发送的报文大小为100字节。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] datasize 100
