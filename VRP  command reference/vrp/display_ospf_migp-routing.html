display ospf migp-routing

命令功能

display ospf migp-routing命令用来显示OSPF的MIGP路由信息。

命令格式

display ospf [ process-id ] migp-routing [ [ ip-address [ mask | mask-length ] ] | [ interface interface-type interface-number ] ] [ nexthop nexthop-address ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

process-id：OSPF进程号。整数形式，取值范围是1～65535。

ip-address：目的IP地址，点分十进制格式。

mask：指定子网掩码，点分十进制格式。

mask-length：指定掩码长度。整数形式，取值范围是0～32。

interface-type：接口类型。

interface-number：接口号。

nexthop nexthop-address：显示下一跳IP地址为此指定值的路由信息。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

通过display ospf migp-routing命令可以查看出接口为TE-Tunnel的路由的实际物理接口。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

Destination

目的地址

Cost

到达目的地址的开销

Type
目的网络的类型。根据目的网络类型不同，可将路由分为区域间路由（Inter-area）和区域内路由（Stub和Transit）。
  • Stub：router-lsa发布的路由，对应非广播网络、非NBMA网络的直连路由
  • Transit：network-lsa发布的路由

NextHop

到达目的地址的下一跳地址

AdvRouter

发布路由器

Area

区域号

Total Nets

区域内部、区域间、ASE和NSSA区域的网络总数

Intra Area

区域内部路由总数（即Stub类型和Transit类型的总和）

Inter Area

区域间路由总数

ASE

ASE区域路由总数

NSSA

NSSA区域路由总数
