single-hop-reserve slot

命令功能

single-hop-reserve slot命令用来设置指定接口板上创建单跳BFD会话的个数，并为其预留资源，以确保在资源不足的情况下优先建立单跳会话。

undo single-hop-reserve slot命令用来释放指定接口板上为单跳BFD会话预留的资源。

缺省情况下，接口板上没有为单跳BFD会话预留资源。

命令格式

single-hop-reserve session-number slot slot-id

undo single-hop-reserve slot slot-id

参数说明

session-number：预留的单跳BFD会话个数。整数形式，最小值是1，最大值是接口板支持的最大会话个数。

slot-id：接口板的编号。

缺省级别

2：配置级

使用指南

执行该命令以前，必须全局使能BFD。

为接口板预留单跳BFD会话，必须满足该条件：预留资源个数 － 该接口板已建立单跳BFD会话个数＜＝该接口板剩余资源数，否则配置失败。

当“接口板剩余资源数 ＝ 预留资源个数 － 该接口板已建立单跳BFD会话个数”时，该接口板上将不再建立多跳BFD会话。

使用实例

# 为3号接口板预留30个单跳BFD会话。

[Quidway] system-view
[Quidway] bfd
[Quidway-bfd] single-hop-reserve 30 slot 3
