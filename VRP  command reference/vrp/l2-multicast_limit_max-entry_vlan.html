l2-multicast limit max-entry vlan

命令功能

l2-multicast limit max-entry vlan命令用来配置VLAN或端口＋VLAN的组播组数量限制。可以使用不同的VLAN段同时配置多个VLAN。对于不需要做数量限制的组播组，可以使用ACL的permit规则定义范围区分出来。

undo l2-multicast limit max-entry vlan命令用来取消对VLAN或VLAN＋端口的组播组数量限制。在配置undo时，若输入VLAN段的范围不在配置的范围之内，仅处理配置范围内的VLAN。

缺省情况下，不对任何VLAN或端口＋VLAN组播组进行数量限制。

命令格式

l2-multicast limit max-entry count vlan { { vlan-id [ to vlan-id ] }

&<1-10> } [ except acl-number ]

undo l2-multicast limit max-entry vlan { { vlan-id [ to vlan-id ] }

&<1-10> }

参数说明

max-entry count：组播组限制数量。整数形式，取值范围是1～8000。

vlan vlan-id：需要配置数量限制的VLAN编号。整数形式，取值范围是1～4094。

except：排除不需要做数量限制的组播组。

acl-number：ACL号。整数形式，取值范围是2000～2999。

缺省级别

2 ：配置级

使用指南

此命令为覆盖式命令。

使用实例

# 将VLAN 10到20，VLAN 30的基于VLAN的组播组数量限制配置为1000。

<Quidway> system-view
[Quidway] l2-multicast limit max-entry 1000 vlan 10 to 20 30

# 将VLAN 10到20，VLAN 30的基于端口+VLAN组播组数量限制配置为1000。

<Quidway> system-view
[Quidway] interface GigabitEthernet1/0/0
[Quidway-GigabitEthernet1/0/0] l2-multicast limit max-entry 1000 vlan 10 to 20 30
