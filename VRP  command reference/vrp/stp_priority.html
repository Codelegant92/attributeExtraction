stp priority

命令功能

stp priority命令用来配置交换机在指定生成树实例中的优先级。

undo stp priority命令用来恢复交换机在指定生成树实例中优先级的缺省值。

命令格式

stp [ instance instance-id ] priority priority

undo stp [ instance instance-id ] priority

参数说明

instance instance-id：指定生成树实例。instance-id表示生成树实例的编号，取值范围是0～48，取值为0表示的是CIST。

priority priority：交换机的优先级。取值范围是0～61440，步长为4096，即交换机可以配置16个优先级取值，如0、4096、8192等。缺省值是32768。优先级值越小，则交换机的优先级越高。

使用指南

注意：

如果当前交换机已配置为根交换机或者备份根交换机，就不能再配置优先级。如果要配置交换机的优先级，必须先去使能根交换机或者备份根交换机功能。

交换机的优先级是生成树计算的重要依据。每台交换机在不同生成树实例中可以配置不同的优先级。

在使用stp priority命令或undo stp priority命令时，如果不输入instance instance-id参数，则所作的配置在CIST实例上生效。

使用实例

# 配置交换机在生成树实例1中的优先级为4096。

<Quidway> system-view
[Quidway] stp instance 1 priority 4096
