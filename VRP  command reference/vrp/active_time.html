active time

命令功能

active time命令用来配置RMEP激活时间。

undo active time命令用来取消配置的RMEP激活时间。

缺省情况下，RMEP激活时间为0秒。

命令格式

active time time

undo active time

参数说明

time time：RMEP激活时间。取值范围是0～600，单位是秒。

使用指南

现如今，802.1ag协议分为draft7草案版本和standard2007标准版本，本命令只支持standard2007标准版本使用。

当在本端设备上执行remote-mep ccm-receive enable命令使能接收某个RMEP发送的CCM消息功能后，

  • 如果通过CC探测发现本端MEP与该RMEP之间出现连通性故障，则本端设备上提示RMEP连通性告警。

  • 如果本端MEP与RMEP之间物理连通正常，但是由于对端设备没有及时配置MEP或者MEP配置延后，此时进行CC探测，若本端MEP连续3个CCM发送周期没有接收到远端MEP发送的CCM消息，则认为和远端MEP之间发生了连通性故障，在本端设备上提示RMEP连通性告警。

    可以看出，此告警为误告警。为了解决这个问题，提出了RMEP激活时间这个概念。

若设备配置了RMEP激活时间，当在本端设备上使能接收某个RMEP发送的CCM消息功能后，等待配置的RMEP激活时间到达后，本端设备开始接收CCM报文。也就是说RMEP激活时间是预留给用户对远端MEP配置的时间。

如果配置的RMEP激活时间到达后，本端设备上的MEP连续3个CCM周期内没有收到CCM报文，则认为MEP之间发生了连通性故障，本端设备上才提示RMEP连通性告警。

如果本端设备上配置RMEP激活时间采用缺省值，说明只要设备上使能了接收某个RMEP发送的CCM消息功能，便无需等待，本端设备上的MEP会立即接收对端MEP发送的CCM报文。

说明：
如果MA或MAC Tunnel内配置了RMEP，就不能够再配置RMEP激活时间。

使用实例

# 配置RMEP激活时间为30s。

<Quidway> system-view
[Quidway] cfm md md1
[Quidway-md-md1] ma ma1
[Quidway-md-md1-ma-ma1] active time 30
