ip netstream aggregation

命令功能

ip netstream aggregation命令用来进入NetStream聚合配置模式。

命令格式

ip netstream aggregation { as | as-tos | destination-prefix | destination-prefix-tos mpls-label| prefix | prefix-tos | protocol-port | protocol-port-tos | source-prefix | source-prefix-tos }

参数说明

as：自治系统聚合。根据NetStream 流的源、目的自治系统号、入接口索引、出接口索引，4个关键值进行分类。

as-tos：自治系统-ToS聚合。根据NetStream 流的源、目的自治系统号、入接口索引、出接口索引、ToS，5个关键值进行分类。

destination-prefix：目的前缀聚合。根据NetStream 流的目的自治系统号、目的掩码长度、目的前缀、输出接口索引，4个关键值进行分类。

destination-prefix-tos：目的前缀-ToS聚合。根据NetStream 流的目的自治系统号、目的掩码长度、目的前缀、ToS、输出接口索引，5个关键值进行分类。

mpls-label标签聚合。根据NetStream流的标签号进行聚合分类。

prefix：前缀聚合。根据NetStream 流的源、目的自治系统号，源、目的掩码长度、源前缀、目的前缀、输入接口索引、输出接口索引，8个关键值进行分类。

prefix-tos：前缀-ToS聚合。根据NetStream 流的源、目的自治系统号、源、目的掩码长度、源、目的前缀、ToS、输入接口索引、输出接口索引，9个关键值进行分类。

protocol-port：协议-端口聚合。根据NetStream 流的协议号、源端口、目的端口，3个关键值进行分类。

protocol-port-tos：协议-端口-ToS聚合。根据NetStream 流的协议号、源端口、目的端口、ToS、输入接口索引、输出接口索引，6个关键值进行分类。

source-prefix：源前缀聚合。根据NetStream 流的源自治系统号、源掩码长度、源前缀、输入接口索引，4个关键值进行分类。

source-prefix-tos：源前缀-ToS聚合。根据NetStream 流的源自治系统号、源掩码长度、源前缀、ToS、输入接口索引，5个关键值进行分类。

使用指南

当需要对采集到的流量进行分类时，使用此命令进入对应的聚合模式，进行聚合配置。对流量聚合后不仅可以对流量进行分类，还可以减少发送到网管的统计信息的数量，减轻网络的负担。

进入聚合视图后，只有执行命令enable后，采集到的流统计信息才能按照配置的聚合模式进行输出。

使用实例

# 进入AS聚合模式。

<Quidway> system-view
[Quidway] ip netstream aggregation as

# 进入MPLS聚合模式。

<Quidway> system-view
[Quidway] ip netstream aggregation mpls-label
