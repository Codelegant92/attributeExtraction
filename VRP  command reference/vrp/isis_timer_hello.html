isis timer hello

命令功能

isis timer hello命令用来指定接口发送Hello报文的间隔时间。

undo isis timer hello命令用来恢复缺省设置。

缺省情况矿下，接库发送Hello报文的间隔时间是10秒。

命令格式

isis timer hello { hello-interval1 | millisecond-timer hello-interval2 } [ level-1 | level-2 ]

undo isis timer hello [ level-1 | level-2 ]

参数说明

hello-interval1：Hello报文发送的间隔时间。整数类型，取值范围是3～255，单位是秒。缺省值是10秒。

millisecond-timer hello-interval2：Hello报文发送的间隔时间。整数类型，取值范围是100～255000，单位是毫秒。缺省值是10000毫秒。

level-1：Level-1 Hello报文的发送间隔。

level-2：Level-2 Hello报文的发送间隔。

说明：

如果没有指定级别，则默认级别为Level-1和Level-2。

参数level-1和level-2仅在广播接口上是可配置的，而且必须先在接口上使能IS-IS。

使用指南

只有使能了Smart-Hello功能，Hello报文时间间隔才能设置成毫秒级（millisecond-timer参数）。如果Smart-Hello功能被取消，相应的Hello报文时间间隔将恢复到默认值。

在广播链路上，Level-1和Level-2的Hello报文会分别发送，其间隔时间也要分别设置。在点到点链路上只有一种Hello报文，所以不需要使用参数level-1和level-2。发送间隔越短，就需要占用越多的系统资源来发送Hello报文，因此，间隔时间需要根据实际情况设定。

说明：

如果通过isis circuit-type命令将广播接口模拟为P2P接口或者通过undo isis circuit-type命令将该接口恢复为广播接口，则IS-IS发送Hello报文的间隔时间恢复为缺省值。

使用实例

# 设置Level-2的Hello报文在接口GigabitEthernet1/0/0上每20秒发送一次。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] isis timer hello 20 level-2
