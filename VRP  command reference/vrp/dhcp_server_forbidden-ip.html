dhcp server forbidden-ip

命令功能

dhcp server forbidden-ip命令用来配置DHCP地址池中不参与自动分配的IP地址范围。

undo dhcp server forbidden-ip命令用来删除指定的不参与自动分配的IP地址范围。

缺省情况下，地址池中所有IP地址都参与自动分配。

命令格式

dhcp server forbidden-ip start-ip-address [ end-ip-address ]

undo dhcp server forbidden-ip start-ip-address [ end-ip-address ]

参数说明

start-ip-address：指定不参与自动分配的IP地址段的起始IP地址。

end-ip-address：指定不参与自动分配的IP地址段的结束IP地址，应与start-ip-address在同一网段，并且不能小于start-ip-address。如果不指定该参数，表示只有一个IP地址，即start-ip-address。

使用指南

有些地址分配给其他的服务，如分配给DNS就不能在进行分配，可以使用dhcp server forbidden-ip命令用来配置DHCP地址池中不参与自动分配的IP地址范围。

多次执行本命令，可以配置多个不参与自动分配的IP地址段。

在使用undo dhcp server forbidden-ip命令删除设置时，必须确保参数与原先配置的完全相同，即不能仅删除原先配置的部分地址。

使用实例

# 将10.110.1.1到10.110.1.63之间的IP地址保留，不参与地址自动分配。

<Quidway> system-view
[Quidway] dhcp server forbidden-ip 10.110.1.1 10.110.1.63
