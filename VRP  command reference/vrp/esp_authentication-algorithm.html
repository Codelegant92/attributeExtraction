esp authentication-algorithm

命令功能

esp authentication-algorithm命令用来设置ESP协议采用的认证算法。

undo esp authentication-algorithm命令用来设置ESP协议不对报文进行认证。

缺省情况下使用md5，即MD5认证算法。

命令格式

esp authentication-algorithm { md5 | sha1 }

undo esp authentication-algorithm

参数说明

md5：采用MD5认证算法，密钥长度128位。

sha1：采用SHA-1认证算法，密钥长度160位。

缺省级别

2：配置级

使用指南

MD5算法的计算速度比SHA-1算法快，而SHA-1算法的安全强度比MD5算法高。

ESP协议允许对报文同时进行加密和认证，或只加密，或只认证。

注意：undo esp authentication-algorithm命令不是恢复认证算法为缺省算法，而是设置认证算法为空，即不认证。当认证算法不为空时，undo esp authentication-algorithm命令才起作用。

ESP协议采用的加密算法和认证算法不能同时设置为空。

在安全隧道的两端设置的安全策略所引用的提议必须设置成采用同样的认证算法。

使用实例

# 设定提议prop1采用ESP协议并使用SHA-1认证算法。

<Quidway> system-view
[Quidway] ipsec proposal prop1
[Quidway-ipsec-proposal-prop1] transform esp
[Quidway-ipsec-proposal-prop1] esp authentication-algorithm sha1
