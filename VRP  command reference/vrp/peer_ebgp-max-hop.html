peer ebgp-max-hop

命令功能

peer ebgp-max-hop命令用来配置允许BGP同非直连网络上的对等体建立EBGP连接，并同时可以指定允许的最大跳数。

undo peer ebgp-max-hop命令用来取消已有的配置。

缺省情况下，只能在物理直连链路上建立EBGP连接。

命令格式

peer { group-name | ipv4-address | ipv6-address } ebgp-max-hop [ hop-count ]

undo peer { group-name | ipv4-address | ipv6-address } ebgp-max-hop

参数说明

group-name：对等体组的名称。

ipv4-address：对等体的IPv4地址。

ipv6-address：对等体的IPv6地址。

说明：

  • ipv4-address参数仅在BGP视图和BGP-VPN实例视图下生效。

  • ipv6-address参数仅在BGP视图和BGP-VPN6实例视图下生效。

hop-count：最大跳数，范围为1～255，缺省值为255。如果指定的最大跳数为1，则不能同非直连网络上的对等体建立EBGP连接。

视图

BGP视图、BGP-VPN实例视图、BGP-VPN6实例视图

缺省级别

2：配置级

使用指南

如果在EBGP连接的其中一端配置了该命令，另一端也需要配置该命令。

使用实例

# 允许同不直接相连网络上的EBGP对等体1.1.1.2建立连接。

<Quidway> system-view
[Quidway] bgp 100 
[Quidway-bgp] peer 1.1.1.2 as-number 200
[Quidway-bgp] peer 1.1.1.2 ebgp-max-hop
