if-match rtp

命令功能

if-match rtp命令用来定义RTP协议端口匹配规则。

undo if-match rtp命令用来删除RTP协议端口匹配规则。

命令格式

if-match [ not ] rtp start-port min-rtp-port-number end-port max-rtp-port-number

undo if-match [ not ] rtp start-port min-rtp-port-number end-port max-rtp-port-number

参数说明

min-rtp-port-number：指定发起RTP报文的最小的UDP目的端口号，取值范围是2000～65535。

max-rtp-port-number：指定发起RTP报文的最大的UDP目的端口号，取值范围是2000～65535。

使用指南

该命令用来匹配落在指定RTP端口号范围内的RTP报文，即匹配所有在min-rtp-port-number与 max-rtp-port-number之间的偶数UDP端口号的报文。

在同一个类下多次重复使用该命令，新的配置并不覆盖以前的配置，所有的配置都将生效。

使用实例

# 定义类class1的匹配规则为：匹配RTP端口号在16384和32767之间的偶数UDP端口号的报文。

<Quidway> system-view
[Quidway] traffic classifier class1
[Quidway-classifier-class1] if-match rtp start-port 16384 end-port 32767
