rpr protection restore

命令功能

rpr protection restore命令用来配置本地节点的保护恢复模式，即在本地节点WTR超时后是否恢复到IDLE状态。

undo rpr protection restore命令用来恢复缺省值。

缺省情况下，使用revertive恢复模式。

命令格式

rpr protection restore { revertive | non-revertive }

rpr protection restore do-restore { west | east }

undo rpr protection restore

参数说明

revertive：当环上的故障排除后，节点由保护模式恢复到IDLE状态。

non-revertive：当环上的故障排除后，节点的保护模式不变。即仍为保护模式，不返回到正常模式。

do-restore：用户强制恢复到正常模式。

west：本地节点的西向连接。

east：本地节点的东向连接。

缺省级别

2：配置级

使用指南

使用rpr protection restore revertive设定自动恢复模式。

使用rpr protection restore non-revertive设定手动恢复模式。

使用rpr protection restore do-restore来手动恢复保护模式。

使用实例

# 将本地节点配置为可恢复模式。

<Quidway> system-view
[Quidway] interface ring-if 1
[Quidway-ring-if1] rpr protection restore revertive

# 将本地节点配置为不可恢复模式。

<Quidway> system-view
[Quidway] interface ring-if 1
[Quidway-ring-if1] rpr protection restore non-revertive

# 强制恢复本节点西向的保护模式。

<Quidway> system-view
[Quidway] interface ring-if 1
[Quidway-ring-if1] rpr protection restore do-restore west
