encapsulation（ATM-PVC视图）

命令功能

encapsulation命令用来为PVC指定ATM AAL5封装类型。

undo encapsulation命令用来恢复缺省封装。

缺省情况下，为aal5snap封装。

命令格式

encapsulation aal5-encap

undo encapsulation

参数说明

aal5-encap：AAL5封装类型。其可能的值如下：

  • aal5snap：LLC/SNAP（逻辑链接控制/子网访问协议）封装类型

  • aal5mux：MUX复用封装类型

  • aal5muxnonstandard：非标准兼容的MUX复用封装类型，用于与其他厂商设备互通

  • aal5nlpid：RFC1490封装类型

视图

ATM-PVC视图、ATM-Class视图

缺省级别

2：配置级

使用指南

只有aal5snap封装支持InARP协议，当采用aal5mux、aal5muxnonstandard和aal5nlpid封装时不能配置InARP。

如果配置了InARP，必须先删除InARP后才能将PVC的AAL5封装类型改变为aal5mux、aal5muxnonstandard或aal5nlpid。

另外，某些类型的封装可能并不支持部分应用方式（即IPoA、IPoEoA、PPPoA和PPPoEoA中的一种或几种），当出现不能支持的情况时，系统会给出提示。

使用实例

下面两种做法都可以指定PVC的AAL5封装类型为aal5snap。

# 指定PVC 1/32的AAL5封装类型为aal5snap。

<Quidway> system-view
[Quidway] interface atm 1/0/0
[Quidway-Atm1/0/0] pvc 1/32
[Quidway-atm-pvc-Atm1/0/0-1/32] encapsulation aal5snap

# 指定PVC 1/33的AAL5封装类型为aal5snap。

[Quidway-atm-pvc-Atm1/0/0-1/33] undo encapsulation
