peer next-hop-invariable

命令功能

peer next-hop-invariable命令配置不同AS域的PE间向EBGP对等体发布路由时不改变下一跳；向IBGP对等体发布引入的IGP路由时使用IGP路由的下一跳地址。

undo peer next-hop-invariable命令用来恢复缺省配置。

缺省情况下，BGP发言者在向EBGP对等体发布路由时和向IBGP对等体发布引入的IGP路由时将下一跳改为自己的接口地址。

命令格式

peer { ipv4-address | group-name } next-hop-invariable

undo peer { ipv4-address | group-name } next-hop-invariable

参数说明

ipv4-address：对等体的IPv4地址。

group-name：对等体组的名称。

缺省级别

2：配置级

使用指南

当配置这条命令后，仅对于引入的IGP路由且发往IBGP对等体时，不更改该IGP路由的下一跳地址。

peer next-hop-invariable和peer next-hop-local是两条互斥命令。

使用实例

# 向EBGP对等体发布路由时不改变下一跳。

<Quidway> system-view
[Quidway] bgp 100 
[Quidway-bgp] peer 1.1.1.2 as-number 200 
[Quidway-bgp] ipv4-family vpnv4 
[Quidway-bgp-af-vpnv4] peer 1.1.1.2 enable
[Quidway-bgp-af-vpnv4] peer 1.1.1.2 next-hop-invariable

# 向IBGP对等体发布引入的IGP路由时使用IGP路由的下一跳属性。

<Quidway> system-view
[Quidway] bgp 100 
[Quidway-bgp] peer 1.1.1.2 as-number 100
[Quidway-bgp] ipv4-family unicast
[Quidway-bgp-af-ipv4] peer 1.1.1.2 next-hop-invariable
