test-failtimes

命令功能

test-failtimes命令用来设置当NQA测试连续测试失败达到一定的次数以后，需要向网管发送Trap信息。

undo test-failtimes命令用恢复缺省设置。

缺省情况下，测试失败一次就会发送一条Trap信息。

命令格式

test-failtimes times

undo test-failtimes

参数说明

times：测试失败的次数，取值范围是1～15。

缺省级别

使用指南

在测试时，每发一个包进行测试称为一次探测（probe）。

一次测试可能要发多个包进行测试，即，包含多次探测。在默认情况下，对于一次测试，只要有一次或一次以上探测成功，即认为该测试是成功的；若全部探测失败，则认为该测试失败。

配置了test-failtimes命令后，是否发送Trap消息还需要判断是否打开了相应的send-trap开关。

不能对正在运行的测试例更改此项配置。

使用实例

# 设置测试例user test的连续测试失败数。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] test-failtimes 10
