display multicast ipv6 routing-table

命令功能

display multicast ipv6 routing-table命令用来查看IPv6组播路由表信息。

命令格式

display multicast ipv6 routing-table [ ipv6-source-address [ ipv6-source-mask-length ] | ipv6-group-address [ ipv6-group-mask-length ] | incoming-interface { interface-type interface-number | register } | outgoing-interface { { exclude | include | match } { interface-type interface-number | register | none } } ] * [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

ipv6-source-address：指定组播源的IPv6地址。

ipv6-source-mask-length：指定组播源地址的掩码长度。取值范围是0～128。

ipv6-group-address：指定组播组的IPv6地址，通过掩码指定一个组播组的范围，显示该范围内的组播组对应的路由表信息。取值范围是FF00::0～FFFF：FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。

ipv6-group-mask-length：指定组地址的掩码长度。取值范围是8～128。

incoming-interface：指定组播转发项的入接口。

interface-type interface-number：指定接口类型和编号。

register：表示IPv6 PIM-SM的注册接口。

outgoing-interface：表示组播转发项的出接口。

include：显示下游接口列表中包含指定出接口的（S，G）表项。

exclude：显示下游接口列表中不包含指定出接口的（S，G）表项。

match：显示下游接口列表中只有一个接口且该接口与指定出接口相同的（S，G）表项。如果没有指定接口，则显示下游接口列表为空的（S，G）表项。

none：显示不存在下游的路由项。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

该命令显示IPv6组播路由表的信息，而display multicast ipv6 forwarding-table命令显示IPv6组播转发表的信息。

组播转发表项与组播路由表项保持一致。

  • 每一个（S，G）项有一个唯一的Upstream（上游），表示组播数据应该从此接口到达。

  • 每一个（S，G）项有一个Downstream List（下游接口列表），表示组播数据应该从哪些接口向外转发。

使用display multicast ipv6 routing-table命令：

  • 如果指定ipv6-source-address，则显示指定组播源对应的IPv6组播路由表信息。

  • 如果指定ipv6-source-mask-length，则显示指定地址范围内的组播源对应的IPv6组播路由表信息。

  • 如果指定ipv6-group-address，则显示指定组播组对应的IPv6组播路由表信息。

  • 如果指定ipv6-group-mask-length，则显示指定地址范围内的组播组对应的IPv6组播路由表信息。

  • 当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。
