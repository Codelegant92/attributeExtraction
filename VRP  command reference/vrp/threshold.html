threshold

命令功能

threshold命令用来设置往返时延RTD、单向时延OWD的阈值。

undo threshold命令用来取消设置往返时延RTD、单向时延OWD的阈值。

缺省情况下，没有配置阈值。

命令格式

threshold { owd-ds owd-ds-vale | owd-sd owd-sd-value | rtd rtd-value }

undo threshold { owd-ds owd-ds-value | owd-sd owd-sd-value | rtd rtd-value }

参数说明

owd-ds owd-ds-value：从目的到源的单向时延阈值。整数形式，取值范围是1～60000，单位是毫秒。

owd-sd owd-sd-value：从源到目的的单向延时阈值。整数形式，取值范围是1～60000，单位是毫秒。

rtd rtd-value：往返时延阈值。整数形式，取值范围是1～60000，单位是毫秒。

缺省级别

2：配置级

使用指南

单向时延OWD的阈值，只适用于Jitter、ICMPJitter、使能接口板发包的ICMPJitter、使能接口板发包的Jitter类型的测试例。

使用实例

# 设置往返时延阈值2ms。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway -nqa-user-test] threshold rtd 2

# 设置源到目的的单向时延阈值为1ms。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway -nqa-user-test] threshold owd-sd 1
