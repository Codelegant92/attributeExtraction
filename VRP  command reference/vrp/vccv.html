vccv

命令功能

vccv cc命令用来使能VC的连接性检测。

undo vccv命令用来去使能VC的连接性检测。

缺省情况下，系统没有使能VC的连接性检测。

命令格式

vccv cc { alert | cw } * cv { bfd | lsp-ping } *

undo vccv

参数说明

cc：控制字通道（Control Channel）。

alert：使能MPLS Router Alert通道。

cw：使能控制字通道。

cv：连接性检测（Connectivity Verification）。

bfd：采用BFD方式对VC进行连接性检测。

lsp-ping：采用LSP-PING方式对VC进行连接性检测。

使用指南

VCCV属性支持控制字功能。配置通过控制字方式来进行VC的连接性检测前，需要先使用命令control-word，使能控制字功能。

VC的连接性检测有自动方式和手工方式。自动方式包括BFD和OAM，手工方式包括LSP-PING和TraceRoute。检测使用的通道包括控制字通道和MPLS Router Alert通道。

动态PW和静态PW支持VC的连接性检测。

使用实例

# 使pwt模板通过LSP-PING方式对VC进行连接性检测，并使能控制字通道。

<Quidway> system-view
[Quidway] pw-template pwt
[Quidway-pw-template-pwt] vccv cc cw cv lsp-ping

# 使pwt模板通过BFD方式对VC进行连接性检测，并使能控制字通道。

<Quidway> system-view
[Quidway] pw-template pwt
[Quidway-pw-template-pwt] vccv cc cw cv bfd
