ipv6 nd ra prefix

命令功能

ipv6 nd ra prefix命令用来配置RA报文中的前缀消息。

undo ipv6 nd ra prefix命令用来取消在RA报文中指定的前缀。

缺省情况下，RA报文中只包含接口上使用ipv6 address命令配置的地址前缀。

命令格式

ipv6 nd ra prefix { ipv6-address prefix-length | ipv6-address/prefix-length } valid-lifetime preferred-lifetime [ no-autoconfig ] [ off-link ]

undo ipv6 nd ra prefix { ipv6-address prefix-length | ipv6-address/prefix-length }

参数说明

ipv6-address prefix-length或ipv6-address/prefix-length ：指定IPv6地址和前缀长度，其中前缀长度为整数形式，取值范围是1～128。通过IPv6地址和前缀长度，可以计算出RA报文携带的IPv6前缀。

valid-lifetime：指定有效存活时间。整数形式，单位为秒，取值范围是0～4294967295。

preferred-lifetime：指定首选存活时间。整数形式，单位为秒，取值范围是0～4294967295。首选存活时间不能大于有效存活时间。

no-autoconfig：指定A-Flag标志位，若使用该参数，则表示配置的前缀是用于无状态地址配置的。

off-link：指定O-Flag标志位，若使用该参数，则表示配置的前缀不是分配给本地链路的。

使用指南

如果使用ipv6 nd ra prefix命令配置了前缀，则只发布指定的前缀。

此命令支持Ethernet接口及其子接口、GigabitEthernet接口及其子接口、POS接口以及Tunnel接口。

此命令指定的前缀不能为链路本地地址（fe80::）、组播地址（ff00::）、未指定地址（::）以及其它接口已经使用的前缀（包括接口地址前缀和RA消息携带的前缀）。

在配置该命令前，需要先在接口视图下执行ipv6 enable命令，使能IPv6功能。

使用实例

# 配置GE接口上RA报文的前缀消息。

<Quidway> system-view
[Quidway] interface gigabitethernet 1/0/0
[Quidway-GigabitEthernet1/0/0] ipv6 nd ra prefix 2001:10::100/64 100 10
[Quidway-GigabitEthernet1/0/0] ipv6 nd ra prefix 4001:10::100/128 1000 400 no-autoconfig
[Quidway-GigabitEthernet1/0/0] ipv6 nd ra prefix 3001:10::100/64 1000 400 off-link
