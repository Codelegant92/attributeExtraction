snmp-agent trap enable feature-name msdp

命令功能

snmp-agent trap enable feature-name msdp命令用来打开MSDP相关事件的告警开关。

undo snmp-agent trap enable feature-name msdp命令用来关闭MSDP相关事件的告警开关。

命令格式

snmp-agent trap enable feature-name msdp { non-excessive all | trap-name { establish | backward } }

undo snmp-agent trap enable feature-name msdp { [ non-excessive | excessive ] all | trap-name { establish | backward } }

参数说明

non-excessive all：打开MSDP模块所有非海量事件的告警开关。

trap-name：打开指定类型MSDP事件的告警开关。

establish：打开建立MSDP peer事件的告警开关。

backward：打开拆除MSDP peer事件的告警开关。

non-excessive：关闭MSDP模块所有非海量事件的告警开关。

excessive：关闭MSDP模块所有海量事件的告警开关。

all：关闭MSDP模块所有事件的告警开关。

使用指南

目前定义的MSDP非海量事件有：建立MSDP peer和拆除MSDP peer。

当用户希望打开MSDP模块所有事件的告警开关时，可以选定non-excessive all批量处理；当用户希望打开某个或几个事件的告警开关时，可以指定trap-name。

使用实例

# 在系统视图下打开MSDP peer连接建立的告警开关。

<Quidway> system-view
[Quidway] snmp-agent trap enable feature-name msdp trap-name establish

# 在系统视图下关闭MSDP peer连接建立的告警开关。

<Quidway> system-view
[Quidway] undo snmp-agent trap enable feature-name msdp trap-name establish
