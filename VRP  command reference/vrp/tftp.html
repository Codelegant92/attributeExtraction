tftp

命令功能

tftp命令用来上传文件到TFTP服务器或下载文件至本地。

命令格式

tftp [ -a source-ip-address | -i interface-type interface-number ] host-name { get | put } source-filename [ destination-filename ]

tftp ipv6 [ -a source-ip-address ] { ipv6-address | host-name } [ -i interface-type interface-number ] { get | put } source-filename [ destination-filename ]

参数说明

-a source-ip-address：本端路由器的IP地址。

-i interface-type interface-number：本端路由器出接口的接口类型和接口编号。如果远端主机的IPv6地址是链路本地地址，则必须指定出接口。

host-name：指定TFTP服务器的IP地址或者主机名，字符串形式，长度是1～20。

get：表示进行下载文件操作。

put：表示进行上传文件操作。

source-filename：指定源文件名，取值范围1～64个字符。

destination-filename：指定目标文件名，字符串形式，长度范围是1～64。缺省情况下，源文件名与目的文件名相同。

使用指南

如果未指定本地文件的文件名，则以原来的文件名保存。

用于通过TFTP方式向IPV6主机传送文件，缺省的文件传输模式为二进制传输模式。此版本不支持ASCII模式的文件传输。可以任意指定目的文件。缺省情况下，源文件名与目的文件名相同。

通过指定源地址，用户可以以指定的IP地址与服务端通信，从而达到进行安全校验的目的。

使用实例

# 将TFTP服务器根目录下的vrpcfg.txt文件下载到本地硬盘。TFTP服务器的IP地址为1.1.254.2，下载到本地之后以文件名vrpcfg.bak保存。

<Quidway> tftp 1.1.254.2 get vrpcfg.txt hda1:/vrpcfg.bak

# 将存储设备Flash根目录下的文本文件vrpcfg.txt上传到TFTP服务器缺省路径下。TFTP服务器的IP地址为1.1.254.2，vrpcfg.txt文件在TFTP服务器上以文件名vrpcfg.bak保存。

<Quidway> tftp 1.1.254.2 put flash:/vrpcfg.txt vrpcfg.bak

# 提供链路本地地址和接口名称，从对端TFTP服务器获取数据。

<Quidway> tftp ipv6 fe80::250:daff:fe91:e058 -i gigabitethernet 6/0/0 get filetoget filetoget
