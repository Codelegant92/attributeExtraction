policy-based-route

命令功能

policy-based-route命令用来创建或修改策略路由和策略点。

如果命令参数中所指出的策略路由或策略点不存在，则创建该策略路由或策略点，否则，只能修改它。

策略点的模式由参数 { permit | deny }的值决定。策略点的顺序号node-id决定策略点优先级，顺序号的值越小则优先级越高，相应策略优先执行。

undo policy-based-route命令用来删除策略路由或策略点。

命令格式

policy-based-route policy-name { permit | deny } node node-id

undo policy-based-route policy-name [ permit | deny | node node-id ]

参数说明

policy-name：指定策略名称。字符串形式，长度范围是1～19。

permit:：策略点的模式，表示对满足匹配条件的报文进行策略路由。

deny：策略点的模式，表示对满足匹配条件的报文不进行策略路由。

node-id:：指定策略点的顺序号。整数形式，取值范围是0～65535。
