apply ip-address next-hop（Route-Policy视图）

命令功能

apply ip-address next-hop命令用来设置路由的下一跳地址。

undo apply ip-address next-hop命令用来取消设置。

缺省情况下，不设置下一跳地址。

命令格式

apply ip-address next-hop { peer-address | ipv4-address }

undo apply ip-address next-hop { peer-address | ipv4-address }

参数说明

ipv4-address：下一跳地址。

peer-address：指定对等体地址为下一跳。

视图

Route-Policy视图

缺省级别

2：配置级

使用指南

在BGP中，通过策略设置路由信息的下一跳地址分两种情况。

  • IBGP：

    对于IBGP对等体，配置的入口策略或者出口策略均可以生效。如果策略中配置的下一跳地址是不可达的，那么IBGP对等体也会将该路由加入到BGP路由表中，但不是有效路由。

  • EBGP：

    对EBGP对等体使用策略修改下一跳地址时，一般配置为入口策略。这是因为如果配置为出口策略，这条路由到达EBGP对等体后会因为下一跳不可达而被丢弃。当EBGP对等体是通过物理连接直接建立时，该策略不能生效，即不能修改路由信息的下一跳地址。

使用实例

# 定义一个apply子句，设置下一跳地址为193.1.1.8。

<Quidway> system-view
[Quidway] route-policy policy permit node 10
[Quidway-route-policy] apply ip-address next-hop 193.1.1.8
