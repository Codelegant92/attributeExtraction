mac-limit rule-name（系统视图）

命令功能

mac-limit rule-name命令用来配置全局MAC地址学习限制规则。

undo mac-limit rule-name命令用来删除全局MAC地址学习限制规则。

命令格式

mac-limit rule-name rule-name { action { discard | forward } | alarm { disable | enable } | maximum max rate interval }*

undo mac-limit rule-name rule-name

参数说明

rule-name rule-name：全局MAC 地址限制规则名。字符串形式，1～31字符。规则名不能有空格、问号、减号。

action：当MAC地址表项数目达到限制后，系统所采取的动作。

discard：当MAC地址表项数目达到限制后，使用新MAC地址的报文被丢弃。

forward：当MAC地址表项数目达到限制后，使用新MAC地址的报文被继续转发，但不加入MAC地址表。

alarm：当MAC地址表项数目达到限制后，是否发送告警。

disable：当MAC地址表项数目达到限制后，不发送告警。

enable：当MAC地址表项数目达到限制后，以syslog方式提示告警。

maximum max ：最多可以学习的MAC数量。整数形式，取值范围是0～64512。当配置值为0时，表示不对学习数量进行限制。

rate interval：进行MAC学习的速率。整数形式，取值范围是0～1000，单位是毫秒。当配置值为0时，表示不对学习速率进行限制。

使用指南

设备支持4096条全局MAC地址学习限制规则。

全局MAC地址学习限制规则可以应用在多个接口上，实现不同的接口上共用一条MAC地址学习限制规则的功能。一条全局MAC地址学习限制规则最多应用在128个接口上。

如果同一个单板上的接口共用一条全局MAC地址学习限制规则，那么这些接口上的流量之和受该全局MAC地址学习限制规则控制。

创建全局MAC地址学习限制规则必须配置有效的最大值和最大速率。

使用实例

# 配置全局MAC学习限制规则，规则名为name1。数量限制配置为10000，速率为10毫秒，超出限制告警。

<Quidway> system-view
[Quidway] mac-limit rule-name name1 maximum 10000 rate 10 alarm enable
