cfm md

命令功能

cfm md命令用来创建维护域MD，并进入MD视图。如该MD已存在，则本命令用来进入MD视图。

undo cfm md命令用来删除MD。

命令格式

cfm md md-name [ format { no-md-name | dnsname-and-mdname | mac-address | md-name } ] [ level level ]

undo cfm md { all | name md-name }

参数说明

md md-name：指定MD的名称，用来标识一个维护域。字符串形式，不支持空格、“-”和“?”，长度范围是1～43。MD的名称用于标识MD，同一设备上的MD的名称不能重复。

no-md-name：表示发送的报文中的MA ID域中没有MD Name字段。

dnsname-and-mdname：表示用户在配置MD时使用DNS名称＋MD名称作为MD名称。

mac-address：表示用户在配置MD时不用指定MAC地址。

md-name：表示用户在配置MD时输入MD名称，报文中的MA ID域中含有MD名称。

level level：指定MD的级别。整数形式，取值范围是0～7，值越大MD的优先级越高。缺省值是0级。

all：表示删除所有MD。

使用指南

可以把由一个ISP管理的所有具备CFM功能的设备配置在一个MD内。

说明：

低级别MD的802.1ag协议报文进入高级别的MD后被丢弃，高级别MD的802.1ag协议报文可以穿越低级别的MD。相同级别的MD的802.1ag协议报文不能彼此穿越。

802.1ag standard2007标准版本规定，用户在配置MD时可指定MD名称类型。每个维护域只能有一种类型名称和一个级别。可以使用本命令进入一个已经存在的MD视图，但是不能使用本命令更改已经存在的MD的名称输入类型和级别。若需要更改已经存在的MD的名称输入类型和级别，需先删除该MD，然后再重新创建MD。

802.1ag draft7草案版本中规定创建了一个MD后，不能再使用本命令修改该MD的级别，如要修改，需先删除该MD，然后再重新创建MD。

使用cfm enable（系统视图）命令全局使能CFM功能后才能使用本命令。

一台路由器设备上最多可创建64个MD。

删除一个MD时，设备自动删除该MD下的所有MA、MEP、MIP、RMEP的相关配置。

MA、MEP、RMEP必须在MD内创建，因此在创建MA、MEP、RMEP前必须先创建MD。

使用实例

# 创建名称为mdcustomer、级别为4级的MD。

<Quidway> system-view
[Quidway] cfm enable
[Quidway] cfm md mdcustomer level 4

# 将mdcustomer的级别从4改为6。

<Quidway> system-view
[Quidway] undo cfm md name mdcustomer
[Quidway] cfm md mdcustomer level 6

# 创建名称为mdcustomer、名称输入类型为md-name、级别为4级的MD。

<Quidway> system-view
[Quidway] cfm enable
[Quidway] cfm md mdcustomer format md-name level 4
