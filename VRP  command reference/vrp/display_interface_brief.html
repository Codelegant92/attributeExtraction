display interface brief

命令功能

display interface brief命令用来显示设备上当前所有接口的简要信息，包括接口的物理状态、协议状态、接收方向最近一段时间的带宽利用率、发送方向最近一段时间的带宽利用率、接收的错误报文数和发送的错误报文数。

命令格式

display interface brief [ main ] [ | { begin | include | exclude } regular-expression ]

参数说明

main：显示主接口简要信息。

|：使用正则表达式过滤输出的显示信息。

begin：显示以包含指定正则表达式的行开始的配置信息。

exclude：显示不包含指定正则表达式的行的配置信息。

include：显示包含指定正则表达式的行的配置信息。

regular-expression：正则表达式。字符串形式，支持空格，区分大小写。

项目

描述

Interface
接口名称和接口编号。所有接口按字典序显示。如果Eth-Trunk、IP-Trunk、MFR、MP-Group、IMA-Group或Ring-If等逻辑接口下有成员接口，则其成员接口在该逻辑接口下按字典序显示，并且每行首字符缩进两个空格。可显示的接口类型与display interface命令中可显示的接口类型一致，包括但不限于：

  • ATM

  • AUX

  • BRI

  • Dialer

  • Ethernet

  • Eth-Trunk

  • GigabitEthernet

  • IMA-Group

  • IP-Trunk

  • Logic-Channel

  • Loopback

  • MFR

  • MP-Group

  • NULL

  • POS

  • Ring-If

  • RPR

  • Serial

  • Virtual-Ethernet

  • Virtual-Template

PHY
接口的物理状态。可能的物理状态有：

  • up：表示该接口物理层处于正常启动的状态。

  • down：表示该接口物理层出现故障。

  • *down：Administratively down，管理员在该接口执行了shutdown（接口视图）命令。

  • ^down：standby，表示该接口是备份接口。

  • (l)：loopback，表示该接口启用环回功能。

  • (b)：BFD down，表示该接口物理层处于BFD Down状态。

Protocol
接口的链路协议状态。可能的链路协议状态有：

  • up：表示该接口的数据链路层处于正常启动状态。

  • down：表示该接口链路协议出现故障。

  • (l)：loopback，表示该接口启用环回功能。

  • (s)：spoofing，表示该接口启用spoofing功能。

  • (b)：BFD down，表示该接口数据链路层处于BFD Down状态。

  • (d)：dampening suppressed，表示该接口处于协议模块抑制状态。

  • 空白：接口无链路协议状态。

InUti

接口接收方向最近300秒内的平均带宽利用率。接口接收方向最近300秒内的平均带宽利用率=接口接收方向最近300秒内平均速率/接口带宽。不足0.01%且有流量时显示0.01%。当接口带宽变小后（例如以太网接口通过speed命令调整接口带宽，或IP-Trunk等接口的成员接口状态变为Down或解除绑定），由于通信流量没有即时调整，可能会出现带宽利用率为100%。“--”表示该类型的接口不支持带宽利用率的显示。

OutUti

接口发送方向最近300秒内的平均带宽利用率。接口发送方向最近300秒内的平均带宽利用率=接口发送方向最近300秒内平均速率/接口带宽。不足0.01%且有流量时显示0.01%。当接口带宽变小后（例如以太网接口通过speed命令调整接口带宽，或IP-Trunk等接口的成员接口状态变为Down或解除绑定），由于通信流量没有即时调整，可能会出现带宽利用率为100%。“--”表示该类型的接口不支持带宽利用率的显示。

InErrors

接口接收的错误报文数。在用户视图下执行reset counters interface命令或计数达到最大值0xFFFFFFFF后，相应接口的该计数被置为零。

OutErrors

接口发送的错误报文数。在用户视图下执行reset counters interface命令或计数达到最大值0xFFFFFFFF后，相应接口的该计数被置为零。
