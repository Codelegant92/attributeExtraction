ipv6 summary

命令功能

ipv6 summary命令用来设置IS-IS生成IPv6聚合路由。

undo ipv6 summary命令用来取消聚合。

缺省情况下，系统不做路由聚合。

命令格式

ipv6 summary ipv6-address prefix-length [ avoid-feedback | generate_null0_route | tag tag | [ level-1 | level-1-2 | level-2 ] ] *

undo ipv6 summary ipv6-address prefix-length [ level-1 | level-1-2 | level-2 ]

参数说明

ipv6-address：需要聚合IPv6路由的地址范围。

prefix-length：IPv6路由的地址前缀长度。

avoid-feedback：避免通过路由计算学习到聚合路由。

generate_null0_route：为防止路由循环而生成NULL0路由。

tag tag：管理标签号。整数形式，取值范围是1～4294967295。

level-1：只对Level-1区域的路由进行聚合。

level-1-2：对Level-1区域以及Level-2区域的路由都进行聚合。

level-2：只对Level-2区域的路由进行聚合。

说明：

如果命令中没有指定Level，缺省为level-2。

使用指南

可以将有相同下一跳的路由聚合为一条路由，这样一方面减小路由表规模，另外可以减少本路由器生成的LSP报文大小和LSDB的规模。其中，被聚合的路由可以是IS-IS协议发现的路由，也可以是被引入的IPv6路由。另外，聚合后路由的开销值取所有被聚合路由中学到IPv6路由的最小开销值。

使用此命令前，必须先使能IS-IS进程的IPv6能力。配置过程请参见isis ipv6 enable命令。

使用实例

# 设置一条2002::/32的聚合路由。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] ipv6 summary 2002:: 32
