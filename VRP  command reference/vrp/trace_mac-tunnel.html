trace mac-tunnel

命令功能

trace mac-tunnel命令用来定位MAC tunnel的连通性故障，这种操作称为MAC tunnel Trace。

命令格式

trace mac-tunnel [ ttl ttl | -t timeout ] *

参数说明

ttl ttl：指定跟踪的最大跳数。ttl表示跟踪的最大跳数，整数形式，取值范围是1～255，缺省值是64。

-t timeout：指定等待应答的超时时间。timeout表示等待应答的超时时间，整数形式，取值范围是1～65535，单位是ms，缺省值是2000ms。

视图

MAC Tunnel视图

缺省级别

0：参观级

使用指南

当需要定位MAC tunnel的连通性故障时可使用本命令。

MAC tunnel Trace由MAC tunnel的一端发起，目的地是该MAC tunnel的另一端。

使用本命令时，被检测的MAC tunnel必须已使用cfm enable（MAC Tunnel视图）命令使能了CFM功能。

使用实例

# 定位名称为tu1的MAC tunnel的连通性故障。

<Quidway> system-view
[Quidway] mac-tunnel tunnel-name tu1
[Quidway-mac-tunnel-tu1] cfm enable
[Quidway-mac-tunnel-tu1] trace mac-tunnel

表1 trace mac-tunnel命令输出信息描述表

项目

描述

Hops

跳数。

Mac

中间节点或目的节点所在设备的桥MAC地址。

Ingress

中间节点或目的节点接收trace报文的入接口。

Ingress Action
入接口对Trace报文的处理。

  • IngOK表示入接口成功转发Trace报文。

  • 本字段为空表示入接口转发Trace报文不成功。

Reply Action
本设备对Trace报文的回复，取值包括：

  • RlyFDB：报文在此设备被成功转发到下一跳设备。

  • RlyHit：报文到达目的设备。

Forwarded
Trace报文是否被继续转发，取值包括：

  • Forwarded：Trace报文被继续转发。

  • Not Forwarded：Trace报文未被继续转发。

Egress

中间节点转发Trace报文的出接口。

Egress Action
出接口对Trace报文的处理，取值包括：

  • EgrOK表示出接口成功转发Trace报文。

  • 本字段为空表示出接口转发未转发Trace报文或转发不成功。
