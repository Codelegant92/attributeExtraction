mpls te vpn-binding vpn-instance flow-queue

命令功能

mpls te vpn-binding vpn-instance命令用来将BGP MPLS IP VPN静态绑定TE。

undo mpls te vpn-binding vpn-instance命令用来删除BGP MPLS IP VPN静态绑定TE。

命令格式

mpls te vpn-binding vpn-instance vpn-instance-name cir cir pir pir [ flow-queue flow-queue-name ]

undo mpls te vpn-binding vpn-instance vpn-instance-name

参数说明

vpn-instance vpn-instance-name：vpn-instance名称。字符串格式，长度范围是1～31。

cir cir：VPN的承诺信息速率。取值范围是100～10000000，单位是kbit/s。

pir pir：VPN的峰值信息速率。取值范围是100～10000000，单位是kbit/s。

flow-queue flow-queue-name：Flow-queue模板名称。字符串格式，长度范围是1～31。

使用指南

mpls te vpn-binding vpn-instance命令用来将BGP MPLS IP VPN绑定到TE隧道上，并配置VPN的CIR、PIR和Flow-queue 模板。其中CIR限制VPN的承诺信息速率，PIR限制VPN的峰值信息速率，Flow-queue模板配置VPN中各种优先级流量的带宽比例。

同一个TE隧道接口下，该命令与多CT配置和模板配置互斥。

当TE隧道同时配置了热备份和RRVPN，且主LSP处于热备份Inuse状态时，不能同时修改主隧道的Modify属性和RRVPN的子隧道属性，否则配置无法生效，即命令mpls te commit执行不成功。其中，主隧道的Modify属性包括：
  • 环路检测
  • RRO记录
  • 资源允许被抢占
  • 允许抢占别的LSP的资源
  • 亲和属性
  • 亲和属性掩码
  • 带宽
  • 隧道备份类型
  • 跳数限制
  • CAR策略
  • 路由锁定
  • 显示路径
  • 快速重路由属性及Bypass隧道的带宽
  • 自动快速重路由的Bypass隧道的属性

使用实例

# 在TE下绑定VPN1，CIR为2000kbit/s，PIR为3000kbit/s，Flow-queue模板使用已经配置的fq1。

<Quidway> system
[Quidway] interface Tunnel 2/0/7
[Quidway- Tunnel2/0/7] mpls te vpn-binding vpn-instance vpn1 cir 2000 pir 3000 flow-queue fq1
