destination-address

命令功能

destination-address命令用来配置NQA测试例的目的地址。

undo destination-address命令用来取消NQA测试例的目的地址。

命令格式

destination-address ipv4 ip-address [ lsp-masklen masklen | lsp-loopback loopback-address ] *

destination-address mac mac-address

destination-address remote-mep mep-id remote-mep

destination-address url urlstring

undo destination-address

参数说明

ipv4 ip-address：设置目的地址为IPv4类型。地址采用点分十进制格式。

lsp-masklen masklen：设置LSP IPv4地址前缀的掩码长度。取值范围是0～32。

lsp-loopback loopback-address：设置MPLS Ping报文头中127/8系列的IP地址。

mac mac-address：设置单播mac地址。地址采用H-H-H格式，每个H代表一个十六位数字。

remote-mep mep-id remote-mep ：设置对端ID。整数形式，取值范围是1～8191。

url urlstring：设置目的地址为URL类型。url字符串长度范围是1～230。

使用指南

只有LSP类型的测试例可以配置LSP参数。

只有HTTP和DNS类型的测试例可以将目的地址配置为URL类型。

DNS类型测试例的目的地址只能是URL，不能是IPv4地址。

不能修改正在执行的测试例的目的地址。

使用实例

# 设置测试例user test的目的地址。

<Quidway> system-view
[Quidway] nqa test-instance user test
[Quidway-nqa-user-test] test-type jitter
[Quidway-nqa-user-test] destination-address ipv4 1.1.1.1
