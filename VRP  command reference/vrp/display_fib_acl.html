display fib acl

命令功能

display fib acl命令用来过滤显示FIB信息，根据所输入的访问控制列表数字把通过了该过滤规则的FIB表项按照一定格式显示出来。

命令格式

display fib [ vpn-instance vpn-instance-name ] acl acl-number [ verbose ]

参数说明

vpn-instance vpn-instance-name：VPN实例的名称。字符串形式，区分大小写，长度范围是1～31。

acl-number：指定ACL规则编号。整数形式，取值范围是2000～2999。

verbose：表示查看转发信息表的详细信息。

使用指南

在配置该命令时，如果找不到编号为2000～2999之间的ACL，则提示如下内容：

Error: Number ACL not configured or no rule exist in the ACL

Route entry matched by access-list

与该过滤规则匹配的FIB表项

Summary counts

总计数

Destination/Mask

目的地址/掩码长度

Nexthop

下一跳

Flag

当前标志，G、H、U、S、D的组合。G表示Gateway（网关路由）H表示Host（主机路由）U表示Up（可用路由）S表示Static（静态路由）D表示Dynamic（动态路由）

TimeStamp

时间戳，表示该表项存在的时间，单位是秒。

Interface

到目的地址的出接口。

TunnelID

表示转发表项索引。用于连接上行板和下行板之间的转发表项。该值不为0时，表示匹配该项的报文通过MPLS隧道转发。该值为0时，表示报文不通过MPLS隧道转发。
