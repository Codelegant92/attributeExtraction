acl（用户界面视图）

命令功能

acl命令用来通过引用访问控制列表，对提供各种终端服务的用户界面设置呼入呼出权限限制。

undo acl命令用来取消用户界面的呼入呼出权限限制。

缺省情况下，不对用户界面的呼入呼出进行限制。

命令格式

acl [ ipv6 ] acl-number { inbound | outbound }

undo acl [ ipv6 ] { inbound | outbound }

参数说明

acl-number：地址访问控制列表号，取值范围是2000～3999。

inbound：表示对用户界面的呼入进行限制，即限制某个地址或地址段的用户登录到路由器。

outbound：表示对用户界面的呼出进行限制，即限制已经登录的用户登录到其他路由器。

缺省级别

3：管理级

使用指南

在执行该命令前，需要先执行acl命令和rule命令配置完毕访问控制列表。

一个用户界面只能配置一个同类的ACL，不会冲突。即IPv4的inbound和outbound，IPv6的inbound和outbound，共4类，每类最多只能配一个。

如果ACL编号是2000～2999，则表示使用基本访问控制列表限制源地址；如果ACL编号是3000～3999，则表示使用高级访问控制列表同时限制源地址和目的地址。

配置生效后，该用户界面的所有用户均受此ACL限制。

使用实例

# 在VTY0用户界面上，限制Telnet呼出。

<Quidway> system-view
[Quidway] user-interface vty 0
[Quidway-ui-vty0] acl 2000 outbound

# 在VTY0用户界面上，取消对Telnet呼出的限制。

<Quidway> system-view
[Quidway] user-interface vty 0
[Quidway-ui-vty0] undo acl outbound
