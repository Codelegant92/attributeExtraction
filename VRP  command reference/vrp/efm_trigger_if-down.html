efm trigger if-down

命令功能

efm trigger if-down命令用来配置EFM OAM和当前接口联动。

undo efm trigger if-down命令用来取消EFM OAM和当前接口联动的功能。

缺省情况下，接口不和EFM OAM联动。

命令格式

efm trigger if-down

undo efm trigger if-down

参数说明

无

视图

Ethernet接口视图、GE接口视图

缺省级别

2：配置级

使用指南

如果使能了EFM OAM和当前接口联动功能，当EFM OAM检测到链路连通性故障后，相应接口的接口协议状态变为efm Down，接口下除EFM协议报文外的所有其他报文都无法被转发，二三层业务将全部被阻塞。因此，一旦启用EFM OAM和当前接口联动功能，将可能对业务产生很大影响。如果当前接口通过EFM OAM检测到链路连通性故障恢复，则恢复该接口的所有转发报文，并通知该接口上的所有二三层业务作相应处理。

故障恢复后，接口协议状态重新变为Up。

此外，请务必确保两端EFM OAM协议状态变为detect之后再配置EFM OAM和当前接口联动功能。

如果由于人为误操作导致二三层业务被阻塞，可通过在接口视图下执行命令undo efm trigger if-down恢复业务。

当前接口必须已使用efm enable命令使能了EFM OAM。

使用实例

# 配置EFM OAM和接口GigabitEthernet1/0/1联动。

<Quidway> system-view
[Quidway] efm enable
[Quidway] interface gigabitethernet1/0/1
[Quidway-GigabitEthernet1/0/1] efm enable
[Quidway-GigabitEthernet1/0/1] efm trigger if-down
