display interface ip-trunk

命令功能

display interface ip-trunk命令用来查看IP-Trunk接口的状态信息。

命令格式

display interface ip-trunk [ trunk-id ] [ | count ] [ | { begin | include | exclude } regular-expression ]

参数说明

trunk-id：端口聚合的IP-Trunk ID，取值范围是0～63。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。字符串格式，支持空格，区分大小写。

使用指南

查看IP-Trunk接口的信息时，可以查看到该接口下捆绑的成员端口的状态和权重信息。

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

项目

描述

IP-Trunk1 current state :
捆绑接口IP-Trunk1的当前状态。

  • UP：接口处于正常启动的状态。

  • DOWN：接口在物理上出现故障。

  • Administratively down：如果网络管理员在该接口执行shutdown命令，将显示状态为Administratively down。

Line protocol current state :

接口当前的链路状态。

Last line protocol up time:

接口的协议状态最近一次变为Up的时间。

Hash arithmetic :
散列算法的依据。有两种：

  • According to flow：根据流进行散列

  • According to packet all：根据所有包进行散列

The Maximum Transmit Unit is

接口的最大传输单元。

Link layer protocol is

成员接口的链路层协议类型。

Physical is IP_TRUNK

捆绑接口的类型是IP-Trunk。

Last 300 seconds input/output rate

这一字段显示了在前5分钟内通过接口发送和接收的字节速率和报文速率。

Input/Output

接口收到或发送报文的方向。

packets

接口接收或发送总报文数。

bytes

接口接收或发送总字节数。

unicast

接口接收或发送的单播报文的数。

broadcasts

接口接收或发送的广播报文的数。

multicasts

接口接收或发送的组播报文的数。

errors

接口在物理层检测时发现的错误报文数目。

drops

在物理层丢弃的报文数。

unknown protocol

接收报文时，在物理层检测时发现的不能识别的协议报文数。

PortName

IP-Trunk接口中的成员口名称。

Status

当前成员端口的状态。

Weight

成员端口的权重。

The Number of Ports in Trunk :

IP-Trunk接口中成员端口的数量。

The Number of UP Ports in Trunk :

IP-Trunk接口中成员端口状态是Up的端口数量。
