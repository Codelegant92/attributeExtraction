ping mac-8021ag

命令功能

ping mac-8021ag命令用来检测当前MA内本地MEP和远端MEP或MIP之间的连通性，这种检测称为802.1ag MAC Ping。

命令格式

ping mac-8021ag { mac mac-address | remote-mep mep-id mep-id } [ -c count | interface interface-type interface-number | -s packetsize | -t timeout ] *

参数说明

mac mac-address：指定目的节点所在设备的桥MAC地址。mac-address表示目的节点所在设备的桥MAC地址，格式为H-H-H，其中H为1至4位的十六进制数。不能为广播MAC地址或组播MAC地址。这里的目的节点可以是MEP，也可以是MIP。

remote-mep mep-id mep-id：指定RMEP的ID。mep-id表示RMEP的ID，整数形式，取值范围是1～8191。该RMEP在当前设备上必须已使用remote-mep命令创建，且创建时指定了该RMEP所在设备的桥MAC地址。

-c count：指定Ping的次数。count表示Ping的次数，整数形式，取值范围是1～4294967295，缺省值是5。

interface interface-type interface-number：指定本设备发送Ping报文的出接口。interface-type表示接口类型，取值包括：ethernet、eth-trunk和gigabitethernet。interface-number表示接口编号。

-s packetsize：指定发送的Ping报文的大小。packetsize表示Ping报文的大小，整数形式，取值范围是95～1500，单位是字节，缺省值是95。该大小是指去掉二层报文头后的报文大小。

-t timeout：指定等待应答报文的超时时间。timeout表示等待应答报文的超时时间，整数形式，取值范围是1～65535，单位是ms，缺省值是2000ms。

使用指南

当需要手动检测两台或多台设备间的连通性时，可使用本命令。

802.1ag MAC Ping操作由当前MA内的本地MEP发起，目的节点可以是MEP，也可以是MIP。

进行802.1ag MAC Ping操作时：

  • 当前MA必须已关联了VLAN或VSI。

  • 当前MA必须已配置了MEP。

  • 如果指定出接口，则该接口不能是配置了inward型MEP的接口。

使用实例

# 从macustomer内的MEP ping桥MAC地址为0001-0300-0204的MIP，Ping两次，Ping报文的大小为112字节。

<Quidway> system-view
[Quidway] cfm enable
[Quidway] cfm md mdcustomer
[Quidway-md-mdcustomer] ma macustomer
[Quidway-md-mdcustomer-ma-macustomer] ping mac-8021ag mac 0001-0300-0204 -c 2 -s 112
Reply from 0001-0300-0204: byte = 112 time = 9ms
