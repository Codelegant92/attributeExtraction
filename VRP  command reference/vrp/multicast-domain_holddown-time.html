multicast-domain holddown-time

命令功能

multicast-domain holddown-time命令用来配置从Switch-MDT切换回Share-MDT的延迟时间。

undo multicast-domain holddown-time命令用来恢复延迟时间为缺省值。

缺省值是60秒。

命令格式

multicast-domain holddown-time interval

undo multicast-domain holddown-time

参数说明

interval：表示切换回Share-MDT的延迟时间。整数形式，取值范围是0～180，单位是秒。

使用指南

如果该VPN中尚未配置share-group，则不允许配置本命令。

在某些情况下，私网组播数据的转发速率会在切换阈值上下振荡。为了避免组播数据流在Switch-MDT与Share-MDT之间频繁切换，当系统经过计算发现转发速率低于阈值后，并不立即执行切换，而是启动holddown计时器，超时时间为该命令的配置值。在holddown计时器超时前，系统将继续检测数据转发速率。如果速率始终低于切换阈值则切换回Share-MDT，否则继续使用Switch-MDT进行报文转发。

在实际的切换中，从流量变化到设备感知存在1～30秒的额外时间滞后（即：从数据转发速率发生变化，到系统启动holddown计时器，存在1～30秒的延时），而这种时间滞后是不计入holddown-time中的。因此，从数据转发速率发生变化，到系统完成切换，总的延迟时间将较holddown时间长1～30秒。

将interval设置为0时，系统发现转发速率低于阈值后，立即从Switch-MDT切换回Share-MDT。

使用实例

# 配置切换回Share-MDT的延迟时间为80秒。

<Quidway> system-view
[Quidway] ip vpn-instance mytest
[Quidway-vpn-instance-mytest] multicast-domain holddown-time 80
