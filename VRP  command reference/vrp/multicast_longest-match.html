multicast longest-match

命令功能

multicast longest-match命令用来配置按照最长匹配来选择RPF路由。

undo multicast longest-match命令用来恢复缺省配置。

缺省情况下，选择优先级最高的路由作为RPF路由。

命令格式

multicast longest-match

undo multicast longest-match

参数说明

无

视图

系统视图、VPN实例视图

缺省级别

2：配置级

使用指南

当存在多条最优单播路由时，选取RPF路由的规则是：

  • 缺省情况下，选择优先级最高的一条作为RPF路由。

  • 如果配置了multicast longest-match，则选择掩码最长的一条作为RPF路由。

  • 如果配置了multicast load-splitting，则在多条最优单播路由之间实现组播负载分担。

使用实例

# 配置公网实例按照最长匹配原则选择路由。

<Quidway> system-view
[Quidway] multicast longest-match

# 配置VPN实例mytest按照最长匹配原则选择路由。

<Quidway> system-view
[Quidway] ip vpn-instance mytest
[Quidway-vpn-instance-mytest] multicast longest-match
