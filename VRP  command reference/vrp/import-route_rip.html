import-route (RIP)

命令功能

import-route命令用来从其它路由协议引入路由。

undo import-route命令用来取消从其它路由协议引入路由。

命令格式

import-route protocol [ process-id ] [ cost { cost | transparent }] [ route-policy route-policy-name ]

undo import-route protocol [ process-id ]

参数说明

protocol：可引入的外部路由协议名称，目前RIP可引入的路由包括：direct、static、rip、ospf、isis和bgp。

process-id：进程号。对于路由协议rip、ospf、isis来说，需要指定进程号。

cost：引入路由的开销值。整数形式，取值范围是0～15。

transparent：该参数只在引入BGP路由时有效，引入路由的开销值为BGP路由的MED值。

route-policy route-policy-name：引入路由时指定路由策略名称。字符串形式，不支持空格，区分大小写，长度范围是1～40。

缺省级别

2：配置级

使用指南

通过配置路由策略命令，可以在路由引入时只引入特定的路由，并可设置路由的属性。

可以通过以下三条命令设置引入路由的开销值，其优先级依次递减：

  • 通过apply cost命令设置的路由开销值。

  • 通过import-route（RIP）命令设置的引入路由的开销值。

  • 通过default-cost (RIP)命令设置的缺省路由开销值。

说明：

RIP协议规定的Tag字段长度为16bits，其他路由协议的Tag字段长度为32bits。如果在引入其他路由协议时，应用的路由策略中使用Tag，则应确保Tag值不超过65535，否则将导致路由策略失效或者产生错误的匹配结果。

使用实例

# 从IS-IS进程7中引入路由，并设置其开销值为7。

<Quidway> system-view
[Quidway] rip 1
[Quidway-rip-1] import-route isis 7 cost 7
