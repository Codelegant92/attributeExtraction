mip create-type（MD视图）

命令功能

mip create-type命令用来配置MD或默认MD内的MIP生成规则。

undo mip create-type命令用来将MD或默认MD内的MIP生成规则恢复为缺省值。

缺省情况下，MD或默认MD内的MIP生成规则为none，即不生成MIP。

命令格式

mip create-type { default | explicit | none }

undo mip create-type

参数说明

default：如果端口不存在更高级别的MEP，并且不存在更低级别的MIP，则可以在该端口上创建MIP。在本类型下，接口上没有配置MEP也可创建MIP。

explicit：如果端口存在更低级的MEP但不存在更高级别的MEP，而且不存在更低级别的MIP，则可以创建MIP。在本类型下，接口上只有已配置了更低级别的MEP才可能创建MIP。

none：如果端口的MIP的生成规则为none类型，即不自动创建MIP。

使用指南

现如今，802.1ag协议分为draft7草案版本和standard2007标准版本，本命令只支持standard2007标准版本使用。

当MD中的设备都使能了CCM发送功能后，各个MEP定期发送CCM报文。当某个MEP连续3个CCM发送周期没有接收到另一个远端MEP发送的CCM报文，则认为和该MEP之间发生了连通性故障。为了确定故障的位置，需要用MIP节点来定位。

在MD或默认MD内配置MIP生成规则后，所属MD或默认MD内的所有端口都使用所配置的MIP生成规则生成MIP节点。

配置了MIP的生成规则为default类型或explicit类型后，设备会根据规则自动创建MIP。

说明：

本命令仅支持所属MD或默认MD内的二层端口上配置MIP创建规则。

使用实例

# 配置一个名称为customer的MD内MIP生成规则为none。

<Quidway> system-view
[Quidway] cfm md customer level 1
[Quidway-md-customer] mip create-type none

# 配置默认级别MD内MIP生成规则为default。

<Quidway> system-view
[Quidway] cfm default md
[Quidway-default-md] mip create-type default
