peer sa-request-policy

命令功能

peer sa-request-policy命令用来配置路由器响应由指定MSDP对等体发出的SA请求的策略。一旦SA请求通过过滤，立即回复SA消息。

undo peer sa-request-policy命令用来恢复缺省配置。

缺省情况下，路由器响应所有MSDP对等体的所有SA请求。

命令格式

peer peer-address sa-request-policy [ acl basic-acl-number ]

undo peer peer-address sa-request-policy

参数说明

peer-address：表示发出该SA请求消息的MSDP对等体地址。点分十进制形式。

acl basic-acl-number：表示基本IP访问控制列表号，描述组播组地址。整数形式，取值范围是2000～2999。

缺省级别

2：配置级

使用指南

如果没有指定访问控制列表，则忽略来自该MSDP对等体的所有SA请求。如果指定了访问控制列表，只处理符合该列表的组的SA请求消息，其他的被忽略。

peer sa-request-policy命令与acl命令配合使用。在ACL视图下，通过rule命令的source参数定义MSDP对等体转发SA消息的组地址。

使用实例

# 在公网实例中，配置过滤SA请求消息的访问控制列表，SA消息来自MSDP peer 175.58.6.5，只接收组地址范围是225.1.1.0/24的活动源的SA请求消息。

<Quidway> system-view
[Quidway] acl number 2001
[Quidway-acl-basic-2001] rule permit source 225.1.1.0 0.0.0.255
[Quidway-acl-basic-2001] quit
[Quidway] msdp
[Quidway-msdp] peer 175.58.6.5 sa-request-policy acl 2001
