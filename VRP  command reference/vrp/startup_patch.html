startup patch

命令功能

startup patch命令用来指定下次启动的补丁包文件名。

命令格式

startup patch file-name [ slave-board ]

参数说明

file-name：指定下次启动的补丁包文件名，字符串形式，格式为[ drive-name ][ file-name ]，长度范围1～48。如果未指定drive-name，则此值为默认的设备名。

slave-board：指定下次启动后备用主控板的补丁包文件名。

使用指南

指定补丁包文件名，下次启动后系统会自动加载该补丁。

执行该命令时，建议与命令patch-state run配合使用。

说明：

目前此命令仅在NE40E、NE80E、NE5000E上可用。

使用实例

# 指定下次启动的补丁包文件。

<Quidway> startup patch cfcard:\patch.pat
