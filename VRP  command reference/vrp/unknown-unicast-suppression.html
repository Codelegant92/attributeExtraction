unknown-unicast-suppression

命令功能

unknown-unicast-suppression命令用来对接口的单播报文进行速率限制。

undo unknown-unicast-suppression命令用来取消对接口的单播报文进行速率限制。

缺省情况下，不对接口的单播报文进行速率限制。

命令格式

unknown-unicast-suppression { limit-percent | value limit-value } { inbound | outbound }

undo unknown-unicast-suppression { inbound | outbound }

参数说明

limit-percent：速率限制占端口带宽的百分比。整数形式，取值范围是0～100，缺省值为100，表示不限制单播报文速率。

value limit-value：指定端口带宽的速率限制值。十进制整数形式，取值范围是0～1000，单位是Mpbs。

inbound：限制上行入口未知单播报文的速率。

outbound：限制下行出口未知单播报文的速率。

视图

Ethernet接口视图、GE接口视图、Eth-Trunk接口视图

缺省级别

2：配置级

使用指南

当单播流量超过配置的阈值时，系统将丢弃多余的组播流量，使组播流量降低到合理的范围，保证网络业务的正常运行。

说明：
  • 本命令只能用于NE40E、NE80E、NE5000E产品。
  • 对于NE40E、NE80E、NE5000E产品，流量抑制会申请一组CAR资源，每块单板分配给二层接口的CAR资源最多为512组。当耗尽单板的CAR资源后再配置此命令，命令行将提示CAR资源已经耗尽，配置失败。

使用实例

# 配置端口上行未知单播速率限制为50%。

<Quidway> system view
[Quidway] interface gigabitethernet 6/0/0
[Quidway-GigabitEthernet6/0/0] portswitch
[Quidway-GigabitEthernet6/0/0] unknown-unicast-suppression 50 inbound
