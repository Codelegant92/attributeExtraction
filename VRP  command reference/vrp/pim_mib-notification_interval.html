pim mib-notification interval

命令功能

pim mib-notification interval命令用来设置PIM发送各种事件Trap报文的时间间隔。

undo pim mib-notification interval命令用来恢复PIM发送各种Trap报文的时间间隔为缺省值。

命令格式

pim mib-notification interval { interface-election-dr election-value | invalid-join-prune jp-value | invalid-register register-value | neighbor-loss loss-value | rp-mapping-change change-value }

undo pim mib-notification interval { all | interface-election-dr | invalid-join-prune | invalid-register | neighbor-loss | rp-mapping-change }

参数说明

interface-election-dr election-value：表示接口当选DR的Trap最小发送间隔。整数形式，取值范围是0～65535，单位是秒。缺省值是65535秒。

invalid-join-prune jp-value：表示无效的Join/Prune报文Trap的最小发送间隔。整数形式，取值范围是10～65535，单位是秒。缺省值是65535秒。

invalid-register register-value：表示收到无效注册报文Trap的最小发送间隔。整数形式，取值范围是10～65535，单位是秒。缺省值是65535秒。

neighbor-loss loss-value：表示邻居丢失Trap的最小发送间隔。整数形式，取值范围是0～65535，单位是秒。缺省值是0秒。

rp-mapping-change change-value：表示RP-Mapping改变Trap的最小发送间隔。整数形式，取值范围是0～65535，单位是秒。缺省值是65535秒。

使用指南

pim mib-notification interval用来设置发送Notification的间隔。当事件发生时，如果距离上一次发送Notification的间隔小于设定值，则不发送Notification。

发送间隔缺省值是0秒的事件发生时立即发送告警。

发送间隔缺省值是65535秒的事件发生时不会发送告警。使用此命令将间隔设置为较小的值后，事件发生时才会发送告警。

使用实例

# 在PIM模式下配置发送无效的Join/Prune报文告警间隔。

<Quidway> system-view
[Quidway] multicast-mib
[Quidway-multicast-mib] pim mib-notification interval invalid-join-prune 50
