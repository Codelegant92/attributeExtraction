import-route (OSPFv3)

命令功能

import-route命令用来引入外部路由信息，在执行引入外部路由前，要求引入路由的OSPFV3进程是active的。

undo import-route命令用来取消对外部路由信息的引入。

缺省情况下，不引入其它协议的路由信息。

命令格式

import-route protocol [ process-id ] [ cost cost | type type | route-policy route-policy-name ] *

undo import-route protocol [ process-id ]

参数说明

protocol：可引入的源路由协议，目前可为direct、bgp、static、ripng、isis或ospfv3。

process-id：引入的源路由协议进程号，当引入路由协议为ripng、isis或ospfv3时，需要指定该值。

cost cost：引入路由的开销值。整数形式，取值范围是0～16777214。

type type：引入路由的类型，可以取1或2。

route-policy route-policy-name：指定路由策略名称，只有满足指定匹配条件的路由才被引入。字符串形式，长度范围是1～40个字符。

使用实例

# 指定引入RIPng路由为第二类路由，且路由开销值为50。

<Quidway> system-view
[Quidway] ospfv3 1
[Quidway-ospfv3-1] import-route ripng 1 type 2 cost 50

# OSPFv3进程100引入OSPFv3进程160发现的路由。

<Quidway> system-view
[Quidway] ospfv3 100
[Quidway-ospfv3-100] import-route ospfv3 160
