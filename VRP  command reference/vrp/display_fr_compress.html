display fr compress

命令功能

display fr compress命令用来查看帧中继FRF.9压缩的统计信息，如果不指定接口，将显示所有接口上的FRF.9压缩统计信息。

命令格式

display fr compress [ interface interface-type interface-number ]

参数说明

interface-type：查看指定接口类型帧中继FRF.9压缩信息。

interface-number：查看指定接口类型下指定接口编号的帧中继FRF.9压缩信息。不能指定为子接口，只能指定为主接口。如果指定主接口的接口编号，会同时显示该主接口下的子接口的FRF.9压缩信息。

使用指南

在监控接口的状态或检查接口的故障原因时，可执行此命令获取接口的状态信息和统计信息。用户可以根据这些信息进行流量统计和接口的故障诊断等。

使用实例

# 查看帧中继压缩的统计信息。

<Quidway> display fr compress interface Serial 1/0/0
Serial1/0/0.1 -DLCI:200
  xmt:
    Compressed packets  0
    Uncompressed packets  10
  rcv:
    Compressed packets  Total/NotDiscard/Decompress 0/0/0
    Uncompressed packets  Total/NotDiscard 8/8
  Compressed packet:
    Original bytes  xmt/rcv 0/0
    Compressed bytes  xmt/rcv 0/0
    1 min avg ratio xmt/rcv  0.000/0.000
    5 min avg ratio xmt/rcv  0.000/0.000
表1 display fr compress命令输出信息描述

项目

描述

Serial1/0/0.1 -DLCI:

帧中继接口编号和接口上配置的数据链路标识。

xmt:

发送报文的字节数。

Compressed packets:

发送的报文中被压缩的字节数。

uncompressed packets:

发送的报文中没有被压缩的字节数。

rcv:

接收报文的字节数。

Compressed packets Total/NotDiscard/Decompress

接收到的报文中压缩字节数 接收到的报文总数/没有丢弃的报文数/解压缩的报文数。

Original bytes xmt/rcv

原始报文数：接收/发送。

compressed bytes xmt/rcv

被压缩的发送/接受的字节数。

1 min avg ratio xmt/rcv

1分钟发送/接受的平均压缩率。

5 min avg ratio xmt/rcv

5分钟发送/接受的平均压缩率。
