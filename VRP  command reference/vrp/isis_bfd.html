isis bfd

命令功能

isis bfd 命令用来在指定接口上配置BFD会话的参数值。

undo isis bfd 命令用来恢复指定接口上BFD会话参数的缺省值。

命令格式

isis bfd { min-rx-interval receive-interval | min-tx-interval transmit-interval | detect-multiplier multiplier-value } *

undo isis bfd { min-rx-interval | min-tx-interval | detect-multiplier } *

参数说明

min-rx-interval receive-interval：指定期望从对端接收BFD报文的最小接收间隔。receive-interval取值范围是100～1000，单位是毫秒。缺省值是1000毫秒。

min-tx-interval transmit-interval：指定向对端发送BFD报文的最小发送间隔。transmit- interval取值范围是100～1000，单位是毫秒。缺省值是1000毫秒。

detect-multiplier multiplier-value：指定本地检测倍数。multiplier-value取值范围为3～50，缺省值是3。

说明：

不同产品，上述最大、最小值可能不同。

使用指南

本地的min-rx-interval值与对端的min-tx-interval值进行协商得到receive-interval，如果在receive-interval × multiplier-value时间间隔内没有收到对方发送的BFD报文，就宣告邻居Down掉。

接口上创建BFD会话的前提条件是：配置全局BFD特性和执行了isis bfd enable命令。

接口配置的BFD特性优先级高于进程配置的BFD特性优先级。如果打开了接口的BFD开关，则按照接口上BFD参数建立BFD会话。

只有在该接口上使能了BFD特性后，所配置的BFD会话的参数才能生效。

使用实例

# 在POS 1/0/0接口上使能BFD特性，并指定最小接收间隔为400ms，本地检测倍数为4。

<Quidway> system-view
[Quidway] interface pos1/0/0
[Quidway-Pos1/0/0] isis bfd enable
[Quidway-Pos1/0/0] isis bfd min-rx-interval 400 detect-multiplier 4
