display ipsec sa

命令功能

display ipsec sa命令用来查看安全联盟的相关信息。

命令格式

display ipsec sa [ brief | remote ip-address | policy policy-name [ seq-number ] | duration ]

参数说明

brief：表示显示所有的安全联盟的简要信息。

remote ip-address：表示显示对端地址为ip-address的安全联盟的信息。

policy：表示显示由指定安全策略创建的安全联盟的信息。

policy-name：指定安全策略的名字。

seq-number：指定安全策略的顺序号。

duration：表示显示全局的安全联盟生存周期。

使用指南

display ipsec sa brief命令可以快速查看所有已经建立的安全联盟参数，显示所有的安全联盟的简要信息，显示格式为简要格式，简要信息包括：

  • 源地址

  • 目的地址

  • 安全参数索引

  • VPN实例名称

  • IPSec采用的安全协议

  • 安全协议采用的认证算法和加密算法

使用remote和policy参数均显示安全联盟的详细信息，显示格式是先显示安全策略的部分信息，再显示此安全策略中的安全联盟的详细信息。

使用duration参数显示全局的安全联盟的生存时间，包括“基于时间”和“基于流量”两种类型。

当未指定任何参数时显示所有的安全联盟的信息。

主要字段

描述

Src Address

本端的IP地址

Dst Address

对端的IP地址

SPI

安全参数索引

VPN

VPN实例名称

Protocol

IPSec采用的安全协议

Algorithm

安全协议采用的认证算法和加密算法，其中，以“E:”开头表示加密算法；以“A:”开头表示认证算法

主要字段

描述

Interface

应用了安全策略的接口

path MTU

从该接口发送出去的最大IP数据报文长度

ipsec policy

采用的安全策略，包括策略名、顺序号和协商方式

connection id

安全通道标识符

in use settings

IPSec采用的模式，有两种：传输（transport）和隧道（tunnel）模式

tunnel local

本端的IP地址

tunnel remote

对端的IP地址

inbound

输入端SA的信息

proposal

安全策略所引用的提议

sa remaining key duration

安全联盟剩余的存活时间

max received sequence-number

接收的报文最大序列号

outbound

输出端SA的信息

max sent sequence-number

发送的报文最大序列号
