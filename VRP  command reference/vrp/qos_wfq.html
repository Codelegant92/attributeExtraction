qos wfq

命令功能

qos wfq命令用来在接口上应用加权公平队列或修改加权平均队列的参数。

undo qos wfq命令用来恢复缺省拥塞管理机制FIFO。

命令格式

qos wfq [ queue-length max-queue-length [ queue-number total-queue-number ] ]

undo qos wfq

参数说明

max-queue-length：队列的最大长度，即每个队列中数据包的最大个数，超出后数据包将被丢弃，取值范围是1～1024，缺省值为64。

total-queue-number：队列的总数目，可取的值为：16、32、64、128、256、512、1024、2048、4096，缺省值为256。

使用指南

当接口没有应用WFQ策略时，使用该命令可以在接口上应用WFQ策略，同时可指定WFQ的参数。

如果接口已经应用了WFQ策略，使用该命令可以修改WFQ的参数。

使用实例

# 在接口GE 6/0/0上应用WFQ，并设置队列长度为100，总队列个数设置为512个。

<Quidway> system-view
[Quidway] interface GigabitEthernet6/0/0
[Quidway-GigabitEthernet6/0/0] qos wfq queue-length 100 queue-number 512
说明：

链路层使用X.25协议的物理接口不可以应用加权公平队列。其他物理接口都可以应用加权公平队列。在NE16E/08E/05的 ATM口上应用加权公平队列需要进入PVC视图。
