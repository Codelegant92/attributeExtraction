bfd bind mpls-te

命令功能

bfd bind mpls-te命令用来配置BFD检测TE隧道或与TE隧道绑定的主用或备用LSP。

undo bfd cfg-name命令用来删除指定的BFD会话。

缺省情况下，Tunnel隧道没有使用BFD检测。

命令格式

bfd cfg-name bind mpls-te interface tunnel tunnel-number [ te-lsp [ backup ] ]

undo bfd cfg-name

参数说明

cfg-name：BFD配置名。字符串形式，不支持空格，长度范围是1～15。

tunnel-number：BFD会话绑定的Tunnel接口名称。

te-lsp：BFD检测与Tunnel隧道绑定的主LSP。

backup：BFD检测与Tunnel隧道绑定的备用LSP。

缺省级别

使用指南

在配置BFD检测TE隧道之前，必须配置Tunnel接口。

目前BFD只能够检测信令类型为CR-Static或RSVP-TE的TE隧道。

BFD可以检测TE隧道，也可以检测与TE隧道绑定的主用或备用LSP。

BFD检测与Tunnel绑定的主用或备用LSP时，如果LSP的状态为Down，则不能建立BFD会话。

BFD检测TE隧道时，如果TE隧道的状态为Down，则能够创建BFD会话，但BFD会话不能Up。

一个Tunnel可能有多个LSP，当BFD检测Tunnel时，只有全部LSP都出现故障时，BFD会话的状态才为Down。

说明：

对于同一个TE隧道或同一条LSP，不能同时配置MPLS OAM和BFD检测。

使用实例

# 创建BFD会话检测Tunnel隧道，Tunnel接口是Tunnel 1/0/10。

<Quidway> system-view
[Quidway] bfd 1to4tunnel bind mpls-te interface Tunnel 1/0/10
[Quidway-bfd-lsp-session-1to4tunnel]

# 创建BFD会话检测与Tunnel绑定的主用LSP。

[Quidway] bfd 1to4rsvp bind mpls-te interface Tunnel 1/0/10 te-lsp 
[Quidway-bfd-lsp-session-1to4rsvp]

# 创建BFD会话检测与Tunnel绑定的备用LSP。

[Qudiway] bfd 1to4backup bind mpls-te interface Tunnel 1/0/10 te-lsp backup
[Quidway-bfd-lsp-session-1to4backup]
