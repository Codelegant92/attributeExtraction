ospf timer dead

命令功能

ospf timer dead命令用来设置OSPF的邻居失效时间。

undo ospf timer dead命令用来恢复邻居失效时间的缺省值。

缺省情况下，p2p、broadcast类型接口的OSPF邻居失效时间为40秒，p2mp、NBMA类型接口的OSPF邻居失效时间为120秒。

命令格式

ospf timer dead interval

undo ospf timer dead

参数说明

interval：OSPF邻居失效的时间。整数形式，取值范围是1～235926000，单位是秒。

使用指南

OSPF邻居的失效时间是指：在该时间间隔内，若未收到邻居的Hello报文，就认为该邻居已失效。dead interval的值至少应为Hello interval的值的4倍，同一网段上的路由器的dead interval值也必须相同。

OSPF协议不支持NULL接口的配置。

使用实例

# 配置接口pos1/0/0上的邻居失效时间为60秒。

<Quidway> system-view 
[Quidway] interface pos 1/0/0
[Quidway-Pos1/0/0] ospf timer dead 60
