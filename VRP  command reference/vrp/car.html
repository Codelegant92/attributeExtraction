car

命令功能

car命令用来为流行为配置使用流量监管。

undo car命令用来取消流量监管配置。

命令格式

car cir cir [ cbs cbs ebs ebs ] [ green action [ red action ] ]

undo car

参数说明

cir：承诺信息速率。

cir：流量的平均速率，取值范围是8000～155000000 bps。

cbs：承诺突发尺寸，每个时间间隔可发送的位数，取值范围是15000～155000000bit。cbs的取值应大于等于cir取值的1/2，当cir>30000bit/s时，缺省取值是cir的1/2，当cir<30000bit/s时，缺省取值是15000bit。

ebs：过度突发尺寸，取值范围是0～155000000bit，取值范围是0～155000000bit，缺省取值是0，即只采用一个令牌桶监管。

green：数据包的流量符合承诺速率时对数据包采取的动作，缺省值为pass。

red：数据包的流量不符合承诺速率时对数据包采取的动作，缺省值为discard。

action：对数据包采取的动作，有以下几种：

  • discard：丢弃数据包。

  • remark-dscp-pass：设置新的DSCP值new-dscp，并发送，取值范围是0～63。

  • remark-prec-pass：设置新的IP优先级new-precedence，并发送，取值范围是0～7。

  • remark-mpls-exp-pass：设置新MPLS报文的EXP域，并发送，取值范围是0～7。

  • pass：发送数据包。

使用指南

接口上应用的策略中使用car时，可以应用到接口的入或出方向。

接口上应用配置了CAR的策略将导致原有的qos car命令失效。

如果多次使用该命令在同一个策略的类上配置，最后一次配的将覆盖以前配置的。

使用实例

# 为流行为配置使用流量监管。报文正常流量为38400bit，在第一时间可以有2倍于正常流量的突发流量通过，以后速率小于等于38400bit时正常发送，大于38400bit时，报文优先级改为0并发送。
