peer public-as-only（MBGP）

命令功能

peer public-as-only命令用来配置发送MBGP更新报文时不携带私有自治系统号。

undo peer public-as-only命令用来配置发送BGP更新报文时会携带私有自治系统号。

缺省情况下，发送MBGP更新报文时，会携带私有自治系统号。

命令格式

peer { group-name | peer-address } public-as-only

undo peer { group-name | peer-address } public-as-only

参数说明

group-name：指定MBGP对等体组的名称。

peer-address：指定MBGP对等体的地址。点分十进制形式。

视图

BGP-IPv4组播地址族视图

缺省级别

2：配置级

使用指南

该命令只能应用于EBGP对等体。

AS编号的取值范围是1～65535，其中公有AS编号范围是1～64511，私有AS编号范围是64512～65534，65535则作为保留AS编号在特殊应用中使用。

公有AS编号可以直接在Internet上使用，私有AS编号不能直接发布到Internet上，它仅在内部路由域中使用。

缺省情况下，MBGP在向对等体发布路由时会携带私有AS编号。某些情况下，私有AS编号信息并不需要被传播出去，这时可以使用此命令配置AS_Path属性中仅携带公有AS编号。

说明：

一般情况下，MBGP在发送MBGP更新报文时都是携带该自治系统号的（可能是公有的AS号，也可能是私有的AS号）。当发送MBGP更新报文时，为了使外部路由器忽略私有AS号，需配置发送MBGP更新报文时不携带私有AS号。

使用实例

# 配置当MBGP向对等体组test发送更新报文时不携带私有自治系统号。

<Quidway> system-view
[Quidway] bgp 100
[Quidway-bgp] ipv4-family multicast
[Quidway-bgp-af-multicast] peer test enable
[Quidway-bgp-af-multicast] peer test public-as-only
