static-lsp ingress tunnel-interface

命令功能

static-lsp ingress tunnel-interface命令用来创建一条与指定TE隧道绑定的静态LSP。

undo static-lsp ingress tunnel-interface命令用来删除与指定TE隧道绑定的静态LSP。

命令格式

static-lsp ingress tunnel-interface tunnel tunnel-number destination ip-address { nexthop next-hop-address | outgoing-interface interface-type interface-number } out-label out-label

undo static-lsp ingress tunnel-interface tunnel tunnel-number

参数说明

tunnel tunnel-number：信令协议为static的TE隧道接口。

destination ip-address：静态LSP的目的IP地址。

nexthop next-hop-address：下一跳IP地址。本地IP地址不能被指定为下一跳。

outgoing-interface interface-type interface-number：去往目的地址的出接口。

out-label out-label：出标签值。整数形式，取值范围由许可证文件决定。取值范围是16～1023。

使用指南

MPLS OAM可以对绑定到TE隧道的静态LSP进行连通性检测，这类静态LSP也可以用作OAM的反向通道。

在创建这类静态LSP时，Ingress节点上必须通过static-lsp ingress tunnel-interface命令将LSP与TE隧道绑定，Transit节点和Egress节点上的配置则与普通静态LSP相同。

使用实例

# 配置一条静态LSP，绑定到TE隧道Tunnel1/0/1。

<Quidway> system-view
[Quidway] static-lsp ingress tunnel-interface Tunnel1/0/1 destination 3.3.3.3 nexthop 10.1.2.2 out-label 20
