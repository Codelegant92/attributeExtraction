display ipv6 routing-table

命令功能

display ipv6 routing-table命令用来显示IPv6路由表的信息。

命令格式

display ipv6 routing-table [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table acl acl6-number [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table ipv6-address prefix-length [ longer-match ] [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table ipv6-address1 prefix-length1 ipv6-address2 prefix-length2 [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

display ipv6 routing-table ipv6-prefix ipv6-prefix-name [ verbose ] [ | count ] [ | { begin | exclude | include } regular-expression ]

参数说明

verbose：显示当前路由表中所有路由的详细信息，包括激活和未激活的路由信息。

acl：使用ACL6对输出的信息进行过滤。如果指定的ACL6不存在，将显示所有激活路由的信息。

acl6-number：基本访问控制列表ACL6的编号。

longer-match：仅显示与特定网络/掩码匹配的路由。

ipv6-address：IPv6目的地址。

prefix-length：前缀长度。

ipv6-address1 / ipv6-address2：IPv6地址。ipv6-address1和ipv6-address2共同决定一个地址范围，只有在此范围内的路由才会被显示。

prefix-length1/prefix-length2：前缀长度。

ipv6-prefix ipv6-prefix-name：指定IPv6前缀列表的名称。

|：使用正则表达式对输出信息进行过滤。

count：统计符合输出条件的信息行数。

begin：从包含匹配字符串的行开始输出。

exclude：只输出不包含匹配字符串的行。

include：只输出包含匹配字符串的行。

regular-expression：对输出信息进行匹配的正则表达式。

使用指南

显示内容包括目的地址、前缀长度、协议类型、优先级、度量值、下一跳和输出接口、隧道ID、路由携带的标签值、路由生成的时间、路由状态。

该命令（不含verbose参数）只能显示当前使用的路由，即最佳路由。

在使用display ipv6 routing-table ipv6-address prefix-length [ longer-match ] [ verbose ]显示路由信息时，可通过调整参数组合显示所需信息：

  • 指定参数ipv6-address prefix-length：显示与目的地址精确匹配的IPv6路由。

  • 指定参数ipv6-address prefix-length longer-match：显示与目的地址在输入的前缀范围内匹配的IPv6路由。前缀长度为0时，显示IPv6路由表中所有路由。

例如路由表中有4条路由，2000::20/128、2000::/100、2000::/64和1000::/64。

  • 如果使用带前缀长度的精确匹配display ipv6 routing-table 2000:: 64，只有路由2000::/64会显示。

  • 如果使用不带前缀长度的精确匹配display ipv6 routing-table 2000::，只有路由2000::/100会显示。

  • 如果使用前缀长度非零的longer-match匹配display ipv6 routing-table 2000:: 127 longer-match，只有路由2000::/100和2000::/64会显示。

  • 如果使用前缀长度为零的longer-match匹配display ipv6 routing-table 2000:: 0 longer-match，则4条路由都会显示。

当显示信息太多不方便查看时，可以选择begin、exclude或include参数指定匹配条件来过滤显示信息。

项目

描述

Destination

目的网络/主机的地址

PrefixLength

前缀长度

NextHop

下一跳IPv6地址

Preference

优先级

RelayNextHop

迭代的下一跳

Tag

路由管理标识

Neighbour

邻居的接口IP地址

ProcessID

路由协议的进程号

Interface

下一跳可达的出接口

Protocol

路由协议名称

State

该路由的当前状态

Cost

路由开销

Tunnel ID

隧道ID，值为0x0表示没有使用隧道或隧道建立不成功

Label

路由携带的标签值

Entry ID

路由在路由表中检索的关键字

Flags

路由标志位信息

Refernce Cnt

路由被引用的次数

Age

路由生成时间
