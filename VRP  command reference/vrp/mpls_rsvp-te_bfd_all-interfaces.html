mpls rsvp-te bfd all-interfaces

命令功能

mpls rsvp-te bfd all-interfaces命令用来配置RSVP-TE所有接口会话参数。

undo mpls rsvp-te bfd all-interfaces命令用来恢复为缺省配置。

命令格式

mpls rsvp-te bfd all-interfaces { min-tx-interval tx-interval | min-rx-interval rx-interval | detect-multiplier multiplier }*

undo mpls rsvp-te bfd all-interfaces { min-tx-interval | min-rx-interval | detect-multiplier }*

参数说明

min-tx-interval tx-interval：配置BFD会话参数最小发送时间间隔。整数形式，单位是毫秒，取值范围是10～1000，单位是毫秒。缺省值是10毫秒。

min-rx-interval rx-interval：配置BFD会话最小接收时间间隔。整数形式，取值范围是10～1000，单位是毫秒。缺省值是10毫秒。

detect-multiplier multiplier：配置BFD会话本地检测的倍数。BFD会话本地检测倍数的值。整数形式，取值范围是3～50，缺省值为3。

使用指南

BFD检测参数的取值范围及缺省值在BFD模块统一定义，不同产品可能有差异。

本地BFD报文实际发送时间间隔＝MAX { 本地配置的发送时间间隔，对端配置的接收时间间隔 }；本地实际接收时间间隔＝MAX { 对端配置的发送时间间隔，本地配置的接收时间间隔 }；本地检测时间＝本地接收时间间隔×对端配置的BFD检测倍数。

本地BFD报文实际发送时间间隔＝MAX { 本地配置的发送时间间隔，对端配置的接收时间间隔 }；本地实际接收时间间隔＝MAX { 对端配置的发送时间间隔，本地配置的接收时间间隔 }；本地检测时间＝本地接收时间间隔×对端配置的BFD检测倍数。

例如，

  • 本地配置的发送时间间隔为200ms，接收时间间隔为300ms，检测倍数为4

  • 对端配置的发送时间间隔为100ms，接收时间间隔为600ms，检测倍数为5

  • 本地实际的发送时间间隔为MAX { 200ms，600ms }＝600ms，接收时间间隔为MAX { 100ms，300ms }＝300ms，检测时间间隔为300ms×5＝1500ms

  • 对端实际的发送时间间隔为MAX { 100ms，300ms }＝300ms，接收时间间隔为MAX { 200ms，600ms }＝600ms，检测时间间隔为600ms×4＝2400ms

使用实例

<Quidway> system-view
