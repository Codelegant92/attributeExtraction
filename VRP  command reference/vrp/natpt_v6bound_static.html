natpt v6bound static

命令功能

natpt v6bound static命令用来配置IPv6到IPv4地址的静态映射。

undo natpt v6bound static命令用来删除IPv6到IPv4地址的静态映射。

命令格式

natpt v6bound static ipv6-address ipv4-address

undo natpt v6bound static ipv6-address ipv4-address

参数说明

ipv6-address：将IPv6源地址映射为IPv4地址，与IPv4域中的IPv4主机相通。地址格式为X:X::X:X。

ipv4-address：指定要映射的IPv4地址，点分十进制格式。

使用指南

当报文从IPv6域传输到IPv4域时，在接口上使能NAT-PT，IPv6报文的源地址需要转换为等效的IPv4地址，静态配置就为输入的IPv6报文提供了这个等效的IPv4地址。实际上，缺省情况下也同时添加了一条反方向的等效规则，因此当IPv4地址转化为IPv6地址时同样适用。

说明：

当执行undo natpt v6bound static命令时，某些会话可能仍然会使用映射直到会话超时。建议删除IPv6侧的静态映射的同时清除NAT-PT动态映射。

使用实例

# 配置IPv6地址2001::1到IPv4地址2.3.4.5的映射。

<Quidway> system-view
[Quidway] natpt v6bound static 2001::1 2.3.4.5
