display controller cpos t1

命令功能

display controller cpos t1命令用来查看指定CPOS的指定T1的物理层配置信息。

命令格式

display controller cpos cpos-number t1 t1-number

参数说明

t1-number：CPOS的T1通道号。整数形式，取值范围是1～84。

与display controller cpos命令相比，本命令可以显示对应的低阶通道的错误和告警信息，以及T1帧的错误和告警信息。

表1 display controller cpos t1命令输出信息描述

项目

描述

Cpos1/0/0 current state :

CPOS接口当前的物理状态。

Frame-format SDH, multiplex AU-3, clock master, loopback not set

CPOS接口的物理层信息：帧格式为SDH、采用AU-3的复用路径、主时钟模式（使用内部时钟信号）、没有使能环回。

Tx: J0: 0x01, J1: "NetEngine", C2: 0x02

发送的开销字节。

Rx: J0: 0x01, J1: "NetEngine", C2: 0x02

接收的开销字节。

Regenerator section:

再生段的告警和错误统计。

Multiplex section:

复用段的告警和错误统计。

Higher order path(VC-3-2):

此T1通道所属高阶通道的告警和错误统计，VC-3-2表示第2个VC-3。

Lower order path:

低阶通道的告警和错误统计。

Alarm:Error:

告警统计和错误统计。可能的错误类型有以下几类。

  • FRED：Receive Loss of Basic Frame Alignment，接收到基本帧丢失位。也可以认为是收到有红色告警错误的帧。

  • COFA：Change of Frame Alignment，帧对齐改变。

  • SEF：Severely Errored Frame，严重错帧，连续4个帧同步错误将产生一个SEF。

  • FERR：Framing Bit Error，指有Ft/Fs/FPS/FAS错误的帧。

  • CERR：CRC Error，循环冗余校验错。

  • BERR：PRBS Bit Error（随机码测试位错，只用于测试）。

  • BIP：Bit-Interleaved Parity，比特交叉奇偶校验。

  • REI：Remote Error Indication，远端错误指示。

上面前三种错误（FRED、COFA、SEF）统称为Alarm Error，简写为AERR。

Cpos4/0/0 CT1 2 is up

CPOS接口1/0/0的T1通道1当前的物理状态。

Frame-format ESF, clock master, loopback not set

T1通道的物理层信息：帧格式为ESF、主时钟模式（使用内部时钟信号）、没有使能环回。

T1 framer(2-1-1):

T1的复用过程，2-1-1依次表示此T1通道所属的VC-3编号、TUG-2编号和TU-11编号，具体的计算原理可参考与本手册配套的配置指南。
