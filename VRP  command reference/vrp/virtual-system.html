virtual-system

命令功能

virtual-system命令用来配置IS-IS进程的虚拟系统ID。没有该ID，将不会生成扩展LSP。一个IS-IS进程最多可配置50个虚拟系统ID。

undo virtual-system命令用来删除虚拟系统ID。

命令格式

virtual-system virtual-system-id

undo virtual-system virtual-system-id

参数说明

virtual-system-id：IS-IS进程的虚拟系统ID。长度是6字节（48比特），格式是XXXX.XXXX.XXXX。

缺省级别

2：配置级

使用指南

虚拟ID和普通ID一样，在整个路由域中必须唯一。

virtual-system命令是和lsp-fragments-extend命令配合使用的，只有使能了LSP分片扩展，并用reset isis all命令重启了IS-IS进程后，配置的虚拟系统ID才会生效。

如果没有使能LSP分片扩展和重启IS-IS进程，则只能对虚拟系统ID进行配置，但不会生效。

使用实例

# 设置IS-IS进程1的虚拟系统ID为2222.2222.2222。

<Quidway> system-view
[Quidway] isis
[Quidway-isis-1] network-entity 10.0000.0000.0001.00
[Quidway-isis-1] virtual-system 2222.2222.2222
