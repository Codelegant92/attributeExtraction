natpt prefix

命令功能

natpt prefix命令用来配置全局或接口的NAT-PT前缀。

undo natpt prefix命令用来取消配置。

命令格式

natpt prefix prefix [ interface interface-type interface-number [ nexthop ipv4-addresse ] ]

undo natpt prefix prefix

参数说明

prefix：指定IPv6地址前缀，格式为X:X::X:X，前缀长度只能为96位。

interface-type interface-number：指定接口类型及接口编号。

ipv4-address：指定下一跳的IPv4地址。

使用指南

如果接口和下一跳均未配置，则此前缀为全局前缀，否则为接口前缀。

收到IPv6侧报文后，检查报文目的地址的前缀。

  • 如果为全局前缀，根据FIB转发转换后的报文。
  • 如果为接口前缀，根据接口或下一跳转发转换后的报文。

由于首先检查IPv4侧报文在接收接口的NAT-PT前缀，因此接口NAT-PT前缀有较高的优先级。

说明：

  • NAT-PT前缀必须不同于IPv6接口地址前缀。

  • 使用undo natpt prefix命令可能会影响某些动态映射的转换。

使用实例

# 配置全局的NAT-PT前缀。

<Quidway> system-view
[Quidway] natpt prefix 2001::

# 配置特定接口的NAT-PT的前缀。

[Quidway] natpt prefix 3001:: interface gigabitethernet 2/0/0

# 配置含有下一跳的NAT-PT前缀。

[Quidway] natpt prefix 3001:: interface pos 3/0/0 nexthop 128.23.0.1
