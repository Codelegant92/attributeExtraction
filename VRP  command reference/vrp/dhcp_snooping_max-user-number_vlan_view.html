dhcp snooping max-user-number（VLAN视图）

命令功能

dhcp snooping max-user-number命令用来限制指定VLAN或VLAN＋端口下用户数的最大值。

undo dhcp snooping max-user-number命令用来取消限制指定VLAN或VLAN＋端口下用户数的最大值。

缺省情况下，VLAN或VLAN＋端口下没有配置对用户数量的限制。

命令格式

dhcp snooping max-user-number user-number [ interface interface-type interface-number ]

undo dhcp snooping max-user-number [ interface interface-type interface-number ]

参数说明

max-user-number user-number：用户数的最大数量。十进制整数形式，取值范围是1～16384。

interface interface-type interface-number：接口类型和接口编号。支持此功能的接口包括Ethernet接口、GigabitEthernet接口、Eth-Trunk接口。

缺省级别

2：配置级

使用指南

当配置的DHCP Snooping静态绑定表或动态生成的绑定表表项达到配置的最大用户数限制数量，任何用户将无法成功申请到IP地址。

为了抑制用户恶意申请IP地址，可配置限制用户数的功能。该功能基于VLAN或VLAN＋端口实现。

当用户需要修改已经配置的用户数限制数量时，重新配置即可，只有当前配置生效。

执行本命令前必须先在系统视图下执行dhcp snooping enable命令使能全局的DHCP Snooping功能。

说明：
如果当前VLAN是RRPP的控制VLAN，则无法使能用户数达到告警阈值后的告警功能。

使用实例

# 限制VLAN 100内接口GE1/0/1上用户数数量为2000。

<Quidway> system-view
[Quidway] dhcp snooping enable
[Quidway] vlan 100
[Quidway-vlan100] dhcp snooping max-user-number 2000 interface gigabitethernet 1/0/1
