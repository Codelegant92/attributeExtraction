multicast routing-enable

命令功能

multicast routing-enable命令用来使能组播功能。

undo multicast routing-enable命令用来恢复缺省配置。

缺省情况下，没有使能组播功能。没有使能组播功能时，路由器不转发组播报文。

命令格式

multicast routing-enable

undo multicast routing-enable

参数说明

无

视图

系统视图、VPN实例视图

缺省级别

2：配置级

使用指南

只有在VPN实例或公网实例内使能组播功能，才能配置该实例下的其它所有组播命令。

要使能VPN实例的组播功能，必须满足以下两个条件：

  • VPN实例的索引号小于或等于组播VPN最大索引号，目前组播VPN最大索引号是1023。

  • 组播VPN实例的个数没有达到上限，目前系统最多支持1024个组播VPN实例。

使用undo multicast routing-enable命令，将清除公网实例或VPN实例上所有的组播配置。如果实例下正在运行组播业务，则组播业务将会中止。如果需要在实例上恢复组播业务，必须重新配置被清除掉的组播命令。

使用实例

# 启动公网实例的组播功能。

<Quidway> system-view
[Quidway] multicast routing-enable

# 启动VPN实例mytest的组播功能。

<Quidway> system-view
[Quidway] ip vpn-instance mytest
[Quidway-vpn-instance-mytest] route-distinguisher 100:1
[Quidway-vpn-instance-mytest] multicast routing-enable
